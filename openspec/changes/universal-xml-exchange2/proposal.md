## Why

Первая версия подсистемы обмена (`universal-xml-exchange`) реализовала pull-модель без контроля жизненного цикла сеанса: правила задавались именем макета на источнике, подтверждение доставки отсутствовало, «зависшие» выгрузки не отслеживались. Нужна двухфазная модель: статусы профиля на источнике, выгрузка только из «Новое», подтверждение приёмником через `ConfirmDataReceipt`, правила обмена в хранилище профиля и журналирование переходов.

## What Changes

1. **Справочник `рг_НастройкиОбменаXML`** — расширение объектной модели: `Префикс`, `ПрефиксБазыПриемника`, `Источник` (роль сервер/клиент), `Статус` (строка с кодами жизненного цикла: пусто = не готов, `Новое`, `Выполняется`, `Выполнено`, `Ошибка`), `ПравилаОбменаДанными` (хранилище XML-правил); для приёмника — `АдресСервиса`, `Логин`; для источника — ТЧ `Параметры` (Имя/Значение). Перечисление `рг_СтатусыОбмена` удаляется (extend 2026-06-29). При записи профиля-источника проверяется наличие правил в хранилище.
2. **Веб-сервис `rgExchangeService`** — операция `GetData` срабатывает только при статусе «Новое» и атомарно переводит профиль в «Выполняется»; ZIP содержит правила и данные. **BREAKING:** новая операция `ConfirmDataReceipt` завершает сеанс (статус «Выполнено»).
3. **Обработка `рг_УниверсальныйОбменДаннымиXML`** — встроенный движок выгрузки/загрузки в расширении (as-built export; API «Универсальный обмен данными XML»). Сервер создаёт экземпляр через `Обработки.рг_УниверсальныйОбменДаннымиXML.Создать()`. Макет epf и контейнер `ргУниверсальныйОбменXML` **не** используются (extend 2026-06-25).
4. **Бизнес-процесс обмена** — инициация на источнике (загрузка правил, параметры, кнопка «Подготовить к обмену» вместо ручного выбора «Новое»); на приёмнике — получение ZIP, загрузка, вызов `ConfirmDataReceipt` при успехе.
5. **Наблюдаемость и надёжность** — запись переходов статусов и ошибок в журнал регистрации; контроль «зависших» обменов через статус «Выполняется».

### Modified Capabilities

- `exchange-settings`: ролевая модель профиля (источник/приёмник), статусы сеанса, правила в хранилище значений, префиксы идентификации, проверки при записи
- `exchange-web-service`: условный `GetData` по статусу, двухфазное завершение через `ConfirmDataReceipt`
- `exchange-export`: выгрузка из правил хранилища профиля, ZIP «правила + данные», транзакционный перевод статуса
- `exchange-import`: загрузка пакета на приёмнике, вызов подтверждения доставки, обработка ошибок без подтверждения

## Impact

- **Расширение:** `src/ЗУП/cfe/рг_УниверсальныйОбменXML/` — справочник, веб-сервис, обработка, общий модуль (перечисление статусов удаляется в S4)

- **ADR-0003:** контракт движка **согласован** с as-built — встроенная `рг_УниверсальныйОбменДаннымиXML` (extend 2026-06-25); ADR при archive уточнить
- **Миграция:** существующие профили с `ИмяМакетаПравил` потребуют переноса правил в хранилище и настройки префиксов (если уже введены в эксплуатацию)
