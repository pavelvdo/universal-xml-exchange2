# Verification — «Движок обмена завершил выгрузку с ошибками»

## Прогресс приёмки S2

Цепочка WS **доходит до выгрузки**: профиль найден, статус «Новое»→«Выполняется» записан (fix privileged). Падение в `ВыгрузитьДанныеДвижком` → `Движок.ВыполнитьВыгрузку()` → `ФлагОшибки = Истина`.

## Причина (verified по коду)

Не инфраструктура и не права — **ошибка выполнения правил обмена** во встроенной `рг_УниверсальныйОбменДаннымиXML`. Детали в протоколе движка; в WS наружу уходит только общее сообщение (стр. 827).

После ошибки `ПолучитьДанныеОбмена` переводит профиль-источник в статус **«Ошибка»** — для повтора нужно вручную вернуть **«Новое»**.

## Диагностика на ИБ-источнике

1. Обработка `рг_УниверсальныйОбменДаннымиXML` → те же **правила** (из профиля) и **параметры** (ТЧ профиля) → **Выгрузка** → открыть **протокол** — там конкретная строка ошибки.
2. Профиль-источник: правила не пустые; ТЧ `Параметры` — имена совпадают с параметрами в XML правил.
3. Правила соответствуют **этой** конфигурации/данным (нет ссылок на отсутствующие объекты).
4. Журнал регистрации на источнике — события подсистемы обмена с текстом ошибки выгрузки.
5. После исправления: статус профиля **«Новое»** → повтор обмена.

## Улучшение (не в scope текущего fix)

Проброс текста протокола движка в исключение WS / журнал — отдельная задача extend.
