# Приложение 2. Форматы навигационных ссылок

> **Agent ref:** Форматы навигационных ссылок в 1С:Предприятие. Внешние и внутренние ссылки, параметры, адрес хоста, форматы по типам объектов.
> **Ключевые разделы:** форматы ссылок, адрес хоста, внутренние ссылки, параметры (ref, extdata, vrn, stngs, Z), e1cib/e1ccs.

---

## 2.1. Описание формата навигационной ссылки

### 2.1.1. Общая информация и виды ссылок

Навигационные ссылки бывают **внешние** (применение вне 1С:Предприятие) и **внутренние** (внутри платформы, избранное, история).

**Состав навигационной ссылки:**
- **Адрес хоста** — расположение информационной базы относительно клиента; метод `ПолучитьНавигационнуюСсылкуИнформационнойБазы()`.
- **Внутренняя ссылка** — расположение данных внутри базы.
- **Параметры ссылки** — опционально, конкретизируют поведение.

**Виды внутренних ссылок:** объект ИБ, реквизит объекта, реквизит ТЧ объекта, реквизит/запись регистра, отчёт, обработка, раздел, команда раздела, временное хранилище, обсуждение системы взаимодействия.

**Формирование внешней ссылки:**
- Открытие формы / навигация: адрес хоста + внутренняя ссылка. Типы: объект ИБ, отчёт, обработка, раздел, команда раздела.
- Ресурс (без открытия формы): адрес хоста + внутренняя ссылка. Типы: реквизит объекта/записи регистра, временное хранилище.

**Рекомендация:** при работе из разных клиентов запоминать внутренние ссылки; внешние, созданные в веб-клиенте, могут не работать в тонком клиенте (и наоборот).

### 2.1.2. Параметры навигационной ссылки

Параметры указываются после `?`; несколько параметров разделяются `&`. Параметр: `имя=значение`. Массив — значения через `,`. Ссылка как значение — предваряется именем типа через `:` (например: `ДокументСсылка.РасходнаяНакладная:a9b000055d49b45e11db8c4c9d5c4223`).

Пример: `e1cib/data/Справочник.Товары?ref=949b0050569f397311e8bb08fb974c0a` — внутренняя ссылка на элемент справочника Товары с параметром `ref`. При нескольких одноимённых параметрах используется первый (слева направо). Неизвестные параметры игнорируются.

**extdata** — передача параметров в открываемый объект; значение параметра — идентификатор в хранилище внешних данных навигационных ссылок. Интерпретация зависит от открываемого объекта.

### 2.1.3. Формат адреса хоста

- **Файловая ИБ:** UNC — `e1c:/file/dbsrvr/bases/mybase`; локальный диск — путь с буквой диска в том же формате.
- **Клиент-сервер:** строка адреса сервера и базы по правилам платформы.
- **Через веб-сервер:** URL веб-сервера и путь к публикации.

**Разделители (разделённая ИБ):** параметр `Z` после `?`. Тонкий клиент — все разделители сеанса; веб-клиент и тонкий через веб — только разделители без признака «не указывать в адресе» (настройка публикации). Указание разделителей задаётся в default.vrd.

## 2.2. Внутренние ссылки

### 2.2.1. Общая информация

Принадлежность: **e1cib** — данные в ИБ; **e1ccs** — данные на сервере системы взаимодействия.

Общий формат внутренней ссылки: `<принадлежность>/data/<путь к метаданным>/<идентификатор или структура>` (ниже уточняется по типу).

### 2.2.2. Ссылки на объекты информационной базы

**2.2.2.1. Ссылка на объект ИБ**  
Формат: `<путь к метаданным>/<идентификатор ссылки>`. Пример пути: `Документ.РасходТовара`.

**2.2.2.2. Ссылка на реквизит объекта ИБ**  
Формат: `<путь к метаданным>/<имя реквизита>/<идентификатор ссылки>`.

**2.2.2.3. Ссылка на реквизит табличной части**  
Формат: `<путь к метаданным>/<имя табличной части>/<имя реквизита>/<идентификатор ссылки>/<индекс строки ТЧ>`.

**2.2.2.4. Ссылка на запись регистра**  
Формат: `<путь к метаданным регистра>/<имя ключевого поля>/<значение>` (для периодических добавляется измерение Период). Значения отбора кодируются по RFC 1738 (% + два hex-символа).

**2.2.2.5. Ссылка на реквизит записи регистра**  
Формат: `<путь к метаданным>/<имя реквизита>/<имя ключевого поля>/<значение>`; кодирование значений — как выше.

**2.2.2.6. Ссылка на отчёт**  
Формат: `<путь к метаданным отчёта>`. Параметры: **vrn** — ключ варианта (передаётся в КлючВарианта формы); **stngs** — ключ в хранилище внешних данных с настройками компоновки (Вариант формы). При одновременном указании vrn и stngs используется только stngs.

**2.2.2.7. Ссылка на обработку**  
Формат: `<путь к метаданным>`, например `Обработка.ФормированиеЗаказа`.

**2.2.2.8. Ссылка на раздел**  
Формат: `<имя раздела>` (подсистема первого уровня), например `ПродажиТоваров`.

**2.2.2.9. Ссылка на точку навигации (стандартная команда)**  
Формат: `<имя раздела>/<имя команды>`, например `Справочник.Товары.ОткрытьСписок`.

**2.2.2.10. Ссылка на временное хранилище**  
Формат: идентификатор временного значения.

**2.2.2.11. Ссылка на запись таблицы внешнего источника**  
Формат: `<путь к метаданным таблицы ВнешнийИсточникДанных>.<таблица>/<имя ключевого поля>/<значение>`. Кодирование значения — по RFC 1738.

**2.2.2.12. Ссылки на команды глобального командного интерфейса**  
Формат: имя команды (стандартная — `Справочник.Товары.Создать`, общая — `ОбщаяКоманда.ПоискОбъектов`, команда — `Справочник.Товары.Команда.ПечатьШтрихКода`). Параметры: **cmdprm** — параметр команды (массив через `,`; составной тип — тип и значение через `:`); **exdata** — ключ в хранилище внешних данных, структура подставляется в ПараметрыВыполненияКоманды. Оба параметра необязательны.

**2.2.2.13. Ссылка на динамический список**  
Формат: `<имя списка>` (например `Справочник.Товар`, `Документы.РасходТоваров`). **stngs** — ключ хранилища с типом НастройкиТаблицыДинамическогоСписка (фиксированные/пользовательские настройки, режим отображения, текущая строка). Если ТекущаяСтрока = Неопределено, учитываются ВосстанавливатьТекущуюСтроку и НачальноеОтображениеСписка.

**2.2.2.14. Ссылка на начальную страницу**  
Формат по правилам платформы для начальной страницы.

### 2.2.3. Ссылки на объекты системы взаимодействия

**2.2.3.1. Ссылка на обсуждение**  
Формат: идентификатор обсуждения в системе взаимодействия. Режим совместимости 8.3.2 и младше — один формат; старше 8.3.2 — другой (по документации платформы).

**2.2.3.2. Ссылка на сообщение**  
Формат: идентификатор сообщения в системе взаимодействия. Не поддерживается получение ссылок на сервисные и не отправленные сообщения.
