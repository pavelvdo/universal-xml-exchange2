# Приложение 9. Работа с XBase

> **Agent ref:** Работа с форматом XBase (DBF) в 1С:Предприятие.  
> **Ключевые разделы:** поля и записи, структура таблицы, индексы, запись изменений, удаление записей, создание БД и индексов, ограничения.

---

## 9.1. Поля и записи

Термины для объекта встроенного языка XBase: запись — строка (например, одна запись справочника); поле — колонка с именем, типом, длиной и точностью. Значение поля для конкретной записи — значение поля. Таблица — набор записей с одной и той же структурой (набор полей); структура таблицы = состав полей; структура записи совпадает со структурой таблицы.

## 9.2. Таблица, структура таблицы, файл базы данных

Таблица в формате DBF хранится в отдельном файле. Термин «файл базы данных» или «файл БД» здесь означает файл таблицы (.dbf).

## 9.3. Индексы, выражение индекса и фильтр, индексный файл

Индексы используются для упорядочивания и поиска по одному или нескольким полям. У индекса: имя, признак уникальности, выражение индекса и фильтр (выражения на специальном языке; по ним определяется порядок и попадание записи в индекс). Уникальный индекс не допускает повторяющихся значений индексного выражения. Индексы хранятся в индексном файле; один индексный файл может содержать несколько индексов.

## 9.4. Запись изменений в базу данных

Изменение свойств объекта в памяти не записывается в файлы сразу. При включённом автосохранении запись в файлы происходит при смене позиции (переход к следующей записи, поиск по ключу и т.д.). При выключенном — только при вызове соответствующего метода. Объект XBase может быть связан не более чем с одним индексным файлом. Изменения в БД, сделанные в сеансе с другим индексным файлом, чем открытый ранее, требуют переиндексации (обновления индексного файла).

## 9.5. Работа с индексными файлами

См. п. 9.3 и 9.4: один индексный файл на объект; при смене индексного файла между сеансами нужна переиндексация.

## 9.6. Удаление записей

Удаление записи — пометка в служебном поле, физически запись не стирается. На помеченные удалёнными записи позиционирование не происходит, если не включён режим просмотра удалённых записей. Есть свойство и методы для определения «удалена ли запись» и восстановления. Метод сжатия базы физически уничтожает помеченные удалёнными записи; метод очистки базы — все записи. После них восстановление невозможно.

## 9.7. Создание базы данных, индекса, индексного файла

Объект XBase позволяет создавать новую БД произвольной структуры, новые индексы и новый индексный файл. Изменение структуры БД возможно только для объектов, не связанных с существующей БД (новая БД). Создание индексов и индексного файла допустимо и для уже открытой БД.

## 9.8. Ограничения

- Основное назначение XBase — экспорт/импорт в формате DBF.
- Поля типа memo не поддерживаются.
- Поддерживается только монопольный доступ к файлам.
- Поддерживаются индексные файлы формата CDX. Использование индексных файлов, созданных внешними программами (и наоборот), не рекомендуется из-за возможной несовместимости версий.
