# Срез S1: Профили обмена и регламентные задания

**Сценарий:** администратор создаёт/редактирует профиль обмена (URL, логин, имя макета правил, параметры фильтрации), пароль сохраняется в безопасном хранилище БСП, при включении расписания создаётся/обновляется/удаляется задание `рг_ВыполнениеОбменаXML`.
**Primary acceptance:** создать элемент `рг_НастройкиОбменаXML` → заполнить адрес сервиса, логин, имя макета, строки параметров, указать пароль → записать → переоткрыть: пароль не виден в реквизите, профиль сохранён.
**Приёмка:** ручной тест в ИБ с развёрнутым расширением.
**Связь со spec:** exchange-settings — Scenario «Создание профиля с обязательными реквизитами», Scenario «Запись пароля при сохранении профиля», Scenario «Настройка параметров фильтрации», Scenario «Включение регламентного обмена», Scenario «Отключение регламентного обмена».
**Зависимости:** нет.

## 1. Метаданные (ручное конфигурирование)

- [x] S1.1 Ручное конфигурирование (Конфигуратор): справочник `рг_НастройкиОбменаXML` с реквизитами `АдресСервиса`, `Логин`, `ИмяМакетаПравил`, `ИспользоватьРасписание`, `Расписание` (ХранилищеЗначения), `ИндентификаторРегламентногоЗадания` (UUID) — профиль подключения к источнику (D4; spec exchange-settings «Профиль обмена хранит параметры подключения»)
- [x] S1.2 Ручное конфигурирование (Конфигуратор): табличная часть `Параметры` с реквизитами `ИмяПараметра` (Строка 50) и `ЗначениеПараметра` (составной тип: Строка/Число/Дата/Булево) — произвольные параметры фильтрации (spec exchange-settings «Табличная часть произвольных параметров»)
- [x] S1.3 Ручное конфигурирование (Конфигуратор): формы `ФормаЭлемента` и `ФормаСписка`; на форме элемента — нехранимые реквизиты `Пароль` (режим «Пароль»), `ПарольИзменен` (Булево) и **`Расписание`** (реквизит **формы**, произвольный/без типа — как `РассылкиОтчетов`, `<Type/>` в Form.xml; **не** ХранилищеЗначения, **не** `DataPath>Объект.Расписание`); команду `РасписаниеРегламентногоОбменa`. Реквизит объекта `Расписание` (ХранилищеЗначения) — только в метаданных справочника; запись — в BSL через `ТекущийОбъект.Расписание` в `ПередЗаписьюНаСерверe`. Выгрузить `Form.xml`
- [x] S1.4 Ручное конфигурирование (Конфигуратор): общий модуль `рг_УниверсальныйОбменXMLСервер` (флаги «Сервер», «Вызов сервера») — серверная точка входа подсистемы (D3)
- [x] S1.5 Ручное конфигурирование (Конфигуратор): регламентное задание `рг_ВыполнениеОбменаXML` без предопределённого расписания, метод `рг_УниверсальныйОбменXMLСервер.ВыполнитьОбменПоПрофилю` — шаблон программных заданий по профилю (D3; зависит от S1.8)
- [x] S1.6 Выгрузить объекты S1 в `src/ЗУП/cfe/рг_УниверсальныйОбменXML/` — убедиться в наличии `Catalogs/рг_НастройкиОбменаXML/Ext/ObjectModule.bsl`, `Forms/ФормаЭлемента/Ext/Form/Module.bsl`, `CommonModules/рг_УниверсальныйОбменXMLСервер/`, `ScheduledJobs/рг_ВыполнениеОбменаXML/`
- [x] S1.11 Ручное конфигурирование (Конфигуратор): выдать роли `рг_ИспользованиеОбменаXML` права **только** на справочник `рг_НастройкиОбменаXML` (чтение, добавление, изменение, удаление, интерактивные операции); **не** назначать права на регламентное задание — в платформе 1C тип `ScheduledJob` недоступен в правах роли; управление заданиями — в коде S1.9 через `РегламентныеЗаданияСервер` в привилегированном режиме (D3); выгрузить `Roles/рг_ИспользованиеОбменаXML/Ext/Rights.xml`

## 2. Серверная логика (BSL)

- [x] S1.7 В модуле объекта `рг_НастройкиОбменаXML` (`Ext/ObjectModule.bsl`): при записи сохранять пароль через `ОбщегоНазначения.ЗаписатьДанныеВБезопасноеХранилище(Ссылка, Пароль, "Пароль")`, получая значение из `ДополнительныеСвойства` (ключ `"Пароль"`) — пароль не хранится в открытом виде (spec exchange-settings «Пароль хранится в безопасном хранилище БСП»)
- [x] S1.8 В общем модуле `рг_УниверсальныйОбменXMLСервер` (`Ext/Module.bsl`): экспортная процедура `ВыполнитьОбменПоПрофилю(Профиль)` — точка входа регламентного задания; при вызове из регламента ссылка на профиль передаётся параметром задания; полный сеанс обмена подключается в S2.14 (D3; spec exchange-import «Инициация загрузки пользователем или регламентом»)
- [x] S1.9 В модуле объекта `рг_НастройкиОбменаXML` (`Ext/ObjectModule.bsl`): при записи в привилегированном режиме создавать/обновлять/удалять регламентное задание `рг_ВыполнениеОбменаXML` через `РегламентныеЗаданияСервер` по флагу `ИспользоватьРасписание`, синхронизируя расписание из реквизита `Расписание`, сохраняя UUID задания в `ИндентификаторРегламентногоЗадания`, передавая в параметры задания ссылку на профиль (D3; spec exchange-settings «Управление регламентным обменом по профилю»)

## 3. Форма профиля (BSL)

- [x] S1.10 В модуле формы: расписание в **реквизите формы** `Расписание` (не `Объект.Расписание`); диалог БСП; `ПередЗаписьюНаСерверe` — `ТекущийОбъект.Расписание = Новый ХранилищеЗначения(Расписание)` и `ДополнительныеСвойства.Расписание` (паттерн `РассылкиОтчетов`); без `Тип("…")`

- [x] S1.12 Убрать `Объект.Расписание` из модуля формы; загрузка в реквизит формы `Расписание` из БД / `ЗначениеКопирования` / регламентного задания

- [x] S1.13 Проверка заполненности — `РасписаниеЗаполнено()` (Строка), без `Тип("РасписаниеРегламентногоЗадания")` — форма + объект

## 4. Приёмка

- [x] S1.accept Принять срез S1 «Профили обмена и регламентные задания» — профиль создаётся, пароль защищён, регламент управляется:
  - **Primary (обязательно):** создать элемент `рг_НастройкиОбменаXML` → заполнить адрес сервиса, логин, имя макета, строки параметров, указать пароль → записать → переоткрыть: пароль не виден в реквизите, профиль сохранён
  - Scenario «Создание профиля с обязательными реквизитами» (включено в Primary): адрес, логин и имя макета сохранены и доступны для последующего сеанса обмена
  - Scenario «Запись пароля при сохранении профиля» (включено в Primary): пароль записан в безопасное хранилище, в открытом реквизите пароля нет
  - Scenario «Настройка параметров фильтрации» (опционально): добавить строки ТЧ `Параметры` с примитивными значениями → записать → строки сохранены после переоткрытия
  - Scenario «Включение регламентного обмена» (опционально): включить флаг и задать расписание → записать → задание `рг_ВыполнениеОбменаXML` создано/обновлено
  - Scenario «Отключение регламентного обмена» (опционально): снять флаг или удалить профиль → связанное задание `рг_ВыполнениеОбменаXML` удалено/деактивировано

# Срез S2: Сквозной обмен данными (источник + приёмник)

**Сценарий:** оператор на приёмнике выбирает профиль и запускает обмен; приёмник вызывает `GetData`; источник выгружает по правилам из макета; приёмник распаковывает ZIP и загружает данные обработкой `ргУниверсальныйОбменXML`.
**Primary acceptance:** на источнике опубликован `rgExchangeService` → на приёмнике открыть обработку `ргУниверсальныйОбменXML` → выбрать профиль → запустить обмен → данные из источника появились в базе-приёмнике.
**Приёмка:** ручной end-to-end тест на двух ИБ.
**Связь со spec:** exchange-web-service, exchange-export, exchange-import; exchange-settings — Scenario «Использование пароля при обмене».
**Зависимости:** S1.

## 1. Метаданные источника и приёмника (ручное конфигурирование)

- [ ] S2.1 Ручное конфигурирование (Конфигуратор): создать обработку `ргУниверсальныйОбменXML` на базе штатной `УниверсальныйОбменДаннымиXML` (`src/ЗУП/cf/DataProcessors/УниверсальныйОбменДаннымиXML/`) с доработками по ТЗ; добавить форму ручного запуска обмена (выбор профиля, команда «Выполнить обмен») (D6; spec exchange-import «Инициация загрузки пользователем или регламентом»)
- [ ] S2.2 Ручное конфигурирование (Конфигуратор): при adopt обработки `ргУниверсальныйОбменXML` скопировать вспомогательные макеты движка (`ОбщиеПроцедурыФункции`, `ПараметрыОбработчиков` из эталона cf) и добавить макет(ы) правил обмена (XML, КД 2.1) — правила выгрузки на источнике с обработчиками преобразования примитивов в ссылки (spec exchange-export «Выгрузка через обработку ргУниверсальныйОбменXML»; design § Manual Configuration)
- [ ] S2.3 Ручное конфигурирование (Конфигуратор): создать SOAP-веб-сервис `rgExchangeService` с операцией `GetData` (параметры `ИмяМакетаПравил` String, `Параметры` тип XDTO `Structure` пакета `http://v8.1c.ru/8.1/data/core`; возврат `base64Binary`), аутентификация HTTP Basic на уровне публикации (D2, D5)
- [ ] S2.4 Выгрузить обработку `ргУниверсальныйОбменXML`, веб-сервис `rgExchangeService` и изменённые объекты в `src/ЗУП/cfe/рг_УниверсальныйОбменXML/` — убедиться в появлении модулей обработки, формы и веб-сервиса
- [ ] S2.5 Ручное конфигурирование (Конфигуратор): дополнить права роли `рг_ИспользованиеОбменаXML` на обработку `ргУниверсальныйОбменXML` и веб-сервис `rgExchangeService`; выгрузить `Roles/рг_ИспользованиеОбменаXML/Ext/Rights.xml` — права на объекты среза S2 (D6; design § Manual Configuration)

## 2. Контракт движка (static, gate перед BSL выгрузки/загрузки)

- [ ] S2.6 Верифицировать по коду (Read/Grep по `src/ЗУП/cf/DataProcessors/УниверсальныйОбменДаннымиXML/`) публичный контракт штатной обработки — имена параметров обмена, методов выгрузки/загрузки и обязательных вспомогательных макетов (`ОбщиеПроцедурыФункции`, `ПараметрыОбработчиков`); зафиксировать в `design.md` § Engine Contract и подтвердить отсутствие автоматического поиска ссылок по примитивам в ядре `рг_УниверсальныйОбменXMLСервер` (D6; spec exchange-export «Без автоматического поиска ссылок»). **Блокирует S2.7–S2.13.**

## 3. Серверная выгрузка на источнике (BSL)

- [ ] S2.7 В общем модуле `рг_УниверсальныйОбменXMLСервер` (`Ext/Module.bsl`): добавить экспортную функцию выгрузки — создать `Обработки.ргУниверсальныйОбменXML.Создать()`, загрузить правила из указанного макета, подставить параметры из XDTO-структуры, активировать все ПВД и сформировать XML-файл (spec exchange-export «Загрузка правил из макета», «Подстановка параметров перед выгрузкой», «Активация всех ПВД»; зависит от S2.6)
- [ ] S2.8 В общем модуле `рг_УниверсальныйОбменXMLСервер` (`Ext/Module.bsl`): упаковать сформированный XML-файл выгрузки в ZIP-архив и вернуть его как Base64 — сжатый пакет для передачи клиенту (spec exchange-export «Упаковка результата в ZIP»)
- [ ] S2.9 В модуле веб-сервиса `rgExchangeService` (`Ext/Module.bsl`): реализовать операцию `GetData(ИмяМакетаПравил, Параметры)` как тонкую обёртку над функцией выгрузки общего модуля (spec exchange-web-service «Операция GetData»)

## 4. Приёмник: сеанс обмена (BSL)

- [ ] S2.10 В общем модуле `рг_УниверсальныйОбменXMLСервер` (`Ext/Module.bsl`): реализовать подготовку сеанса по профилю — чтение настроек справочника, чтение пароля через `ОбщегоНазначения.ПрочитатьДанныеИзБезопасногоХранилища`, формирование XDTO-структуры параметров из ТЧ `Параметры` (`ИмяПараметра`, `ЗначениеПараметра`) (spec exchange-import «Чтение настроек профиля»)
- [ ] S2.11 В общем модуле `рг_УниверсальныйОбменXMLСервер` (`Ext/Module.bsl`): вызвать веб-сервис источника через WS-прокси с HTTP Basic, передать имя макета и XDTO-структуру параметров, получить Base64 ZIP (D5; spec exchange-import «Вызов веб-сервиса и передача параметров»)
- [ ] S2.12 В общем модуле `рг_УниверсальныйОбменXMLСервер` (`Ext/Module.bsl`): распаковать полученный ZIP во временный каталог сервера и загрузить XML через `Обработки.ргУниверсальныйОбменXML.Создать()`, гарантируя удаление временных файлов после сеанса (spec exchange-import «Распаковка и загрузка XML»; зависит от S2.6)
- [ ] S2.13 В общем модуле `рг_УниверсальныйОбменXMLСервер` (`Ext/Module.bsl`): обрабатывать ошибки соединения и некорректного ответа — сообщение пользователю при ручном запуске и запись в журнал регистрации при регламентном (spec exchange-import «Обработка ошибок соединения»)
- [ ] S2.14 В общем модуле `рг_УниверсальныйОбменXMLСервер` (`Ext/Module.bsl`): дополнить процедуру `ВыполнитьОбменПоПрофилю(Профиль)` (S1.8) вызовом полного сеанса обмена приёмника (spec exchange-import «Регламентный запуск»)

## 5. Форма обработки (BSL)

- [ ] S2.15 В модуле формы обработки `ргУниверсальныйОбменXML` (`Ext/Form/Module.bsl`): по команде «Выполнить обмен» для выбранного профиля вызывать сеанс обмена серверного модуля и показывать пользователю результат или ошибку (spec exchange-import «Ручной запуск обмена»)

## 5. Приёмка

- [ ] S2.accept Принять срез S2 «Сквозной обмен данными» — данные из источника загружены в приёмник одним запуском:
  - **Primary (обязательно):** на источнике опубликован `rgExchangeService` → на приёмнике открыть обработку `ргУниверсальныйОбменXML` → выбрать профиль → запустить обмен → данные из источника появились в базе-приёмнике
  - Scenario «Использование пароля при обмене» (включено в Primary): обмен проходит аутентификацию паролем из безопасного хранилища без ввода пароля вручную
  - Scenario «Публикация сервиса» (опционально): администратор публикует `rgExchangeService` на источнике → приёмник обращается к нему по URL из профиля
  - Scenario «Передача параметров фильтрации» (опционально): задать параметры в ТЧ профиля → запустить обмен → выгружены только данные по этим фильтрам
  - Scenario «Регламентный запуск» (опционально): дождаться/инициировать срабатывание задания `рг_ВыполнениеОбменаXML` → обмен выполнен без участия пользователя
  - Scenario «Недоступность веб-сервиса» (опционально): указать недоступный URL → запустить обмен → сообщение об ошибке, загрузка не выполнена
