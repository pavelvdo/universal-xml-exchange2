> **Agent ref:** Временное хранилище (сессия, адрес, время жизни). Файлы: клиент–сервер, асинхронные методы, веб-клиент, расширение работы с файлами. ## 21.1. Временное хранилище

Временное хранилище — специализированное хранилище, привязанное к сеансу. Назначение: временное хранение данных при клиент-серверном взаимодействии до переноса в базу. Совместно с механизмом работы с файлами позволяет: поместить данные с клиента во временное хранилище ИБ, перенести их в БД и получить обратно на компьютер пользователя. Типичные задачи: хранение изображений товаров, файлов, связанных с документами.

В веб-клиенте при передаче файла с клиента на сервер файл сначала помещается во временное хранилище, затем на сервере используется при записи объекта в БД.

**Ограничение:** во временное хранилище нельзя помещать данные типов, не поддерживающих сериализацию (в т.ч. в составе коллекций).

**Время жизни данных:**

- **Привязка к форме:** жизнь данных = жизнь формы; при удалении формы связанные данные очищаются.
- **Без привязки к форме:** очистка выполняется при следующем запросе формы, при следующем серверном вызове из клиентского общего модуля, при контекстном/неконтекстном клиентском вызове из формы, при серверном вызове из модуля команды (очистка — после окончания вызова; если вызов только помещал значение — очистка не выполняется до его окончания).

То есть можно поместить значение во временное хранилище в одном вызове и использовать в следующем; после использования оно автоматически удаляется до завершения серверного вызова.

Данные идентифицируются уникальным адресом (возвращают методы записи). Проверка адреса: метод глобального контекста `ЭтоАдресВременногоХранилища()`. Временное хранилище одного сеанса недоступно из другого. Исключение: передача данных из фонового задания в сеанс, инициировавший задание (см. 21.3.9).

## 21.2. Особенности работы с файлами в асинхронном режиме и веб-клиенте

Рекомендуется передавать файл на сервер и уже на сервере загружать данные в нужный формат (например, в реквизит объекта).

Файловые операции на стороне клиента (включая перенос файлов клиент–сервер и работу с временным хранилищем) **асинхронны** во всех клиентских приложениях. На сервере предпочтительны синхронные методы; на клиенте — асинхронные. При ошибках файловых операций используется конструкция `Попытка … Исключение`.

**Веб-клиент:**

1. Без расширения работы с файлами доступны методы `ПолучитьФайл()`, `НачатьПомещениеФайла()`, `НачатьПомещениеФайлов()` (с ограниченной параметризацией).
2. В веб-клиенте доступны только асинхронные методы работы с файлами (не зависит от настройки «Режим использования модальности»).
3. При установке расширения работы с файлами расширяется параметризация асинхронных методов; синхронная работа с файлами в веб-клиенте недоступна.
4. Расширение не поддерживает HTTPS с клиентским сертификатом.

Расширение требуется, когда файловая операция не сопровождается интерактивным действием пользователя (например, помещение файла по пути из кода). Указание файла через диалог выбора расширения не требует.

**Примечания:** для Internet Explorer рекомендуется MSXML 4.0 или 6.0; расширение для IE устанавливается в `%APPDATA%\1C\1СEWebExt\FileSystemExtIE`.

## 21.3. Способы работы с файлами и временным хранилищем

### 21.3.1. Общая информация

Код, который должен выполниться после помещения файла, следует размещать в обработчике обратного вызова метода `НачатьПомещениеФайла()`, чтобы поведение в тонком и веб-клиенте совпадало.

### 21.3.2. Помещение данных из файла во временное хранилище

#### 21.3.2.1. Один файл

**Клиент:** метод `ПоместитьФайлНаСерверАсинх()`. По завершении (через `Ждать`) возвращается объект `ОписаниеПомещенногоФайла`.

Параметры:

- **Адрес** — адрес во временном хранилище; если не указан, создаётся новый элемент, адрес передаётся в обработчик. Если указан — данные по старому адресу удаляются.
- **Описание файла:** не указано (диалог по умолчанию); полный путь (в веб-клиенте нужно расширение); объект `СсылкаНаФайл` (после перетаскивания на форму); объект `ПараметрыДиалогаПомещенияФайлов`.
- **Уникальный идентификатор формы** — при указании корректного идентификатора элемент хранилища удаляется автоматически после следующего серверного вызова; при несоответствии — при завершении клиентского сеанса. Рекомендуется явно удалять неиспользуемое значение методом `УдалитьИзВременногоХранилища()`.

События метода: оповещение перед началом помещения (можно отказаться, например при превышении размера); оповещение о ходе выполнения (можно прервать передачу через `ОтказОтПомещенияФайла`). Фильтры диалога выбора не гарантируют тип выбранных файлов — лишь упрощают выбор.

**Сервер:** синхронно — `ПоместитьФайл()`; асинхронно (обратные вызовы) — `НачатьПомещениеФайлаНаСервер()`.

#### 21.3.2.2. Несколько файлов

Метод `ПоместитьФайлыНаСерверАсинх()`: общая схема как у помещения одного файла. Описание файлов: строка с маской (веб-клиент — нужно расширение) или массив объектов (`ОписаниеПередаваемогоФайла` с путём или `СсылкаНаФайл` при перетаскивании). В одном массиве — значения одного типа. Обработчики получают массивы; можно отказаться от всех файлов или от части; передача файлов последовательная. Результат `Ждать` — массив `ОписаниеПомещенногоФайла`.

**Сервер:** синхронно — `ПоместитьФайлы()`; асинхронно — `НачатьПомещениеФайловНаСервер()`.

#### 21.3.2.3. Данные из памяти

`ПоместитьВоВременноеХранилище()` — размещение в хранилище значения из оперативной памяти (не из файла). Без параметра «Адрес» данные размещаются по новому адресу, возвращаемому функцией. Размещённые данные привязываются к форме или очищаются при серверном вызове в зависимости от параметров.

**Внимание:** фактическая сериализация при помещении не выполняется — помещается ссылка на значение в кеше (до 20 минут), затем значение сериализуется, записывается в хранилище сеансовых данных и удаляется из кеша. Адрес, по которому уже вызван `УдалитьИзВременногоХранилища()`, нельзя повторно использовать как параметр `Адрес` метода `ПоместитьВоВременноеХранилище()` — будет исключение.

### 21.3.3. Получение данных из информационной базы в файл

#### 21.3.3.1. Один файл

`ПолучитьФайлССервераАсинх()`: получает файл из ИБ и сохраняет в локальную ФС. Параметры: адрес данных в ИБ (навигационная ссылка) — обязательный; полное имя файла для сохранения или объект `ПараметрыДиалогаПолученияФайлов`. В зависимости от варианта: с диалогом — результат обещания `Неопределено`, расширение в веб-клиенте не обязательно; без диалога — результат типа `ОписаниеПереданногоФайла`, в веб-клиенте нужно расширение. У пользователя должно быть право «Просмотр» на реквизит с сохраняемыми данными.

#### 21.3.3.2. Несколько файлов

`ПолучитьФайлыССервераАсинх()`: массив описаний получаемых файлов (`ОписаниеПередаваемогоФайла`), описание диалога сохранения, при необходимости — параметры архива. Результат — объект `Ожидание`; при диалоге результат выполнения — `Неопределено`; без диалога — массив `ОписаниеПереданногоФайла`. В веб-клиенте: вариант с диалогом не требует расширения (но использует его для настроек); вариант с фиксированным каталогом требует расширения.

`ПараметрыДиалогаПолученияФайлов`: `ВыборКаталога` = Истина — один каталог для всех файлов (в веб-клиенте игнорируется); Ложь — диалог для каждого файла. `Заголовок`, `Каталог` — заголовок и начальный каталог (в веб-клиенте начальный каталог игнорируется).

**Архив:** параметр `ПараметрыПолученияАрхиваФайлов`. Режим: `ПолучатьАрхивВсегда` — архив всегда; `ПолучатьАрхивПриНеобходимости` — только при получении нескольких файлов и при отсутствии расширения. Имя архива — свойство `ИмяАрхива`. Файлы сначала сохраняются во временный каталог сервера, упаковываются платформой, архив передаётся на клиент. Размещение: по `ОписаниеПередаваемогоФайла.Имя` — имя без каталога (в корне/каталоге), с относительным путём (создаётся структура), абсолютное имя (сохранение по пути; в архиве воссоздаётся структура).

### 21.3.4. Получение данных из временного хранилища

`ПолучитьИзВременногоХранилища(Адрес)` — извлечение данных по адресу, выданному при помещении.

**Внимание:** на сервере значение получается по ссылке на объект в кеше. До 20 минут с момента помещения или последнего обращения значение в кеше, затем записывается на диск и удаляется из кеша; при следующем обращении загружается с диска. После сериализации/десериализации ссылки внутри значения на другие объекты не восстанавливаются.

### 21.3.5. Удаление данных из временного хранилища

`УдалитьИзВременногоХранилища(Адрес)` — удаление данных по адресу. После сохранения данных в реквизите объекта ИБ хранящиеся во временном хранилище можно удалить.

### 21.3.6. Проверка адреса на принадлежность временному хранилищу

`ЭтоАдресВременногоХранилища(Адрес)` — возвращает Истина, если адрес указывает на временное хранилище.

### 21.3.7. Получение адреса реквизита

Для доступа к данным в реквизите через файловые методы нужен адрес в ИБ. `ПолучитьНавигационнуюСсылку(КлючОбъекта, ИмяРеквизита)` возвращает адрес значения в ИБ. Для реквизита табличной части в имя реквизита добавляется имя табличной части и точка (например, `"ТЧ.Реквизит"`).

### 21.3.8. Разрешение на выполнение группы операций с файлами

В веб-клиенте при нескольких файловых операциях (например, сохранение документа и запуск приложения) можно уменьшить число запросов пользователю: `ЗапроситьРазрешениеПользователяАсинх()` показывает список планируемых операций и запрашивает разрешение на группу. При разрешении операции выполняются без дополнительных запросов; при отказе — по одному запросу на операцию. В тонком и толстом клиенте метод возвращает Истина без диалога. В веб-клиенте требуется расширение работы с файлами.

Разрешение запрашивается для конкретного набора параметров; при отличии параметров при реальном вызове запрос повторится. Для нескольких одинаковых операций (в т.ч. с теми же параметрами) нужно указать соответствующее количество элементов в массиве параметров. Интерактивные операции из запроса исключаются. Разрешения действуют до выполнения разрешённого вызова или до окончания исполнения встроенного языка.

Методы, для которых можно запрашивать разрешение: `ПоместитьФайлНаСерверАсинх`, `ПоместитьФайлыНаСерверАсинх`, `ПолучитьФайлССервера`, `ПолучитьФайлыССервера`, `ЗапуститьПриложениеАсинх`, `УдалитьФайлыАсинх`, `ПоискФайловАсинх`, `СоздатьКаталогАсинх`, `СоздатьДвоичныеДанныеИзФайлаАсинх`, `ФайловыеПотоки.СоздатьВременныйФайлАсинх`, `ФайловыеПотоки.СоздатьАсинх`, `ФайловыеПотоки.ОткрытьАсинх`, `ФайловыеПотоки.ОткрытьДляЧтенияАсинх`, `ФайловыеПотоки.ОткрытьДляЗаписиАсинх`, `ФайловыеПотоки.ОткрытьДляДописыванияАсинх`, `ДвоичныеДанные.ЗаписатьАсинх`. Синхронный вариант: `ЗапроситьРазрешениеПользователя()`; асинхронный: `НачатьЗапросРазрешенияПользователя()`.

### 21.3.9. Работа с временным хранилищем в фоновом задании

Передача данных из фонового задания в сеанс, запустивший задание: в родительском сеансе поместить во временное хранилище пустое значение методом `ПоместитьВоВременноеХранилище()`, указав параметр `Адрес`. Этот адрес передать в фоновое задание параметрами. В фоновом задании использовать тот же адрес как параметр `Адрес` метода `ПоместитьВоВременноеХранилище()` — результат будет скопирован в сеанс, из которого запущено задание. Данные из фонового задания доступны в родительском сеансе только после завершения задания.

### 21.3.10. Доступ к стандартным каталогам

- **Рабочий каталог данных пользователя** — каталог, привязанный к пользователю и ИБ (разные ИБ — разные каталоги). Метод `РабочийКаталогДанныхПользователяАсинх()`. Каталог создаётся при первом обращении; при невозможности создания — исключение. Используется для хранения файлов, отчётов, данных для внешних систем и т.п.
- **Каталог документов** — каталог ОС для постоянного хранения данных пользователя (отчёты, печатные формы). Метод `КаталогДокументовАсинх()`. Расположение зависит от ОС (см. синтакс-помощник).

Синхронные аналоги: `РабочийКаталогДанныхПользователя()`, `КаталогДокументов()`. Асинхронные (обратные вызовы): `НачатьПолучениеРабочегоКаталогаПользователя()`, `НачатьПолучениеКаталогаДокументов()`.

## 21.4. Работа с картинками

### 21.4.1. Параметры картинки и операции преобразования

Объект `Картинка` (конструктор по имеющимся данным) даёт доступ к свойствам: `Ширина()`, `Высота()` (в точках); `ГлубинаЦвета()` (системное перечисление); `ОттенкиСерого()` (PNG, TIFF); `КоличествоКадров()` (многостраничный TIFF); `ПлотностьПоГоризонтали()`, `ПлотностьПоВертикали()` (DPI, по умолчанию 72); `ФорматКартинки()`; свойство `НаборВариантов` (набор картинок). Для набора картинок методы `Ширина`, `Высота`, `ПлотностьПоГоризонтали`, `ПлотностьПоВертикали`, `РазмерФайла` не имеют смысла.

**Преобразования:** объект `ОбрабатываемаяКартинка` (создаётся по исходной картинке). Методы выполняются последовательно над текущим состоянием картинки в объекте: `УстановитьРазмер()`, `УстановитьФормат()`, `Масштабировать()`, `Повернуть()`, `УстановитьГлубинуЦвета()`, `ПреобразоватьВОттенкиСерого()`, `УстановитьПлотность()` (метаданные DPI; если формат не хранит — без эффекта), `ВыделитьОбласть()` (фрагмент замещает картинку), `ВыделитьЗначимуюЧастьКартинки()`, `ПолучитьКартинки()` (массив страниц многостраничного TIFF). Результат — получить объект `Картинка` из `ОбрабатываемаяКартинка` и сохранить в файл или использовать иначе.

**Отображение на форме:** элемент «Поле картинки» отображает картинку по адресу значения (картинка или двоичные данные) во временном хранилище или в БД. В свойстве «Данные» задаётся реквизит строкового типа; его значение интерпретируется как адрес картинки.
