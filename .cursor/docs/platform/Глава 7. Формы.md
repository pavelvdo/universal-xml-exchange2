## 7.1. Общая информация

В редакторе формы определяются: реквизиты формы, параметры формы, команды формы, модуль формы, элементы формы, командный интерфейс.

ВНИМАНИЕ! Формы одновременно присутствуют и на стороне сервера, и на стороне клиента. Об этом всегда следует помнить при разработке формы.

## 7.2.2. Типы данных формы

Совокупность всех реквизитов формы — данные формы. Для отображения и редактирования служат элементы формы, связанные с реквизитами.

ВНИМАНИЕ! Все данные формы должны быть описаны в виде реквизитов. Не допускается использование переменных модуля формы в качестве источников данных для элементов формы.

Основной реквизит формы определяет стандартную функциональность формы (расширение формы).

ВНИМАНИЕ! Основной реквизит у формы может быть только один.

Типы данных формы:
- Типы, существующие на тонком и веб-клиенте (Число, СправочникСсылка.Товары, ГрафическаяСхема, ТабличныйДокумент).
- Типы в круглых скобках в редакторе — прикладные типы, преобразуемые в данные формы, например (СправочникОбъект.Товары).
- Динамический список — отображение произвольной выборки из БД; механизм на системе компоновки данных (сортировка, отбор, поиск, группировка, условное оформление).

Специальные типы данных формы:
- ДанныеФормыСтруктура — набор свойств произвольного типа (структуры, коллекции). Представление в форме: СправочникОбъект и т.п.
- ДанныеФормыКоллекция — список по индексу или идентификатору; НомерСтроки не равен индексу для ТЧ и наборов записей. Представление: табличная часть.
- ДанныеФормыСтруктураСКоллекцией — структура и коллекция одновременно. Представление: набор записей.
- ДанныеФормыДерево — иерархические данные.

ВНИМАНИЕ! Не рекомендуется использовать данные формы в качестве параметров процедур/функций при передаче управления клиент→сервер и в качестве возвращаемых значений при передаче на клиента.

ВНИМАНИЕ! Прикладные объекты доступны только на сервере; объектами данных форм можно пользоваться и на сервере, и на клиенте.

Ограничения реквизитов формы:
- Только типы с возможностью XDTO-сериализации.
- Запрещено присваивание Массив и Соответствие реквизитам формы.
- Для ДанныеФормыСтруктура, ДанныеФормыЭлементКоллекции, ДанныеФормыЭлементДерева, ДанныеФормыСтруктураСКоллекцией — запрещено присваивание Массив и Соответствие.
- Рекомендуется ФиксированныйМассив, ФиксированноеСоответствие вместо Массив/Соответствие в коллекциях.
- Не рекомендуется Структура и СписокЗначений в реквизитах реквизитов формы.
- В реквизитах типа Произвольный и в коллекциях не допускаются прикладные объекты (ДокументОбъект, СправочникОбъект и т.д.).

Свойство реквизита **Использовать всегда**:
- Реквизит под динамическим списком: включено — всегда считывается из БД; выключено — только при наличии видимого элемента формы, связанного с реквизитом.
- Реквизит под коллекцией движений: включено — движения считываются; выключено — не считываются (если нет элемента на движения).
- Остальные реквизиты: включено — реквизит всегда в данных формы; выключено — только при наличии элемента формы, связанного с реквизитом (видимость не учитывается).

ПРИМЕЧАНИЕ 1. Свойство у родительского реквизита действует на все подчиненные.
ПРИМЕЧАНИЕ 2. Для объектных типов реквизиты Ссылка, ПометкаУдаления, ЭтоГруппа всегда в данных формы независимо от «Использовать всегда».
ПРИМЕЧАНИЕ 3. Запрещено присваивание реквизитам данных формы значений других объектов данных формы.

Колонки реквизитов, не связанные с данными (дополнительные колонки ТЧ/наборов записей): создаются при создании данных формы; доступны на клиенте и сервере. Для ДинамическийСписок — параметры списка (основная таблица, настройки).

ВНИМАНИЕ! Колонки реквизитов, не связанные с данными, не участвуют в преобразовании между данными формы и объектами ИБ.

Обновление отображения: изменения данных формы отображаются элементами после окончания выполнения кода или после вызова ОбновитьОтображениеДанных().

ПРИМЕЧАНИЕ. В веб-клиенте ОбновитьОтображениеДанных() обновляет элементы только после окончания выполнения кода.

## 7.2.3. Преобразование данных прикладных объектов в данные формы и обратно

Глобальные методы (сервер):
- ЗначениеВДанныеФормы() — прикладной объект → данные формы.
- ДанныеФормыВЗначение() — данные формы → прикладной объект.
- КопироватьДанныеФормы() — копирование между данными формы (доступен и на клиенте).

Методы объекта ФормаКлиентскогоПриложения (сервер):
- ЗначениеВРеквизитФормы() — прикладной объект → указанный реквизит формы.
- РеквизитФормыВЗначение() — реквизит формы → прикладной тип.

ПРИМЕЧАНИЕ. При стандартных действиях (открытие формы, Записать) в форме с основным реквизитом преобразование выполняется автоматически.

ВНИМАНИЕ! Колонки реквизитов, не связанные с данными, не участвуют в преобразовании. Колонки, отсутствующие в объекте, при преобразовании в данные формы очищаются. Внутреннее состояние объекта (например, УстановитьСсылкуНового()) в данные формы не переносится.

Первый параметр РеквизитФормыВЗначение() и ДанныеФормыВЗначение() — только реквизиты типов: ДанныеФормыСтруктура, ДанныеФормыКоллекция, ДанныеФормыСтруктураСКоллекцией, ДанныеФормыДерево. В преобразуемом объекте должны существовать все колонки данных формы.

## 7.2.4. Свойства реквизитов

- **Основной реквизит** — данный реквизит является основным, определяет расширение формы.
- **Сохраняемые данные** — при изменении: попытка блокировки связанного реквизита, установка Модифицированность; в режиме «Только просмотр» формы элементы, связанные с реквизитом, тоже только просмотр.
- **Проверка заполнения** (Выдавать ошибку) — для примитивных типов, СписокЗначений, ДеревоЗначений, ТаблицаЗначений. Проверка аналогична ЗначениеЗаполнено(); для ТЧ — хотя бы одна строка.

ПРИМЕЧАНИЕ. Проверка заполнения табличных частей: табличная часть заполнена при наличии хотя бы одной строки.

## 7.3. Специальные типы реквизитов формы

### 7.3.1. Динамический список

Формирование запроса: задание основной таблицы (свойство Основная таблица) или произвольный запрос (свойство Произвольный запрос). Без основной таблицы снижается эффективность и недоступны команды, связанные с основной таблицей.

Параметр формы **Отбор**: имя свойства структуры в параметре должно совпадать с именем поля отбора динамического списка. Массив/фиксированный массив/список значений в отборе → условие В Списке.

Свойство **Динамическое считывание данных** — порционное считывание без кеша на сервере (способ 1) или страницами с кешем (способ 2/3) в зависимости от основной таблицы и вида ключа.

Свойство **Получение представлений для невидимых полей**: Истина — представления для всех ссылочных полей на клиенте; Ложь — только для видимых элементов (может ускорить список при большом числе ссылочных полей).

Настройки динамического списка (КомпоновщикНастроек):
- Настройки — из конфигуратора; быстрый доступ: Список.Порядок.
- ПользовательскиеНастройки — изменения пользователя в 1С:Предприятие.
- ФиксированныеНастройки — из кода и параметров формы; быстрый доступ: Список.Отбор, Список.Параметры, Список.УсловноеОформление.

Итоговые настройки комбинируются: пользовательские целиком/поэлементно, фиксированные добавляются «как есть». Одноименные настройки в фиксированных и пользовательских недопустимы.

Без основной таблицы доступны свойства **Вид ключа** (Значение поля, Ключ строки, Номер строки, Авто) и **Поля ключа**. Ключ должен обеспечивать уникальность строк запроса.

Ограничения произвольного запроса: не поддерживаются ПЕРВЫЕ; отбор/сортировка/группировка по реквизитам ТЧ, полям представлений, ВерсияДанных, ИмяПредопределенныхДанных, неограниченная Строка, ДвоичныеДанные и др.; не поддерживается УПОРЯДОЧИТЬ ПО (задавать через настройки списка). Не поддерживаются в качестве основной таблицы: табличные части объектов, таблицы регистрации изменений, последовательностей, перерасчетов и др.

Поиск: полнотекстовый поиск по основной таблице; строка поиска до 1000 символов, до 32 слов; разбиение по пробелам; для полей — условия по типу (ПОДОБНО для строки, = для числа, дата в диапазоне дня и т.д.). Получение данных для отчета: ПолучитьИсполняемуюСхемуКомпоновкиДанных(), ПолучитьИсполняемыеНастройкиКомпоновкиДанных().

Обработчик **ПриПолученииДанныхНаСервере** (внеконтекстный): параметры ИмяЭлемента, Настройки (копия настроек списка, Настройки.ДополнительныеСвойства для передачи данных), Строки (коллекция строк для отображения). Вызывается после получения данных списком; отбор, сортировка, условное оформление уже применены. Изменения в Строки.Данные и Строки.Оформление. Для полей, изменяемых в обработчике, рекомендуется исключать их из группировки/порядка/отбора методами УстановитьОграниченияИспользованияВГруппировке(), УстановитьОграниченияИспользованияВПорядке(), УстановитьОграниченияИспользованияВОтборе().

ПРИМЕЧАНИЕ. Примеры в документации не являются законченными; демонстрируют механизмы работы с динамическим списком.

### 7.3.2. Форматированный документ

Реквизит формы типа ФорматированныйДокумент, элемент — Поле форматированного документа. В ИБ хранить в ХранилищеЗначения. API: закладки (положение символа), ПолучитьЗакладкуПоПозиции(), ПолучитьПозициюПоЗакладке(), ПолучитьГраницыВыделения(), УстановитьГраницыВыделения(); Элементы (ПараграфФорматированногоДокумента), ПолучитьЭлементы(), СформироватьЭлементы(). Для редактирования: УстановитьHTML() при чтении, ПолучитьHTML() перед записью.

### 7.3.3. Планировщик

Реквизит типа Планировщик. Измерения — коллекция Планировщик.Измерения (помимо шкалы времени). Элементы — Планировщик.Элементы.Добавить(Начало, Конец); свойства: Текст, ЗначенияИзмерений (ФиксированноеСоответствие), Значение (уникальный идентификатор), Расписание (для повторяющихся). Шкала времени: Планировщик.ШкалаВремени.Положение, Элементы, Единица, Кратность, Формат. Периоды отображения: Планировщик.ТекущиеПериодыОтображения. События: ПередСозданием, ПередНачаломБыстрогоРедактирования, ПередНачаломРедактирования (переопределение форм); ОбработкаФормированияКоманд (контекстное меню, ОписаниеКомандыПланировщика); НажатиеНаДействиеПланировщика. ПроверкаПеретаскиванияВнутри() — параметры ДействиеПеретаскивания, ПараметрыПеретаскивания, ВариантИзмененияГраниц, Отказ.

### 7.3.4. Диаграммы

Типы: Диаграмма, ДиаграммаГанта, Дендрограмма. Реквизит Диаграмма на форме — поле вида Поле диаграммы. Заполнение: серии и точки (Диаграмма.Серии.Добавить(), Диаграмма.Точки.Добавить()), УстановитьЗначение(Точка, Серия, Значение); или ИсточникДанных (таблица значений). УстановитьТочку()/УстановитьСерию() — поиск по ключу или создание. ПолучитьЗначение(Точка, Серия). Свойства: ОбластьПостроения, ОбластьЗаголовка, ОбластьЛегенды, ОтображатьЗаголовок, ОтображатьЛегенду. События: Выбор, ОбработкаРасшифровки, ПриАктивизация (выделение — ПолучитьВыделенныеЭлементы()). ПриРедактированииЗначения — параметр СостояниеРедактирования (Завершено, Отменено, НеЗавершено).

Диаграмма Ганта: Серии, Точки (иерархия), ПолучитьЗначение(Точка, Серия) → ЗначениеДиаграммыГанта с коллекцией интервалов (Интервал.Начало, Интервал.Конец). УстановитьТочку()/УстановитьСерию() по ключу. Интервалы фона: ДиаграммаГанта.ИнтервалыФона. Связи между интервалами: ИнтервалДиаграммыГанта.Добавить(другой интервал). ЗначениеДиаграммыГанта.Редактирование = Истина — интерактивное редактирование. Свойства расширения поля: РежимВыделенияЗначений, ТекущееЗначение, ВыделенныеЗначения; РежимВыделенияИнтервалов, ТекущийИнтервал, ВыделенныеИнтервалы. События: ПриАктивизацииЗначения, ПриАктивизацииИнтервала, ПриОкончанииРедактированияИнтервала. Дополнительные данные — колонки реквизита формы (ИзменитьРеквизиты), Значение.Данные["ИмяКолонки"].

Дендрограмма: Элементы, Связи (ПервыйЭлемент, ВторойЭлемент, Расстояние).

## 7.4. Параметры формы

Параметры (закладка Параметры): перечень и типы; свойство **Ключевой параметр** — участие в ключе уникальности формы (поиск уже открытой формы). При открытии формы параметры передаются в структуре; не переданный параметр добавляется платформой со значением Неопределено. Анализ — в ПриСозданииНаСервере() (коллекция Параметры).

ВНИМАНИЕ! После вызова ПриСозданииНаСервере все неключевые параметры формы удаляются из коллекции Параметры. Неключевые параметры, нужные для дальнейшей работы, следует сохранять в данных формы.

Стандартные параметры формы — перечень в справке по расширению формы (выбор из формы выбора, открытие форм объектов, стандартные команды и т.д.).

## 7.5. Команды формы

Группы команд: созданные разработчиком (обработчик в модуле формы); стандартные (расширение основного реквизита и реквизитов-списков при наличии связанного элемента формы); глобальные (глобальный командный интерфейс, параметризованные — при наличии источников параметров на форме).

Свойство команды **Действие** — обработчик (только клиентские процедуры/функции без параметров). **Изменяет сохраняемые данные** — при выполнении: блокировка основного реквизита, установка Модифицированность; в режиме «Только просмотр» формы элементы команды тоже только просмотр. Для параметризованной глобальной команды с параметром от реквизита с «Сохраняемые данные» при новом объекте — попытка записи с вопросом пользователю.

ПРИМЕЧАНИЕ. При автозаполнении командных панелей/контекстных меню стандартные команды не добавляются, если в элементе уже есть вручную добавленные кнопки с теми же командами (кроме команд из фрагмента глобального командного интерфейса).

Стандартные команды диалога: Да, Нет, ОК, Отмена и т.д. — закрытие формы, возврат КодВозвратаДиалога (модальная) или только закрытие (немодальная). Имя обработчика — по свойству команды Имя. Команда ПоказатьВСписке — метод ВыполнитьПереход(Объект) у формы.

## 7.6. Модуль формы

Директивы компиляции: **НаКлиенте** — выполнение на клиенте, переменная на время жизни клиентской части; **НаСервере** — на сервере, переменная только на время серверного вызова; **НаСервереБезКонтекста** — на сервере вне контекста формы (нет доступа к контексту формы, нет передачи данных формы); **НаКлиентеНаСервереБезКонтекста**. Отсутствие директивы — по умолчанию НаСервере. Не допускается несколько директив перед одним методом/переменной; одноименные методы/переменные, отличающиеся только директивой. Обработчики команд формы — только в клиентских методах.

ВНИМАНИЕ! В серверной переменной формы невозможно сохранить данные между двумя вызовами серверной стороны формы.

При контекстном вызове клиент→сервер данные формы передаются на сервер, выполняется вызов, данные возвращаются на клиент (существенное время). Внеконтекстный вызов быстрее. Не рекомендуется сохранять ссылки на реквизиты формы в переменных на время ИзменитьРеквизиты() (сервер) или любого контекстного серверного вызова (клиент). Рекомендуется использовать директивы препроцессора внутри процедур и функций.

ПРИМЕЧАНИЕ. Обработка инструкций препроцессора выполняется до обработки директив компиляции.

## 7.7. Элементы формы

Типы элементов: поле формы, таблица формы, кнопка, группа формы, декорация, дополнение элемента формы. Элементы образуют иерархию. Поле и таблица всегда связаны с данными формы; при отсутствии связи или недоступности реквизита на клиенте поле не видно и удаляется при создании формы.

### 7.7.2. Общие свойства элементов формы

**Данные** — реквизит формы для поля; **Команда** — для кнопки. **Только просмотр** — у элемента или у группы (распространяется на подчиненные). **Видимость** и **Пользовательская видимость** — результирующая видимость = И AND по обоим для пользователя. **Пропускать при вводе** — пропуск при обходе Enter (Да/Нет/Авто). **Положение заголовка** — вывод заголовка элемента. **Сочетание клавиш** — фокус на элемент (не для поля надписи и декорации).

Связь с реквизитами:
- **Обычная**: ПутьКДанным = имя реквизита.
- **Текущие данные таблицы**: путь вида Объект.Товары.Цена — данные текущей строки таблицы.
- **Определенная строка таблицы**: путь вида Контрагенты[3].Долг — по индексу (или иерархия индексов). Не поддерживается для динамических списков.
- **Через ссылку**: Объект.Товары.Товар.Артикул — реквизит по ссылке (только просмотр). Для составного типа (СправочникСсылка и т.д.) получение реквизитов через ссылку недоступно.
- **Итоги коллекций**: Объект.Товары.ИтогСумма, количество строк и т.д. — только просмотр.

ПРИМЕЧАНИЕ. Данная связь не поддерживается для динамических списков.
ПРИМЕЧАНИЕ. Для реквизитов составного типа недоступно получение реквизитов через ссылку.

Свойство **Вид** — способ отображения (поле ввода, надпись, флажок и т.д.). **Расширенная подсказка** — у группы, кнопки, декорации. **Картинки** — из конфигуратора (в т.ч. из файла) или программно (пустая или из библиотеки картинок).

### 7.7.3. Форма

Элемент типа Форма — корень иерархии, единственный. События: ПриСозданииНаСервере (отказ — форма не создается), ПриОткрытии. **Режим открытия окна**: Независимый, Блокировать окно владельца, Блокировать весь интерфейс. **Вертикальная прокрутка**: Использовать, ИспользоватьБезРастягивания, ИспользоватьПриНеобходимости, Авто. **Сохранять настройки окна** — размер, положение, ширина колонок, разделители. **Вариант масштаба** (Такси): Авто, Обычный, Компактный. **Масштаб** (10–400%). **ТолькоПросмотр** (только программно) — недоступность стандартных команд записи, проведения и т.д. **Сохранение данных формы в настройках** — колонка Сохранение у реквизитов; сохраняемые реквизиты в хранилище (свойство формы Хранилище настроек или хранилище данных форм).

ПРИМЕЧАНИЕ. Недоступно указание Сохранение для табличных частей объектов.

### 7.7.4. Поле

Виды поля: поле ввода, надпись, флажок, картинка, переключатель, текстовый документ, табличный документ, календарь, индикатор, полоса регулирования, диаграмма, диаграмма Ганта, дендрограмма, графическая схема, географическая схема, HTML-документ, форматированный документ, планировщик, PDF-документ. В таблице — только поле ввода, картинки, надписи, флажка.

**Обновление отображаемых данных**: при изменении данных в обработчике ожидания/внешнем событии поле может обновиться из реквизита после контекстного вызова. **ОбновлениеТекстаРедактирования** (поле ввода): ПриИзмененииЗначения — обновлять только при отличном значении реквизита; Никогда — не обновлять. **ОбновитьОтображениеДанных**(СписокЭлементов) — обновить только указанные элементы (ГруппаФормы, ТаблицаФормы, ПолеФормы); пустой массив — не обновлять ни один. Имеет смысл в обработчике внешнего события, ожидания, оповещения.

Представление в поле: по реквизиту; при режиме выбора из списка — колонка Представление списка выбора (в таблице при отображении ячейки — представление реквизита; при редактировании — список выбора). **Отображение предупреждения при редактировании** и **Предупреждение при редактировании** — предупреждение при начале редактирования. **АвтоОтметкаНезаполненного** — автоматическая актуализация ОтметкаНезаполненного. **ПодсказкаАвтоЗаполнения** (веб/мобильный) — АдресЭлектроннойПочты, Имя, НомерТелефона и т.д. Контекстное меню: обработчик ОбработкаФормированияКоманд; параметр Команды — массив ОписаниеКомандыПоляВвода; Параметры.Источник (ОбластьВвода, МножественноеЗначение), Параметры.Идентификатор для множественного значения.

### 7.7.6. Таблица

Отображение: динамический список, табличная часть, список значений и т.д. **Положение командной панели**, **Состав команд**. Для динамического списка: **Начальное отображение дерева** (Раскрывать все уровни — при создании формы на сервере только первый уровень, остальное при открытии). **Обновление при изменении данных**: Авто / Не обновлять (явно Обновить()). Параметр формы **МножественныйВыбор** → свойство таблицы. **ВариантУправленияВысотойЭлемента**: В строках формы (Высота таблицы), В строках таблицы (ВысотаВСтрокахТаблицы), По содержимому. **ФиксацияВТаблице** (у поля в таблице или группы колонок): Нет, Лево, Право. **Поиск при вводе символов**: Авто, Использовать, Не использовать. **НачальноеОтображениеСписка**: Начало, Конец, Авто (для списков с датой/периодом — в конец). Картинка строк: ПутьКДаннымКартинкиСтроки, КартинкаСтрок (индекс в запросе).

ПРИМЕЧАНИЕ. Если таблица отображает динамический список и Начальное отображение дерева = Раскрывать все уровни, при создании формы на сервере получается только первый уровень строк, остальные — при открытии формы.

### 7.7.7. Кнопка

Свойство **Команда** — привязка к команде. **Положение в командной панели**: В дополнительном подменю, В командной панели, В командной панели и дополнительном подменю, Авто. **Уникальность команды** — для размещения двух кнопок одной стандартной команды в разных местах панели с автозаполнением. **Вид**: Обычная кнопка, Кнопка командной панели, Гиперссылка, Гиперссылка командной панели. **Отображение фигуры**: Авто, Всегда, При активности, Нет. **Фигура кнопки**. Для команды «Создать по параметру» — свойство кнопки **Параметр** (тип документа).

### 7.7.8. Группа

Виды: Обычная группа, Командная панель, Страницы (закладки), Группа колонок. **Источник команд** (у командной панели) — форма, таблица, поле табличного документа/графической схемы. **Состав команд**. Обычная группа: оформление (Нет, Слабое/Обычное/Сильное выделение); **Поведение**: Обычное, Свертываемая (свойство Свернута), Всплывающая (заголовок обязателен). Методы расширения группы: Показать(), Скрыть(), Скрыта(). **Только просмотр** группы — на все подчиненные. **Разрешить изменение состава** — настройка формы пользователем. **Данные заголовка** — отображение в заголовке; при заданном Заголовок — в скобках после заголовка.

### 7.7.9. Дополнение элемента формы

Для таблицы и поля PDF: Отображение строки поиска, Состояние просмотра, Управление поиском; для поля PDF — свойства PDF. **Источник дополнения элемента формы** — связь дополнения с элементом.

## 7.9. Открытие и модификация формы

Открытие: ОткрытьФорму(ИмяФормы, Параметры, Владелец, …) или ПолучитьФорму(…), затем Форма.Открыть(). Модальное открытие — для доступа к реквизитам после закрытия использовать ПолучитьФорму() и Открыть().

**ОбработкаПолученияФормы** (модуль менеджера объекта) — переопределение открываемой формы при открытии по умолчанию (по имени формы). Не вызывается для общих форм, форм констант, хранилищ настроек, внешних отчетов и обработок.

ПРИМЕЧАНИЕ. Обработчик ОбработкаПолученияФормы не вызывается при попытке открытия общих форм, форм констант, форм хранилищ настроек, внешних отчетов и внешних обработок.

Модификация свойств элементов: коллекция **Элементы** (без иерархии), **Родитель**, **ПодчиненныеЭлементы**. Установка ТолькоПросмотр, Доступность, Видимость у родителя влияет на подчиненные (сложение по И). Изменение свойства **Вид** у элемента с расширением создает новое расширение (свойства по умолчанию).

ПРИМЕЧАНИЕ. Свойства элементов формы, относящиеся к отображаемым данным (ПутьКДанным, ПутьКДаннымЗаголовка и т.д.), можно изменять только для новых элементов или после смены вида элемента.

ПРИМЕЧАНИЕ. При программном изменении состава элементов формы возможны вызовы обработчиков событий (например, ПриАктивизацииСтроки).

Модификация формы (только на сервере): **ИзменитьРеквизиты**(ДобавляемыеРеквизиты, УдаляемыеРеквизиты) — пакетное изменение реквизитов; **ПолучитьРеквизиты**(ИмяРеквизита) — получить массив для изменения; для программно добавленных реквизитов обращение только ЭтотОбъект.ИмяРеквизита. **Команды.Добавить**(Имя) — новая команда; Свойства: Действие (имя обработчика), Заголовок; обработчик — &НаКлиенте. **Элементы.Добавить**(Имя, ТипЭлемента) — новый элемент; для поля: ПутьКДанным, Вид; для кнопки: ИмяКоманды. РеквизитФормы(Имя, Тип, Родитель, Заголовок, ИспользоватьВсегда).

ПРИМЕЧАНИЕ. Один обработчик может обслуживать несколько программно добавляемых команд.

Обработчик ожидания: при изменении данных формы в нем вызывать ОбновитьОтображениеДанных(СписокЭлементов), чтобы не терять ввод в редактируемом поле. Не рекомендуется контекстные серверные вызовы, меняющие данные формы, при возможном конфликте с вводом пользователя.

Перетаскивание: события НачалоПеретаскивания (источник), ПроверкаПеретаскивания (приемник), Перетаскивание (приемник), ОкончаниеПеретаскивания (источник). ПараметрыПеретаскивания: Значение, Действие, ДопустимыеДействия. Элементы-источники/приемники: таблица, поле табличного документа, календарь, поле картинки, декорация-картинка, поле планировщика. Свойства: Разрешить начало перетаскивания, Разрешить перетаскивание. Способ перетаскивания файлов: Как файл (Файл; веб-клиент не поддерживает), Как ссылка на файл (СсылкаНаФайл). СсылкаНаФайл не сериализуется, не хранится в реквизитах/ИБ, не передается клиент–сервер, создается только платформой при перетаскивании.

Модальность: модальные окна (ОткрытьФормуМодально(), Вопрос() и т.д.) — не поддерживаются в веб-клиенте. Блокирующие окна — ОткрытьФорму(…, РежимОткрытияОкнаФормы.БлокироватьОкноВладельца) с обратным вызовом при закрытии. При закрытии приложения в ПередЗакрытием, ПриЗакрытии параметр ЗавершениеРаботы = Истина; в этих обработчиках при ЗавершениеРаботы запрещены серверные вызовы и открытие форм.
