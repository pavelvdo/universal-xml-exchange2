# Глава 30. Расширение конфигурации

> **Agent ref:** Расширения конфигурации — доработка без изменения основной конфигурации. Ключевое: заимствование и собственные объекты; свойства контролируемое / проверяемое / модифицируемое; аннотации расширения методов — &Перед, &После, &Вместо (и ПродолжитьВызов), &ИзменениеИКонтроль (#Вставка/#Удаление); формы в расширениях (заимствование элементов, объединение результирующей формы); права и основные роли; API — РасширенияКонфигурации, Записать(), ПроверитьВозможностьПрименения(), ЕстьИзмененияРасширениямиКонфигурации().
> **Разделы:** 30.1 Общая информация, 30.2 Управление, 30.3 Заимствование и собственные объекты, 30.4 Объекты расширения (модули, аннотации, формы, права), 30.5–30.7 РИБ, профили, подключение, 30.8 Встроенный язык.

## 30.1. Общая информация

### 30.1.1. Термины и определения

- **Расширяемая конфигурация** — основная конфигурация ИБ, к которой подключается расширение.
- **Расширение конфигурации** — набор объектов метаданных (заимствованных и собственных), подключаемых к расширяемой конфигурации.
- **Собственный объект** — самодостаточный объект метаданных (в основной конфигурации или в расширении).
- **Заимствованный объект** — объект расширяемой конфигурации, добавленный в расширение.
- **Расширяемый объект** — собственный объект, для которого в заимствованном объекте изменены параметры (свойства, формы и т.д.).
- **Расширяющий объект** — заимствованный объект с изменениями относительно расширяемого. Только контролируемые свойства без изменений не делают объект расширяющим.
- **Результирующий объект** — собственный объект + объединение всех расширяющих объектов. В работающей конфигурации все объекты — результирующие.
- **Расширяющее свойство** — свойство заимствованного объекта, изменяющее одноимённое свойство расширяемого.
- **Контролируемое свойство** — при несовпадении значения с расширяемой конфигурацией расширение не подключается.
- **Проверяемое свойство** — при несовпадении выдаётся предупреждение, расширение подключается.
- **Модифицируемое свойство** — значение в результирующем объекте берётся из расширения (при нескольких расширениях — из последнего в списке).
- **Расширение данных** — расширение структуры хранимых данных (реквизиты и т.д.) с помощью заимствованных и собственных объектов расширения.

Свойство заимствованного объекта не может быть одновременно контролируемым и модифицируемым.

### 30.1.2. Краткое описание механизма

Механизм расширения позволяет дорабатывать расширяемую конфигурацию без её изменения и без снятия с поддержки. Объекты расширения могут менять поведение объектов основной конфигурации, дополнять структуру хранения (реквизиты) и создавать собственные объекты с собственными данными. Сопоставление объектов выполняется по имени (и при включённой опции — по внутренним идентификаторам); изменение имени в основной конфигурации или в расширении может привести к невозможности подключения расширения.

### 30.1.3. Назначение расширения

- **Исправление** — исправление ошибок, допускаются «опасные» возможности (в т.ч. &Вместо). Привязка к версии; при обновлении нужен анализ применимости. Несколько таких расширений допустимы при отсутствии конфликтов (например, не расширять один метод по-разному).
- **Адаптация** — адаптация под клиента; «опасные» возможности не рекомендуются; минимальное количество таких расширений.
- **Дополнение** — новые возможности с минимальной привязкой к версии (например, новый набор отчётов); не должны учитывать другие расширения; количество произвольное.

При разделённой ИБ у расширения можно указать область действия: вся ИБ или область данных.

### 30.1.4. Особенности и ограничения

Подключение расширения (из конфигуратора или из встроенного языка): в небезопасном режиме или при разрешении профиля безопасности на доступ к расширению. Сохранённое в файл расширение подключается через стандартную функцию «Управление расширениями конфигурации» или через программный интерфейс.

**Не поддерживается создание в расширении:** общие реквизиты-разделители, хранилища настроек, языки.

**Не поддерживается расширение:** добавление реквизитов/ТЧ для плана видов характеристик, плана счетов, плана видов расчетов; изменение структуры регистров (только состав регистраторов). Документы расширения могут быть регистраторами только для собственных регистров расширения. Для собственных регистров накопления не поддерживаются агрегаты.

**Контролируемые свойства (особо):** состав плана обмена; предопределённые элементы справочников, ПВХ, плана счетов, плана видов расчётов.

**Расширение данных:** один экземпляр расширения на ИБ/область; после подключения расширения, расширяющего данные, не поддерживается фоновая реструктуризация; тестирование и исправление — только структуры расширяемой конфигурации.

## 30.2. Управление расширениями

### 30.2.1. В режиме Конфигуратора

Диалог **Конфигурация → Расширения конфигурации**: создание, удаление, открытие конфигурации расширения, сохранение в файл. Команды дублируются в командной панели, контекстном меню и меню **Действия → Конфигурация**. Специализированные команды: **Проверка возможности применения** (текущее расширение) и **Проверка возможности применения всех расширений**. При проверке: совпадение контролируемых свойств; наличие всех заимствованных объектов; для &ИзменениеИКонтроль — проверка соответствия текста метода (неблокирующая: при несовпадении метод не расширяется, расширение подключается).

**Создание расширения:** указываются имя, синоним и **префикс имён** (добавляется к новым отчётам, обработкам, подсистемам, обработчикам в расширяющих формах). Автоматически копируются и помечаются как контролируемые: основной режим запуска, назначение использования, основной язык, режим совместимости интерфейса и режим совместимости. Автоматически создаётся роль \<Префикс\>ОсновнаяРоль с доступом к собственным объектам расширения и устанавливается в свойство **ОсновныеРоли**.

**Сохранение в файл:** **Конфигурация → Сохранить конфигурацию в файл** в диалоге расширений → файл .cfe для подключения к прикладному решению (через стандартную функцию или API).

### 30.2.2. В режиме 1С:Предприятие

Подключение расширения — через стандартную функцию «Управление расширениями конфигурации» или программный интерфейс (при наличии прав и режиме/профиле, разрешающем доступ к расширению).

### 30.2.3. Удаление расширений

Интерактивно: для расширения без расширения данных — подтверждение и удаление; для расширения с расширением данных — сначала снять флажок **Активно**, затем удалить. Из встроенного языка: **РасширениеКонфигурации.Удалить()** без дополнительного контроля; проверки — **Активно**, **ИзменяетСтруктуруДанных()**.

## 30.3. Работа с расширением

### 30.3.1. Создание заимствованного объекта

Команда контекстного меню объекта: **Добавить в расширение**. При этом заимствуются все родительские объекты (если ещё не заимствованы); подчинённые — нет. При заимствовании формы заимствуется только форма (или объект + форма, если объект не был в расширении); при заимствовании реквизита формы с типом из другого объекта заимствуются реквизиты этого объекта и объекты по типам. Параметрическая команда не тянет за собой тип параметра — при необходимости объект типа заимствовать отдельно.

### 30.3.2. Работа с заимствованным объектом

У заимствованного объекта в панели свойств отображаются только свойства, которые могут быть контролируемыми, проверяемыми или модифицируемыми. Модифицируемое свойство в результирующем объекте получает значение из расширения (при нескольких расширениях — из последнего). Неуказанное модифицируемое свойство ведёт себя как в обычной конфигурации (например, основная форма по умолчанию).

**Соответствие по внутренним идентификаторам:** свойство расширения **Поддерживать соответствие объектам расширяемой конфигурации по внутренним идентификаторам** (для новых расширений включается автоматически) и свойство заимствованного объекта **Объект расширяемой конфигурации** позволяют при переименовании в основной конфигурации выполнить проверку применимости и применить одно из решений: переименовать с сохранением соответствия; сохранить имя, изменив соответствие; выбрать соответствие; отключить проверку; очистить соответствие; установить значение из объекта; удалить объект.

### 30.3.3. Собственный объект

Создание и редактирование — как в основной конфигурации. Уникальность имён контролируется при подключении расширения (среди основной конфигурации и всех расширений) и при разработке (в рамках своего расширения и основной конфигурации).

### 30.3.4. Блокировка и совместная разработка

Одновременно нельзя выполнять взаимоисключающие действия: например, при открытом расширении в конфигураторе его нельзя удалить из списка в режиме 1С:Предприятие. Расширение может храниться в отдельном хранилище конфигурации (не в одном с расширяемой конфигурацией и не несколько расширений в одном хранилище).

## 30.4. Объекты расширения

### 30.4.1. Свойства расширения

Назначение, префикс имён, соответствие по внутренним идентификаторам — см. выше. Описание: краткая/подробная информация, авторские права, адреса сведений о поставщике и конфигурации. Возможна замена картинок (основной раздел, логотип, заставка) и расширение **Основной стиль**. **Основные роли** — только собственные роли расширения, только доступ к собственным объектам расширения. **Режим совместимости расширения конфигурации** — аналогично основной конфигурации, ограничивает возможности при разработке расширения.

### 30.4.2. Программные модули

#### 30.4.2.1. Общая информация

Расширяются практически все программные модули управляемого приложения; глобальные серверные модули расширять нельзя. Расширение привилегированного общего модуля выполняется в непривилегированном режиме (если иное не разрешено профилем безопасности). Собственные общие модули в расширении — без привилегий и глобального серверного. Расширяющий модуль находится в одном пространстве имён с расширяемым: доступны переменные и методы расширяемого контекста; экспортные методы и переменные расширения попадают в публичный контекст расширяемого модуля. Тело модуля расширения и область переменных расширять нельзя, но можно задать свои переменные и своё тело модуля; тело модуля расширения выполняется после модуля расширяемой конфигурации.

#### 30.4.2.2. Способы расширения (аннотации)

Перехват методов реализуется аннотациями. Общий вид: `&Аннотация("ИмяМетода") Процедура Префикс_Имя() … КонецПроцедуры`. Имя расширяющего метода должно быть уникально; параметры должны совпадать с расширяемым методом (включая **Знач**); значения по умолчанию берутся из расширяемого метода. Изменение параметра в расширяющем методе видно всем последующим в цепочке.

- **&Перед("ИмяМетода")** — сначала выполняется метод расширения, затем расширяемый метод.
- **&После("ИмяМетода")** — сначала расширяемый метод, затем метод расширения.
- **Обрамление** — методы с &Перед и &После: сначала &Перед, затем расширяемый метод, затем &После.
- **&Вместо("ИмяМетода")** — полная замена вызова. Оригинал вызывается через **ПродолжитьВызов**(фактические параметры). Для **функций** доступна только эта аннотация. Рекомендуется использовать только при невозможности применить Перед/После.
- **&ИзменениеИКонтроль** — точечная модификация текста расширяемого метода. При заимствовании метод копируется в расширение; изменения помечаются директивами **#Вставка … #КонецВставки** и **#Удаление … #КонецУдаления**. При применении проверяется соответствие текста расширяемого метода (пробелы/пустые строки игнорируются); при несовпадении в конфигураторе можно запустить трёхстороннее слияние для восстановления. В рабочей конфигурации метод с &ИзменениеИКонтроль полностью подменяет расширяемый метод; другие аннотации работают уже с этим результатом. **Каждый метод может быть расширен с помощью &ИзменениеИКонтроль только один раз** (в одном расширении). Синтаксический контроль в конфигураторе вне областей #Вставка игнорирует ошибки «переменная не определена».

Порядок при нескольких расширениях: первым выполняется расширение, зарегистрированное **последним** (ближе к расширяемой конфигурации). Цепочка: &Перед из последнего расширения → … → &Перед из первого → расширяемый метод → &После из первого → … → &После из последнего. Не рекомендуется расширять один и тот же метод с &Вместо в нескольких расширениях без вызова ПродолжитьВызов(); с безусловным ПродолжитьВызов() несколько таких расширений допустимы.

#### 30.4.2.3. Редактирование программного модуля

**Добавить в расширение** — курсор в методе; при отсутствии объекта в расширении объект добавляется. Для не-модуля формы предлагается выбор аннотации (Перед, После, Вместо, ИзменениеИКонтроль). Для функций — только Вместо. Для модуля формы — уточнение: расширение как обработчик события/команды или как обычный метод. Ограничения: для процедуры недоступны Вместо/ИзменениеИКонтроль при уже существующих Перед/После; при существующем Вместо другая аннотация недоступна.

### 30.4.3. Форма

При заимствовании формы заимствуются только **элементы** формы; реквизиты, команды и параметры отображаются, но не заимствуются явно до добавления в расширение. Заимствованная форма запоминается как «сохранённая». Изменения свойств элементов в основной конфигурации попадают в расширение после **Обновить расширение формы**. Реквизиты/команды/параметры, не добавленные в расширение, наследуют изменения из основной конфигурации; добавленные в расширение участвуют в трёхстороннем сравнении при конфликтах. Результирующая форма: объединение расширяемой и расширяющих форм по компонентам (по имени); при конфликте изменений берётся компонента из расширения. Условное оформление и командный интерфейс объединяются по своим правилам. Модуль формы расширяется с теми же аннотациями; обработчики в формах создаются через панель свойств, без явных аннотаций в коде.

### 30.4.4. Права доступа

При создании расширения создаётся роль с доступом к собственным объектам и подставляется в **ОсновныеРоли**. При подключении можно установить **Использовать основные роли для всех пользователей**: тогда минимальный набор прав задаётся основными ролями расширения для всех пользователей. Иначе основные роли действуют только при отсутствии пользователей в конфигурации. Заимствованная роль позволяет изменять права и RLS для объектов расширяемой конфигурации (не корневые административные права). Собственная роль — только для собственных и заимствованных объектов расширения. Расширение прав доступа в заимствованной роли в клиент-серверном варианте регулируется профилем безопасности (флажок расширения прав, список ролей, ограничивающих расширение прав).

### 30.4.5. Кратко по объектам метаданных

**Отчеты и обработки:** заимствование с возможностью добавления реквизитов, ТЧ; у отчёта можно переопределить основную схему компоновки данных. **XDTO:** заимствование пакета или части; URI и состав при нескольких расширениях — из последнего. **Web-сервис:** дополнение пакетов XDTO; публикация собственных — через настройки публикации или default.vrd. **HTTP-сервис:** контроль уникальности корневого URL собственных сервисов. **Предопределённые элементы:** контроль/проверка состава; заимствование с переносом имени, кода, наименования, иерархии, типа (ПВХ), вида и забалансовости (план счетов). **Табличный документ:** способ расширения — объединять или замещать; команды отображения изменений, сохранённой таблицы, результата, обновления расширения. **Планы обмена:** контролируемость состава; заимствование с переносом состава и настроек; собственный план обмена не участвует в РИБ. **Справочники, документы, перечисления:** добавление реквизитов/ТЧ (где разрешено); документ — регистратор только для регистров своей «части» (расширение/основная). **Параметры сеанса, функциональные опции, критерии отбора, подписки:** заимствование и собственные; у подписок — обобщённые типы для собственных. **Общие реквизиты:** только заимствованные; у разделителя — управление составом и контроль вхождения объектов. **Макеты:** заимствованные можно модифицировать; табличный документ — по правилам объединения/замещения. **Нумераторы, регламентные задания, внешние источники, БП и задачи:** заимствование и собственные с ограничениями (нумераторы — см. документацию; регламентные задания в РИБ — по области данных/ИБ). **Определяемые типы, сервисы интеграции:** заимствование и собственные; у сервисов интеграции — добавление каналов и правка модуля.

**Расширяемые свойства объектов:** длина кода/наименования/номера, тип кода/номера, локализуемые реквизиты (заголовок, формат, подсказка, представления и т.д.) — по правилам результирующего значения (максимум, последнее расширение и т.д.). Состав типов реквизитов при расширении формируется без потерь для обоих вариантов; список нерасширяемых типов и объектов см. в документации платформы.

## 30.5. Расширение и распределённая информационная база

Расширения могут передаваться между узлами (план обмена с флажком **Включать расширения конфигурации**). У расширения — свойство **Используется в распределённой информационной базе**. В начальный образ переносятся расширения с этим флагом; при обмене действуют те же ограничения. В периферийном узле запрещено редактировать и удалять расширения, добавленные в главном. **РасширениеКонфигурации.ГлавныйУзел** — Неопределено (добавлено локально) или ссылка на главный узел (получено по обмену). Порядок применения при старте: сначала расширения с главного узла, затем добавленные в текущем узле.

## 30.6. Профили безопасности

Работоспособность методов расширения может ограничиваться профилем безопасности. При подключении расширения задаётся **Безопасный режим** или **Профиль безопасности** (клиент-сервер). В профиле доступ к расширению настраивается в элементе «Внешние модули». Для небезопасного расширения при запрете в профиле возможно добавление в исключения (контрольная сумма расширения). Для изменения настроек безопасного режима при подключении нужно право **Администрирование** в дополнение к **АдминистрированиеРасширенийКонфигурации**.

---

## 30.7. Подключение расширений и время работы

Подключение к сеансу — до **УстановкаПараметровСеанса**, до конца сеанса. Обновление расширения в ИБ не обновляет уже запущенные сеансы. Порядок подключения: по назначению (Исправление → Адаптация → Дополнение), внутри назначения — сначала область ИБ, затем область данных. При подключении проверяются контролируемые свойства; при несовпадении расширение не подключается. Проверка применимости из кода: **РасширениеКонфигурации.ПроверитьВозможностьПрименения()**, **РасширенияКонфигурации.ПроверитьВозможностьПримененияВсех()**. При расширении данных создание таблиц выполняется при **РасширениеКонфигурации.Записать()**; при удалении расширения — удаление таблиц. Поведение при отключённых расширениях, расширяющих данные (запись, чтение, типы реквизитов) — см. документацию платформы.

## 30.8. Работа с расширениями из встроенного языка

Требуется право **АдминистрированиеРасширенийКонфигурации**; для **Записать()** — небезопасный режим. Менеджер — **РасширенияКонфигурации** (глобальный контекст).

- **РасширенияКонфигурации.Создать()** — создать расширение без записи в ИБ; запись — **Записать()** (при необходимости после проверок).
- **РасширенияКонфигурации.Создать(ДвоичныеДанные)** — создать из файла и записать в ИБ.
- **РасширениеКонфигурации.Записать()** — записать/обновить расширение в ИБ.
- **РасширениеКонфигурации.Удалить()** — удалить.
- **РасширениеКонфигурации.Активно**, **ИзменяетСтруктуруДанных()** — состояние и признак расширения данных.
- **РасширениеКонфигурации.ПроверитьВозможностьПрименения()** — проверка применимости одного расширения.
- **РасширенияКонфигурации.ПроверитьВозможностьПримененияВсех()** — всех расширений.
- **РасширенияКонфигурацииИзменены()** — признак изменения расширений в сеансе (аналог КонфигурацияИзменена() для расширений).
- **\<Объект метаданных\>.ЕстьИзмененияРасширениямиКонфигурации()** — Истина, если объект модифицирован расширениями (свойство, новый дочерний объект, изменение дочернего).
- **\<Объект метаданных\>.РасширениеКонфигурации()** — расширение конфигурации или расширяемая конфигурация, которым принадлежит объект.
