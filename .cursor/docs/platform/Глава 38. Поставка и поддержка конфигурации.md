# Глава 38. Поставка и поддержка конфигурации

> **Agent ref:** поставка (.cf / .cfu), комплект поставки (1cv8.efd, setup), поддержка конфигурации, правила изменения объектов, варианты поставки, шаблоны, базовые каталоги.
> **Разделы:** 38.1–38.5.
> ---

## 38.1. Общая информация

- **Поставка:** полная — файл .cf; обновления — файл .cfu.
- **Поддержка конфигурации:** обновление средствами конфигуратора файлами поставки; цель — сохранение логической целостности.
- **Комплект поставки:** дистрибутив с программой установки (Windows: setup.exe, Linux: setup, macOS: setup.app) и файлами поставки в архиве 1cv8.efd.

## 38.2. Правила изменения объектов (настройка поставки)

| Режим | Смысл |
|-------|--------|
| Изменения разрешены | Любое изменение в пользовательской конфигурации (по умолчанию). |
| Изменения не рекомендуются | Изменения не рекомендуются поставщиком. |
| Изменения запрещены | Полностью запрещены; объект нельзя снять с поддержки. |

**Исходный текст модулей:** при снятом флажке «Включать исходный текст» в поставку попадает только скомпилированный текст; функциональность сохраняется, исходный текст недоступен. **Исключения** (исходники исключать нельзя): модуль управляемого приложения, модули форм, модули команд, модули с директивами препроцессора, клиентские общие модули управляемого режима.

## 38.3. Создание файлов поставки и обновлений

В свойствах конфигурации (Разработка): **Поставщик**, **Версия**. Рекомендуемый формат версии: `<версия>.<подверсия>.<релиз>.<сборка>`. Для файла обновления можно включить прежние обновления («Добавить из предыдущих версий»). Текстовые файлы комплекта — кодировка UTF-8.

## 38.4. Комплект поставки (описание)

Описание комплекта — дерево: **шаблоны конфигураций**, **варианты поставки**, **базовые каталоги**, **языки**.

- **Шаблоны:** текущая конфигурация, текущая ИБ, отдельный файл, маска каталога. Свойства: Поставщик, Имя, Версия, размещение манифеста.
- **Варианты поставки:** комбинации файлов (полная поставка или только обновления).
- **Базовые каталоги:** переменные среды или каталоги для переносимых путей.
- **Языки:** локализуемые наименования; каталог ресурсов должен соответствовать каталогам платформы.

## 38.5. Поддержка и обновление

Новая версия типовой конфигурации поставляется как файл поставки и/или файл обновления. Обновление конфигурации БД выполняется в конфигураторе. При включённой поддержке в ИБ хранится конфигурация поставщика.
