# Verification — «Нарушение прав доступа» при Записать профиля (WS)

## Прогресс приёмки

Профиль по префиксам **найден** (топология ИБ исправлена). Падение на переводе статуса «Новое» → «Выполняется» в `НачатьВыгрузкуGetData`, строка 551.

## Цепочка ошибки

1. `GetData` → `НачатьВыгрузкуGetData` → `ОбъектПрофиля.Записать()`.
2. Подписка/обработчик `ПередЗаписью` типовой ЗУП: `УдалениеПомеченныхОбъектовСлужебный` читает параметр сеанса `ВыполняетсяУдалениеОбъектов`.
3. WS выполняется под пользователем HTTP Basic (логин из профиля приёмника) **без прав** на этот параметр сеанса → «Нарушение прав доступа!».

## Это не ошибка поиска префиксов

Роль `рг_ИспользованиеОбменаXML` (`setForNewObjects=true`) даёт права на объекты расширения, но **не отключает** типовые подписки БСП при записи справочника.

## Обход для теста (сейчас)

На **источнике** пользователь WS (логин в профиле приёмника):

- роль **`рг_ИспользованиеОбменаXML`** + типовая роль с правами записи справочников/сеанса (или временно **Полные права** / администратор для smoke-теста).

Повторить «Выполнить обмен».

## Исправление в коде (рекомендация для apply/extend)

В `НачатьВыгрузкуGetData` и `УстановитьСтатусПрофиляИсточника`: запись статуса профиля из WS-контекста в **`УстановитьПривилегированныйРежим(Истина)`** (с `Попытка`/`Исключение` и сбросом режима). Опционально `ОбменДанными.Загрузка = Истина` для пропуска проверок правил при смене только статуса.

## Следующий шаг

Smoke с расширенной ролью WS-пользователя **или** `/opsx:apply universal-xml-exchange2` — fix привилегированной записи статуса.
