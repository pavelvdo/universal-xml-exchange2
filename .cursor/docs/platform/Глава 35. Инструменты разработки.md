# Глава 35. Инструменты разработки

> **Agent ref:** Редактор форм (реквизиты, команды, элементы, ролевая настройка). Редактор модулей (группировка, форматирование, переход к определению, контекстная подсказка). Конструкторы: запроса, движений регистров, печати, ввода на основании. Проверка конфигурации: `/CheckConfig`, параметры (`-IncorrectReferences`, среды синтаксиса, `-UnreferenceProcedures`, `-HandlersExistence`, `-ExtendedModulesCheck` и др.).
> **Разделы:** 35.1 Редактор формы · 35.2 Редактор текстов · 35.9 Конструктор запроса · 35.10–35.12 Конструкторы · 35.13 Проверка конфигурации

---

## 35.1. Редактор формы

Закладки: Реквизиты, Команды, Элементы, Параметры, Модуль, Командный интерфейс.

**Ролевая настройка:** свойства Просмотр, Редактирование (реквизиты), Пользовательская видимость (элементы), Использование (команды). Изменение реквизита или команды может потребовать удаления связанных элементов; при удалении команд предлагается удалить связанные элементы.

---

## 35.2. Редактор текстов (модули)

Подсветка синтаксиса: Текст → Встроенный язык. Недопустимые по XML 1.0 символы при вводе игнорируются.

**Группировка.** Синтаксические конструкции (Если…КонецЕсли, Пока…КонецЦикла, Процедура…КонецПроцедуры и т.д.) группируются автоматически. Области — директивы препроцессора `#Область … #КонецОбласти` (#Region … #EndRegion). Имя области — по правилам встроенного языка. Области могут быть вложенными, не должны пересекаться с другими синтаксическими конструкциями. Перед `#Область` допускается комментарий. Горячие клавиши: Ctrl+Num-/+, Ctrl+Shift+Num-/+, Ctrl+Shift+R — обновить группировки.

**Форматирование.** Синтаксический отступ — смещение вправо внутри управляющих конструкций (табуляция). Обычный отступ — выравнивание по левой границе предыдущей строки. Текст → Блок → Форматировать — форматирование выделенного блока. Пробелы в конце строк удаляются при записи модуля. Блок → Добавить/Удалить комментарий, Добавить/Удалить перенос строки (символ «|»).

**Переход.** Текст → Процедуры и функции — список процедур и функций. F12 — переход к определению переменной/процедуры/функции/объекта метаданных. Ctrl+«-» — возврат. Поиск мест использования — по всей конфигурации.

**Контекстная подсказка.** Ctrl+Пробел или Ctrl+M; после «.» — по настройке. В списке: системные объекты, свойства, методы, объекты конфигурации, переменные и процедуры/функции общих модулей и модулей объектов. Пиктограммы: свойства глобального контекста, свойства объектов, экспорт переменных, локальные переменные, процедуры/функции (глобальные, объектов, экспорт, локальные).

---

## 35.3–35.8. Прочие редакторы

Редакторы табличных документов, карты маршрута, картинок — детали интерфейса в документации платформы.

---

## 35.9. Конструктор запроса

Генерация текста запроса и кода обработки результата: обход результата, вывод в табличный документ, вывод в диаграмму. Закладки: Таблицы, Поля, Условия, Порядок, Итоги; для виртуальных таблиц — параметры. Пакет запросов — последовательность запросов. При выводе в табличный документ создаётся макет в объекте или общий макет (при вызове из общего модуля).

---

## 35.10. Конструктор движений регистров

Для документов. Выбор регистра, тип движения (для накопления), счета/вид корреспонденции (для бухгалтерии). Формулы атрибутов по реквизитам документа. Автозаполнение при совпадении имён реквизитов и измерений/ресурсов.

---

## 35.11. Конструктор печати

Создание макета с именованными областями и процедуры печати. Объекты: справочники, документы, журналы, планы видов характеристик, счетов, расчётов, обмена. Результат: макет, команда печати, процедура в модуле менеджера. Права команды — аналогично праву «Просмотр».

---

## 35.12. Конструктор ввода на основании

Для справочников, документов, планов видов характеристик, счетов, видов расчёта, обмена, бизнес-процессов. Формулы заполнения реквизитов результирующего объекта по реквизитам объекта-основания.

---

## 35.13. Проверка конфигурации (/CheckConfig)

- **Некорректные ссылки** — ссылки на удалённые объекты, некорректные ссылки на предопределённые данные. Параметр: `-IncorrectReferences`.
- **Синтаксический контроль модулей по средам:** Тонкий клиент (`-ThinClient`: Клиент, НаКлиенте, ТонкийКлиент), Веб-клиент (`-WebClient`), Мобильный клиент (`-MobileClient`), Мобильный автономный (`-MobileClientStandalone`), Сервер (`-Server`), Внешнее соединение (`-ExternalConnection`, `-ExternalConnectionServer`), Мобильное приложение (`-MobileAppClient`, `-MobileAppServer`), Толстый клиент управляемый (`-ThickClientManagedApplication`, `-ThickClientServerManagedApplication`), Толстый клиент обычное приложение (`-ThickClientOrdinaryApplication`, `-ThickClientServerOrdinaryApplication`).
- **Поставка без исходников** — проверка генерации образов модулей. Параметр: `-DistributiveModules`.
- **Логическая проверка:** неиспользуемые процедуры/функции (`-UnreferenceProcedures`), существование обработчиков (`-HandlersExistence`), пустые обработчики (`-EmptyHandlers`), расширенная проверка (`-ExtendedModulesCheck`), поиск модальности (`-CheckUseModality` с `-ExtendedModulesCheck`), синхронные вызовы (`-CheckUseSynchronousCalls` с `-ExtendedModulesCheck`).
- **Мобильное приложение:** неподдерживаемая функциональность (`-UnsupportedFunctional`), проверка подписи (`-MobileClientDigiSign`).

Настройки проверки можно сохранять. При проверке конфигурации, подключённой к хранилищу, рекомендуется захватить корневой объект конфигурации.
