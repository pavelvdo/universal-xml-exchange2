# Приложение 11. Правила автоматического формирования имен элементов формы

> **Agent ref:** Правила автоматического формирования имен элементов управляемой формы.  
> **Ключевые разделы:** Элементы формы, имена, командные панели, контекстное меню.

---

Контекстное меню и автоматическая командная панель доступны через свойства «Элементы» с префиксами **КоманднаяПанель**, **КонтекстноеМеню** и **Форма** на обоих вариантах встроенного языка. При ручном формировании имён (например, при программном создании элементов) рекомендуется следовать тем же правилам; не менять имена, сформированные системой при разработке формы. Имя из пути к связанным данным формируется как путь с удалёнными символами «.».

1. **Элементы, связанные с данными (поля и таблицы), в т.ч. основной реквизит.** Имя — полный путь к данным; для основного реквизита его имя из имени элемента удаляется. Пример: табличная часть «Товары» основного реквизита → имя таблицы **Товары**. Таблица ТЧ «Товары» реквизита ДопОбъект → **ДопОбъектТовары**.

2. **Колонки и текущие данные таблиц (кроме таблиц основного реквизита).** Правило: `<ИмяТаблицы><ПутьКДаннымРеквизита − ПутьКДаннымТаблицы>`. Пример: реквизит Количество ТЧ Товары основного реквизита → **ТоварыКоличество**; тот же реквизит у ДопОбъект → **ДопОбъектТоварыКоличество**.

3. **Таблицы, связанные с основным реквизитом.** Если таблица отображает основной реквизит — как п.1. Если связана с реквизитом основного реквизита — как п.2. Пример: колонка «Дата» в таблице Список (динамический список) → **СписокДата**. Таблица ТЧ Товары, реквизит Количество → элемент **Товары**, колонка **ТоварыКоличество**.

4. **Предопределённые элементы.** Контекстное меню: `<ИмяРодителя>КонтекстноеМеню`. Командная панель: `<ИмяРодителя>КоманднаяПанель`. Пример: командная панель формы → **ФормаКоманднаяПанель**; контекстное меню реквизита ТоварыКоличество → **ТоварыКоличествоКонтекстноеМеню**.

5. **Группа, связанная с источником команд:** `<ИмяЭлементаИсточника>Действия`.

6. **Кнопка в форме (не в командных панелях/контекстном меню):** `<ИмяКоманды>`. Префикс элемента не добавляется. Пример: команда формы «Записать» → **Записать**; команда «Изменить» ТЧ Товары → **Изменить**.

7. **Кнопка в автоматической командной панели с Автозаполнением:** `<ИмяВладельцаКоманднойПанели><ИмяКоманды>`. Пример: команда формы «Записать» → **ФормаЗаписать**.

8. **Кнопки СоздатьПоПараметру и ПоказатьВСписке (несколько штук)** в командной панели с Автозаполнением: **Форма** + **ИмяКоманды** + **ИмяОбъектаКонфигурации**.

9. **Кнопка в контекстном меню с Автозаполнением:** `<ИмяВладельцаКонтекстногоМеню>КонтекстноеМеню<ИмяКоманды>`. Пример: «Подобрать» в контекстном меню поля Дата → **ДатаКонтекстноеМенюПодобрать**.

10. **Группа в командной панели/контекстном меню без своего источника команд:** `<ИмяИсточникаКомандРодителя><ИмяГруппы>`. Имена групп не влияют на имена дочерних элементов.

11. **Группа в таблице, не связанной с основным реквизитом:** `<ИмяТаблицыФормы><ИмяГруппы>`. Пример: группа ЦенаИКоличество в таблице Товары → **ТоварыЦенаИКоличество**.

12. **Неавтоматические командные панели:** с источником — как п.7; без источника — имена кнопок и групп не модифицируются.

13. **Группа кнопок в командной панели/контекстном меню с источником команд:** `<ИмяРодителяКоманднойПанели><ИмяГруппы>`. Префикс «КоманднаяПанель» не добавляется. Пример: группа с источником Форма в контекстном меню колонки УслугиКоличество → **УслугиКоличествоКонтекстноеМенюФормаДействия**.
