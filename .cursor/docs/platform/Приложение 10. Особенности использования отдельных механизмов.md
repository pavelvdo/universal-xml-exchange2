# Приложение 10. Особенности использования отдельных механизмов

> **Agent ref:** Особенности использования отдельных механизмов платформы 1С:Предприятие.  
> **Ключевые разделы:** HTML, формы, файлы, табличный документ, экспорт, почта.

---

## Работа с HTML-документом

- Элемент `<TABLE>` не может быть дочерним для `<P>`. Конструкция `<P><TABLE></TABLE></P>` после обработки платформой превращается в `<P></P><TABLE></TABLE>` (по стандарту HTML — элемент P).

## Форма

- В обработчике ожидания не обрабатывается Ctrl+Break и не выполняется перерисовка экрана. Не размещать в нём код, препятствующий комфортной работе пользователя.
- Оповещение сервера о закрытии формы и снятие блокировки объекта происходят при: закрытии формы с заблокированным объектом; получении формы с сервера (ОткрытьФорму, ПолучитьФорму); вызове серверных методов формы; закрытии 20 форм; ожидании системы более 20 секунд.
- При выполнении команд динамического списка (удаление, пометка на удаление, проведение/отмена документа, перенос элемента справочника в другую группу) также происходит оповещение сервера.
- Открытие формы списка по ОткрытьЗначение() или кнопке «Открыть» в толстом клиенте (управляемый режим) — при: режиме редактирования «В списке»; наличии обычной формы списка как основной/вспомогательной; разрешённом использовании обычных форм в управляемом приложении; отсутствии управляемой формы списка/объекта как основной/вспомогательной.

## Работа с файлами

- В Windows длина полного пути к файлу не может превышать 260 символов.

## Табличный документ

- **Размещение текста:** у непустой ячейки при свойстве «Размещение текста» = Авто ячейки объединяются до следующей заполненной ячейки или до конца таблицы по горизонтали.
- **Экспорт в HTML:** если текст выходит за границу ячейки по горизонтали, при экспорте в HTML ячейки объединяются до следующей заполненной или до конца таблицы.
- **Экспорт в PDF:** подбор шрифта для символов, отсутствующих в текущем шрифте. В Windows используется запись реестра `HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft\Windows NT\CurrentVersion\FontLink\SystemLink` (шрифт Microsoft Sans Serif); при отсутствии символов — подбор по диапазону UNICODE (см. таблицу ниже). В Linux и веб-клиенте шрифт подбирается из доступных в системе.
- **Экспорт в Excel:** текст, выровненный по центру и выходящий за границу ячейки, обрезается по границам ячейки. Рекомендация: объединённая ячейка под весь текст или каждая строка многострочного текста — в своей строке таблицы. При нескольких форматах строк разбиение по страницам выполняется по колонкам первой строки; колонки других форматов могут обрезаться. Длина текста в одной ячейке не должна превышать 32 767 символов — иначе ошибка при сохранении.
- **Картинки в ячейках (экспорт в Excel/ODS):** при положении текста «Слева»/«Справа» текст в остальных ячейках колонки обрезается; при «Сверху»/«Снизу» — в строке. При выводе текста поверх картинки используется рисунок типа «текст»; ориентация текста в части случаев не сохраняется.
- **SVG:** при экспорте в HTML/HTML4 и Office OpenXML — преобразование в растровое PNG; в ODS без фильтров — SVG, с фильтрами — как в OpenXML; в HTML5 — SVG вставляется в документ; в PDF — векторный экспорт (элементы с фильтрами — растровая картинка, по умолчанию 600 dpi).
- **Колонтитулы с форматированными строками:** в Word — игнорируются переводы строк и картинки; в Excel — игнорируются цвета текста и фона; суммарная длина колонтитула ≤ 255 символов (включая служебную информацию); XLS — картинки не экспортируются; XLSX — только первая картинка. В ODS — игнорируются изображения и цвет фона.
- **Области:** при экспорте в XLS и Microsoft Office OpenXML имена областей исходного табличного документа не экспортируются.

## Работа с почтой

- В Windows отключение использования протокола OpenSSL при работе с почтой выполняется через системный реестр: ключ `HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Control\SecurityProviders\SCHANNEL\Protocol` установить в 0 (тип DWORD); при отсутствии — создать.
