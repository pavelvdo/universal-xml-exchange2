# Приложение 7. Особенности поведения системы в различных режимах

> **Agent ref:** Особенности поведения системы в различных режимах работы 1С:Предприятие.  
> **Ключевые разделы:** низкая скорость соединения, режим совместимости (по версиям), веб-клиент, разделение данных, ОС (Linux, macOS), защита от опасных действий.

---

## 7.1. Особенности режима низкой скорости соединения

- Начальная страница: формы рабочего стола не отображаются автоматически (гиперссылка «Показать рабочий стол»); в Такси начальная страница отображается сразу.
- Формы: при создании получают представления ссылочных данных и значения «через точку» без дополнительных серверных вызовов.
- Оповещение о закрытии формы и снятие блокировки — при ОткрытьФорму/ПолучитьФорму, вызове серверных методов, закрытии 20 форм, ожидании >20 с, при командах динамического списка (удаление, пометка удаления, проведение, перенос в группу).
- Настройки сохраняются отложенно: при изменении — при следующем открытии формы, или при накоплении 20 отложенных, или через 20 мин бездействия, или при закрытии приложения.
- Нет картинок панели разделов; в поле ввода не формируется список выбора во время ввода по строке.
- Кеш списков быстрого выбора и ввода по строке: не более 20 списков на тип, не более 200 списков всего; очистка каждые 20 мин; повторный выбор с теми же параметрами без смены поля — новый запрос к серверу.
- Поиск в динамическом списке: при низкой скорости — после потери фокуса строкой поиска (при нормальной — в процессе набора с задержкой).

## 7.2. Особенности режима совместимости

Поведение ряда механизмов зависит от свойства конфигурации **Режим совместимости**. Для каждой версии перечислены отличия: при включённом режиме совместимости с указанной версией — одно поведение, при выключенном (текущее) — другое. Ниже — сводка по подразделам; полный перечень пунктов см. в документации платформы.

### 7.2.1. Предыдущие версии (8.1)

Основной режим запуска «Управляемое приложение» несовместим с режимом совместимости 8.1. Отличия: подсистемы и свойство Состав (8.1); заполнение обычных форм (только при вводе на основании и Заполнить()); ПроверитьЗаполнение() недоступен; справка из формы — диалог выбора раздела; таблицы журналов без виртуального поля Тип; конструктор запроса показывает таблицы/поля без права просмотра; СКД не проверяет интерактивные права на таблицы; картинки из библиотеки во внешних отчётах/обработках — внешние; свойства ДоступноеПолеКомпоновкиДанных и др.; нет ВерсияДанных для ряда объектов; интерактивные права не связаны с Редактирование; выгрузка модуля приложения в Конфигурация.МодульПриложения.txt; форматирование Число/Дата/Булево по региональным настройкам; вид сравнения Содержит в СКД — _, %, [ как спецсимволы; ВЫРАЗИТЬ в строку — фиксированная длина с пробелами; настройки печати табличных документов общие; запрос с РАЗЛИЧНЫЕ и УПОРЯДОЧИТЬ ПО по выражению не в выборке — допускается; поля остатка вложенного запроса во внешнем запросе — как обычные; Строка фиксированного размера — до 1024 символов; регистры по умолчанию без разделения итогов; полные имена модулей в техжурнале без конкретизации; блокировки — проверка только первого значения по элементу; конструктор формы/макета — синоним/имя меняются вместе; редактор картинок без альфа-канала; выделение активной колонки; Tab/Shift+Tab в пустой таблице — по колонкам; свойства цветов группировки табличного документа; нестандартная рамка/цвет — без скруглений; ПередЗакрытием при закрытии по таймауту не вызывается; выгрузка журнала в XML — UserName полное; пользователь без админправ может менять ЗапрещеноИзменятьПароль. Состав таблиц: регламентные задания, константы, настройки регистров накопления/бухгалтерии — при совместимости в одной таблице, без — по отдельным. Справка по подсистеме в содержании; СКД: один ресурс в диаграмму, нет группировки по полям-ресурсам; диагностика ключевых значений динамического списка; параметр Период Получить() периодического регистра сведений — при совместимости необязателен; размер элемента формы и шрифт; ВерсияДанных — пустая строка / 12 пробелов; обновление текста в поле ввода и ОбновлениеТекстаРедактирования; УстановитьЗначение() и ИспользоватьИзИсточникаДанных() — приоритет источника / исключение; ключи настроек командного интерфейса (Подсистема.desktop/… vs РабочийСтол/…); реквизиты от функциональных опций в проверяемых; чтение таблиц при открытии формы — все / только видимые; имя в источнике данных внешнего источника в SQL — кавычки всегда / только при спецсимволах; тип колонки вычисляемого поля СКД при выводе в коллекцию; одно поле СКД в разных группировках — разные колонки (в обоих режимах). С 8.2.16: произвольные представления; дополнительные данные иерархии. С 8.3.2: функциональная опция с привилегированным режимом — проверка прав при совместимости. С 8.3.3: ОсновныеРоли — одна роль / несколько; настройки динамического списка и функциональные опции; имя кнопки Отмена (ФормаОтмена / ФормаОтменитьРедактирование).

### 7.2.2.–7.2.22. С версиями 8.3.5–8.3.25

Для каждой подверсии (8.3.5, 8.3.6, … 8.3.25) заданы отличия по темам: формы (в т.ч. ключ настроек, ПриОткрытии/ПриЗакрытии, период и отображение таблицы), динамические списки (основной реквизит, доступные настройки, отбор, поиск, ключевые поля), разделители и регистры сведений, СКД (вычисления, макеты, отбор, группировки, условное оформление, вывод в коллекцию), блокировки и нумерация, внешние источники и NULL, запросы (временные таблицы, В с подзапросом, тип ВЫБОР, деление на NULL), XDTO и почта, ЧтениеДанных (КонвертируемыйРазделительСтрок), расширения (Выполнить(), фоновые задания), прокси (переменные окружения Linux), табличные документы (печать, примечания Excel, шрифты, вертикальное выравнивание), полнотекстовый поиск, стандартное хранилище настроек (упорядочивание выборки, длина ключа/имени пользователя), защита от опасных действий (подстановка параметра в комментарии). Формат каждого пункта в полной документации: «При совместимости с версией X — …; при текущем — …».

## 7.3. Особенности режима совместимости расширения

Проверка применимости расширений перед применением; добавление справочников/регистров сведений и реквизитов/ТЧ в расширении; порядок свойств планов счетов; СУММА()/СРЕДНЕЕ() для определяемого типа (параметры типа / длина 38); установка текста запроса и ИспользованиеКопийБазыДанных — при совместимости расширения с указанной версией поведение может отличаться. Детали — в документации платформы.

## 7.4. Особенности работы веб-клиента

- Браузеры: в Chrome и Safari нет программной вставки из буфера (с определённых версий — любого программного доступа к буферу); нет автоматической аутентификации ОС; нет изменения колонтитулов при печати. В Safari — нет расширения криптографии. COM — только в IE. Пароли: OS X и iOS до 5.x — только латиница и цифры; с iOS 6 — национальные алфавиты (для совместимости рекомендуется латиница и цифры).
- Firefox: при добавлении в закладки исправить ссылку (удалить текст до http://).
- Chrome: с расширением для 1С возможны сбои буфера обмена при запуске — перезапуск браузера.
- HTTP: часть возможностей недоступна или работает некорректно; в 32-разрядной ОС могут ставиться 64-разрядные расширения.
- Не поддерживаются: прерывание модуля пользователем (ОбработкаПрерыванияПользователя игнорируется); ЗаблокироватьРаботуПользователя(); диагностика ошибки в Вычислить() как ошибки выражения; SVG — в части браузеров конвертация в PNG; отладка в ПередЗавершениемРаботыСистемы/ПриЗавершенииРаботыСистемы и ПередЗакрытием/ПриЗакрытии при закрытии окна браузера; смена курсора при длительных операциях; горизонтальная прокрутка колёсом мыши + Shift; в IE и Firefox — обработка клавиш во время серверного вызова (в Chrome и Safari — после); символ & для горячих клавиш не поддерживается; в Safari при длительном запросе возможна ошибка.
- Формы: не рекомендуется открывать/закрывать несколько форм в одном обработчике; при закрытии вспомогательного окна крестиком или Alt+F4 не вызывается ПередЗакрытием; закрытие до полной отрисовки может дать ошибку браузера; при изменении реквизитов формы вне её модуля нужен ОбновитьОтображениеДанных(); в Chrome при свёрнутом окне предупреждение не активирует окно.
- Шрифты: для нестандартного размера указывать и гарнитуру; в Chrome «MS Shell Dlg» даёт «Times New Roman».
- Ограничения: Вопрос() без кнопки Отмена — крестик переоткрывает диалог; панель состояния обновляется в зависимости от браузера; поле индикатора внешне отличается; Дендрограмма, ГрафическаяСхема, ГеографическаяСхема, ДиаграммаГанта — ограниченная функциональность; нет системной информации о клиенте; нет работы с ЧтениеТекста, ЗаписьТекста, Zip, XBase, XML, XDTO на клиенте (рекомендуется сервер).
- Буфер обмена (Chrome, Safari): внутренний буфер сеанса; дублирование в ОС; M+/M− меняют внутренний буфер. Калькулятор: MR, M+, M− — внутренний буфер.
- Табличный документ: перенос текста и размеры печати зависят от браузера; нет черно-белого режима; для нестандартного размера страницы — свойство ТочностьПечати; двусторонняя печать в Chrome — через системный диалог или ТочностьПечати; межстрочный/межсимвольный интервал может отличаться; в ячейке таблицы формы выводятся только целые строки по высоте; при смене источника и перестроении формы порядок ПриИзмененииСтроки и перезагрузки не определён; в формах с вертикальной прокруткой отключать растягивание по вертикали у таблиц, табличного документа, диаграммы; ПолучитьHTML() форматированного документа — пустой список картинок; выделение текста в IE и тонком клиенте скрывает цвет; нет открытия из файла табличного документа с ссылками в расшифровках; нет прокрутки колёсом в табличном документе, поле табличного документа, таблице формы; выравнивание по правому краю при не помещающемся тексте — по левому; в Chrome и Safari возможны артефакты рамок при объединениях; точечные рамки в IE и Chrome могут отображаться сплошной; для точной печати в Chrome и Firefox — ТочностьПечати. Нет режима сравнения файлов, списка последних файлов, перетаскивания файлов в формы. Ограниченный набор шрифтов. Меню Сервис — F10 (Alt не поддерживается). Ctrl+W в Chrome — закрытие окна; Ctrl+F4 — закрытие вкладки; Ctrl+N — новое окно. Масштаб и dpi — только 100 % и стандартный. Переключение окон: в Chrome нет Ctrl+Tab/Ctrl+Shift+Tab; F6/Ctrl+F6 работают. Модальное окно: в IE блокирует все родительские, в Firefox — только родительское, в Chrome окна не блокируются. Из модальной формы все открываемые формы — модальные. Минимальный размер окна задаётся браузером. Несколько мониторов с разным разрешением — координаты могут меняться. Нет вертикальной командной панели. Страницы с другим хостом — ограничения обработки событий и вызова команд; рекомендуется хранить страницы в ИБ. Выбор из списка (ВыбратьИзСписка, ПоказатьВыборИзСписка) — отдельное окно; ВыбратьИзМеню() и меню расшифровки СКД — отдельное окно. ПоместитьФайл() может вернуть Истина при ошибке. В Safari размер окон калькулятора и календаря — не меньше минимального. Отмена/возврат в многострочных полях — по правилам браузера; Ctrl+щелчок по слову — по-разному в браузерах; в поле текстового документа стрелки в первой/последней позиции строки — переход на другую строку; нет режима замены; перенос только по словам; Shift+Del удаляет всю строку; недопустимые по XML 1.0 символы в Chrome и Safari — замена на «?» и блокировка поля; в Firefox в подсказке переводы строк — пробелы. Язык интерфейса в Safari — по браузеру, если не задан параметром L. При отладке имена переменных в точках останова — с учётом регистра. Не поддерживаются: печать на текущий принтер; в поле графической схемы — поиск и замена; в текстовом документе — переход к строке, закладки, подменю Текст; горячие клавиши для поля картинки (масштаб, прокрутка), графической схемы (прокрутка), иерархического списка (свернуть/развернуть узлы), табличного документа (группы, прокрутка, выделение), текстового документа (Ctrl+L в части браузеров). В OS X при неанглийской раскладке ограничения по клавишам Alt/Cmd в Chrome и Safari.

## 7.6. Особенности поведения при включённом режиме разделения данных

**7.6.1. Изменение значений разделителей** — через параметры сеанса; при доступе через веб-сервер возможно блокирование через default.vrd. После смены значений разделителей рекомендуется ОбновитьИнтерфейс().

**7.6.2. Нумерация объектов** — в рамках каждой уникальной комбинации значений разделителей; при определении номера используются значения разделителей сеанса или объекта.

**7.6.3.–7.6.31** (оперативная отметка времени, предопределённые данные, права, пользователи, планы обмена, функциональные опции, регламентные и фоновые задания, web-сервисы, константы, регистры сведений/накопления/бухгалтерии/расчёта, параметры ИБ, список сеансов/соединений, журнал регистрации, настройки и избранное, история, OData, запросы, блокировки, монопольный режим, удаление помеченных, полнотекстовый поиск, расширения, история данных, система взаимодействия, сервис интеграции, политики паролей): поведение каждого механизма учитывает разделители и области данных; детали — в документации платформы.

## 7.8. Особенности работы клиентского приложения под управлением macOS

Нет 32-разрядной версии. Нет файловой ИБ на сетевом ресурсе; нет кластера на macOS; нет публикации на веб-сервере. Нет COM (COMОбъект, Automation-сервер, внешние компоненты на COM). Нет внешних источников данных на клиенте и в конфигураторе; нет аутентификации ОС. Нет консоли администрирования кластера (используются ras и rac). Нет объекта Почта — использовать ЗапуститьПриложение() с URL mailto: (RFC 2368). Нет метафайлов Windows (WMF, EMF). Нет работы с панелью меню macOS. В сочетаниях клавиш вместо Control использовать Cmd.

## 7.9. Мобильная платформа

Возможности платформы для ПК не переносятся в мобильную версию автоматически; изменения мобильной версии — в V8Update.htm. Поддерживаются: подсистемы, константы, справочники, документы, журналы, регистры накопления (без разделения итогов и режима агрегатов), сведений, обработки, перечисления, права и роли (с ограничениями), пользователи (с ограничениями), функциональные опции, параметры сеанса, планы обмена (кроме РИБ), блокировки, подписки на события, запросы, динамический список (с ограничениями), Web-сервисы (без создания в решении), общие картинки, команды, макеты, модули, формы, СКД, условное оформление, XDTO (без создания пакетов), полнотекстовый поиск, языки, отладка. Не поддерживаются: переключение интерфейсов, бухучёт, периодические расчёты, бизнес-процессы и задачи, общие реквизиты, RLS, хранилища настроек, внешние источники, автоматизированное тестирование, перетаскивание, стандартные функции, часть элементов форм, расширенное редактирование, пользовательская настройка форм, сохранение/восстановление данных форм, информационная панель, история, оповещения (ПоказатьОповещениеПользователя), Состояние(), справка, регламентные задания. Поставка без исходников: модуль компилируется для ПК; при разном количестве параметров методов для мобильной и ПК при выгрузке в мобильное приложение — ошибка. Особенности: в iOS нет Выполнить() и Вычислить(); в iOS до 5.x пароли только латиница и цифры (с 6 — национальные; для совместимости рекомендуется латиница и цифры); настройки динамического списка только из встроенного языка; на Android анимация может отключаться на слабых устройствах. Не поддерживаются: сканирование штрихкодов камерой (кроме ПК под Windows), календари, контакты, журнал звонков, печать, SMS, неинтерактивный набор номера, сертификаты клиента.

## 7.10. Механизм защиты от опасных действий

**7.10.1. Общая информация.** Сторонний код (внешние отчёты/обработки, расширения) может выполнять деструктивные действия. Механизм выдаёт предупреждение при: загрузке внешней обработки/отчёта или расширения; загрузке/обновлении конфигурации или расширения; доступе из внешней обработки/расширения к загрузке другой обработки/отчёта, исполнению команды ОС, управлению пользователями, загрузке внешней компоненты. Пользователь разрешает или запрещает; при разрешении действие может потребоваться повторить. Разрешение запоминается в разрезе пользователя, объекта (по хешу) и действия; при следующем использовании того же объекта запрос не повторяется. Запрет не запоминается. В неинтерактивном контексте (например, фоновое задание) неразрешённое действие блокируется с записью в журнал. Проявление: при интерактивной загрузке внешней обработки/отчёта; при интерактивном подключении расширения; при обновлении конфигурации из файла; при вызове Создать()/Подключить() менеджера внешних обработок/отчётов; при попытке команды ОС (ЗапускПриложения, НачатьЗапускПриложения, КомандаСистемы); при Записать()/Удалить() ПользовательИнформационнойБазы; при ПодключитьВнешнююКомпоненту/НачатьПодключениеВнешнейКомпоненты; при ПолучитьCOMОбъект(), конструкторе COMОбъект().

**7.10.2. Отключение.** 1) Снять флажок «Защита от опасных действий» (ЗащитаОтОпасныхДействий) у пользователя. 2) Параметр ЗащитаОтОпасныхДействий методов Создать() и Подключить() менеджеров внешних обработок/отчётов — загрузка без запроса. 3) Параметр DisableUnsafeActionProtection в conf.cfg по шаблонам строк соединения — для всех пользователей подходящих ИБ. Порядок проверки: сначала флажок пользователя; затем conf.cfg; затем явное отключение при подключении обработки/отчёта; затем свойство расширения ЗащитаОтОпасныхДействий.
