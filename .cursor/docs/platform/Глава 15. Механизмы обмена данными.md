# Глава 15. Механизмы обмена данными

> **Agent ref:** Механизмы обмена: универсальный обмен (XML, планы обмена), распределенные ИБ. Ключевое: 15.1 цели, 15.2 универсальные механизмы (XML-сериализация, планы обмена, регистрация изменений), 15.3 распределенные ИБ (иерархия узлов, сообщения, обработчики).
> **Ссылки:** ЧтениеXML, ЗаписьXML, ЗаписатьXML(), ПрочитатьXML(), ПланыОбменаМенеджер, ЗаписьСообщенияОбмена, ЧтениеСообщенияОбмена, ЗарегистрироватьИзменения(), УдалитьРегистрациюИзменений(), ВыбратьИзменения().

---

## 15.1. Цели и задачи

Механизмы обмена данными — набор средств «1С:Предприятие» для обмена данными между ИБ и с внешними системами. Два уровня:
- **Универсальные механизмы** — чтение/запись XML, XML-сериализация; произвольные схемы обмена; формат на базе XML; пригодны для обмена с не-1С системами.
- **Распределенные информационные базы** — иерархия узлов ИБ с синхронизацией конфигурации и данных; базируются на универсальных механизмах, плюс передача изменений конфигурации в подчиненные узлы.

К универсальным механизмам относятся: средства чтения/записи документов XML, XML-сериализация.

## 15.2. Универсальные механизмы обмена данными

### 15.2.1. Общая информация

См. 15.1. Универсальные механизмы могут использоваться вместе или по отдельности; обменивающиеся ИБ могут иметь разные конфигурации.

### 15.2.2. Средства чтения и записи документов XML

Участники обмена обмениваются сообщениями в формате XML. Объекты платформы: **ЧтениеXML**, **ЗаписьXML**, **ПреобразованиеXSL**; для FastInfoset — **ЧтениеFastInfoset**, **ЗаписьFastInfoset**. Эти средства не задают способов представления данных 1С в XML.

### 15.2.3. XML-сериализация

**Задача:** чтение/запись объектов данных 1С в/из XML. Каждый объект данных представляется элементом XML. Типы делятся на простые (только текстовое содержимое) и сложные (вложенные элементы).

Тип данных XML: имя типа + пространство имен. Источники типа: W3C XML Schema Datatypes (`http://www.w3.org/2001/XMLSchema`), предопределенные типы 1С (`http://v8.1c.ru/data`), типы из метаданных (без пространства имен). Явное задание типа — атрибут `xsi:type` (пространство `http://www.w3.org/2001/XMLSchema-instance`) или имя корневого элемента.

**Пространства имен для примеров:** `xmlns:xsd="http://www.w3.org/2001/XMLSchema"`, `xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"`, `xmlns:v8="http://v8.1c.ru/data"`.

#### 15.2.3.1. Представление данных в XML-сериализации

##### 15.2.3.1.1. Простые типы в XML

| Тип 1С | Тип XML (пространство) | Примечание |
|--------|------------------------|------------|
| Число | decimal (xsd) | XML Schema Part 2 |
| Строка | string (xsd) | как есть |
| Дата | dateTime (xsd) | YYYY-MM-DDTHH:MM:SS |
| Булево | boolean (xsd) | false/true |
| ДвоичныеДанные | base64Binary (xsd) | Base64, RFC 2045 |
| NULL | Null (v8) | пустой элемент |
| УникальныйИдентификатор | UUID (v8) | ISO-11578, DCE 1.1 |
| ХранилищеЗначения | ValueStorage (v8) | файл + Base64 |
| Ссылки на объекты БД | CatalogRef.*, DocumentRef.* и т.д. | без namespace; значение = UUID ссылки |
| Ссылки на перечисления | EnumRef.* | без namespace; значение = имя значения перечисления |

Примеры фрагментов:
- Число: `<decimal>45684.087</decimal>`, `<Data xsi:type="xsd:decimal">64793.01</Data>`
- Строка: `<Name>Иванов</Name>`, `<Data xsi:type="xsd:string">Это такая строка</Data>`
- Дата: `<dateTime>2008-11-21T12:00:00</dateTime>`
- Булево: `<boolean>false</boolean>`, `<Posted>true</Posted>`
- Ссылка: `<CatalogRef.Банки>911b5b8b-11f5-4993-9673-2c9a7a8995d5</CatalogRef.Банки>`, `<Data xsi:type="CatalogRef.Банки">...</Data>`
- Перечисление: `<EnumRef.ВидыАдресов>Юридический</EnumRef.ВидыАдресов>`

##### 15.2.3.1.2. Сложные типы в XML

- **Тип** — элемент с текстом имени типа XML (с префиксом namespace).
- **ОписаниеТипов** — тип TypeDescription (v8): вложенные элементы Types, NumberQualifiers, StringQualifiers, DateQualifiers.
- **КонстантаМенеджерЗначения.*** — тип ConstantValueManager.<Имя>, вложенный элемент Value.
- **Объекты БД** — англоязычное имя типа объекта; реквизиты — элементы по имени реквизита, при неоднозначности — `xsi:type`; табличные части — элемент по имени ТЧ, строки — элементы Row.
- **Наборы записей** — элемент Filter (отбор), элемент Records с вложенными Record.
- **УдалениеОбъекта** — тип ObjectDeletion (v8), вложенный v8:Ref (ссылка на объект).

Пример константы:
```xml
<ConstantValueManager.НазваниеОрганизации>
  <Value>ООО "Мебиус"</Value>
</ConstantValueManager.НазваниеОрганизации>
```

Пример УдалениеОбъекта:
```xml
<v8:ObjectDeletion xmlns="http://v8.1c.ru/data">
  <v8:Ref xsi:type="CatalogRef.Банки">60c5cec3-7f6f-4ec3-9620-e757fe3614ca</v8:Ref>
</v8:ObjectDeletion>
```

#### 15.2.3.2. Доступ к XML-сериализации из встроенного языка

**Простые значения:** `XMLСтрока(Значение)` — строка для текста элемента; `XMLЗначение(Тип, Строка)` — значение из строки.

**Преобразование типов:** `XMLТип(Тип)` → ТипДанныхXML или Неопределено; `ИзXMLТипа(ТипДанныхXML)` или `ИзXMLТипа(ИмяТипа, ПространствоИмен)` → тип 1С или Неопределено.

**Запись в XML:** `ЗаписатьXML(ЗаписьXML, Значение[, ИмяКорневогоЭлемента[, ПространствоИмен]][, НазначениеТипаXML])`. Значение — либо конкретный объект (элемент справочника, документ, константа и т.д.), либо объект метаданных (справочник, документ, регистр и т.д.) — тогда записываются все доступные текущему пользователю данные этого объекта.

Примеры:
```bsl
XML.ЗаписатьНачалоЭлемента("object");
ЗаписатьXML(XML, КлючевойТовар.ПолучитьОбъект());
XML.ЗаписатьКонецЭлемента();

XML.ЗаписатьНачалоЭлемента("meta");
ЗаписатьXML(XML, Метаданные.Справочники.Товары);
XML.ЗаписатьКонецЭлемента();
```

**Чтение из XML:** `ПрочитатьXML(ЧтениеXML[, ТипЗначения])`. Тип можно не указывать — определяется по `xsi:type` или имени элемента. Считанные объекты БД, менеджеры констант, наборы записей в БД не записываются автоматически — для записи нужно вызвать `Записать()`. При чтении объекта по ссылке: если объект найден в БД — его реквизиты/ТЧ перезаписываются из XML; если не найден — создается новый с этой ссылкой. Неполное чтение: атрибут `dataPart="true"` — считываются ключевые поля, объект читается/создается в БД, затем реквизиты заполняются из XML по имени.

Дополнительно: `ВозможностьЧтенияXML(ЧтениеXML)`, `ПолучитьXMLТип(ЧтениеXML)`.

### 15.2.4. Планы обмена

Планы обмена — центр механизмов обмена. В конфигурации может быть несколько планов обмена. Составляющие: **инфраструктура сообщений**, **служба регистрации изменений**.

#### 15.2.4.1. Узлы планов обмена

Элементы плана обмена — узлы (участники обмена). Один узел соответствует данной ИБ («этот узел»), остальные — внешние участники. У каждого узла должен быть уникальный код (идентификация при обмене). Код и Наименование плана обмена не могут иметь длину 0.

Пример настройки двух ИБ: в одной — узел «Офис» (этот) + узел «Склад1»; во второй — узел «Склад1» (этот) + узел «Офис».

#### 15.2.4.2. Инфраструктура сообщений

Сообщение — документ XML. Элемент Message (пространство `http://v8.1c.ru/messages`), вложенные Header и Body. Header: ExchangePlan, To, From, MessageNo, ReceivedNo. Body — произвольное содержимое (формат задается разработчиком).

Пример структуры сообщения:
```xml
<v8msg:Message xmlns:v8msg="http://v8.1c.ru/messages">
  <v8msg:Header>
    <v8msg:ExchangePlan>УдаленныеСклады</v8msg:ExchangePlan>
    <v8msg:To>Склад1</v8msg:To>
    <v8msg:From>Офис</v8msg:From>
    <v8msg:MessageNo>20</v8msg:MessageNo>
    <v8msg:ReceivedNo>15</v8msg:ReceivedNo>
  </v8msg:Header>
  <v8msg:Body><!-- тело --></v8msg:Body>
</v8msg:Message>
```

#### 15.2.4.3. Служба регистрации изменений

Регистрируются изменения для последующей передачи только измененных данных. Для каждого элемента данных — запись: ключ элемента, узел-получатель, номер сообщения (при первой отправке). После подтверждения приема записи удаляются.

Регистрация возможна для: констант; объектов БД (справочники, документы, планы счетов, видов характеристик/расчета, бизнес-процессы, задачи); наборов записей регистров, последовательностей, перерасчетов. Ключ: для константы — идентификатор, для объекта — ссылка, для набора с регистратором — регистратор, для регистра сведений без регистратора — измерения (и период при периодическом). В состав плана обмена включаются объекты метаданных; свойство **Авторегистрация** — автоматическая регистрация при записи/удалении или только ручная.

#### 15.2.4.4. Доступ к механизмам планов обмена из встроенного языка

**ПланыОбменаМенеджер:** ЗарегистрироватьИзменения(), УдалитьРегистрациюИзменений(), ИзменениеЗарегистрировано(), ВыбратьИзменения(), СоздатьЧтениеСообщения(), СоздатьЗаписьСообщения().

**ПланОбменаМенеджер.<Имя>:** метод ЭтотУзел() — ссылка на узел данной ИБ. Реквизит ЭтотУзел можно менять (например после восстановления из XML). Смена «этого узла»: только через ОбменДанными.Загрузка = Истина для нового узла и сброс для старого, желательно в транзакции.

**ПланОбменаОбъект.<Имя>:** НомерОтправленного, НомерПринятого.

**ПараметрыОбменаДанными.**Получатели (НаборУзлов): список узлов для регистрации при записи/удалении. Заполняется автоматически перед ПередЗаписью/ПередУдалением (если Автозаполнение = Истина). В обработчике можно добавлять/удалять узлы. Автозаполнение = Ложь — управление списком вручную в коде.

Пример смены «этого узла»:
```bsl
НачатьТранзакцию();
ТекущийЭтотУзел = ПланыОбмена.ОбменДанными.ЭтотУзел();
НовыйОбъект = НовыйЭтотУзел.ПолучитьОбъект();
НовыйОбъект.ОбменДанными.Загрузка = Истина;
НовыйОбъект.ЭтотУзел = Истина;
НовыйОбъект.Записать();
ТекущийОбъект = ТекущийЭтотУзел.ПолучитьОбъект();
ТекущийОбъект.ЭтотУзел = Ложь;
ЗафиксироватьТранзакцию();
```

##### 15.2.4.4.1. Регистрация изменений

**Авторегистрация:** в ПередЗаписью()/ПередУдалении() можно изменить Получатели (например исключить узел).

Пример исключения узла из получателей:
```bsl
Процедура ПередЗаписью()
    Узел = ПланыОбмена.УдаленныеСклады.НайтиПоКоду("Особый");
    ОбменДанными.Получатели.Удалить(Узел);
КонецПроцедуры
```

**ЗарегистрироватьИзменения(Узел | МассивУзлов[, Данные]).** Данные: не указаны — все зарегистрированные для узла; ссылка/УдалениеОбъекта — один объект; объект метаданных — все данные объекта; массив ссылок/УдалениеОбъекта — переданные объекты. Для массива второй параметр обязателен.

**УдалитьРегистрациюИзменений(Узел | МассивУзлов[, Данные | НомерСообщения]).** Для одного узла можно передать номер сообщения — удаляются записи с номером ≤ указанного (подтверждение приема).

**ИзменениеЗарегистрировано(Узел, Данные)** — проверка регистрации.

##### 15.2.4.4.2. Запись сообщений обмена

**ЗаписьСообщенияОбмена:** НачатьЗапись(ЗаписьXML, УзелПолучателя), ЗакончитьЗапись(), ПрерватьЗапись(). После НачатьЗапись() в тело сообщения записываются данные (например через ЗаписатьXML()). ВыбратьИзменения(Узел, НомерСообщения) возвращает ВыборкаДанных (Следующий(), Получить()); Получить() для удаленного объекта возвращает УдалениеОбъекта. Порядок в выборке: константы, планы видов характеристик, счета, прочие ссылочные, наборы записей регистров, последовательности.

Типовой фрагмент записи сообщения с зарегистрированными изменениями:
```bsl
ЗаписьXML = Новый ЗаписьXML();
ЗаписьXML.ОткрытьФайл(ИмяФайлаСообщения);
Узел = ПланыОбмена.УдаленныеСклады.НайтиПоКоду(КодУзла);
ЗаписьСообщения = ПланыОбмена.СоздатьЗаписьСообщения();
ЗаписьСообщения.НачатьЗапись(ЗаписьXML, Узел);
Выборка = ПланыОбмена.ВыбратьИзменения(Узел, ЗаписьСообщения.НомерСообщения);
Пока Выборка.Следующий()
    Данные = Выборка.Получить();
    ЗаписатьXML(ЗаписьXML, Данные);
КонецЦикла;
ЗаписьСообщения.ЗакончитьЗапись();
```

##### 15.2.4.4.3. Чтение сообщений обмена

**ЧтениеСообщенияОбмена:** НачатьЧтение(ЧтениеXML[, ДопустимыйНомерСообщения]), ЗакончитьЧтение(), ПрерватьЧтение(). ДопустимыйНомерСообщения: Любой, Следующий (строго +1 к макс. принятому), Больший (по умолчанию). НачатьЧтение проверяет план обмена, отправителя/получателя, допустимость номера.

Типовой фрагмент чтения с записью в БД:
```bsl
ЧтениеXML = Новый ЧтениеXML();
ЧтениеXML.ОткрытьФайл(ИмяФайлаСообщения);
ЧтениеСообщения = ПланыОбмена.СоздатьЧтениеСообщения();
ЧтениеСообщения.НачатьЧтение(ЧтениеXML);
ПланыОбмена.УдалитьРегистрациюИзменений(ЧтениеСообщения.Отправитель, ЧтениеСообщения.НомерСообщения);
Пока ВозможностьЧтенияXML(ЧтениеXML)
    Данные = ПрочитатьXML(ЧтениеXML);
    Данные.ОбменДанными.Отправитель = ЧтениеСообщения.Отправитель;
    Данные.ОбменДанными.Загрузка = Истина;
    Данные.Записать();
КонецЦикла;
ЧтениеСообщения.ЗакончитьЧтение();
```

##### 15.2.4.4.4. Гарантированная доставка

При гарантированной доставке: после ЗакончитьЗапись() сразу вызывать УдалитьРегистрациюИзменений(Узел, НомерСообщения); при приеме — только ДопустимыйНомерСообщения.Следующий. В большинстве случаев предпочтительнее повторная отправка до подтверждения.

##### 15.2.4.4.5. Разрешение коллизий

Коллизия — один и тот же элемент изменен в двух узлах. Стратегия «главный — подчиненный»: при приеме проверять ИзменениеЗарегистрировано(); записывать данные в зависимости от роли узла (например реквизит Главный у узла плана обмена). Пример: при чтении если у отправителя Главный или изменение не зарегистрировано у получателя — записать, иначе не записывать.

##### 15.2.4.4.6. Соответствия пространств имен

После НачатьЗапись() соответствия пространств имен не заданы. Чтобы не дублировать их в теле, после НачатьЗапись() вызвать ЗаписьXML.ЗаписатьСоответствиеПространстваИмен("xsd", "http://www.w3.org/2001/XMLSchema") и аналогично для xsi, v8.

## 15.3. Распределенные информационные базы

Распределенная ИБ — совокупность узлов (ИБ 1С) с синхронизацией конфигурации и данных, иерархическая структура. У узла — один главный и произвольное число подчиненных. Корневой узел — без главного. Обмен только с «соседями» (главный и подчиненные). Изменения конфигурации — только в корневом узле, затем распространение по иерархии. Изменения данных — в любом узле, синхронизация распространением изменений. Последовательности документов: участие в последовательности имеет смысл только в одном узле. Состав данных обмена регулируется составом плана обмена и обработчиками «по горизонтали».

**ВНИМАНИЕ.** Изменения конфигурации разрешены только в корневом узле распределенной ИБ.

### 15.3.1. Общие принципы

План обмена для распределенной ИБ должен иметь свойство **Распределенная информационная база**. Содержимое сообщений регламентировано протоколом (в отличие от произвольного тела при универсальном обмене). Формирование и прием сообщения — одним методом (ЗаписатьИзменения / ПрочитатьИзменения). Обработчики в модуле плана обмена: ПриОтправкеДанныхПодчиненному, ПриОтправкеДанныхГлавному, ПриПолученииДанныхОтПодчиненного, ПриПолученииДанныхОтГлавного.

### 15.3.2. Главный и подчиненный узлы. Разрешение коллизий. Начальный образ

Узел знает только главный и подчиненные. Коллизии: по умолчанию приоритет у главного узла; от подчиненного — не записывать, если изменение уже зарегистрировано у получателя; от главного — записать и удалить регистрацию. Начальный образ подчиненного узла создается из главного: перенос конфигурации, при необходимости расширений, создание узлов в плане обмена в образе, перенос данных по правилам плана (и ПриОтправкеДанныхПодчиненному). Повторное создание начального образа допустимо; записи регистрации для этого узла удаляются.

### 15.3.3. Сообщение обмена в распределенной ИБ

Тело сообщения — один элемент Changes (пространство v8de). Внутри: Signature, Config (изменения конфигурации, Digest1/Digest2), необязательно ConfigurationExtensions, Nodes (только от главного к подчиненному — данные главного и подчиненного узлов), Data (измененные элементы данных в XML-сериализации). Конфигурирование: состав плана обмена определяет номенклатуру данных. При добавлении объекта в состав уже работающей РИБ существующие данные не регистрируются автоматически — нужна отдельная регистрация при необходимости.

### 15.3.4. Обработчики событий плана обмена (РИБ)

**ПриОтправкеДанныхПодчиненному(ЭлементДанных, ОтправкаЭлемента):** ОтправкаЭлемента: Авто (по умолчанию), Удалить (в сообщение — УдалениеОбъекта или пустой набор записей), Игнорировать (ничего не помещать). **ПриОтправкеДанныхГлавному** — аналогично для отправки главному.

**ПриПолученииДанныхОтПодчиненного(ЭлементДанных, ПолучениеЭлемента, ОтправкаНазад):** ПолучениеЭлемента: Авто (записать, если нет регистрации у отправителя), Игнорировать; ОтправкаНазад = Истина — зарегистрировать изменение для отправителя. **ПриПолученииДанныхОтГлавного** — тот же набор параметров; для подчиненного от главного элемент записывается независимо от регистрации (Авто и Принять дают один эффект).

### 15.3.5. Работа с распределенной ИБ из встроенного языка

- **СоздатьНачальныйОбраз(ПодчиненныйУзел, СтрокаСоединения)** — создание начального образа в пустую или несуществующую ИБ. Рекомендуется монопольный режим на главном узле при интенсивных изменениях.
- **Запись сообщения:** ЗаписьСообщения.НачатьЗапись(ЗаписьXML, Узел); ПланыОбмена.ЗаписатьИзменения(ЗаписьСообщения[, МаксЭлементовВТранзакции]); ЗаписьСообщения.ЗакончитьЗапись(). МаксЭлементовВТранзакции = 0 (по умолчанию) — одна транзакция.
- **Чтение сообщения:** ЧтениеСообщения.НачатьЧтение(ЧтениеXML); ПланыОбмена.ПрочитатьИзменения(ЧтениеСообщения[, МаксЭлементовВТранзакции]); ЧтениеСообщения.ЗакончитьЧтение(). Рекомендуется не указывать второй параметр НачатьЧтение (ДопустимыйНомерСообщения.Больший) — логика РИБ допускает потерю сообщений, но не повторный прием.
- **УстановитьГлавныйУзел(Узел | Неопределено)** — установка/сброс главного узла. **ГлавныйУзел()** — ссылка на главный узел или Неопределено для корневого. Для сброса главного узла также используется параметр командной строки /ResetMasterNode.

**ПрочитатьОписаниеИзменений()** / **ПрочитатьИзмененияКонфигурацииИРасширенийКонфигурации()** — анализ содержимого сообщения (КонфигурацияИзменена, РасширенияКонфигурацииИзменены и т.д.); первый метод не позволяет продолжить чтение сообщения после вызова.

### 15.3.6. Интерактивная работа и сценарии

Действия доступны через меню «Еще» формы списка плана обмена: создание начального образа, запись изменений в файл, чтение изменений из файла. Сценарии реализуются обработчиками ПриОтправке*/ПриПолучении*: поведение по умолчанию (все изменения распространяются, коллизии по главному/подчиненному); распределение по подчиненным (фильтрация по реквизиту узла, например Склад — отправлять подчиненному только «свои» документы, при получении от подчиненного при смене реквизита регистрировать отправку назад УдалениеОбъекта); приоритет подчиненного узла (ПриПолученииДанныхОтПодчиненного — ПолучениеЭлемента = Принять; ПриПолученииДанныхОтГлавного — при зарегистрированном изменении Игнорировать).
