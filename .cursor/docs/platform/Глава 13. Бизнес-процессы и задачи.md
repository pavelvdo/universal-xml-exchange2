## 13.1. Основные понятия

## 13.2. Общая часть

Бизнес-процессы объединяют операции в цепочки взаимосвязанных действий (например: выписка счета → приём оплаты → отпуск товара). Формализуют процедуры обработки событий и участие исполнителей.

Карта маршрута описывает логику и жизненный цикл БП от точки старта до точки завершения — схема последовательности точек маршрута. Движение бизнес-процесса — последовательное выполнение этой цепочки.

Точка маршрута — этап жизненного цикла, связанный с одной автоматической или ручной операцией.

Задачи — учёт заданий по исполнителям, отражение продвижения БП по точкам маршрута. Создаются бизнес-процессами, другими объектами ИБ или пользователями.

Объекты конфигурирования механизма БП: Бизнес-процессы, Задачи, Регистр сведений (адресация), Параметр сеанса. Реквизиты адресации и измерения регистра обычно имеют ссылочный тип (СправочникСсылка) — добавляются справочники.

## 13.3. Маршрутизация

БП создаёт задачи при переходе на точки маршрута; задача при выполнении оповещает БП — бизнес-процесс переходит на следующую точку. Регистр сведений (Адресация) используется для групповых точек; задачи отбираются по текущему исполнителю (параметр сеанса).

Виды маршрутизации:

- **Жёсткая** — строгая карта без условных и параллельных переходов, фиксированные адресаты по точкам. Без свободной и условной маршрутизации.
- **Свободная** — адресаты точки не заданы, определяются программно или интерактивно в ходе жизненного цикла.
- **Условная** — проверка условий, переход по ветвям (бинарно или множественный выбор варианта).
- **Параллельная** — разделение на параллельные ветви с возможностью слияния (ожидания); ветви выполняются независимо.

## 13.4. Система адресации

Назначение — персональная и ролевая адресация задач. Реквизиты адресации задачи задают размерность адресного пространства. Определение исполнителя — свойства задачи: Адресация, Основной реквизит адресации, Текущий исполнитель. Разыменование — определение основного реквизита адресации по остальным.

Регистр сведений Адресация хранит соответствие исполнителя (основной реквизит) структурным подразделениям, рабочим группам, функциям и т.д. (остальные реквизиты адресации). Один реквизит адресации назначается основным — конкретный сотрудник.

Пример: регистр с измерениями Роль и Сотрудник, записи «Кассир — Иванов», «Менеджер — Петров». В точке маршрута в адресации задана только роль Кассир → формируется одна задача (Роль — Кассир, Сотрудник — не заполнен). При просмотре задач сотрудником Ивановым система по регистру отобразит ему эту задачу.

Инициализация текущего исполнителя (в модуле сеанса или при старте):

```bsl
Процедура УстановкаПараметровСеанса(ТребуемыеПараметры)
	Пользователь = Справочники.Сотрудники.НайтиПоНаименованию(ИмяПользователя());
	ПараметрыСеанса.ТекущийИсполнитель = Пользователь;
КонецПроцедуры
```

Последовательность настройки двух БП с общей системой адресации:

1. Исходить из 3-мерной адресации: сотрудник, роль, подразделение.
2. Создать справочники по каждому измерению, заполнить предопределёнными значениями.
3. Регистр сведений — по одному измерению на справочник (тип — ссылка на справочник).
4. Параметр сеанса ТекущийИсполнитель, тип СправочникСсылка.Сотрудники.
5. Инициализировать параметр при запуске.
6. Создать задачу; свойство Адресация — созданный регистр.
7. Реквизиты адресации задачи: Сотрудник, Роль, Подразделение (тип — ссылка на справочник; в «Реквизиты адресации» — ссылка на измерение регистра).
8. Основной реквизит адресации — Сотрудник.
9. Первый БП: свойство Задача — ссылка на задачу; карта маршрута — реквизиты адресации точек из предопределённых данных справочников.
10. Второй БП — та же задача, своя карта маршрута.

## 13.5. Старт бизнес-процесса

Старт — вызов метода Старт() или кнопка «Стартовать и закрыть» в форме объекта.

```bsl
БП = БизнесПроцессы.Согласование.СоздатьБизнесПроцесс();
БП.Дата = ТекущаяДата();
БП.Записать();
БП.Старт();
```

При старте: (1) обработчик ПередСтартом точки старта; (2) продвижение по карте до точки действия; (3) формирование задач. БП может быть записан без старта (старт позже).

## 13.6. Выполнение задач

ВыполнитьЗадачу() — проверка выполнения, затем задача помечается выполненной, БП уведомляется; при выполнении условий БП переходит на следующую точку.

Последовательность при ВыполнитьЗадачу():

1. Начало транзакции  
2. ПередВыполнением (задача)  
3. ПередВыполнением (точка маршрута)  
4. Задача.Выполнена = Истина  
5. ПриВыполнении (задача)  
6. Запись задачи  
7. ПриВыполнении (точка маршрута)  
8. Переход БП на следующую точку  
9. Формирование задач по новой точке  
10. Завершение транзакции  

## 13.7. Разделение и слияние

Точка разделения — один вход, неограниченное число выходов; несколько параллельных ветвей. Точка слияния — выполнение дальше только после прохождения всех входящих ветвей (синхронизация веток своей точки разделения).

Разделение без слияния допускается — параллельные ветви до завершения БП. Слияние без разделения не допускается: при проверке карты — «Не все линии, вошедшие в точку слияния, вышли из точки разделения».

## 13.8. Ручное управление

### 13.8.1. Признак завершённости бизнес-процесса

Установка вручную: БП считается завершённым даже при невыполненных задачах; выполнение этих задач не двигает БП. Признак можно установить у нестартованного БП — старт станет невозможен.

Снятие признака завершения у завершённого БП: задачи остаются выполненными, БП не завершён, невыполненных задач нет, повторный старт невозможен (уже стартованный). Чтобы вернуть БП на нужную точку — снять признаки выполнения у нужных задач.

### 13.8.2. Признак выполнения задачи

Ручная установка Выполнена = Истина не вызывает ПередВыполнением/ПриВыполнении и не продвигает БП; может остановить БП. Снятие признака у задачи незавершённого БП может дать параллельный поток. Снятие у задачи завершённого БП — задача видна как невыполненная, но её выполнение не двигает БП.

### 13.8.3. Удаление задач

Удаление невыполненных задач незавершённого БП может остановить его. Удаление всех задач незавершённого БП переводит его в статус нестартованного.

### 13.8.4. Добавление задач

Ручная задача по завершённому БП — БП остаётся завершённым, выполнение задачи не двигает маршрут. Новая задача по нестартовавшему БП — он получает статус стартованного, выполнение задачи продвигает маршрут. Новая задача по стартовавшему незавершённому БП — распараллеливание.

## 13.9. Условный переход

Точка условного перехода — обработчик проверки условия обязателен; иначе при проверке карты: «Точка условия не имеет обработчика события "Проверка условия"». Результат Истина — ветка Да, иначе — Нет. По умолчанию — Ложь.

```bsl
Процедура ОграничениеСкидкиПроверкаУсловия(ТочкаМаршрутаБП, Результат)
	Если ПолучитьСкидкуПоСчету() > ПолучитьОбычнуюСкидку() Тогда
		Результат = Истина;
	Иначе
		Результат = Ложь;
	КонецЕсли;
КонецПроцедуры
```

## 13.10. Выбор варианта

Точка выбора варианта — многовариантный выбор. Обработчик обязателен; иначе: «Точка выбора варианта не имеет обработчика события Выбор варианта». Обработчик устанавливает параметр Результат одним из вариантов. Неустановленный Результат — ошибка и отмена транзакции.

```bsl
Процедура ВыборВарианта(ТочкаВыбораВарианта, Результат)
	Если ВидОплаты = Перечисления.ВидОплаты.Наличная Тогда
		Результат = ТочкаВыбораВарианта.Варианты.Наличная;
	ИначеЕсли ВидОплаты = Перечисления.ВидОплаты.Безналичная Тогда
		Результат = ТочкаВыбораВарианта.Варианты.Безналичная;
	ИначеЕсли ВидОплаты = Перечисления.ВидОплаты.Взаимозачет Тогда
		Результат = ТочкаВыбораВарианта.Варианты.Взаимозачет;
	КонецЕсли;
КонецПроцедуры
```

ВидОплаты — реквизит бизнес-процесса.

## 13.11. Формирование задач

Задачи создаются только в точках действия и точках вложенных БП. В остальных точках (условие, разделение, слияние, обработка) БП переходит к следующей точке без создания задач.

В точке действия/вложенного БП — одна или несколько задач. Несколько — если точка групповая: по регистру Адресация отбираются записи по реквизитам адресации точки, для каждой создаётся задача.

СтандартнаяОбработка = Истина: разыменование реквизитов адресации точки, формирование одной или списка задач (групповая); у каждой — Наименование, ссылка на БП, точка маршрута, реквизиты адресации.

Последовательность при переходе на точку (например, «Выписка счета»):

1. Начало транзакции  
2. ПередСозданиемЗадач() (передаётся пустой массив — можно сформировать задачи вручную и отказаться от стандартной обработки)  
3. Формирование списка задач  
4. ПриСозданииЗадач() (задачи ещё не записаны; можно донастроить — срок, приоритет, добавить задачи; Отказ = Истина — прерывание и исключение)  
5. Запись всех задач  
6. По каждой задаче: ПередЗаписью → запись → ПриЗаписи  
7. Завершение транзакции  

Автозаполнение при формировании БП: Наименование (из точки), ссылка на БП, точка маршрута, реквизиты адресации (из точки).

## 13.12. Проверка выполнения

Для автоматического выполнения (например, при проведении документа): ПроверитьВыполнение() у задачи, затем ВыполнитьЗадачу().

```bsl
Если Задача.ПроверитьВыполнение() Тогда
	Задача.ВыполнитьЗадачу();
КонецЕсли;
```

Последовательность ПроверитьВыполнение(): вызов метода → ОбработкаПроверкиВыполнения() у задачи (Результат = Ложь → метод возвращает Ложь) → ОбработкаПроверкиВыполнения() у точки маршрута → возврат результата; при Истина вызывается ВыполнитьЗадачу().

## 13.13. Выполнение вложенных процессов

Точка маршрута «Вложенный бизнес-процесс» — основной БП ждёт завершения вложенного, затем переход к следующей точке.

Последовательность: Начало транзакции → ПередСозданиемВложенныхБизнесПроцессов() → ПередСозданиемЗадач() для точки → при СтандартнаяОбработка массив задач → ПриСозданииЗадач() → запись задач и массив вложенных БП → ПриСозданииВложенныхБизнесПроцессов() → запись и старт вложенных БП → Завершение транзакции. Запись и старт вложенных БП — после ПриСозданииВложенныхБизнесПроцессов().

## 13.14. Завершение бизнес-процесса

БП автоматически Завершен = Истина при достижении точки завершения и отсутствии невыполненных задач.

Свойство Ведущая задача (вложенный БП): при завершении вложенный БП помечает эту задачу выполненной → продвижение основного БП.

При переходе на точку завершения — обработчик ПриЗавершении(). Отказ = Истина — прерывание, задача по точке остаётся невыполненной.

Ручная установка Завершен = Истина — прерывание хода БП без вызова обработчиков (кроме ПередЗаписью/ПриЗаписи), без выполнения ведущей задачи.

## 13.15. Стандартные реквизиты бизнес-процессов и задач

**Бизнес-процессы:** ПометкаУдаления (Булево), Номер (Строка/Число), Дата (Дата), Завершен (Булево), ВедущаяЗадача (ЗадачаСсылка.<Имя задачи>), Ссылка (БизнесПроцессСсылка.<Имя БП>).

**Задачи:** ПометкаУдаления (Булево), Номер (Строка/Число), Дата (Дата), Наименование (Строка), Выполнена (Булево), БизнесПроцесс (БизнесПроцессСсылка.<Имя>), ТочкаМаршрута (ТочкаМаршрутаБизнесПроцессаСсылка.<Имя>), Ссылка (ЗадачаСсылка.<Имя задачи>).

## 13.16. Бизнес-процессы с несколькими точками старта

Несколько точек старта — выбор по внешним условиям. Старт() при этом должен вызываться с указанием точки (иначе ошибка). Интерактивно: команды старта по точкам. Для вложенных БП: в ПриСозданииВложенныхБизнесПроцессов() записывать и стартовать каждый БП с нужной точкой.

```bsl
Процедура ВложенноеСогласованиеПриСозданииВложенныхБП(ТочкаМаршрутаБП, ФормируемыеПроцессы, Отказ)
	Для каждого БизнесПроцесс из ФормируемыеПроцессы Цикл
		БизнесПроцесс.Записать();
		Точки = БизнесПроцессы.СогласованиеДокумента.ТочкиМаршрута;
		БизнесПроцесс.Старт(Точки.УпрощенноеСогласование);
	КонецЦикла
КонецПроцедуры
```

## 13.17. Обратная связь

Внешние объекты могут влиять на БП через оповещения. Пример: задача требует проведения документа; при проведении — автоматическое выполнение задачи.

В форме документа при проведении:

```bsl
Процедура ПослеЗаписи(Отказ)
	Если БылоПроведение Тогда
		Оповестить("ПроведениеДокумента", , ЭтотОбъект.Ссылка);
	КонецЕсли;
КонецПроцедуры
```

В модуле управляемого приложения — подписка на оповещение и вызов общего модуля (например РаботаСБизнесПроцессами.ВыполнитьЗадачуПоДокументу(Источник)): выбор задач по документу, для каждой ПроверитьВыполнение() и при Истина — ВыполнитьЗадачу(). Альтернатива — регламентное задание по расписанию: отбор задач по исполнителю, проверка и автоматическое выполнение.

## 13.19. Редактирование бизнес-процесса

Свойство Задача — ссылка на объект задачи. Без задачи при сохранении конфигурации: «Не выбрана задача бизнес-процесса». Автонумерация, Длина номера, Тип номера (Число/Строка), Контроль уникальности, Периодичность — как у документов.

Привилегированный режим при создании задач (закладка Права): формирование задач в привилегированном режиме (сервер и файловый вариант; на толстом клиенте клиент-серверного — не устанавливается). По умолчанию управляемое приложение — Истина, обычное — Ложь.

Допускаются дополнительные реквизиты и табличные части (документы на согласование, резолюции, участники и т.д.).

## 13.20. Редактирование задачи

Адресация — непериодический регистр сведений, измерения связаны с реквизитами адресации задачи (ролевая адресация). Основной реквизит адресации — один из реквизитов (конкретный исполнитель); если не указан — определяется по регистру (Адресация). Текущий исполнитель — параметр сеанса.

Автопрефикс номера: Не использовать / Номер бизнес-процесса. Группа «Реквизиты адресации» — связь с измерениями регистра для разыменования. Длина номера, Тип номера, Контроль уникальности, Автонумерация — по аналогии с другими объектами. Наименование задач от БП = наименование точки маршрута. Табличные части — по необходимости (например, документы для согласования).
