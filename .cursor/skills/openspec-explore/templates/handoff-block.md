# Шаблон блока `## Постановка ЗНИ` (SSOT)

Единственный источник каркаса финального блока explore. Дубли каркаса в других файлах запрещены — только ссылка сюда. Команды `/opsx:new` и `/opsx:extend` подхватывают блок из чата (приоритет 1) или handoff-файла (приоритет 2). Legacy-заголовок `## Для /opsx:ff` распознаётся читателями как синоним, но не генерируется.

## Каркас

```markdown
## Постановка ЗНИ

**Симптом:**
<2–3 предложения мини-абзацем: что наблюдалось, в каких условиях.>

**Корневая причина:** `модуль:строка` — [verified | hypothesis: план верификации]
<2–3 предложения мини-абзацем: почему это происходит.>

**Что менять:**
<2–3 предложения мини-абзацем: подход к исправлению, без псевдокода.>

**Файлы:** <список путей в backticks>
**Тема маркера:** <опционально; черновик comment_suffix — только описание, без ФИО и даты>
**Приёмка:** <1–2 предложения: как проверить исправление>
**Связь с архивом:** extends `<change>` | новый change | unrelated (одна строка пояснения)
**Срезы (черновик):** <опционально; 1–3 предполагаемых вертикальных среза, например «S1 фикс ядра, S2 UI»>
**Открытые решения:** <опционально; только неблокирующие вопросы — new перенесёт в design.md § Открытые вопросы>
**Architect / verify:** required | not-required | report: temp/reports/architecture-YYYY-MM-DD-<slug>.md
```

## Правила сборки (assembly rules — обязательны)

1. **Каждое поле — выжимка из конкретного отчёта** `Task` в `temp/reports/*` или решения, принятого в чате. Шаблон — чеклист полей, не текст для копирования; placeholder-текст в чат не выводится.
2. **Симптом** — наблюдаемое поведение из «От вас» + уточнения из отчётов. **Не** копировать дословно слот «Вопрос» брифа (это исследовательский вопрос, не симптом).
3. **Объём:** Симптом, Корневая причина, Что менять — мини-абзацы по 2–3 предложения. Однострочные пункты-телеграммы запрещены (HALT).
4. **Маркер причины** обязателен: `[verified]` (код:строка, трасса, воспроизведение) или `[hypothesis: <план верификации>]`. Hypothesis без плана запрещена.
5. **Architect / verify** — машиночитаемое значение: `required` (триггеры сработали, отчёта нет), `not-required` (точечный фикс) или `report: <путь>` (отчёт уже создан). Прозу «да/нет» не использовать.
6. **Блокирующая развилка не зарывается в поля.** Если решение заказчика блокирует постановку (выбор A/B меняет «Что менять» или «Приёмку») — блок НЕ печатается: сначала карточка решения, END TURN, блок собирается после ответа с вшитым решением. Неблокирующие вопросы — в «Открытые решения».
7. **Опциональные поля** (Тема маркера, Срезы, Открытые решения) выводятся только при непустом содержимом.

## Композиция финального сообщения (HALT-правила)

- Блок `## Постановка ЗНИ` **заменяет** синтез «Итог / Вердикт / Дальше» в финальном сообщении — не печатать оба.
- **Один** «Следующий шаг» на сообщение: `/opsx:new <name>`. Дублирующих строк «Дальше:» / «Следующий шаг:» с другими действиями быть не должно.
- Содержание не дублируется: то, что вошло в поля блока, не повторяется буллетами выше по сообщению.
