# Verification — S2.accept: профиль не найден по префиксам

## Симптом

GetData доходит до источника; SOAP-ответ:

`Профиль-источник не найден по префиксам "ПРИЕМ1" / ""`

## Корневая причина (verified)

1. **Данные:** на приёмнике в профиле заполнен только `Префикс` = `ПРИЕМ1`, `ПрефиксБазыПриемника` пустой → в WS уходит `prefReceive=""`.
2. **Форма (дефект S1.3):** в `Form.xml` группа `ГруппаПриемник` содержит только поле `Префикс1` (`Объект.Префикс`); поля `ПрефиксБазыПриемника` **нет** — пользователь не может заполнить второй префикс на форме приёмника.
3. **Валидация:** `ПередЗаписью` объекта проверяет пару префиксов только для `Источник = Истина`; для приёмника префиксы не обязательны → профиль сохраняется с пустым вторым полем.

## Что нужно на стенде

| ИБ | Профиль | Обязательно |
|----|---------|-------------|
| **Источник** | `Источник = Истина` | Та же пара `Префикс` + `ПрефиксБазыПриемника`, правила, статус **«Новое»** |
| **Приёмник** | `Источник = Ложь` | Та же пара префиксов, адрес WS, логин/пароль |

Пример пары: `Префикс` = `ИСТ1`, `ПрефиксБазыПриемника` = `ПРИЕМ1` (на обеих ИБ одинаково).

## Ручное исправление формы (Конфигуратор)

Справочник `рг_НастройкиОбменаXML` → форма `ФормаЭлемента` → группа **ГруппаПриемник** → добавить поле `ПрефиксБазыПриемника` (`Объект.ПрефиксБазыПриемника`) рядом с `Префикс1` → выгрузить в `src/`.

## Рекомендация по постановке

Добавить в `ObjectModule` проверку заполненности обоих префиксов и для профиля-приёмника (`/opsx:extend` или fix в apply).
