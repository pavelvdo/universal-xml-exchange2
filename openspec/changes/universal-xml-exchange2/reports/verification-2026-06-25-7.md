## Резюме

Предыдущая инструкция verify была неточной: у **операции** WS нет отдельной вкладки «Параметры» в палитре свойств — там только тип возвращаемого значения, «Возможно пустое значение», «Имя процедуры».

Параметры — **отдельные объекты метаданных** «Параметр», подчинённые операции в **дереве конфигурации** (см. platform Глава 17: «операции и параметры (типы XDTO), модуль»).

## Как добавить параметры (S2.3)

**Путь в дереве:** Общие → Веб-сервисы → `rgExchangeService` → **Операции** → `GetData` (развернуть ветку).

1. ПКМ на **`GetData`** → **Добавить** → **Параметр**.
2. Параметр 1 — палитра свойств:
   - **Имя:** `Префикс` (как в модуле)
   - **Тип значения:** `string` (`http://www.w3.org/2001/XMLSchema`)
   - **Направление передачи:** Входной
   - **Возможно пустое значение:** нет
3. ПКМ на **`GetData`** → **Добавить** → **Параметр** — второй:
   - **Имя:** `ПрефиксБазыПриемника`
   - остальное — как у `Префикс`
4. То же для операции **`ConfirmDataReceipt`** (два входных `string`).

**Альтернативный UI:** двойной щелчок по `rgExchangeService` → форма объекта → закладка **«Операции»** → выбрать операцию → задать параметры в составе операции (зависит от версии платформы).

**Модуль WS** уже содержит `GetData(Префикс, ПрефиксБазыПриемника)` — имена параметров метаданных **должны совпадать** с аргументами процедуры.

**Критерий готовности:** после выгрузки в `rgExchangeService.xml` под операциями появляются узлы `<Parameter>` (как в cf `WebServices/Exchange.xml`); или в опубликованном WSDL у операций есть аргументы `Префикс`, `ПрефиксБазыПриемника`.
