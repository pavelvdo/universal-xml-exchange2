## ADDED Requirements

### Requirement: SOAP-веб-сервис rgExchangeService

В расширении SHALL быть реализован SOAP-веб-сервис `rgExchangeService` для базы-источника.

#### Scenario: Публикация сервиса

- **WHEN** администратор публикует веб-сервис расширения на источнике
- **THEN** приёмник может обратиться к сервису по URL из профиля обмена

### Requirement: Операция GetData

Сервис SHALL предоставлять операцию `GetData` для получения пакета данных выгрузки.

#### Scenario: Успешный вызов GetData

- **WHEN** клиент вызывает `GetData` с именем макета правил и структурой параметров
- **THEN** сервис возвращает двоичные данные в формате Base64Binary, представляющие ZIP-архив с файлом XML-выгрузки

### Requirement: Входные параметры GetData

Метод `GetData` MUST принимать: имя макета правил обмена (строка); структуру параметров (XDTO-пакет `http://v8.1c.ru/8.1/data/core`, тип `Structure`) с фильтрами для выгрузки.

#### Scenario: Передача параметров фильтрации

- **WHEN** клиент передаёт в `GetData` имя макета и XDTO-структуру с парами ключ–значение
- **THEN** сервер использует эти параметры при инициализации правил обмена перед выгрузкой
