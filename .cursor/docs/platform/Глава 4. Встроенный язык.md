# Глава 4. Встроенный язык

> Agent ref: Встроенный язык 1С — типы данных, операторы, выражения, приемы работы, варианты запуска.
> Ключевые разделы: 4.1 Назначение и характеристика, 4.3 Типы данных, 4.6 Операторы, 4.7 Основные приемы работы, 4.8 Варианты запуска

## 4.1. Назначение и краткая характеристика встроенного языка

Встроенный язык предназначен для описания алгоритмов функционирования прикладной задачи в конфигураторе «1С:Предприятие». Язык предметно-ориентированный; операторы имеют русское и англоязычное написание (можно смешивать). Есть объектно-ориентированные возможности: доступ к свойствам и методам специализированных типов (документы, справочники и т.п.), задаваемых в метаданных. Типизация динамическая: тип переменной определяется значением; явное объявление через Перем необязательно (неявное — первое присваивание). Поддерживаются массивы, структуры, соответствия и другие коллекции.

## 4.2. Формат исходных текстов программных модулей

### 4.2.1. Что такое программный модуль

Программный модуль — текст на встроенном языке с процедурами и функциями, вызываемыми системой в определённые моменты. Не имеет формальных границ «начало/конец модуля». Размещается в точках конфигурации, требующих описания алгоритмов (например, обработчики кнопок). Все процедуры и функции модуля выполняются в едином контексте.

### 4.2.2. Контекст выполнения программного модуля

**4.2.2.1. Глобальный контекст**

Глобальный контекст виден всем модулям: свойства и методы глобального контекста, системные перечисления и наборы значений (КодВозвратаДиалога, Символы и т.д.).

**4.2.2.2. Локальный контекст**

Локальный контекст задаётся местом размещения модуля в конфигурации; виден только этому модулю — набор доступных объектов, свойств и методов.

### 4.2.3. Виды программных модулей

**4.2.3.1. Модуль управляемого приложения**

Выполняется при загрузке конфигурации при старте в режимах: тонкий клиент, веб-клиент, толстый клиент (управляемое приложение). Назначение: обработка начала/окончания сеанса. Недоступен на сервере. Необработанное исключение в обработчиках ПередНачаломРаботыСистемы(), ПриНачалеРаботыСистемы(), ПередЗавершениемРаботыСистемы(), ПриЗавершенииРаботыСистемы() приводит к аварийному завершению системы.

Пример без аварийного завершения:
```bsl
Процедура ПриНачалеРаботыСистемы()
    Попытка а=1/0;
    Исключение Сообщить("Деление на 0");
    КонецПопытки;
КонецПроцедуры
```

Пример с аварийным завершением:
```bsl
Процедура ПриНачалеРаботыСистемы()
    а=1/0;
КонецПроцедуры
```

Доступны из модуля: часть глобального контекста для управляемого приложения; экспортные процедуры/функции клиентских общих модулей; экспортные процедуры/функции серверных неглобальных общих модулей с свойством «Вызов сервера».

**4.2.3.2. Модуль внешнего соединения**

В корне конфигурации. Обработчики событий при старте/окончании работы в режиме внешнего (COM) соединения. Возможно объявление переменных, процедур и функций для внешнего приложения. Через COM доступны: экспортные процедуры/функции модуля внешнего соединения и общих модулей, глобальный контекст. Включение/исключение модулей — свойства общих модулей; фрагментов — инструкции препроцессора. Режим без пользовательского интерфейса.

**4.2.3.3. Модуль сеанса**

Выполняется при старте системы при загрузке конфигурации. Инициализация параметров сеанса; исполняется в привилегированном режиме сервера. Установка параметров — в обработчике УстановкаПараметровСеанса. Содержит только определения процедур и функций, без экспорта. УстановкаПараметровСеанса вызывается до ПередНачаломРаботыСистемы (или ПриНачалеРаботыСистемы для внешнего соединения). Тип сеанса (фоновое задание или иной) можно определить через ПолучитьТекущийСеансИнформационнойБазы() и ПолучитьФоновоеЗадание().

**4.2.3.4. Общие модули**

Отдельная ветка метаданных. Содержат общие алгоритмы, доступные из разных мест конфигурации. В любом общем модуле — объявление процедур и функций, доступных из других модулей. Включение/исключение целиком — свойства общих модулей; фрагментов — инструкции препроцессора.

**4.2.3.5. Модули прикладных объектов**

Имеют объекты: константы, справочники, документы, отчёты, обработки, планы видов характеристик, планы счетов, планы видов расчётов, планы обмена, бизнес-процессы, задачи, регистры. Модуль — свойство объекта; в нём объявление переменных, процедур и функций, доступных при работе с объектом. В контексте доступны реквизиты, табличные части, методы и события объекта.

**4.2.3.6. Модули менеджеров объектов**

Менеджер управляет объектом как объектом конфигурации (создание, формы, макеты). Модуль менеджера расширяет функциональность — процедуры и функции, относящиеся к типу объекта, а не к экземпляру. Контекст: глобальный контекст; экспортные процедуры/функции глобальных и неглобальных общих модулей (серверных); локальный контекст модуля. Переменных и тела модуля быть не может.

Пример: в модуле менеджера справочника Контрагенты — `Функция ПолучитьСписокДебиторов() Экспорт … КонецФункции`. Вызов: `Дебиторы = Справочники.Контрагенты.ПолучитьСписокДебиторов();`

**4.2.3.7. Модули форм**

У каждой формы свой модуль. Контекст: реквизиты формы, ФормаКлиентскогоПриложения, расширение формы по типу основного реквизита, глобальный контекст (в т.ч. неглобальные общие модули), экспорт модуля управляемого приложения. Нужна согласованность директив процедур (&НаКлиенте, &НаСервере и т.д.) и свойств общих модулей (Клиент, Сервер и т.д.).

**4.2.3.8. Модули команд**

Только процедуры и функции; переменных и тела модуля нет. Обработчик ОбработкаКоманды() должен быть с директивой &НаКлиенте. Контекст клиентских процедур: глобальный контекст и локальный контекст модуля команды; серверных — глобальный контекст, экспортные методы глобальных общих модулей (сервер), серверные методы модуля команды. Экспортные методы команды извне модуля недоступны (к командам и их контексту из языка доступа нет).

### 4.2.4. Формат программного модуля

**4.2.4.1. Комментарии**

Комментарий: от `//` до конца строки. Можно с начала строки или после оператора. После `//` на той же строке оператор писать нельзя.
```bsl
А=В; // Это - комментарий
```
Комментарий перед процедурой/функцией может использоваться контекстной подсказкой конфигуратора.

**4.2.4.2. Формат операторов**

Структура модуля: раздел определения переменных (до первого Процедура/Функция/исполняемый оператор) — только Перем; раздел процедур и функций; раздел основной программы (от первого исполняемого оператора вне процедур/функций до конца модуля), выполняется при инициализации модуля.

Операторы разделяются точкой с запятой; конец строки не признак конца оператора. Исполняемый оператор может иметь метку для Перейти. Формат: `~метка:Оператор[(параметры)] [КлючевоеСлово];` или `~метка:А=В;` Метка — идентификатор после тильды, затем двоеточие.

**4.2.4.3. Имена переменных, процедур и функций**

Последовательность букв, цифр и подчёркиваний, начинающаяся с буквы или подчёркивания. Не должны совпадать с зарезервированными словами и с именами свойств/процедур/функций в текущем контексте. Регистр не учитывается.

**4.2.4.4. Язык написания программных модулей**

Язык двуязычный: зарезервированные слова, имена типов, свойств, методов, событий имеют русское и английское имя; в тексте можно смешивать.

**4.2.4.5. Регистры букв**

Регистр при написании имён не имеет значения.

**4.2.4.6. Зарезервированные слова**

Не используются как имена переменных, реквизитов, процедур и функций. Пары (русское / английское): Если/If, Тогда/Then, ИначеЕсли/ElsIf, Иначе/Else, КонецЕсли/EndIf, Для/For, Каждого/Each, Из/In, По/To, Пока/While, Цикл/Do, КонецЦикла/EndDo, Процедура/Procedure, Функция/Function, КонецПроцедуры/EndProcedure, КонецФункции/EndFunction, Перем/Var, Перейти/Goto, Возврат/Return, Продолжить/Continue, Прервать/Break, И/And, Или/Or, Не/Not, Попытка/Try, Исключение/Except, ВызватьИсключение/Raise, КонецПопытки/EndTry, Новый/New, Выполнить/Execute.

**4.2.5. Специальные символы**

| Символ | Описание |
|--------|----------|
| // | Начало комментария до конца строки |
| \| | В строковой константе — продолжение предыдущей строки (перенос) |
| ~ | Начало метки оператора |
| : | Окончание метки |
| ; | Разделитель операторов |
| () | Параметры методов, процедур, функций; выражения |
| [] | Обращение к свойству по строке имени; индекс коллекции |
| , | Разделитель параметров |
| ' ' | Литералы даты |
| . | Десятичная точка; обращение к свойствам/методам |
| + - * / % | Арифметика; + для конкатенации строк |
| > >= < <= = <> | Сравнения |

## 4.3. Типы данных и переменные

### 4.3.1. Общая информация

Тип задаёт множество значений и операции над ними. Переменная — ссылка на значение; при динамической типизации может ссылаться на значения разных типов. Неприсвоенная переменная — Неопределено. В системе всё является объектом (значение = объект). Типы: мутабельные (изменяемые) и иммутабельные (неизменяемые). Примитивные иммутабельные: Неопределено, NULL, Число, Дата, Строка, Булево. Мутабельные (например, объект элемента справочника) допускают изменение состояния. Сравнение: иммутабельные — по значению; мутабельные — по ссылке (равны только сами себе). Ссылочный тип (СправочникСсылка, ПеречислениеСсылка и т.д.) хранит ключ для получения значения из БД (разыменование); иммутабельный.

### 4.3.2. NULL

Используется для отсутствующего значения при работе с БД (например, соединения таблиц). Литерал: NULL.

### 4.3.3. Булево

Значения: Истина (True), Ложь (False). В сравнении можно писать `Если МояПеременная Тогда` вместо `Если МояПеременная = Истина Тогда`.

### 4.3.4. Дата

Дата от 01.01.0001, время до секунды; в переменной — до 0.0001 с. В БД — до секунды. Не рекомендуется использовать дробные числа в арифметике с датами. Литерал: строка цифр в кавычках 'ГГГГММДДччммсс' (год, месяц, день обязательны; время и секунды можно опускать). Примеры: Дата('20170323104525'), Дата(2017, 03, 23, 10, 45, 25).

### 4.3.5. Неопределено

Одно значение — пустое, не принадлежащее другим типам (например, реквизиты с составным типом). Литерал: Неопределено (Undefined).

### 4.3.6. Строка

Строка произвольной длины, Unicode (UTF-16). Суррогатные пары — два вызова Символ() по code unit. Литералы: символы в кавычках; кавычка в строке — `""`. Многострочные: части в кавычках, на следующей строке `|`. Примеры: `"ООО ""Василек"""`, многострочная с `|`.

### 4.3.7. Тип

Идентификация типов (ТипЗнч(), Тип()). Литералов нет.

### 4.3.8. Число

Десятичное число; арифметика: +, -, *, /, %. **ВНИМАНИЕ:** максимальная разрядность — 32 знака. Литерал: цифры с опциональной точкой и знаком [+|-]. Пример: `А = 15; Б = -968.612;`

### 4.3.9. Конвертация значения в строку и обратно

Преобразование в строку: неявное при отображении/присвоении строке или явно через конструктор Строка(значение). Зависит от Формат() и региональных установок. Обратное преобразование (строка → Булево, Число, Дата): канонический вид (как литералы языка) или локализованный (по региональным настройкам). Преобразование «значение → строка → значение» при смене локализации может не восстанавливаться. Канонически: Булево — Истина/True, Ложь/False; Число — десятичное с точкой; Дата — как литерал. Рекомендации: для переносимости не использовать локализуемые представления при сериализации; применять XMLСтрока()/XMLЗначение(); не менять региональные настройки после сохранения текстовых значений.

## 4.4. Оператор присваивания

`<Назначение> = <Источник>;` Назначение — переменная или свойство с записью. Присвоение необъявленной переменной, совпадающей по имени со свойством контекста, может давать ошибку; рекомендуется явно объявлять Перем. Примеры: `А = В;`, `Стр1 = "777";`, `ДатаДокумента = '20020717';`

## 4.5. Выражения языка

Выражение — формула из операций, констант, переменных, функций. Арифметические операции: +, -, *, /, %, унарный минус. Для Дата: +Число (секунды), -Число, Дата-Дата (секунды). Результат с датой — точность до 0.0001 с; при записи в БД дробная часть отбрасывается; рекомендуется округлять. Конкатенация строк: `+`; нестроковые операнды приводятся к строке. Логические: сравнения (=, <>, <, <=, >, >=) и булевы И, Или, Не. Сравнение Число и Булево: Булево приводится к Числу (Истина→1, Ложь→0). Приоритет (по возрастанию): Или, И, Не, сравнения, +-, */%, унарные +-, . и вызовы. Скобки для управления порядком. При вычислении логического выражения вычисляются только нужные части (сокращённое вычисление).

## 4.6. Операторы и синтаксические конструкции

### 4.6.1. Объявление переменной

`Перем Имя1 [Экспорт] [, Имя2, …];` Экспорт — доступ при обращении к контексту модуля; для каждой переменной отдельно. Неявное объявление — первое появление в левой части присваивания; в выражениях недопустимы необъявленные переменные. Области: глобальные (модуль управляемого приложения, Экспорт); переменные модуля; локальные (в процедуре/функции).

### 4.6.2. Создание нового объекта

Вариант 1: `Новый ИмяТипа[(Парам1, ..., ПарамN)]`  
Вариант 2 (для прикладных типов): `Новый(Тип, Параметры)` — Тип из Тип("…") или значение типа Тип; Параметры — массив. Разыменование к результату Новый напрямую применять нельзя — сначала присвоить переменной. Примеры: `Новый Массив(3)`, `Новый(ТипЗнч("КвалификаторыСтроки"), Параметры)`.

### 4.6.3. Процедура

`[Асинх] Процедура Имя([[Знач] Пар1 [=ДефЗнач], ...]) [Экспорт] … [Возврат;] КонецПроцедуры`  
Асинх — асинхронная процедура (допустим Ждать); только на клиенте. Знач — передача по значению; иначе по ссылке; у асинхронных параметры всегда по значению. Параметры по умолчанию можно располагать в любом месте списка. Ключевые слова Процедура/КонецПроцедуры — операторные скобки, точку с запятой после них не ставить.

### 4.6.4. Функция

`[Асинх] Функция Имя(...) [Экспорт] … Возврат Значение; КонецФункции`  
Завершение — Возврат; без Возврат возвращается Неопределено. Асинхронная функция возвращает Обещание. Остальное аналогично процедуре.

### 4.6.5. Условные операторы

Если: `Если выражение Тогда … [ИначеЕсли выражение Тогда …] [Иначе …] КонецЕсли`  
Вычислить по условию: `?(условие, выражение1, выражение2)` — если Истина, вычисляется выражение1, иначе выражение2.

### 4.6.6. Циклы

**Для:** `Для имя = выражение1 По выражение2 Цикл … [Прервать;] [Продолжить;] КонецЦикла` — счётчик от выражение1 до выражение2 с шагом 1.  
**Для каждого:** `Для каждого имя Из коллекция Цикл … КонецЦикла` — обход коллекции.  
**Пока:** `Пока выражение Цикл … КонецЦикла`  
Прервать — выход из цикла; Продолжить — переход к следующей итерации.

### 4.6.7. Безусловный переход

`Перейти ~метка;` Область действия — модуль/процедура/функция. ПРИМЕЧАНИЕ: метка не должна быть на Процедура/Функция; нельзя переходить внутрь Пока/Для/Если/Попытка извне.

### 4.6.8. Выполнение фрагмента кода

`Выполнить(Строка)` — выполняет код из строки. **ВНИМАНИЕ:** не рекомендуется строить на этом существенную функциональность. ПРИМЕЧАНИЕ: код не может содержать объявления процедур/функций и явное Перем.

### 4.6.9. Ожидание завершения асинхронной функции

`[Присваивание = ] Ждать Обещание;` Приостанавливает метод, возвращает управление вызывающему; после завершения Обещания выполнение возобновляется, восстанавливаются локальные переменные. Возвращает результат или генерирует исключение.

### 4.6.10. Исключения

**ВызватьИсключение:** вариант 1 — `ВызватьИсключение выражение` (описание — строка). Вариант 2 — `ВызватьИсключение(Описание, Категория, Код, ДополнительнаяПричина, Причина)`. Категория — из КатегорияОшибки (некоторые значения недопустимы). Инструменты настройки сообщений и отчётов об ошибках описаны в документации платформы.

**Попытка:** `Попытка … Исключение … [ВызватьИсключение;] КонецПопытки` При ошибке в блоке попытки управление — в блок Исключение; вложенные Попытка поддерживаются. Не перехватываются: ошибки компиляции при выполнении; ошибки в ПередЗакрытием() и ПриЗакрытии() управляемой формы. ВызватьИсключение в блоке Исключение передаёт управление внешнему обработчику. Информация об ошибке: ОписаниеОшибки(), ИнформацияОбОшибке().

### 4.6.11. Обработчики событий

**ДобавитьОбработчик Событие, Обработчик;** Событие — выражение.ИмяСобытия; обработчик — метод объекта или процедура/функция в области видимости. Для COM-событий с разным числом параметров — несколько обработчиков с разными именами и сигнатурами.  
**УдалитьОбработчик Событие, Обработчик;** — удаление с проверкой соответствия числа параметров.

## 4.7. Основные приемы работы

### 4.7.1. Обращение к свойствам объектов

Помимо точки: `<Объект>[ИмяСвойства]` — обращение по строке с именем свойства (например, `Спр["Наименование"]`).

### 4.7.2. Дополнение контекста объектов и форм

К объектам и формам извне можно обращаться к экспортным переменным, процедурам и функциям; для форм — также к реквизитам.

### 4.7.3. Сериализация/десериализация

Виды: обычная, в XML, XDTO (XML/JSON). При клиент-серверном вызове: сериализация параметров → передача → десериализация на сервере → вызов → сериализация результата → десериализация на клиенте. Циклические ссылки в параметрах делают вызов невозможным. Если один и тот же объект присвоен нескольким свойствам сериализуемой структуры, после десериализации каждое свойство ссылается на свою копию (объект «размножается»).

### 4.7.4. Передача параметров процедур и функций

По ссылке — изменение формального параметра меняет фактический; по значению (Знач) — копия, но для агрегатных объектов можно менять содержимое (например, Очистить() массив), нельзя заменить сам параметр. Без передачи клиент–сервер: по умолчанию по ссылке; с Знач — по значению. С передачей клиент–сервер: всегда передаются копии; при возврате для параметров без Знач — копии обратно (если несколько формальных параметров получили один фактический — после возврата будет значение «самого правого» изменённого); при исключении на сервере значения параметров на клиенте неопределены. Пропуск параметра — запятая без значения; опущение — только параметры с значением по умолчанию справа. Строки при передаче на сервер не должны содержать символы, недопустимые в XML 1.0; проверка на сервере — НайтиНедопустимыеСимволыXML(). Доступность типа на клиенте/сервере и поддержка сериализации указаны в синтакс-помощнике (раздел Доступность).

### 4.7.5. Работа с коллекциями значений

Коллекции имеют Количество(), Индекс(), Добавить(), Удалить(), обход Для каждого, обращение [индекс] (индекс с 0). При обходе удаление/изменение состава — поведение не определено.

### 4.7.6. Использование номеров и индексов

Номер — с 1 (строка, табличный документ и т.п.). Индекс коллекции — с 0.

### 4.7.7. Работа с системными перечислениями

Доступ как свойство глобального контекста по имени перечисления; значения — ИмяПеречисления.ЗначениеПеречисления.

### 4.7.8. Работа с предопределёнными значениями

**Через менеджер (сервер):** ТипПредопределенногоЗначения.ИмяОбъектаМетаданных.Значение (Справочники, ПланыВидовХарактерик, ПланыСчетов, ПланыВидовРасчета, Перечисления; для БП — ТочкиМаршрута.ИмяТочки).  
**ПредопределенноеЗначение(строка)** — на клиенте, где менеджеры недоступны. Синтаксис строки совпадает с оператором Значение языка запросов (Справочник, ПланВидовХарактерик и т.д., единственное число; ТочкаМаршрута.ИмяТочки; ПустаяСсылка для пустой ссылки).

### 4.7.9. Синхронные и асинхронные методы

В сеансе код выполняется в одном потоке; по умолчанию вызовы синхронные. Асинхронные методы возвращают управление до завершения; реализация: обратный вызов (Начать*/Показать* + ОписаниеОповещения) или обещания (функции с суффиксом Асинх, возвращают Обещание; ожидание — Ждать). Оператор Ждать только в процедурах/функциях с модификатором Асинх. Асинхронные параметры всегда по значению. Асинхронная функция всегда возвращает Обещание; фактический результат — через Ждать. Исключения из асинхронной функции перехватываются в месте Ждать (Попытка … Ждать Обещание Исключение …). Обещание нельзя передать на сервер. В веб-клиенте синхронная техника не поддерживается; для файлов и криптографии нужны расширения браузера (в т.ч. «Расширение для работы с 1С:Предприятием»).

## 4.7.10. Преобразование значений при изменении типа

При присвоении реквизиту другого типа применяются правила преобразования (булево в дату — пустая дата; булево в строку — представление; булево в число — 0/1; строка в булево/дату/число — по каноническому или локализованному виду; число в дату — пустая дата; число в булево — 0→Ложь, 1→Истина; сокращение перечисления — пустая ссылка; уменьшение длины строки — усечение; уменьшение дробной части — округление; отрицательные числа при запрете — 0 и т.д.).

## 4.8. Особенности различных вариантов запуска системы

Система используется в файловом и клиент-серверном вариантах, внешнем соединении, интернет-сервисах.

### 4.8.1. Исполнение процедур и функций

**4.8.1.1. Различие инструкций препроцессора и директив компиляции**

Инструкции препроцессора (#Если и т.д.) действуют на исходный текст (удаляют фрагменты для данного контекста). Директивы компиляции (&НаКлиенте, &НаСервере и т.д.) действуют на структурные единицы (методы, переменные) — в каком контексте компилируется. Модуль формы существует в нескольких экземплярах (клиент/сервер, контекстный/внеконтекстный); препроцессор вырезает текст, затем компиляция по директивам. Пример: #Если НаСервере Тогда &НаКлиенте Процедура … — на сервере текст есть, но процедура не компилируется (директива); на клиенте текст вырезан препроцессором. Методы с &НаКлиентеНаСервереБезКонтекста и &НаКлиентеНаСервере попадают в несколько экземпляров модуля.

**4.8.1.2. Инструкции препроцессора**

Синтаксис: `#Если выражение Тогда` … `#ИначеЕсли` … `#Иначе` … `#КонецЕсли`; также #Область/#КонецОбласти, #Вставка/#КонецВставки, #Удаление/#КонецУдаления (в расширениях с ИзменениеИКонтроль). Символы препроцессора: Сервер, НаСервере, Клиент, НаКлиенте, ТонкийКлиент, МобильныйКлиент, ВебКлиент, ВнешнееСоединение, ТолстыйКлиентУправляемоеПриложение, ТолстыйКлиентОбычноеПриложение, МобильноеПриложениеКлиент, МобильноеПриложениеСервер, МобильныйАвтономныйСервер. Набор доступных символов зависит от варианта (файловый/клиент-серверный) и роли (клиент/сервер). Перед передачей на тонкий/веб-клиент сервер обрабатывает препроцессор (неисполняемый на клиенте текст заменяется пробелами, переносы сохраняются).

**4.8.1.3. Директивы компиляции**

`&НаКлиенте`, `&НаСервере`, `&НаСервереБезКонтекста`, `&НаКлиентеНаСервереБезКонтекста`, `&НаКлиентеНаСервере` — перед описанием переменной, процедуры или функции. НаКлиенте — выполнение на клиенте в контексте формы; переменная — время жизни формы. НаСервере — на сервере в контексте формы; переменная — на время вызова. НаСервереБезКонтекста — на сервере без контекста формы; вызовы только внеконтекстных и общих серверных. НаКлиентеНаСервереБезКонтекста — на клиенте и сервере без контекста; переменные с этой директивой не допускаются; вызов как метод ЭтотОбъект не поддерживается. НаКлиентеНаСервере — на клиенте и сервере. Таблица доступности директив по типам модулей приведена в документации платформы.

### 4.8.2. Аннотации

Для механизма расширения конфигурации: &Перед, &После, &Вместо, &ИзменениеИКонтроль (с параметрами). Пользовательские аннотации не поддерживаются.

### 4.8.3. Особенности использования объектов, свойств и методов

В синтакс-помощнике для каждого объекта указана доступность по режимам (тонкий/веб/мобильный клиент, толстый клиент, сервер, внешнее соединение и т.д.). **ВНИМАНИЕ:** если объект недоступен в режиме, его свойства и методы там тоже недоступны. Если объект недоступен на клиенте, тип этого объекта там также недоступен. Указываются также: сериализуемость, обмен с сервером, сериализация в/из XDTO, кеширование результатов, ограничения в обработчиках (например, запрет серверных вызовов в определённых событиях). Диагностика нарушений: ключ командной строки /EnableCheckServerCalls или флажок конфигуратора «Проверять серверные вызовы в обработчиках событий».
