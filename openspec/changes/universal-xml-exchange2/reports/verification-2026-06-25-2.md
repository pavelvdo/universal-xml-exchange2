## Резюме для разработчика

**universal-xml-exchange2 — apply по срезу S1 пока нельзя:** в `Form/Module.bsl` реквизит объекта `ПравилаОбменаДанными` (тип `ХранилищеЗначения`) читается/пишется как `Объект.ПравилаОбменаДанными`. На управляемой форме такие реквизиты **не попадают** в данные `Объект` — отсюда «Поле объекта не обнаружено».

**Verified root cause** (заказчик): не «не обновлена БД», а **ошибка паттерна формы**.

**Эталон в том же модуле:** `Расписание` — реквизит **формы** (Form.xml id=4), чтение через `ЗначениеРеквизитаОбъекта`, запись в `ПередЗаписьюНаСервере`: `ТекущийОбъект.Расписание = Новый ХранилищеЗначения(Расписание)`.

**S1.3 tasks.md** уже требовал «нехранимый реквизит формы» для загрузки правил — в выгрузке Form.xml реквизита формы для правил **нет** (только индикатор `ПравилаЗагружены` без DataPath).

## Исправление (S1 inside-slice rework)

1. **Конфигуратор:** реквизит формы `ПравилаОбменаДанными`, тип `ХранилищеЗначения` (аналог `Расписание`), не выводить на форму или только индикатор.
2. **BSL:** загрузка XML → реквизит формы; `ПрочитатьПравилаОбмена()` при открытии через `ЗначениеРеквизитаОбъекта`; `ПередЗаписьюНаСервере` → `ТекущийОбъект.ПравилаОбменаДанными = ПравилаОбменаДанными`.
3. Удалить обращения к `Объект.ПравилаОбменаДанными` и workaround `ЕстьРеквизитПравилаОбменаДанными`.

ObjectModule.bsl — без изменений (там `ПравилаОбменаДанными` на объекте корректен).
