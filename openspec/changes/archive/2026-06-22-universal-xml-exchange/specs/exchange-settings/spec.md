## ADDED Requirements

### Requirement: Профиль обмена хранит параметры подключения

Система SHALL хранить профили обмена в справочнике `ргНастройкиОбменаXML`. Для каждого профиля MUST сохраняться: адрес публикации веб-сервиса (URL), имя пользователя для подключения к базе-источнику, строковый идентификатор макета правил обмена на источнике.

#### Scenario: Создание профиля с обязательными реквизитами

- **WHEN** пользователь создаёт элемент справочника и заполняет URL, логин и имя макета правил
- **THEN** система сохраняет профиль и позволяет использовать его для сеанса обмена

### Requirement: Пароль хранится в безопасном хранилище БСП

Система MUST NOT хранить пароль подключения к источнику в открытом виде в реквизитах справочника. Пароль SHALL записываться и читаться через механизм безопасного хранилища паролей БСП.

#### Scenario: Запись пароля при сохранении профиля

- **WHEN** пользователь указывает пароль на форме элемента и записывает элемент
- **THEN** пароль сохраняется в безопасном хранилище, связанном с элементом справочника, а не в открытом реквизите

#### Scenario: Использование пароля при обмене

- **WHEN** система инициирует сеанс обмена по профилю
- **THEN** пароль извлекается из безопасного хранилища для аутентификации при вызове веб-сервиса

### Requirement: Табличная часть произвольных параметров

Справочник SHALL содержать табличную часть параметров обмена в формате «Имя» — «Значение». Допустимые типы значений: строка, число, дата, булево.

#### Scenario: Настройка параметров фильтрации

- **WHEN** пользователь добавляет строки в табличную часть параметров с именем и значением примитивного типа
- **THEN** система сохраняет параметры и передаёт их в сеанс обмена при вызове веб-сервиса

### Requirement: Управление регламентным обменом по профилю

Система SHALL поддерживать привязку регламентного задания обмена к элементу профиля: создание, обновление расписания и удаление при отключении или удалении профиля.

#### Scenario: Включение регламентного обмена

- **WHEN** пользователь включает регламентный обмен на профиле и задаёт расписание
- **THEN** система создаёт или обновляет регламентное задание `ргВыполнениеОбменаXML` для выполнения обмена по этому профилю

#### Scenario: Отключение регламентного обмена

- **WHEN** пользователь отключает регламентный обмен или удаляет профиль
- **THEN** система удаляет или деактивирует связанное регламентное задание
