# Test-case: explore-question → решение «заводим ЗНИ» → блок из отчётов → /opsx:new без трения

Регрессионный сценарий перехода: исследование стартовало как вопрос («как устроено X, надо ли чинить»), по ходу диалога принято решение заводить ЗНИ. Проверяет сборку блока `## Постановка ЗНИ` из отчётов `Task` (не из слотов брифа) и вход `/opsx:new` без лишних вопросов.

## Вход

- **Профиль:** `explore-question`, в финале — переключение на постановку ЗНИ.
- **Вопрос пользователя:** «почему статус операции в журнале опроса не обновляется после повторной отправки — это баг или так задумано?»
- По ходу: один `Task` к `onec-code-explorer`, отчёт в `temp/reports/exploration-YYYY-MM-DD-<slug>.md`; вердикт «нужна правка»; пользователь: «да, заводим ЗНИ».

## Контрольные точки

| # | Шаг | Ожидание | Итог |
|---|-----|----------|------|
| 1 | Финал после «заводим ЗНИ» | Блок `## Постановка ЗНИ` собран по `../templates/handoff-block.md`: поля — выжимки из отчёта explorer (якоря `модуль:строка` в причине), **не** копия слота «Вопрос»; Симптом/Причина/Что менять — мини-абзацы 2–3 предложения | PASS |
| 2 | Композиция финального сообщения | Блок **заменяет** синтез «Итог / Вердикт / Дальше»; в сообщении ровно **один** «Следующий шаг: `/opsx:new <name>`»; содержание блока не продублировано буллетами выше | PASS |
| 3 | Открытая развилка | Блокирующих развилок в полях нет; неблокирующий вопрос (если есть) — в поле «Открытые решения» | PASS |
| 4 | `/opsx:new` без аргумента | B0: одна информирующая строка «Создаю ЗНИ `<slug>` по постановке из чата.» — **без** вопроса про имя, без END TURN; slug derive из темы | PASS |
| 5 | Metadata Gate (основной) | Тема маркера + `defaultDeveloper` → `AskQuestion` про **описание** (`comment_suffix`); контекст ФИО из project; preview `// +++ ФИО дата …`; Other = только описание; «Без описания — только ФИО и дата» вместо Mechanical; никаких других решений в этом вопросе | PASS |
| 5b | Metadata Gate (нет ФИО в project) | Сначала текстовый вопрос только ФИО, затем gate описания | PASS |
| 5c | Metadata Gate (Other с transport-строкой) | Other = `// +++ Иванов …` → soft-reject «укажите только описание» | PASS |
| 6 | Перенос полей | «Открытые решения» из блока попали в `design.md` § Открытые вопросы; «Architect / verify: required» учтён Design Gate | PASS |

## Прогон

1. Запустить `/opsx:explore` с вопросом из входа; после отчёта explorer ответить «да, заводим ЗНИ».
2. Пройти контрольные точки 1–3 (финал explore), затем вызвать `/opsx:new` без аргумента — точки 4–6.
3. При FAIL — править `templates/handoff-block.md` (композиция), `openspec-new-change/SKILL.md` (шаги 1b.0, 1.5), **не ослабляя** HALT.
