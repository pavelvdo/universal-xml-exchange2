# Test-case: extend B1 — уточнение scope (signing-contract)

Регрессия адаптивного брифа для `/opsx:extend` user-path.

## Вход

```
/opsx:extend signing-contract-sign-verify-bytes только для облачных УНЭП
```

Предусловие: change `signing-contract-sign-verify-bytes` существует; `Drift-check: pass` (уточнение в рамках Why, без расширения на локальный УНЭП).

## Контрольные точки

| # | Ожидание | FAIL если |
|---|----------|-----------|
| 1 | Бриф **B1** карточка, заголовок `Бриф: signing-contract-sign-verify-bytes` | `Бриф для исследования`, `Как буду искать`, `Что понял`, нумерованный план артефактов |
| 2 | Слот **Изменение** + строка **Подтвердить?** | Сплошной абзац без слотов; **План** с proposal/design/specs |
| 3 | `Drift-check` и «Соответствие scope» **не в чате** | `Drift-check:` в сообщении пользователю |
| 4 | END TURN на «Подтвердить?» (текстом) | `AskQuestion` для утверждения |
| 5 | После «да» — правки артефактов; план в `debug.md` § Extend | План артефактов был в чат-брифе |

## Эталон B1

```markdown
Бриф: signing-contract-sign-verify-bytes

**Изменение** — исключение полей подписанта из ключевых полей ограничивается только облачными УНЭП, без отката правок S1–S3. Условие зафиксирую явно в требованиях и задачах срезов.

**Затронутое** — поля подписания и Kontur CryptoAPI.

Подтвердить? — «да» или уточните критерий «облачный УНЭП» в вашем контуре.
```
