# Глава 11. Бухгалтерский учет

> Agent ref: Планы счетов, регистры бухгалтерии, субконто, виды учета, проводки, консолидированный учет.
> Ключевые разделы: 11.2 Планы счетов, 11.3 Аналитический учет, 11.5 Записи регистров бухгалтерии, 11.6 Консолидированный учет, 11.8 Регистры бухгалтерии

## 11.1. Обзор

Ведение бухгалтерского учета обеспечивают объекты конфигурации **Планы счетов** и **Регистры бухгалтерии**.

- Несколько планов счетов; для каждого — произвольная иерархия субсчетов большой вложенности. Итоги хранятся раздельно по каждому плану счетов.
- Нумерация счетов: цифры и буквы (гибкая).
- Аналитический учет — для любого счета/субсчета, различные разрезы; объекты аналитики — элементы справочников, документы или произвольные числа, даты, строки.
- Неограниченное число видов учета на план счетов (например, количественный и валютный). Включение вида учета задается при редактировании плана счетов по каждому счету.
- Консолидированный учет: в одной базе — несколько предприятий (структурных подразделений); итоги — по предприятию или по всем.

---

## 11.2. Планы счетов

**План счетов** — совокупность синтетических счетов для группировки информации о хозяйственной деятельности.

- **Несколько планов счетов** — учет по всем одновременно; число планов не ограничено (например, разные стандарты учета).
- **Субсчета** — многоуровневая иерархия «счет – субсчета»; неограниченное число счетов первого уровня и субсчетов любого уровня. Структура кода задается шаблоном (последовательность символов); технически не задает иерархию, но структуру кодов рекомендуется соблюдать.
- **Разделенное редактирование:** в конфигураторе задаются основные характеристики (длина кода и наименования, макс. количество субконто и др.), вводятся предопределенные счета и субсчета, настройка видов учета. Пользователь может добавлять свои счета и субсчета, но не удалять предопределенные.
- **Хранение итогов** — в регистрах бухгалтерии по структуре плана счетов. В языке есть методы получения остатков и оборотов по счетам по любому виду учета, в целом по счету/субсчету или с разбивкой по объектам аналитики.

---

## 11.3. Аналитический учет

Организуется механизмом **субконто**.

- **Субконто** — объект аналитического учета. **Вид субконто** — множество однотипных объектов (например, вид «Организации», элемент — конкретная организация). В конфигураторе создается любое количество видов субконто. Для описания свойств объектов аналитики используется тип **План видов характеристик**.
- **Рекомендация:** для плана видов характеристик, используемого как виды субконто плана счетов, не использовать примитивные типы (влияет на производительность при записи движений регистра бухгалтерии). Только ссылочные типы.
- К счету или субсчету «прикрепляется» один или несколько видов субконто (число ограничено свойством «Макс. количество субконто»). При записях регистра бухгалтерии для корреспондирующих счетов с аналитикой указываются объекты аналитического учета.
- Один справочник может быть назначен разными видами субконто (например, Организации — вид «Поставщики» и вид «Покупатели»).

**Субконто и субсчета.** Субсчета — иерархия, подчиненная синтетическому счету; учет по каждому субсчету отдельно; объединение данных по одному объекту с разных счетов — вручную. Субконто: один список (например, Организации) прикрепляется ко всем нужным счетам; объединение информации по одной организации с разных счетов выполняется платформой автоматически.

**Многоуровневая аналитика** — итоги с разной степенью детализации. При субконто — через назначение видами субконто **План видов характеристик** (тип значения — иерархический справочник или составной тип). При субсчетах — за счет уровней субсчетов.

---

## 11.4. Виды учета

Виды учета (например, количественный, валютный) задаются подчиненными объектами типа **Признак учета объекта План счетов**. Для каждого счета или субсчета при создании указывается признак использования каждого вида учета плана счетов.

---

## 11.5. Записи и наборы записей регистров бухгалтерии

- **Хозяйственная операция** — действие, меняющее состояние средств. Ввод — через документы; учетные данные формируются из документа и фиксируются в регистрах бухгалтерии (порядок задается в конфигураторе на встроенном языке). Операция связана с документом: при проведении документа данные переформируются; при удалении документа удаляются и данные операции.
- **Проводка** = запись регистра бухгалтерии; **набор записей** = операция. При записи можно заполнять реквизиты из документа-регистратора.
- Записи одного документа образуют группу по регистратору и обрабатываются вместе. Структура записи строится динамически (счета, суммы, субконто, количество, валюта и т.д.); в конфигураторе можно добавить дополнительные реквизиты.
- **Активность записей:** «отключенные» записи не участвуют в остатках и оборотах. Используется, например, для плановых операций — включить активность при фактическом совершении.

---

## 11.6. Ведение консолидированного учета

Для организаций со структурными подразделениями (в т.ч. на отдельном балансе) требуется учет всех подразделений в одной базе и консолидированная отчетность. В измерение регистра бухгалтерии добавляется ссылка на структурное подразделение (предприятие). По этому измерению итоги хранятся раздельно; платформа разделяет их автоматически. Средствами языка можно получать итоги по конкретному предприятию или по всем.

## 11.7. Создание плана счетов

### 11.7.1. Свойства планов счетов

Объект конфигурации **План счетов**. Объекты данных — бухгалтерские счета. Редактирование — в окне редактирования и палитре свойств.

- **Маска кода** — структура кода счетов и субсчетов. Символы маски: `!` — в верхний регистр; `9` — цифра; `#` — цифра, «-», «+», пробел; `N` — буквы или цифры; `U` — то же с преобразованием в верхний регистр; `X` — произвольный символ; `@` — буквы/цифры в верхнем регистре или пробел; `^` — только из кода, не с ввода; `h` — шестнадцатеричные. Литерал специального символа — `\` перед ним. Если в маске есть точки и нет запятых, точки при вводе заменяются запятыми.
- **Автопорядок по коду** — при установке упорядочивание по полю «Порядок», а не по коду.
- **Длина порядка** — если > 0, поле «Порядок» используется для упорядочивания по умолчанию. Макс. длина реквизитов Длина кода, Длина наименования, Длина порядка — 628.
- **Реквизиты** — описания счета (например, наименование).
- **Признаки учета** — список видов учета (валютный, количественный и др.).
- **Табличные части** — для стандартных планов не используются; могут использоваться для специальных сценариев (например, связь с другими планами счетов).
- **Виды субконто** — объект типа План видов характеристик.
- **Признаки учета субконто** — список признаков; при настройке аналитики по счетам задается использование вида субконто при формировании предопределенных счетов.

**Данные по счету (предопределенные):** вид счета (Активный, Пассивный, Активный/Пассивный); флажок **Забалансовый** (итоги не входят в баланс); поле **Порядок** (длина из плана счетов; при Автопорядок по коду — упорядочивание по «Порядок»); метод `ПолучитьПорядокКода()` — строка порядка по маске кода. По каждому признаку учета — флажок «Учитывать». По каждому виду субконто — флажок ведения аналитики; если по счету вид учета не ведется, признак учета по субконто должен быть установлен, иначе — по потребности.

Пользователь может добавлять счета и субсчета, редактировать свойства; пометка удаления и удаление (в т.ч. предопределенных) регулируются правами доступа.

### 11.7.2. Организация различных видов учета

Для каждого плана счетов задается произвольное число видов учета; для каждого счета/субсчета при создании — признак использования каждого вида учета.

### 11.7.3. Формирование списка счетов (предопределенные счета)

Бухгалтерские счета создаются в конфигураторе. В свойстве плана счетов **Предопределенные** → Открыть — окно списка счетов. Управление — меню Действия; данные по счету — в окне редактирования счета.

## 11.8. Регистры бухгалтерии

По виду похожи на регистр накопления: измерения, ресурсы, реквизиты; при необходимости — экранные и печатные формы движений. Отличия: связь с планом счетов и поддержка двойной записи. В каждой записи — обязательные реквизиты **Счет Дт** и **Счет Кт** (при поддержке корреспонденции) или **Счет** (без корреспонденции).

### 11.8.1. Свойства регистра бухгалтерии

- **Разрешить разделение итогов** — механизм разделителя итогов для большей параллельности при записи.
- В качестве типа измерения не допускаются: УникальныйИдентификатор, ДвоичныеДанные, строка неограниченной длины.
- **Длина уточнения периода** — разбиение одной секунды на интервалы: 0 — уточнение отключено; 1 — 10 интервалов; 2 — 100; 3 — 1000. Движения в уточнении не входят в конечные остатки периода, но входят в начальные остатки следующего; обороты периодов не искажаются. В оборотах движения уточнения видны, если период анализа захватывает момент этих движений. Уточнение периода логически между периодом и регистратором.

### 11.8.2. Записи регистра бухгалтерии

Чтобы документ изменил бухгалтерские итоги, он должен формировать записи регистра бухгалтерии. Записи группы (по регистратору) обрабатываются вместе; порядок генерации — на встроенном языке. Структура записи динамическая (счета, суммы, субконто, количество, валюта и т.д.); возможны дополнительные реквизиты. Свойство **Активность** — отключенные записи не участвуют в остатках и оборотах. Поведение при работе с формами (автооткрытие списков субконто и т.п.) настраивается свойствами записей регистра бухгалтерии.
