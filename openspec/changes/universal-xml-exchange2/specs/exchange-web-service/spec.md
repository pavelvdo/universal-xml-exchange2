## ADDED Requirements

### Requirement: Операция ConfirmDataReceipt

Сервис SHALL предоставлять операцию `ConfirmDataReceipt` для подтверждения успешной загрузки данных на базе-приёмнике.

#### Scenario: Успешное подтверждение

- **WHEN** клиент вызывает `ConfirmDataReceipt` с префиксами, совпадающими с профилем-источником в статусе «Выполняется»
- **THEN** сервис переводит статус профиля в «Выполнено» и возвращает успешный результат

#### Scenario: Подтверждение при неверном статусе

- **WHEN** клиент вызывает `ConfirmDataReceipt`, а профиль не в статусе «Выполняется» и не в «Выполнено» с теми же префиксами
- **THEN** сервис возвращает ошибку и не меняет статус на «Выполнено»

#### Scenario: Повторное подтверждение после успешного завершения

- **WHEN** клиент повторно вызывает `ConfirmDataReceipt` для профиля уже в статусе «Выполнено» с совпадающими префиксами
- **THEN** сервис возвращает успешный результат без изменения статуса (идемпотентный no-op)

### Requirement: Идентификация профиля по префиксам

Операции `GetData` и `ConfirmDataReceipt` MUST принимать пару строк: `Префикс` и `ПрефиксБазыПриемника` для поиска единственного профиля-источника с `Источник = Истина`.

#### Scenario: Поиск профиля по префиксам

- **WHEN** клиент передаёт корректную пару префиксов, существующую на источнике
- **THEN** сервер находит соответствующий профиль обмена

## MODIFIED Requirements

### Requirement: Операция GetData

Сервис SHALL предоставлять операцию `GetData` для получения пакета данных выгрузки. Операция MUST выполняться только если профиль-источник находится в статусе «Новое». При начале обработки система SHALL атомарно перевести статус в «Выполняется» до формирования ZIP.

#### Scenario: Успешный вызов GetData из статуса Новое

- **WHEN** клиент вызывает `GetData` с префиксами профиля в статусе «Новое»
- **THEN** сервис переводит профиль в «Выполняется», формирует ZIP с правилами и данными и возвращает Base64Binary

#### Scenario: Отклонение GetData при другом статусе

- **WHEN** клиент вызывает `GetData`, а профиль не в статусе «Новое»
- **THEN** сервис не выполняет выгрузку и возвращает ошибку

### Requirement: Входные параметры GetData

Метод `GetData` MUST принимать: `Префикс` (строка) и `ПрефиксБазыПриемника` (строка) для идентификации профиля-источника. Параметры фильтрации SHALL браться из табличной части профиля-источника, а не из тела запроса клиента.

#### Scenario: Использование параметров профиля

- **WHEN** сервер обрабатывает `GetData` для профиля с заполненной табличной частью параметров
- **THEN** значения параметров подставляются в правила обмена перед выгрузкой
