# 10.1. Общие сведения о компоновке данных

> Agent ref: Система компоновки данных: схема компоновки, наборы данных, язык выражений СКД.
> Ключевые разделы: 10.1 Общие сведения, 10.3 Схема компоновки, 10.3.4 Язык выражений
> ---

Система компоновки данных (СКД) предназначена для создания отчётов «1С:Предприятия» по декларативному описанию.

**Возможности:** создание отчёта без программирования; варианты отчёта и пользовательских настроек; автогенерация форм просмотра и настройки; разбиение исполнения на этапы; исполнение этапов на разных компьютерах; программное влияние на выполнение; настройка структуры; совмещение нескольких таблиц; вложенные отчёты.

**Использование:** динамические списки; подготовка данных для обработок.

**Основные элементы СКД:** схема компоновки, наборы данных, настройки, макеты.

- **Схема компоновки данных** — откуда получать данные и как управлять компоновкой. Содержит: текст запроса с инструкциями СКД; наборы данных; доступные поля; связи между наборами; параметры; макеты полей и группировок.
- **Настройки компоновки данных** — отбор, упорядочивание, условное оформление, структура отчёта.
- **Макет компоновки данных** — результат применения настроек к схеме; задание процессору на формирование отчёта.
- **Элемент результата компоновки данных** — результат представляется набором таких элементов; их можно вывести в табличный документ и др.

**Этапы процесса:** наборы данных → связи → отбор → порядок → группировка → компоновка макета.

**Создание схемы:** визуально (конструктор СКД или редактор XML); программно (объекты встроенного языка).

**Подготовка к исполнению** — формирование макета компоновки (запросы, макеты областей). **Исполнение** — получение, агрегация, оформление данных. **Вывод** — в документ в различных форматах.

## 10.3. Схема компоновки данных

**Процессор компоновки данных** — исполнение компоновки. **Процессор вывода в табличный документ** — вывод элементов результата в табличный документ.

Отчёт СКД — иерархическая структура: группировка, таблица, диаграмма, вложенный отчёт.

Загрузка схемы из XML — стандартные средства встроенного языка (например, СериализаторXDTO.ПрочитатьXML). Все выражения в схеме — на **языке выражений СКД**.

### 10.3.1. Составные части схемы компоновки данных

#### 10.3.1.1. Источники данных

Источник — откуда получаются данные (информационная база). Коллекция: свойство **ИсточникиДанных** схемы, элементы ИсточникДанныхСхемыКомпоновкиДанных.

#### 10.3.1.2. Наборы данных

Описание полей набора, полей для отбора и т.п. Свойство **НаборыДанных** схемы. Типы элементов:

- **Набор данных — запрос** (НаборДанныхЗапросСхемыКомпоновкиДанных) — язык запросов.
- **Набор данных — объект** (НаборДанныхОбъектСхемыКомпоновкиДанных) — внешний набор данных (заполняется программно).
- **Набор данных — объединение** (НаборДанныхОбъединениеСхемыКомпоновкиДанных) — объединение нескольких наборов.

Общие свойства: **Имя** (уникальное в схеме), **Поля**. У запроса и объекта — **ИсточникДанных**. У запроса — **Запрос** (текст с расширением языка запросов для СКД). У объединения — **Элементы** (наборы для объединения).

Набор данных — запрос может быть пакетным; результирующий запрос — пакетный; состав полей временной таблицы определяется по использованию в других запросах пакета. В одном наборе — один источник данных.

**Набор данных — объект** — вывод из объекта встроенного языка (таблица значений, результат запроса и т.д.); заполняется программно и передаётся в процессор как внешний набор.

**Набор данных — объединение:** значения полей берутся из полей вложенных наборов по пути к данным.

#### 10.3.1.2.4. Поле набора данных схемы компоновки данных

Свойство **Поля** набора — коллекция ПолеНабораДанныхСхемыКомпоновкиДанных.

#### 10.3.1.3. Связи наборов данных

Свойство **СвязиНаборовДанных** схемы — коллекция СвязьНаборовДанныхСхемыКомпоновкиДанных.

#### 10.3.1.4. Вычисляемые поля

Поля, вычисляемые по выражениям с полями наборов. Используются в настройках как поля набора; **не поддерживаются:** реквизиты вычисляемых полей, иерархическая группировка по вычисляемым полям. Свойство **ВычисляемыеПоляСхемыКомпоновкиДанных**.

#### 10.3.1.5. Поля ресурсов

Поля итогов — значения для групповых записей. Свойство **ПоляИтога** схемы. Каждая запись набора участвует в расчёте ресурса один раз.

#### 10.3.1.6. Параметры

Свойство **Параметры** схемы. Можно задать обязательность и запрет незаполненного параметра.

#### 10.3.1.7. Вложенные схемы

Свойство **ВложенныеСхемыКомпоновкиДанных**.

#### 10.3.1.8. Макеты

Описание макетов для вывода поля или группировки. Свойство **Макеты** схемы (ОписаниеМакетаСхемыКомпоновкиДанных).

#### 10.3.1.9. Макеты полей

Связь поля с макетом — МакетПоляСхемыКомпоновкиДанных. Коллекция в свойстве **МакетыПолей** схемы.

#### 10.3.1.10. Макеты группировок

МакетГруппировкиСхемыКомпоновкиДанных. Свойство **МакетыГруппировок**.

#### 10.3.1.11. Макеты заголовков группировок

Свойство **МакетыЗаголовковГруппировок**.

#### 10.3.1.12. Настройки по умолчанию

Варианты отчёта в схеме содержат настройки по умолчанию. Вариант по умолчанию — первый в списке. Применяются при первом открытии, при «Стандартные настройки» и для программной установки.

### 10.3.2. Работа с несколькими наборами данных

В схеме описываются наборы и **связи** между ними. Связь: набор-источник (родительский) → набор-приёмник (зависимый). В схеме тип связи не задаётся — все связи трактуются как **левые внешние соединения**. В макете тип связи может задаваться компоновщиком (при глобальном отборе по полю зависимого набора — до родителя генерируется **внутренняя** связь).

Если в связи разрешён список параметров — данные зависимого набора получаются порциями по 1000 записей; иначе — по одной.

В выражении источника связи — поля родительского набора; в выражении приёмника — поля зависимого. В одной группировке нельзя использовать поля из несвязанных наборов (исключение — поля-итоги). Несвязанные наборы без общего родителя не могут иметь поля с одинаковыми именами. Данные зависимого набора не получаются без родительского. Обход при нескольких наборах — по последнему зависимому. Если ни одного поля связанного набора не задействовано — набор не включается в макет.

### 10.3.3. Расширение языка запросов для системы компоновки данных

Инструкции в фигурных скобках в тексте запроса.

#### 10.3.3.1. Синтаксические элементы

- **{ВЫБРАТЬ}** — поля, доступные для выбора. Через запятую — псевдонимы полей списка выборки. Суффикс `.*` — дочерние поля (например, Номенклатура.*). Только в первом запросе объединения.
- **{ГДЕ}** — поля для отбора. Используются поля таблиц, не псевдонимы. В каждой части объединения — свой ГДЕ. При незаданных параметрах в результирующий запрос не включается.
- **{ХАРАКТЕРИСТИКИ}** — ТИП, ВИДЫХАРАКТЕРИСТИК (таблица или запрос; ключевое слово РАЗРЕШЕННЫЕ), ПОЛЕКЛЮЧА, ПОЛЕИМЕНИ, ПОЛЕТИПАЗНАЧЕНИЯ, ЗНАЧЕНИЯХАРАКТЕРИСТИК, ПОЛЕОБЪЕКТА, ПОЛЕВИДА, ПОЛЕЗНАЧЕНИЯ.
- **Параметры** виртуальных таблиц — элементы в параметрах (например, даты, измерения) становятся доступными для отбора.

#### 10.3.3.2. Автозаполнение доступных полей

Все поля списка выборки и дочерние становятся доступными для выбора, упорядочивания, группировки, отбора; параметры виртуальных таблиц — для отбора.

### 10.3.4. Язык выражений системы компоновки данных

Используется в схеме (вычисляемые поля, поля итогов, связи и т.д.), в настройках (пользовательские поля), в макете (связи, параметры макета).

**ПРИМЕЧАНИЕ.** В языке выражений СКД нельзя получать поле через точку от выражения. Вычисление выполняется в **безопасном режиме**.

#### 10.3.4.1. Литералы

- **Строка** — в кавычках `"`; внутри — удвоенная кавычка.
- **Число** — например 10.5, 200.
- **Дата** — ДАТАВРЕМЯ(год, месяц, день [, часы, минуты, секунды]).
- **Булево** — Истина, Ложь.
- **Значение** — для перечислений, предопределённых данных: Значение(ИмяЛитерала).
- **Поля** — путь к данным, части через «.»; регистр не учитывается; пробелы/спецсимволы — в квадратных скобках.
- **Параметры** — имя с префиксом `&`.
- **Тип** — Тип("Строка") и т.п.

#### 10.3.4.2. Операции над числами

Унарные +, -; бинарные +, -, *, /, %.

#### 10.3.4.3. Операции над строками

Конкатенация (+). **ПОДОБНО** — шаблон (%, _, [...], [^...]; СПЕЦСИМВОЛ для экранирования); сравнение без учёта регистра.

#### 10.3.4.4. Операции сравнения

=, <>, <, >, <=, >=. НачинаетсяС/НеНачинаетсяС, Подобно/НеПодобно (шаблоны /, %, _). **В** (в список значений), **В** (в набор данных — одно поле). **ЕСТЬ NULL**, **ЕСТЬ Не NULL**.

#### 10.3.4.5. Логические операции

Не, И, Или (операнды — Булево).

#### 10.3.4.6. Агрегатные функции (кратко)

СУММА, КОЛИЧЕСТВО, КОЛИЧЕСТВО(Различные …), МАКСИМУМ, МИНИМУМ, СРЕДНЕЕ; МАССИВ([Различные] Выражение); ТАБЛИЦАЗНАЧЕНИЙ(...); СВЕРНУТЬ(Выражение, НомераКолонок); ПОЛУЧИТЬЧАСТЬ(Выражение, НомерКолонок); УПОРЯДОЧИТЬ(Выражение, НомераКолонок); СОЕДИНИТЬСТРОКИ(...); ГРУППОВАЯОБРАБОТКА(Выражения, ВыраженияИерархии, ИмяГруппировки) — возвращает ДанныеГрупповойОбработкиКомпоновкиДанных.

Каждый, Любой; СтандартноеОтклонениеГенеральнойСовокупности/Выборки; ДисперсияВыборки/ГенеральнойСовокупности; КовариацияГенеральнойСовокупности/Выборки; Корреляция; РегрессияНаклон, РегрессияОтрезок, РегрессияКоличество, Регрессия_R2, РегрессияСреднееX/Y, РегрессияSXX, РегрессияSYY, РегрессияSXY.

**МестоВПорядке**(Порядок, ПорядокИерархии, ИмяГруппировки) — место записи при заданном упорядочивании (с 1). **КлассификацияABC**(Значение, КоличествоГрупп, ПроцентыДляГрупп, ИмяГруппировки).

#### 10.3.4.7. Другие операции

**ВЫБОР** Когда … Тогда … Иначе … Конец.

#### 10.3.4.8. Правила сравнения двух значений

При разных типах — по приоритету типов: NULL (низший), Булево, Число, Дата, Строка, ссылочные, прочие. Ссылочные — по номерам таблиц. При совпадении типов — по правилам типа (Булево, Число, Дата, Строка, ссылки, массивы/таблицы значений, иначе — по строковому представлению или «равенство сам себе» для =, <>; <, <=, >, >= для неподдерживаемых — исключение).

#### 10.3.4.9. Работа со значением NULL

Операция с NULL-операндом → результат NULL. Исключения: И возвращает NULL только если ни один операнд не Ложь; Или — только если ни один не Истина.

#### 10.3.4.10. Приоритеты операций (от низшего)

Или; И; Не; В, ЕСТЬ NULL, ЕСТЬ Не NULL; сравнения; +, - (бинарные); ПОДОБНО, НачинаетсяС и т.д.; *, /, %; унарные +, -.

#### 10.3.4.11. Функции

**Алгебраические:** EXP, LOG, LOG10, POW, SQRT.

**ВРег**, **НРег** — верхний/нижний регистр. **ВЫРАЗИТЬ**(Выражение, "Тип") — выделение типа; при несовпадении типа — NULL. **ВЫЧИСЛИТЬ**(Выражение, Группировка, ТипРасчета) — устаревшая; рекомендуется **ВычислитьВыражение**.

**ВычислитьВыражение**(Выражение, Группировка, ОбластьВычисления, Начало, Конец, Сортировка, ИерархическаяСортировка, ОбработкаОдинаковыхЗначенийПорядка). ОбластьВычисления: ОбщийИтог, Иерархия, Группировка, ГруппировкаНеРесурса. Начало/Конец: Первая([N]), Последняя([N]), Предыдущая([N]), Следующая([N]), Текущая, ОграничивающееЗначение(Выражение). ОбработкаОдинаковыхЗначенийПорядка: Отдельно, Вместе.

**ВычислитьВыражениеСГруппировкойМассив**(Выражение, ВыражениеПолейГруппировки, ОтборЗаписей, ОтборГруппировок) — массив результатов по группировке. **ВычислитьВыражениеСГруппировкойТаблицаЗначений**(Выражения, …) — таблица значений.

**ГОД**, **МЕСЯЦ**, **КВАРТАЛ**, **ДЕНЬ**, **ДЕНЬГОДА**, **ДЕНЬНЕДЕЛИ**, **НЕДЕЛЯ**, **ЧАС**, **МИНУТА**, **СЕКУНДА** — компоненты даты. **ДОБАВИТЬКДАТЕ**(Дата, ТипУвеличения, Величина). **НАЧАЛОПЕРИОДА**, **КОНЕЦПЕРИОДА**(Выражение, ТипПериода). **РАЗНОСТЬДАТ**(Дата1, Дата2, ТипРазности). **ТЕКУЩАЯДАТА**().

**ЕСТЬNULL**(Выражение, ЗначениеПодстановки). **ЗНАЧЕНИЕЗАПОЛНЕНО** — не по умолчанию, не NULL, не пустая ссылка. **ДЛИНАСТРОКИ**, **ЛЕВ**, **ПРАВ**, **ПОДСТРОКА**, **СОКРЛ**, **СОКРП**, **СОКРЛП**, **СТРЗАМЕНИТЬ**, **СТРНАЙТИ**. **НСтр**(ИсходнаяСтрока, КодЯзыка). **ПРЕДСТАВЛЕНИЕ**(Выражение). **СТРОКА**(Выражение). **ФОРМАТ**(Значение, ФорматнаяСтрока). **ОКР**, **ЦЕЛ**. **ТипЗначения**(Выражение). **УНИКАЛЬНЫЙИДЕНТИФИКАТОР**(Ссылка). **УРОВЕНЬ**(), **УРОВЕНЬВГРУППИРОВКЕ**(). **НОМЕРПОПОРЯДКУ**(), **НОМЕРПОПОРЯДКУВГРУППИРОВКЕ**().

Тригонометрические: ACOS, ASIN, ATAN, COS, SIN, TAN (параметр в радианах).

#### 10.3.4.12. Функции общих модулей

Допускаются вызовы функций глобальных и неглобальных общих модулей (Клиент/Сервер по контексту). Только при соответствующем параметре процессора компоновки; в выражениях пользовательских полей — не допускаются.

### 10.3.5. Конструктор схемы компоновки данных

Объект встроенного языка **КонструкторСхемыКомпоновкиДанных** для визуального конструирования схемы; используется в конфигураторе. Для набора-запроса — вложенные наборы из текста запроса оформляются как поля; для набора-объекта — можно добавлять поля — вложенные наборы.

#### 10.3.5.1. Редактирование наборов данных

Этапы: наборы данных → поля наборов → связи → вычисляемые поля → ресурсы → параметры → макеты → вложенные настройки → настройки СКД. Набор-запрос — текст запроса (конструктор запроса или окно конструктора СКД). В запросе набора-объекта в {ВЫБРАТЬ} и {ГДЕ} допустимы вложенные таблицы. Набор-объединение — объединение полей из подчинённых наборов; автозаполнение полей по запросу (флажок Автозаполнение).

#### 10.3.5.2. Редактирование полей набора данных

Заголовок, ограничение доступности, роль, представление, выражения упорядочивания, способ проверки иерархии, тип значения, оформление. Роль/представление/упорядочивание/тип/оформление — только на верхнем уровне иерархии. Заголовок: если не задан — из синонима поля запроса или путь к данным по алгоритму синонимов. Для поля счёта в Вид — ссылка на вид счёта; для измерения — путь к родительскому измерению. Признак «Игнорировать значения NULL» — групповые записи с NULL по полю не включаются. **Обязательное** — поле присутствует всегда, если в настройках задействован хотя бы один элемент набора.

#### 10.3.5.3–10.3.5.6. Связи, вычисляемые поля, ресурсы, параметры

Связи: источник и приёмник — наборы; выражения — поля наборов. Вычисляемые поля: путь, выражение, заголовок, доступность, представление, упорядочивание, тип, оформление, доступные значения. Ресурсы: поля для итогов и выражения; по умолчанию для чисел — Сумма, для остальных — Количество; для иерархической группировки — строка с полем и «Иерархия»; ресурс можно рассчитывать только в разрезе указанной группировки. Параметры: имя, заголовок, типы и значения, выражение, доступность, обязательность, использование, параметры редактирования. Тип СтандартныйПериод: начало 00:00:00, конец 23:59:59.

#### 10.3.5.7. Редактирование макетов

Макет поля, группировки, заголовка группировки, ресурсов таблицы. Для группировки/заголовка — имя группировки или поля, тип макета. Область — координаты в табличном документе; редактирование панелью свойств (Alt+Enter). Параметры и шаблоны ячеек попадают в макет.

#### 10.3.5.8–10.3.5.9. Вложенные схемы, настройки

Вложенные схемы редактируются на отдельной закладке. Настройки по умолчанию задаются в схеме.

#### 10.3.5.9.1. Вложенные поля

**Авто позиция ресурсов:** Не использовать / После всех полей. **Вариант использования группировки:** Авто или Дополнительная информация (вывод одной колонки/строки с полями из Выбранные поля; для группировки-колонки/строки — без полей группировки в этой группировке). Для «Дополнительная информация» игнорируются: расположение полей/итогов/группировок, количество и процент записей и т.п.

Для числовых ресурсов — автоматические вложенные поля (идентификаторы): **ПроцентВИерархии**, **ПроцентВИерархииВКолонкеИлиТочке**, **ПроцентВИерархииВСтрокеИлиСерии**, **ПроцентВГруппе**, **ПроцентВГруппеВКолонкеИлиТочке**, **ПроцентВГруппеВСтрокеИлиСерии**, **ПроцентВКолонкеИлиТочке** (вне таблицы — % общий).

## 10.4. Макет компоновки данных

### 10.4.1. Составные части макета

**МакетКомпоновкиДанных** — инструкция по выполнению компоновки. Содержит: описания макетов областей, тексты запросов, расположение группировок.

- **Источники данных** — свойство `ИсточникиДанных`, коллекция `ИсточникДанныхМакетаКомпоновкиДанных`. Несколько источников могут указывать на одну ИБ по строке соединения.
- **Наборы данных** — свойство `НаборыДанных`. Вложенный набор связан с родительским; при указании условия фильтра связь считается внутренней.
- **Поля набора данных** — свойство с коллекцией `ПолеНабораДанныхМакетаКомпоновкиДанных`. Поля, отсутствующие в описании набора, недоступны (даже если есть в запросе).
- **Значения параметров** — свойство `ЗначенияПараметров`, коллекция `ЗначениеПараметраМакетаКомпоновкиДанных`.
- **Связи наборов** — свойство `СвязиНаборовДанных`, коллекция `СвязьНаборовДанныхМакетаКомпоновкиДанных`.
- **Макеты областей** — свойство `Макеты`, коллекция `ОписаниеМакетаОбластиМакетаКомпоновкиДанных`.

**Тело макета** — элементы: группировка, детальные записи, таблица, диаграмма, макет.

**Группировка таблицы** (`ГруппировкаТаблицыМакетаКомпоновкиДанных`): тело и иерархическое тело содержат только группировки таблицы, детальные записи таблицы, макеты группировки таблицы. Доп. свойства: `МакетОбщихИтогов`, `РасположениеОбщихИтогов`.

**Детальные записи таблицы** — объект `ЗаписиТаблицыМакетаКомпоновкиДанных`.

**Группировка диаграммы** (`ТелоГруппировкиДиаграммыМакетаКомпоновкиДанных`): только макеты группировки диаграммы и группировки диаграммы; макетов заголовка/подвала нет.

### 10.4.2. Макеты областей

**МакетОбластиКомпоновкиДанных** — коллекция `СтрокаТаблицыОбластиКомпоновкиДанных`. Строка — коллекция ячеек (`ЯчейкиТаблицыОбластиКомпоновкиДанных`), элемент — `ЯчейкаТаблицыОбластиКомпоновкиДанных`.

**Элементы макета** — коллекция `ЭлементыМакетаОбластиКомпоновкиДанных` (поля `ПолеОбластиКомпоновкиДанных`). Оформление: `ОформлениеЯчейкиТаблицыОбластиКомпоновкиДанных`, `ОформлениеПоляОбластиКомпоновкиДанных` (формат).

Чтобы в ячейке выводилось значение поля, свойству `Значение` поля области присваивается `ПараметрСистемыКомпоновкиДанных` с именем параметра; параметр добавляется в определение макета с выражением — имя поля или выражение языка СКД.

**Макеты областей диаграммы:** `МакетДиаграммыОбластиКомпоновкиДанных`, `МакетРесурсаДиаграммыОбластиКомпоновкиДанных`, `МакетГруппировкиДиаграммыОбластиКомпоновкиДанных`. Формируются один макет диаграммы, один макета ресурса и несколько макетов группировок (по числу точек и серий).

### 10.4.3. Макеты оформления

**МакетОформленияКомпоновкиДанных** — декларативное описание областей отчета. Коллекция областей: параметры, имя параметра = имя области (из перечня предопределённых). Значение — `ОбластьМакетаОформленияКомпоновкиДанных` (коллекция `ЭлементОбластиМакетаОформленияКомпоновкиДанных`). Элемент: свойство **Уровень** (0 — по умолчанию ко всем макетам данного типа), **Оформление** — имена свойств и значения.

### 10.4.4. Генератор областей макетов

Этапы: расположение группировок → расположение полей группировок → применение макета оформления → условные оформления → объединение ячеек.

**Типы макетов группировки** (`ТипМакетаГруппировкиКомпоновкиДанных`): Авто, Горизонтальный, Вертикальный.

**Расположение группировок** (`РасположениеПолейГруппировки`): Вместе, Отдельно, ОтдельноИТолькоВИтогах.

**Расположение реквизитов** (`РасположениеРеквизитов`): Вместе, Отдельно, ВместеСВладельцем, ВСпециальнойПозиции.

**Расположение папки:** Авто, Горизонтально, Вертикально, ВОтдельнойКолонке.

**Расположение ресурсов** (`РасположениеРесурсов`): Горизонтально, Вертикально.

**Общие итоги таблицы:** `РасположениеОбщихИтоговПоГоризонтали` / `ПоВертикали`: Нет, Начало, НачалоИКонец, Авто.

**Объединение ячеек:** незаполненные справа объединяются с левыми; снизу — с верхними; при обоих вариантах сначала по горизонтали, затем по вертикали.

### 10.4.5. Формирование макета компоновки данных

**КомпоновщикМакетаКомпоновкиДанных** — метод `Выполнить(СхемаКомпоновкиДанных, НастройкиКомпоновкиДанных, ПеременнаяДанныеРасшифровки, МакетОформления)`. Возвращает макет компоновки.

Действия компоновщика: создание макета; модификация запросов наборов данных; формирование фильтров в тексте запроса/описании набора; при необходимости — наборы для иерархии; параметры со значениями пользователя; заполнение тела макета (группировки, таблицы, параметры); использование генератора областей; при переданной переменной — создание объекта `ДанныеРасшифровки`.

Параметр **Группировка** функции `ВычислитьВыражение()`: если указано имя группировки из настроек — сохраняется; иначе ищется родительская группировка, содержащая все поля из параметра, и подставляется её имя. Если не найдена — выражение заменяется на NULL.

Пример:
```bsl
КомпоновщикМакета = Новый КомпоновщикМакетаКомпоновкиДанных;
МакетКомпоновки = КомпоновщикМакета.Выполнить(СхемаКомпоновкиДанных, ИсполняемыеНастройки);
```

## 10.5. Процессор компоновки данных

**ПроцессорКомпоновкиДанных** — на вход макет компоновки данных. Последовательно получают элементы результата методом `Следующий()`.

При инициализации можно указать: внешние наборы данных (структура: имя набора → набор данных); данные расшифровки; признак использования внешних функций в выражениях.

Пример:
```bsl
ПроцессорКД.Инициализировать(МакетКД);
ПроцессорВывода = Новый ПроцессорВыводаРезультатаКомпоновкиДанныхВТабличныйДокумент;
ПроцессорВывода.УстановитьДокумент(ТД);
ПроцессорВывода.НачатьВывод();
Пока Истина Цикл
    Элемент = ПроцессорКД.Следующий();
    Если Элемент = Неопределено Тогда Прервать; КонецЕсли;
    ПроцессорВывода.ВывестиЭлемент(Элемент);
КонецЦикла;
ПроцессорВывода.ЗакончитьВывод();
```
Либо: `ПроцессорВывода.Вывести(ПроцессорКомпоновкиДанных)`.

## 10.6. Функциональные опции и права на просмотр поля

При получении настроек по умолчанию СКД автоматически: недоступные по правам/функциональным опциям поля исключаются из доступных; поля реквизитов выключенных типов удаляются; таблицы запроса при выключенной ФО объекта становятся недоступны; пользовательские поля с недоступными полями удаляются; из группировок/таблиц/диаграмм/порядка/условного оформления удаляются недоступные поля (при опустошении — удаляется вся группировка/таблица/диаграмма/элемент оформления).

При загрузке сохранённой настройки поля не удаляются автоматически (пользователь может заменить или убрать сам).

**КомпоновщикМакета.Выполнить(..., ПроверятьДоступностьПолей)**: при Истина — проверка прав и ФО; при использовании недоступного поля — исключение. При Ложь проверка не выполняется.

## 10.7. Результат компоновки данных

Результат — набор элементов результата; отдельного объекта «результат» нет. Элементы можно получать от процессора или формировать вручную, выводить процессором вывода.

Типы элементов: Начало, Конец, НачалоКонец (с именем макета). Свойства: Макеты, Расположение вложенных (Вертикально/Горизонтально).

### 10.7.1. Вывод в табличный документ

**ПроцессорВыводаРезультатаКомпоновкиДанныхВТабличныйДокумент**: `УстановитьДокумент()`, `НачатьВывод()`, цикл `ВывестиЭлемент()` или `Вывести(ПроцессорКомпоновкиДанных)`, `ЗакончитьВывод()`. Фиксация заголовка таблицы (одна таблица/группировка) настраивается параметрами вывода «Фиксация слева», «Фиксация сверху».

### 10.7.2. Вывод в таблицу и дерево значений

**ПроцессорВыводаРезультатаКомпоновкиДанныхВКоллекциюЗначений**: `УстановитьОбъект(ДеревоИлиТаблица)`. Без вызова — вывод в таблицу значений. Ограничения: только группировки и детальные записи (без таблиц и диаграмм); папки в выбранных полях игнорируются; условное оформление и оформление полей из схемы не используются; из параметров вывода — только расположение общих итогов по вертикали, тип заголовка полей, количество/процент записей; предопределённые макеты не используются. Два вида макетов: `МакетЗаголовкаКоллекцииЗначенийОбластиКомпоновкиДанных`, `МакетКоллекцииЗначенийОбластиКомпоновкиДанных`.

## 10.8. Расчет итогов по полям остатка

**Поле остатка** — в роли проставлен признак «Остаток». Для поля начального остатка в наборе должно быть поле конечного остатка и наоборот. Поля-периоды — непрерывная нумерация с 1. В данных — уникальность комбинации полей-периодов и полей-измерений.

**Алгоритм расчёта итогов по остаткам:** при группировке по полю-периоду: если по всем периодам уже группировка — для каждой комбинации измерений берётся запись, ближайшая к текущему периоду (начальный/конечный остаток по правилам); иначе — первая и последняя записи с периодом = текущему, иначе ближайшая запись. Без группировки по периоду — первые по хронологии записи для неиспользованных измерений = начальный остаток, последние = конечный.

**Бухгалтерские остатки:** учитывается поле-счёт и вид счета (активный/пассивный/АП); итог по Дт/Кт по формулам в зависимости от знака остатка и вида счёта.

Компоновщик макета: при использовании начального остатка добавляет конечный в запрос (и наоборот); при использовании реквизита измерения добавляет само поле-измерение.

## 10.9. Работа с иерархией

### 10.9.1. Иерархические группировки

Для иерархической группировки процессору нужен набор данных для построения иерархии: связь набора с самим собой (поле Родитель → поле Ссылка, параметр Ссылка). Набор может быть в схеме или сгенерирован компоновщиком макета.

### 10.9.2. Иерархические детальные записи

Связь набора с самим собой: выражение-источник — поле группировки (напр. Ссылка), приёмник — поле родителя (напр. Родитель), параметр связи с возможностью списка. Условие связи (напр. ЭтоГруппа). Начальное значение иерархии — напр. `Значение(Справочник.Номенклатура.ПустаяСсылка)`.

### 10.9.3. Один элемент в нескольких родительских записях

Иерархия по регистру/объединению запросов: источник связки — основной набор, приёмник — иерархический набор; иерархическая связь — набор сам с собой (Родитель → Ссылка). Поле иерархического набора, с которым связь, должно иметь то же имя, что и в наборе-источнике.

**Условие В ИЕРАРХИИ:** в глобальном фильтре попадает в запрос как В ИЕРАРХИИ; иначе процессору нужен набор данных со ссылками, удовлетворяющими условию (в схеме или сгенерированный).

## 10.10. Использование СКД при разработке

Схема компоновки — в макете отчёта (Основная схема компоновки данных). Конструктор макета → схема → конструктор схемы или загрузка из XML. Формы: основная, форма настроек, форма редактирования варианта.

**Параметры ввода** (в схеме для полей/параметров): способ ввода в отборе и параметрах (быстрый выбор, форма выбора и т.д.).

**Свойства метаданных:** для набора данных-запроса из объектов метаданных подставляются параметры ввода (маска, связи выбора, форма выбора, формат, быстрый выбор) и параметры вывода (многострочный режим и др.), если не заданы в схеме.

**Фоновое выполнение:** механизм фоновых заданий. Автоформа отчёта в клиент-сервере — фоновое, в файловом — непосредственное. Отключить фоновое: вызов в расширении формы `СкомпоноватьРезультат(РежимКомпоновкиРезультата.Непосредственно)`. При изменении настроек/варианта во время фона — выполнение прекращается, документ отображается приглушённо. Закрытие формы → остановка после ПриЗакрытии. Окно состояния — при разрешении (`РежимАвтоОтображенияСостояния`) и длительности >2 с (4 с при низкой скорости). Интервал опроса завершения — с увеличением в 1,4 раза (макс. 20 с). Система взаимодействия может уведомлять о завершении.

**Сохранение цвета серии диаграммы:** параметры «Тип диаграммы.Сохранение цветов» и «Тип диаграммы.Ключ сохранения цветов». Значения: Для пользователя (хранилище Общее/ЦветаДиаграммы), Для текущего результата, По ключу пользователя (ключ = Общее/ЦветаДиаграммы/ + значение ключа), Не использовать, Авто. Команда «Очистить запомненные цвета диаграмм» в расширении формы.

## 10.11. Особенности использования

- Запрещены имена путей, равные ключевым словам: ВЫБОР, КОГДА, ТОГДА, ИНАЧЕ, КОНЕЦ, ЕСТЬ, Не, ПОДОБНО, СПЕЦСИМВОЛ, РАЗЛИЧНЫЕ.
- Стандартный период: пустая дата начала/окончания → параметры ДатаНачала/ДатаОкончания считаются неустановленными.
- Некорректно: в запросе РАЗЛИЧНЫЕ и в УПОРЯДОЧИТЬ ПО выражение, отсутствующее в списке выборки.
- Запрещена группировка по реквизитам полей-периодов.
- Нет права на просмотр объекта метаданных → все поля таблицы недоступны в СКД.
- Дочерние поля составного типа доступны при праве Просмотр хотя бы на одну таблицу типа и на само поле.
- Из предопределённых макетов при генерации удаляются выражения расшифровки по полям, недоступным по ФО.
- Имя поля иерархического набора в связи должно совпадать с полем набора-источника.
- Поле с ПРЕДСТАВЛЕНИЕ(другое_поле) в выборке — при автозаполнении доступных полей не доступно для настроек.
- Ошибка при группировке по вычисляемому полю с дополнением в периоде.
- Автоотступ в макетах группировки — только для верхней левой ячейки.
- Пакетный запрос: неиспользуемые в последующих запросах поля удаляются из выборки и группировок; запросы без полей и наборы без используемых полей удаляются. Исключение: флажок «Обязательное» у поля в схеме; временная таблица в обязательном соединении не удаляется.
- Вывод в коллекцию: поля разных группировок по одному полю СКД попадают в одну колонку.
- ОБЪЕДИНИТЬ/ОБЪЕДИНИТЬ ВСЕ: отбор по реквизиту применяется только в частях, где реквизит есть; по полю — где выражение не NULL.
- При формировании доступных полей учитывается привилегированный режим.
- Вычисление выражений СКД — в безопасном режиме.
- Табличные части через точку: рекомендуется явно указывать псевдонимы для ТЧ и полей.
- Отбор по ВерсияДанных/ИмяПредопределяемыхДанных с нестандартным сравнением — поле должно быть доступно для выбора, не только для отбора.
- Обязательный параметр для пользователя: использование «Всегда» или секция {ГДЕ} в запросе (автоотключение условий при снятом параметре).
- Поля с режимом пароля в метаданных недоступны в источнике доступных настроек. При вызове КомпоновщикМакета.Выполнить(..., ПроверятьДоступностьПолей = Ложь) из кода поля с паролем использовать можно.
