## 37.1. Общая информация

Групповая разработка — одновременная работа нескольких разработчиков над конфигурацией; объект доступен для изменения только после захвата. «Конфигурация» — основная конфигурация ИБ или любое подключённое расширение.

Создаётся хранилище конфигурации. Объекты по умолчанию закрыты для изменения. Чтобы изменить объект — его нужно захватить; одновременно объект может быть захвачен только одним пользователем. Для добавления объекта захватывают родительский (для новой константы — корневой объект конфигурации). Для удаления незахваченного объекта нужен захват самого объекта, родителя и всех подчинённых. После работы результат помещается в хранилище; остальные получают обновление.

Хранилище обеспечивает целостность метаданных при захвате и помещении. Файл БД хранилища: `1Cv8ddb.1CD`. Формат хранилища — версия 8.3.8. Хранилище для основной конфигурации и для расширения несовместимы. К хранилищу 8.3.10+ не подключается конфигуратор 8.3.7 и младше; конвертация — утилита `cnvdbfl`.

## 37.2. Режимы совместимости хранилища

- **Версии 8.3.2 и младше** — с хранилищем работают любые версии платформы.
- **8.3.3–8.3.5** и выше (включая «Не использовать») — хранилище в оптимизированном формате, ускоренная работа.
Смена режима — только администратор; преобразование может быть длительным.

## 37.3. Права пользователей хранилища

- **Административные** — создание/удаление/отключение пользователей, отмена захвата, смена режима совместимости, оптимизация.
- **Изменение состава версий** — откат и сокращение истории версий.
- **Захват объектов** — без этого права объекты только для просмотра.

## 37.4. Кеш версий

- **Локальный кеш** — на компьютере конфигуратора.
- **Глобальный кеш** — на компьютере с каталогом хранилища.

Кеш ускоряет подключение, открытие версий, сравнение, сохранение версий. По умолчанию размер локального кеша 500 МБ; при превышении удаляются самые старые 50%. Путь по умолчанию: `%APPDATA%\1C\1Cv8\<ИД ИБ>\cfgrepo\cache` (Windows), `~/.1cv8/...` (Linux).

## 37.5. Резервное копирование хранилища

- Режим «8.3.2 и младше» — копировать `1Cv8ddb.1CD`.
- Иной режим — `1Cv8ddb.1CD` и каталог `data`.

Во время копирования подключений к хранилищу быть не должно.

## 37.6. Захват и помещение

Захват — монопольное владение объектом для редактирования. Для захвата родительского объекта нужно получить связанные объекты из хранилища. При помещении можно оставить объекты захваченными (флажок «Оставить захваченными»). Комментарий при помещении описывает изменения; сохраняются последние 20 комментариев.

## 37.7. Отмена захвата

Отмена захвата доступна администратору; пользователь теряет возможность поместить изменения этих объектов в хранилище.
