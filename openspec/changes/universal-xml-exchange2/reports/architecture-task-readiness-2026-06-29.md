## Замечания (WARNING — устранить до apply, не блокируют)

### W1 — Противоречие порядка выполнения метаданные ↔ BSL

- **`design.md § Migration Plan (S4)`** шаг 1: «(Конфигуратор) Удалить перечисление `рг_СтатусыОбмена`; сменить тип `Статус` → Строка(20)…», шаг 3: «(BSL) Функции-константы…». То есть **enum удаляется до** правки BSL.
- **`reports/architecture-extend-coherence-2026-06-29.md` §«План миграции с enum» / «Порядок BSL/метаданных»**: «сначала перевести 7 точек на константы, **затем** удалить enum».
- **Риск:** если выполнить буквально по design (удалить enum раньше S4.4), модуль `рг_УниверсальныйОбменXMLСервер` сохраняет 7 ссылок `Перечисления.рг_СтатусыОбмена.*` → промежуточное состояние с несуществующим объектом метаданных в cfe (нарушение компиляционной целостности расширения).
- **Нумерация задач** (S4.1 метаданные → S4.3/S4.4 BSL) усиливает риск: читается как «сначала удалить enum».
- **Рекомендация:** зафиксировать в `tasks.md` явный безопасный порядок исполнения для S4: **S4.3 → S4.4 (BSL: добавить константы, перевести 7 сравнений; значения совпадают с прежними именами enum) → S4.1 → S4.2 (Конфигуратор: удалить enum, сменить тип, выгрузить)**. Привести `design.md § Migration Plan (S4)` шаг 1↔3 в соответствие с extend-coherence (constants-first).

### W2 — Гигиена delta-spec `exchange-settings`

- В `specs/exchange-settings/spec.md` требование **«Статус жизненного цикла обмена»** присутствует дважды: как `## ADDED Requirements` (строки 26-28, тип «перечисление `рг_СтатусыОбмена`») и как `## MODIFIED Requirements` (строки 81-83, тип «Строка(20)», «Перечисление MUST NOT использоваться»). В файле два заголовка `## MODIFIED Requirements` (строки 54 и 79).
- **Эффект:** внутри одной дельты соседствуют взаимоисключающие версии одного требования (enum vs Строка). MODIFIED по смыслу замещает ADDED, но stale-ADDED-формулировка остаётся в том же файле и может ввести в заблуждение reviewer/archive.
- **Рекомендация:** убрать stale ADDED-формулировку «Статус жизненного цикла обмена» (enum) либо консолидировать в один MODIFIED-блок со строковыми кодами; устранить дублирующий заголовок `## MODIFIED Requirements`.
