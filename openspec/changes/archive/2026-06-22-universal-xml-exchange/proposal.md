## Why

Обмен данными между информационными базами 1С выполняется разрозненно: нет единого клиент-серверного механизма, где база-приёмник инициирует загрузку у базы-источника через веб-сервис, а правила выборки и фильтрации задаются макетами на источнике. Нужна универсальная подсистема с профилями настроек, безопасным хранением паролей и стандартной загрузкой на приёмнике — первый контур на ЗУП КОРП в расширении `рг_УниверсальныйОбменXML`.

## What Changes

1. Справочник `ргНастройкиОбменаXML` — профили обмена: URL веб-сервиса, логин, имя макета правил на источнике, табличная часть параметров (строка, число, дата, булево); пароль через безопасное хранилище БСП; управление регламентными заданиями.
2. SOAP-веб-сервис `rgExchangeService` с операцией `GetData` — приём имени макета и XDTO-структуры параметров, возврат Base64 ZIP с XML-выгрузкой.
3. Обработка `ргУниверсальныйОбменXML` — на базе штатной `УниверсальныйОбменДаннымиXML` ЗУП с доработками по ТЗ; макеты правил обмена (XML); единый движок выгрузки/загрузки на всех базах участников обмена.
4. Общий модуль `ргУниверсальныйОбменXMLСервер` — экспортные методы веб-сервиса, выгрузка/загрузка, обработчик регламентного задания.
5. Регламентное задание `ргВыполнениеОбменаXML` — шаблон программных заданий обмена по профилю.
6. Преобразование примитивных параметров в ссылочные отборы — только в обработчиках правил обмена (КД 2.1); ядро автоматический поиск ссылок не выполняет.

## Capabilities

### New Capabilities

- `exchange-settings`: хранение профилей обмена, параметров фильтрации, безопасное хранение пароля, связь с регламентными заданиями
- `exchange-web-service`: публикация SOAP-сервиса `rgExchangeService`, операция `GetData`, контракт XDTO Structure → Base64 ZIP
- `exchange-export`: выгрузка на базе-источнике по правилам из макета, активация ПВД, упаковка XML в ZIP
- `exchange-import`: инициация обмена на приёмнике, вызов веб-сервиса, распаковка ZIP, загрузка через движок универсального обмена

## Impact

- **Расширение:** `src/ЗУП/cfe/рг_УниверсальныйОбменXML/` — все новые объекты метаданных и BSL
- **Базовая конфигурация:** только чтение эталона `src/ЗУП/cf/`; интеграция с БСП (безопасное хранилище паролей, регламентные задания)
- **Внешние системы:** HTTP(S)-доступ приёмника к опубликованному SOAP-сервису источника; веб-сервер на стороне источника
- **Топология:** две роли одной подсистемы — база-источник (сервер) и база-приёмник (клиент); в одной ИБ возможны обе роли при наличии соответствующих объектов
