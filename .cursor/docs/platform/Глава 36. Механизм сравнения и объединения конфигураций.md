# Глава 36. Механизм сравнения и объединения конфигураций

> **Agent ref:** сравнение конфигураций, объединение, слияние изменений, .cf/.cfe, приоритет основной/загружаемой, объединение модулей и форм.
> **Разделы:** условия сравнения, режимы объединения, управляемые формы, предопределённые данные, расширения, типы данных, загрузка из файла.

---

## 36.1. Общая информация

Сравниваются: общие свойства объектов, реквизиты и реквизиты табличных частей. Отдельно — формы (модули, тексты описаний, макеты).

**Соответствие объектов:** по одинаковым именам — один объект; при разных именах — по внутренним идентификаторам (присваиваются конфигуратором). Формы: по именам → по заголовкам страниц → по картинкам.

**Ограничение:** сравнение и объединение основной конфигурации с расширением не поддерживается.

## 36.2. Условия сравнения

Основная конфигурация — та, с которой запущен Конфигуратор. Загружаемая — из файла (.cf). Расширения: из файла (*.cfe) или из ИБ. Сравнение основной конфигурации и расширения не поддерживается.

## 36.3. Режимы объединения

- **Взять из загружаемой** — объект добавляется или замещается целиком (структура, модули, формы).
- **Объединить с приоритетом основной** — новое добавляется; изменённое сохраняется из основной, из загружаемой только новое.
- **Объединить с приоритетом загружаемой** — изменённое по правилам объединения; при конфликте — из загружаемой.

**Модули:** удалённое при приоритете основной остаётся в тексте; при приоритете загружаемой — в комментариях. Новое добавляется. Изменённое — одно в коде, другое в комментарии по приоритету.

## 36.4. Управляемые формы

Сравнение: модули — по именам процедур; элементы формы — по типу и имени; параметры, команды, реквизиты — по именам. Совпадающие элементы заменяются или объединяются по приоритету. Режим объединения задаётся для формы в целом или отдельно для формы и её модуля.

## 36.5. Предопределённые данные

Сопоставление только по внутреннему идентификатору. Идентификатор сохраняется при объединении и копировании. При обновлении конфигурации БД — сопоставление предопределённых элементов с объектами БД.

## 36.6. Расширения конфигурации

Сравнение/объединение: расширение в .cfe или в ИБ. Не поддерживается сравнение расширения с основной конфигурацией. Для заимствованных объектов учитываются значения и признак контролируемости. При объединении пары заимствованный–заимствованный признак контролируемости берётся из конфигурации, с которой выполняется объединение.

## 36.7. Типы данных

Реквизиты: автоматическое объединение состава типов или ручные правила.

- **Строка:** фиксированная/переменная длина — максимум из двух; неограниченная — из приоритетной конфигурации.
- **Число:** при совпадении точности и признака Неотрицательное — максимальная длина.
- **Дата:** квалификаторы из приоритетной конфигурации.

## 36.8. Объединение модулей

Объединение процедур и функций по именам; соответствие можно задавать вручную или отменять. Результат редактируется до применения. Выполнение объединения — по всей конфигурации (Выполнить).

## 36.9. Загрузка конфигурации из файла

Конфигурация → Загрузить конфигурацию из файла (.cf). Полная замена. При непустой текущей конфигурации — предупреждение о потере изменений, затем запрос на обновление конфигурации БД.
