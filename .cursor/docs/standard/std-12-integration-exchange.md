# Домен: Интеграции и обмен

> Локальная выжимка практик ревью интеграционного BSL. Применяется к коду подготовки JSON, HTTP/RMQ-сообщений, выгрузок, загрузок и обменов с внешними системами.

---

## Сериализация ссылок в JSON

Для передачи ссылки 1С во внешний обмен использовать `XMLСтрока(<Ссылка>)`.

Не собирать GUID вручную через `УникальныйИдентификатор()`, `Строка(...)`, конкатенацию с пустой строкой или `XMLСтрока(<Ссылка>.УникальныйИдентификатор())`.

**Неправильно:**

```bsl
СтруктураСообщения.Вставить("ContractorGUID", Строка(Контрагент.УникальныйИдентификатор()));
СтруктураСообщения.Вставить("OrderGUID", "" + ЗаказПокупателя.УникальныйИдентификатор());
СтруктураСообщения.Вставить("ReasonGUID", XMLСтрока(ПричинаОтмены.УникальныйИдентификатор()));
```

**Правильно:**

```bsl
СтруктураСообщения.Вставить("ContractorGUID", XMLСтрока(Контрагент));
СтруктураСообщения.Вставить("OrderGUID", XMLСтрока(ЗаказПокупателя));
СтруктураСообщения.Вставить("ReasonGUID", XMLСтрока(ПричинаОтмены));
```

`XMLСтрока()` корректно обрабатывает пустые ссылки, поэтому отдельная подстановка `00000000-0000-0000-0000-000000000000` обычно не нужна.

## Имена типовых полей JSON

Для повторяемых сущностей использовать устойчивые английские имена. Это снижает разнобой между обменами и упрощает сопровождение контрактов.

| Данные 1С | Поле JSON |
|-----------|-----------|
| Ссылка | `GUID` |
| Код | `Code` |
| Номер | `Number` |
| Дата | `DateTime` |
| Наименование | `Name` |
| Представление | `Presentation` |
| Пометка удаления | `DeletionMark` |
| Проведен | `Posted` |
| Версия данных | `DataVersion` |
| Номенклатура / товары | `Goods` |
| Контрагент | `Contractor` |
| Партнер | `Partner` |
| Организация | `Organization` |
| Склад | `Warehouse` |
| Валюта | `Currency` |
| Договор | `Contract` |
| Вид операции | `OperationType` |
| Тип документа | `DocumentType` |

Если внешний контракт уже закреплён иначе, он главнее этого словаря. В новом коде не вводить транслитерацию вроде `Kontragent`, `Nomenklatura`, `Summa`.

## Числа в строковом контракте

Если внешнему контракту нужна строка без разделителей групп, использовать форматирование:

```bsl
СтроковоеЧисло = Формат(Сумма, "ЧГ=0");
```

Не удалять неразрывные пробелы после `Строка(Число)`:

```bsl
СтроковоеЧисло = СтрЗаменить(Строка(Сумма), Символы.НПП, "");
```

## Сообщения об ошибках обмена

Блокирующее сообщение должно помогать пользователю или администратору понять, что исправить.

Минимум для сообщения:

1. Что произошло.
2. Какое фактическое значение мешает операции.
3. Какое значение или действие ожидается.

**Неполезно:**

```bsl
ВызватьИсключение "Заказ имеет некорректный статус";
```

**Полезно:**

```bsl
ВызватьИсключение СтрШаблон(
    "Нельзя отправить заказ %1: текущий статус %2, ожидается %3.",
    Заказ,
    ТекущийСтатус,
    ОжидаемыйСтатус);
```

Не превращать это правило в массовую стилистическую проверку всех строк. В ревью оно применяется к сообщениям, которые блокируют пользовательский сценарий, запись, проведение или обмен.
