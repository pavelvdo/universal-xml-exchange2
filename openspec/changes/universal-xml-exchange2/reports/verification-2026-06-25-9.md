# Verification — universal-xml-exchange2 (2026-06-25, runtime defect)

## Вердикт

**Обсуждение:** при приёмке S2 обнаружен дефект реализации (затенение `ПараметрыСеанса`). Исправление внесено в код; повторный end-to-end тест обязателен до закрытия `S2.accept`.

## Симптом

Кнопка «Выполнить обмен» на приёмнике:

```
Поле объекта недоступно для записи (ПараметрыСеанса)
{…рг_УниверсальныйОбменXMLСервер…(993)}: ПараметрыСеанса = Новый Структура;
```

## Корневая причина (verified)

Локальная переменная и параметры функций названы `ПараметрыСеанса` — совпадает с встроенным объектом платформы (параметры сеанса, read-only на сервере). Присвоение `ПараметрыСеанса = Новый Структура` интерпретируется как запись в платформенный объект.

## Исправление

Файл: `CommonModules/рг_УниверсальныйОбменXMLСервер/Ext/Module.bsl`

- Локальная структура сеанса приёмника: `ДанныеСеансаОбмена`
- Функция подготовки: `ПодготовитьДанныеСеансаПриемника`
- Цепочка: `ВыполнитьСеансОбменаПоПрофилюПриемника` → GetData → Confirm — без изменения контракта ключей

## Следующий шаг

1. Загрузить обновлённое расширение в ИБ-приёмник (и источник, если общий каталог).
