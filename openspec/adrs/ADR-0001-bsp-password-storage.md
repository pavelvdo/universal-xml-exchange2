# ADR-0001: Пароль профиля обмена в безопасном хранилище БСП

**Статус:** Accepted
**Дата:** 2026-06-22
**Область:** exchange-settings
**Источник:** openspec/changes/archive/2026-06-22-universal-xml-exchange/reports/architecture-new-2026-06-22.md
**Load-bearing:** yes
**Protects-invariants:**
  - "Пароль подключения к источнику не хранится в открытом реквизите справочника"

## Контекст

Профиль обмена содержит учётные данные для HTTP Basic при вызове веб-сервиса источника. Требование ТЗ — не хранить пароль в реквизитах справочника.

## Решение

Пароль записывается через `ОбщегоНазначения.ЗаписатьДанныеВБезопасноеХранилище(Ссылка, Пароль, "Пароль")` при записи элемента; на форме — нехранимое поле ввода. Владелец — ссылка элемента `рг_НастройкиОбменаXML`.

## Альтернативы

| Вариант | Плюсы | Минусы | Почему отклонён |
|---------|-------|--------|-----------------|
| Реквизит с шифрованием в БД | Проще форма | Не соответствует паттерну БСП и ТЗ | Требование ТЗ |

## Последствия

- Положительные: единый паттерн с другими интеграциями ЗУП/БСП
- Отрицательные: зависимость от регистра БСП «БезопасноеХранилищеДанных»
