# Глава 14. Анализ данных и прогнозирование

> Agent ref: АнализДанных, типы анализа (ОбщаяСтатистика, ПоискАссоциаций, ПоискПоследовательностей, ДеревоРешений, Кластеризация), результат анализа, модель прогноза, ПостроительОтчетаАнализаДанных. Источник: запрос, ТаблицаЗначений, область ячеек табличного документа.
> Ключевые разделы: 14.2 Схема механизма и объекты, 14.2.2 Типы анализа, 14.3 Общая статистика, 14.4 Поиск ассоциаций, 14.5 Поиск последовательностей, 14.6 Дерево решений, 14.7 Кластеризация (метрики, методы, дендрограмма).
> ---

## 14.1. Общая информация

Механизм анализа данных и прогнозирования предназначен для выявления закономерностей в больших объёмах данных и построения прогнозов. Типичные сценарии: группы товаров, покупаемых вместе; последовательности покупок; классификация контрагентов (дерево решений); разбиение объектов на кластеры. Ниже — обзор объектов и параметров платформы.

## 14.2. Схема механизма

Цепочка: **источник данных** → **АнализДанных** (тип анализа, параметры, настройка колонок) → **результат анализа** → вывод через **ПостроительОтчетаАнализаДанных** или создание **модели прогноза** для прогноза по новым данным.

### 14.2.1. Основные объекты

**Источник данных** — результат запроса, таблица значений или область ячеек табличного документа.

**АнализДанных** — объект, выполняющий анализ. Задаются: источник данных, тип анализа, параметры, настройка колонок. Результат — объект результата анализа (тип зависит от типа анализа).

**Типы результата анализа:**
- РезультатАнализаДанныхОбщаяСтатистика
- РезультатАнализаДанныхПоискАссоциаций
- РезультатАнализаДанныхПоискПоследовательностей
- РезультатАнализаДанныхДеревоРешений
- РезультатАнализаДанныхКластеризация

**Настройка колонок анализа данных** — коллекция входных колонок: тип данных, роль колонки, дополнительные настройки (зависят от типа анализа).

**Результат анализа данных** — специальный объект с результатом. Может выводиться в табличный документ через ПостроительОтчетаАнализаДанных, использоваться программно или для создания модели прогноза, сохраняться.

**Модель прогноза** — объект для прогноза по новым данным. Создаётся из результата анализа. Тип модели соответствует типу анализа (например, МодельПрогнозаПоискАссоциаций). Вход: источник данных для прогноза. Выход: таблица значений с прогнозируемыми значениями.

**Настройка входных колонок** — соответствие колонок модели прогноза и колонок выборки прогноза.

**Настройка колонок результата** — управление составом колонок в результирующей таблице модели прогноза.

### 14.2.2. Типы анализа данных

- **Общая статистика** — сбор сводных характеристик по полям источника (дискретные и непрерывные), предварительное исследование.
- **Поиск ассоциаций** — частые группы объектов/значений и правила ассоциаций; поддержка иерархии и объектного/событийного источника.
- **Поиск последовательностей** — цепочки событий во времени (шаблоны последовательностей), в т.ч. по иерархии.
- **Кластерный анализ** — разбиение объектов на кластеры по сходству; параметры: количество кластеров, мера расстояния, метод кластеризации; результат можно вывести в дендрограмму.
- **Дерево решений** — иерархия классифицирующих правил; целевой и входные атрибуты; результат — дерево условий и ошибки классификации.

### 14.2.3. Модели прогноза

Модель создаётся из результата анализа и позволяет выполнять прогноз для новых данных. Пример: модель по поиску ассоциаций по покупкам — на входе текущая накладная, на выходе рекомендуемые товары с вероятностью.

## 14.3. Тип анализа «Общая статистика»

Назначение: предварительный анализ данных. Источник — результат запроса, таблица значений или область ячеек табличного документа.

**Виды данных (по колонкам):** непрерывные (Число, Дата) и дискретные (остальные). Анализ по полям выполняется независимо (без взаимной корреляции).

**Характеристики дискретных данных:** количество значений, количество уникальных значений, мода, частота, относительная частота, накопленная частота, накопленная относительная частота.

**Характеристики непрерывных данных:** количество значений, минимум, максимум, среднее, размах, стандартное отклонение, медиана.

При выводе в табличный документ используются круговые диаграммы для состава полей.

**Пример выполнения анализа и вывода:**

```bsl
Анализ = Новый АнализДанных;
Анализ.ТипАнализа = Тип("АнализДанныхОбщаяСтатистика");
Запрос = Новый Запрос;
Запрос.Текст = "ВЫБРАТЬ Продажи.Номенклатура, Продажи.Количество ИЗ РегистрНакопления.Продажи КАК Продажи";
Анализ.ИсточникДанных = Запрос.Выполнить();
РезультатАнализа = Анализ.Выполнить();
Построитель = Новый ПостроительОтчетаАнализаДанных();
Построитель.Макет = Неопределено;
Построитель.ТипАнализа = Тип("АнализДанныхОбщаяСтатистика");
ТабДок = Новый ТабличныйДокумент;
Построитель.Вывести(РезультатАнализа, ТабДок);
```

## 14.4. Тип анализа «Поиск ассоциаций»

Поиск часто встречаемых вместе групп и правил ассоциаций. Источник: запрос, таблица значений, область ячеек табличного документа.

**Роли колонок источника:**
- **НеИспользуемая** — игнорируется.
- **Объект** — по значению колонки данные относят к одной группе (например, регистратор документа).
- **Элемент** — значения, по которым строятся устойчивые группы и правила ассоциаций.

**Параметры анализа:**
- МинимальныйПроцентСлучаев — минимальный процент случаев для комбинации.
- МинимальнаяДостоверность — минимальный процент соблюдения правила.
- МинимальнаяЗначимость — порог значимости для групп.
- ТипОтсеченияПравил — перечисление ТипОтсеченияПравилАссоциации: Избыточные, Покрытые.

**Результат:** общая информация (количество групп, правил), группы элементов (состав, количество и процент случаев), ассоциативные правила (предпосылка, следствие, процент случаев, достоверность, значимость).

**ТипОтсеченияПравил:** Покрытые — отсекаются правила, покрытые другими по предпосылке или по следствию (без учёта вероятностей). Избыточные — отсекаются правила, избыточные по предпосылке (та же достоверность, что у покрывающего) или по следствию (то же количество случаев, что у покрывающего).

**Пример установки параметра и выполнения:**

```bsl
Анализ.Параметры.ТипОтсеченияПравил.Значение = ТипОтсеченияПравилАссоциации.Избыточные;
РезультатАнализа = Анализ.Выполнить();
Построитель = Новый ПостроительОтчетаАнализаДанных();
Построитель.Макет = Неопределено;
Построитель.ТипАнализа = Тип("АнализДанныхПоискАссоциаций");
ТабДок = Новый ТабличныйДокумент;
Построитель.Вывести(РезультатАнализа, ТабДок);
```

## 14.5. Тип анализа «Поиск последовательностей»

Выявление цепочек событий (шаблонов последовательностей). Источник: запрос, таблица значений, область ячеек табличного документа.

**Роли колонок:**
- **НеИспользуемая** — игнорируется.
- **Последовательность** — объект цепочки (по значению данные относят к одной последовательности).
- **Элемент** — элемент последовательности.
- **Время** — время наступления события (обязательная колонка).

**Параметры (основные):**
- МинимальныйПроцентСлучаев — минимальный процент последовательностей с шаблоном.
- МинимальныйИнтервал — минимальный интервал между событиями (единица измерения, кратность).
- ИнтервалЭквивалентностиВремени — интервал эквивалентности времени (единица, кратность).
- МинимальнаяДлина — минимальная длина последовательности.
- ПоискПоИерархии — для колонок типа Элемент.

**ТипЕдиницыИнтервалаВремениАнализаДанных:** Секунда, Минута (ТекущаяМинута), Час (ТекущийЧас), День (ТекущийДень), Неделя (ТекущаяНеделя), Декада (ТекущаяДекада), Месяц (ТекущийМесяц), Квартал (ТекущийКвартал), Полугодие (ТекущееПолугодие), Год (ТекущийГод).

**Результат:** шаблоны последовательностей — состав, количество случаев, мин/макс/средние интервалы между событиями, процент случаев.

## 14.6. Тип анализа «Дерево решений»

Построение иерархии классифицирующих правил. Нужны: целевой атрибут (по которому строится классификатор) и входные атрибуты.

**Роли колонок:** НеИспользуемая, Прогнозируемая (целевой атрибут).

**Параметры:**
- МинимальноеКоличествоСлучаев — минимум элементов в узле.
- МаксимальнаяГлубина — максимальная глубина дерева.
- ТипУпрощения — тип упрощения дерева (перечисление ТипУпрощенияДереваРешений: НеУпрощать, Упрощать).

**Результат:** дерево решений (каждый узел — условие; переход по ответам от корня к листу), ошибки классификации.

При **Упрощать** узлы по правилам превращаются в листья. Учитываются: Ошибок, ОшибокДочерних, Листов, Случаев; условие упрощения задаётся формулой платформы (например, сравнение (Ошибок + 0,5) и Листов).

**Пример:**

```bsl
Анализ.ИсточникДанных = Запрос.Выполнить();
Анализ.Параметры.ТипУпрощения.Значение = ТипУпрощенияДереваРешений.НеУпрощать;
РезультатАнализа = Анализ.Выполнить();
Построитель = Новый ПостроительОтчетаАнализаДанных();
Построитель.Макет = Неопределено;
Построитель.ТипАнализа = Тип("АнализДанныхДеревоРешений");
ТабДок = Новый ТабличныйДокумент;
Построитель.Вывести(РезультатАнализа, ТабДок);
```

## 14.7. Тип анализа «Кластеризация»

Группировка объектов в кластеры по сходству; в основе — вычисление расстояния между объектами. Источник: запрос, таблица значений, область ячеек. Колонки — входные или неиспользуемые (ТипКолонкиАнализаДанныхКластеризация). Для вывода распределения объектов по кластерам нужно задать параметр **ТипЗаполненияТаблицы** перед выполнением анализа.

**Метрики расстояния (ТипМерыРасстоянияАнализаДанных):**
- Евклидова метрика: √(Σ Wi(Xi − Yi)²)
- Евклидова метрика в квадрате: Σ Wi(Xi − Yi)²
- Метрика города (Манхэттен): Σ Wi|Xi − Yi|
- Метрика доминирования: max Wi|Xi − Yi|

Здесь Xi, Yi — значения атрибутов двух объектов, Wi — весовой коэффициент атрибута (в настройке колонки), i = 1..n.

**Методы кластеризации (МетодКластеризации):**
- **Ближняя связь** — объект присоединяется к группе с минимальным расстоянием до ближайшего объекта группы.
- **Дальняя связь** — к группе с минимальным расстоянием до наиболее дальнего объекта группы.
- **Центр тяжести** — к группе с минимальным расстоянием до центра тяжести группы.
- **k-средних** — задаются начальные центры (первые объекты выборки); объекты распределяются по расстоянию до центров; центры пересчитываются; итерации до стабилизации.

**Дендрограмма:** при использовании алгоритма, отличного от k-средних, и при настройке вывода распределения по кластерам результат можно вывести в виде дендрограммы.

**Пример (метрика и метод):**

```bsl
Анализ.Параметры.МераРасстояния.Значение = ТипМерыРасстоянияАнализаДанных.ЕвклидоваМетрикаВКвадрате;
Анализ.Параметры.МетодКластеризации.Значение = МетодКластеризации.КСредних;
РезультатАнализа = Анализ.Выполнить();
Построитель = Новый ПостроительОтчетаАнализаДанных();
Построитель.Макет = Неопределено;
Построитель.ТипАнализа = Тип("АнализДанныхКластеризация");
ТабДок = Новый ТабличныйДокумент;
Построитель.Вывести(РезультатАнализа, ТабДок);
```

Увеличение веса колонки: через настройку колонок анализа, например для колонки «КоличествоАвтомобилей». Для типов Число и Дата по умолчанию вид данных — Непрерывные, для остальных — Дискретные; при кластеризации допускаются и дискретные колонки.
