# OpenSpec: рг_УниверсальныйОбменXML

Универсальная подсистема обмена данными между информационными базами 1С по правилам обмена через SOAP-веб-сервис: старт на ЗУП КОРП, целевой контур — любая конфигурация на БСП.

## Продукт

**Название проекта:** рг_УниверсальныйОбменXML

**Целевая конфигурация (текущий этап):** 1С:Зарплата и управление персоналом КОРП, редакция 3.1 (версия 3.1.30.197)

**Стратегическая цель:** унифицированный транспорт данных между информационными базами 1С для любой конфигурации на БСП (Библиотека стандартных подсистем); ЗУП — первая площадка разработки и отладки.

**Расширение:** `рг_УниверсальныйОбменXML` («Универсальный обмен XML (РГ)»), префикс `обменXML_`, версия 1.0, путь `src/ЗУП/cfe/рг_УниверсальныйОбменXML/`

**Краткое описание:** Сервис обмена данными между базами 1С через XML-транспорт и правила обмена в макетах: база-приёмник инициирует загрузку, база-источник выгружает по запросу веб-сервиса `rgExchangeService`.

## Назначение

**Проблема:** Обмен данными между базами 1С выполняется разрозненно: нет единого клиент-серверного механизма, где база-приёмник инициирует загрузку у базы-источника через веб-сервис. Правила выборки и фильтрации не стандартизированы (макеты правил обмена на источнике), настройки подключения и параметры фильтрации не собраны в одном профиле, пароли нельзя хранить открытым текстом.

**Цель:** Универсальная подсистема обмена по правилам через SOAP-веб-сервис `rgExchangeService`: база-источник публикует сервис и выгружает данные по правилам из макета; база-приёмник хранит профили в справочнике `ргНастройкиОбменаXML`, вызывает `GetData`, получает ZIP с XML и загружает стандартными механизмами универсального обмена. Первый контур — ЗУП КОРП; перспектива — перенос ядра на любую конфигурацию с БСП.

**Аудитория:**

- разработчики и интеграторы 1С — настройка правил, макетов, профилей обмена;
- пользователи — ручной запуск обмена через обработку `ргУниверсальныйОбменXML`;
- администраторы баз — регламентный обмен, мониторинг подключений, безопасное хранение паролей.

## Область

### Ключевые возможности

- Я настраиваю профиль обмена в справочнике `ргНастройкиОбменаXML`: адрес веб-сервиса, пользователь, имя макета правил на источнике, произвольные параметры фильтрации (строка, число, дата, булево); пароль хранится в безопасном хранилище, не в открытом виде.
- Я инициирую обмен из базы-приёмника (вручную через обработку или регламентным заданием): система читает профиль, подключается к SOAP-веб-сервису `rgExchangeService` и вызывает `GetData`.
- Я передаю в веб-сервис имя макета правил и структуру параметров — на источнике они подставляются в правила обмена перед выгрузкой.
- На базе-источнике обработка `ргУниверсальныйОбменXML` загружает правила из указанного макета, активирует все правила выгрузки данных (ПВД), формирует XML и упаковывает его в ZIP для передачи.
- Я получаю ZIP-архив с XML, распаковываю во временный каталог и загружаю данные стандартными механизмами универсального обмена 1С.
- Я настраиваю преобразование примитивных параметров в ссылочные отборы в обработчиках правил обмена — ядро системы ссылки автоматически не ищет.

### Принципы

- **Клиент инициирует, сервер отдаёт** — база-приёмник запускает сеанс обмена и вызывает веб-сервис; база-источник только выбирает и выгружает данные по запросу.
- **Правила в макетах, не в коде ядра** — логика выборки и конвертации задаётся правилами обмена (.xml) в макетах конфигурации-источника; ядро подставляет параметры и запускает ПВД.
- **Ссылки — в обработчиках правил** — преобразование примитивных параметров в ссылочные отборы выполняют только обработчики правил обмена (КД 2.1); система автоматический поиск объектов не делает.
- **Безопасность учётных данных** — пароль подключения к источнику хранится в безопасном хранилище БСП, не в реквизитах справочника.
- **Стандартная загрузка на приёмнике** — после распаковки ZIP загрузка идёт штатными механизмами универсального обмена 1С.
- **ЗУП — первая площадка, БСП — целевой контур** — разработка и отладка на ЗУП КОРП; архитектура закладывается переносимой на конфигурации с БСП.

### Техническая область

**Состав объектов расширения:**

| Объект | Назначение |
|--------|------------|
| Справочник `ргНастройкиОбменаXML` | Профили обмена (URL, логин, имя макета); пароль через БСП; управление регламентными заданиями |
| Веб-сервис `rgExchangeService` | SOAP, `GetData`: XDTO `Structure` → Base64 ZIP с выгрузкой |
| Обработка `ргУниверсальныйОбменXML` | На базе штатной `УниверсальныйОбменДаннымиXML`; макеты правил (XML); выгрузка/загрузка на всех базах |
| Общий модуль `ргУниверсальныйОбменXMLСервер` | Методы веб-сервиса, выгрузка/загрузка, обработчик регламентного задания |
| Регламентное задание `ргВыполнениеОбменаXML` | Шаблон программных заданий обмена |

**Алгоритм источника:** `GetData` → `Обработки.ргУниверсальныйОбменXML.Создать()` → правила из макета обработки → параметры из XDTO → выгрузка → ZIP → Base64.

**Алгоритм приёмника:** настройки + пароль из безопасного хранилища → XDTO из ТЧ → WS-прокси → распаковка ZIP → загрузка через `Обработки.ргУниверсальныйОбменXML.Создать()`.

## Внешние зависимости

**Базовая конфигурация:** 1С:Зарплата и управление персоналом КОРП, редакция 3.1 (3.1.30.197).

**Платформенные механизмы 1С:**

- SOAP-веб-сервис `rgExchangeService`, метод `GetData`;
- универсальный обмен данными (правила обмена .xml, ПВД — правила выгрузки данных);
- XDTO-пакет `http://v8.1c.ru/8.1/data/core`, тип `Structure`;
- безопасное хранилище паролей БСП.

**Топология:** база-источник (публикует веб-сервис, хранит макеты правил) и база-приёмник (профили, инициатор обмена).

**Сеть:** HTTP(S)-доступ приёмника к URL опубликованного веб-сервиса источника.

**Серверное ПО:** веб-сервер для публикации SOAP-сервиса на стороне источника (конкретная связка — уточняется при внедрении).

- Редактируем только файлы в расширении `src/ЗУП/cfe/рг_УниверсальныйОбменXML/`.
