# Test-case: extend B2 — drift-warning

Регрессия брифа с развилкой для `/opsx:extend` при расширении scope.

## Вход

```
/opsx:extend signing-contract-sign-verify-bytes добавить поддержку локального УНЭП наравне с облачным для исключения полей подписанта
```

Предусловие: в `design.md` Non-Goals или Why ограничивают scope облачным УНЭП → `Drift-check: drift-warning`.

## Контрольные точки

| # | Ожидание | FAIL если |
|---|----------|-----------|
| 1 | Бриф **B2** карточка по `brief-card.md` | B3 с полным исследовательским маршрутом |
| 2 | Слоты **Изменение**, **Почему выбор**, **Варианты** нумерованно (`1.` `2.`) | Нет развилки при drift-warning; варианты **A/B** буквами |
| 3 | `Drift-check` только в internal → `debug.md` | `Drift-check:` в чате |
| 4 | Нет «Как буду искать» / плана артефактов | Нумерованный список proposal/design/tasks |
| 5 | **Подтвердить?** просит номер варианта; голый «да» не запускает работу | «A» / «B» в Подтвердить? |

## Эталон B2

```markdown
Бриф: signing-contract-sign-verify-bytes

**Изменение** — поддержка локального УНЭП наравне с облачным для исключения полей подписанта.

**Почему выбор** — локальные сертификаты сейчас вне области задачи; без выбора нельзя зафиксировать scope в требованиях.

**Варианты**
1. Только облачный УНЭП — поведение для локальных не меняется.
2. Оба типа УНЭП — шире охват, но появятся отдельные условия в требованиях.

Подтвердить? — укажите номер варианта (1 или 2) или уточните.
```
