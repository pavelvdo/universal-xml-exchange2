# Глава 31. Отладка и тестирование прикладных решений

> **Agent ref:** отладка, тестирование, отладочный режим, точки останова, пошаговое выполнение, стек вызовов, протоколы отладки (TCP/IP, HTTP), модульные тесты.

## 31.1. Общая информация

Отладка — обнаружение, локализация и устранение ошибок. Инструменты: отладчик, замер производительности, отображение вызовов сервера, имитация задержек, автоматизированное тестирование.

## 31.2. Отладчик

Возможности: отладка модулей в файловом и клиент-серверном варианте, фоновых заданий; пошаговое выполнение; точки останова; прерывание и продолжение; отладка нескольких модулей; вычисление выражений; изменение переменных; просмотр стека вызовов; остановка по ошибке; редактирование модуля во время отладки.

**Предмет отладки** — контекст встроенного языка: пользователь, тип предмета, сеанс, сетевое имя (TCP/IP), порт (TCP/IP), имя ИБ (HTTP). Типы: Тонкий клиент, Толстый клиент, Сервер, Сервер (файловый), Веб-клиент, Фоновое задание (и файловый вариант), COM-соединение, HTTP/Web-сервис, OData, мобильные (Клиент, Сервер, Мобильный клиент, Автономный сервер, Фоновое задание — при протоколе HTTP). При переназначении сеанса на другой рабочий процесс (кластер) отладка становится невозможной.

**Протоколы:** TCP/IP и HTTP. Порты TCP/IP: 1560–1591 (по умолчанию); HTTP: порт сервера отладки (явно или 1550 в клиент-серверном; 1560–1591 в файловом). Настройка диапазона — debugcfg.xml и параметры компонентов. Закрытые порты делают отладку невозможной или замедляют запуск/работу.

**Запуск отладки:** Отладка → Начать отладку (конфигуратор запускает приложение в отладочном режиме). Или Сервис → 1С:Предприятие при разрешённой отладке в настройках. Для отладки сервера — ключ `-debug` при запуске сервера.

## 31.3. Точки останова, пошаговое выполнение, выражения

Точки останова расставляются в редакторе модуля. Пошаговое выполнение: шаг с заходом, шаг с обходом, шаг с выходом. Вычисление выражений — в окне отладчика; изменение значений переменных во время остановки. Стек вызовов отображает цепочку процедур и функций.

## 31.4. Тестирование

Платформа предоставляет механизмы автоматизированного тестирования: сценарии и проверка соответствия спецификациям.
