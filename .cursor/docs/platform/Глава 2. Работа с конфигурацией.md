# Глава 2. Работа с конфигурацией

> Agent ref: Работа с конфигурацией: дерево объектов, обновление БД, выгрузка/загрузка.
> Ключевые разделы: 2.3 Сохранение, 2.7 Дерево объектов, 2.14 Обновление БД, 2.17 Выгрузка в файлы
> ---

## 2.1. Две конфигурации

В информационной базе хранятся **основная конфигурация** (редактируемая) и **конфигурация базы данных** (определяет структуру таблиц и функциональность для пользователей). Конфигурацию БД в процессе работы пользователей изменить нельзя. Расширения редактируются отдельно от основной конфигурации.

## 2.2. Сохранение и закрытие

- Сохранение возможно при запущенном 1С:Предприятие и при отладке. Признак изменений — «*» в заголовке.
- **Сохранение в файл**: конфигурация сохраняется в файл с расширением `.cf`.
- **Загрузка из файла**: полная замена текущей конфигурации.

## 2.3. Дерево объектов конфигурации

Конфигурация — древовидная структура. Корневые ветви группируют объекты по типам (Документы, Справочники, Общие → Роли и т.д.). Объекты в группе можно упорядочивать. **Поиск**: по имени в открытых ветках; глобальный поиск — от двух символов. Перетаскивание имён объектов в модули и текстовые документы вставляет ссылку на объект.

## 2.4. Создание и удаление объектов

- **Создание**: новому объекту присваивается условное имя (тип + порядковый номер).
- **Копирование** перетаскиванием в другую ветвь: копируются свойства; при совпадении типа — реквизиты, формы, макеты, табличные части.
- **Удаление**: при наличии ссылок выводится список использований; при включённой настройке выполняется проверка использования в модулях.

## 2.5. Ссылки на объект

Поиск ссылок на объект и поиск ссылок в объекте. Задаётся область поиска (свойства, подчинённые объекты). Результат — в окне сообщений с переходом к объекту.

## 2.6. Свойства объекта и переименование

Свойства группируются по категориям: Основные, Подсистемы, Данные, Команды, Формы, Макеты, Права, Планы обмена, Прочее. Закладки зависят от типа объекта: справочник — Иерархия, Владельцы, Нумерация; документ — Нумерация, Движения, Журналы; регистры — Регистраторы и т.д. **Переименование**: поиск вхождений по конфигурации (строковые константы, запросы, тексты RLS); замена вручную или автоматическая.

## 2.7. Конфигурация базы данных и обновление

Конфигурация БД доступна только для просмотра. Отличие основной конфигурации от конфигурации БД отображается в заголовке символами `<!>`.

**Обновление конфигурации БД**: реструктуризация создаёт или изменяет таблицы, индексы, поля. При несовместимых изменениях возможна **потеря данных** — в диалоге перечисляются планируемые действия. Режимы: с диалогом подтверждения и без диалога (обновление на сервере).

## 2.8. Выгрузка в файлы

**Выгрузка в файлы** — сохранение конфигурации в виде набора файлов в каталоге (формат для СКВ). **Загрузка из файлов** — восстановление конфигурации из такого каталога.
