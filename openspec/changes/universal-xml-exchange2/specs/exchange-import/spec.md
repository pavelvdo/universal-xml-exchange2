## ADDED Requirements

### Requirement: Подтверждение доставки ConfirmDataReceipt

После успешной загрузки данных на базе-приёмнике система SHALL вызывать операцию `ConfirmDataReceipt` веб-сервиса источника с префиксами, идентифицирующими сеанс на источнике.

#### Scenario: Успешный цикл с подтверждением

- **WHEN** приёмник успешно загрузил данные из полученного ZIP
- **THEN** система вызывает `ConfirmDataReceipt` и профиль-источник переходит в статус «Выполнено»

#### Scenario: Ошибка загрузки без подтверждения

- **WHEN** загрузка на приёмнике завершилась ошибкой
- **THEN** система не вызывает `ConfirmDataReceipt` и информирует пользователя или записывает ошибку в журнал

#### Scenario: Понятное сообщение об ошибке веб-сервиса

- **WHEN** при вызове `GetData` или `ConfirmDataReceipt` источник возвращает ошибку (например недопустимый статус профиля или ошибка загрузки правил обмена на источнике)
- **THEN** форма обработки на приёмнике показывает пользователю **читаемый** краткий бизнес-текст с источника (кириллица без «?»), без стека вызовов, без служебных обёрток платформы и без фрагмента протокола движка

## MODIFIED Requirements

### Requirement: Чтение настроек профиля

Система SHALL считывать настройки из выбранного элемента справочника `рг_НастройкиОбменаXML` профиля-приёмника: адрес сервиса, логин, учётные данные, префиксы для идентификации сеанса на источнике.

#### Scenario: Подготовка сеанса по профилю приёмника

- **WHEN** начинается сеанс обмена для профиля с `Источник = Ложь`
- **THEN** используются URL, логин, пароль из безопасного хранилища и префиксы для вызова веб-сервиса

### Requirement: Вызов веб-сервиса и передача параметров

Система SHALL устанавливать соединение с веб-сервисом источника, используя сохранённые адрес и учётные данные, и вызывать `GetData` с префиксами профиля. Параметры фильтрации MUST NOT передаваться из приёмника — они задаются на профиле-источнике.

#### Scenario: Успешный запрос данных

- **WHEN** приёмник вызывает `GetData` с корректными префиксами и учётными данными
- **THEN** получен ответ Base64 с ZIP-архивом, содержащим правила и данные

### Requirement: Распаковка и загрузка XML

Полученный ZIP-архив SHALL распаковываться во временный каталог; загрузка данных из XML SHALL выполняться встроенной обработкой `рг_УниверсальныйОбменДаннымиXML` расширения (`Обработки.рг_УниверсальныйОбменДаннымиXML.Создать()`).

#### Scenario: Загрузка после получения архива

- **WHEN** ZIP успешно получен и распакован
- **THEN** данные загружаются в информационную базу-приёмник через объектный модуль обработки расширения
