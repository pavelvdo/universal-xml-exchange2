## 12.1. Общая информация

Периодические расчеты — вычисления, выполняемые с заданной периодичностью, связанные по правилам и взаимно влияющие друг на друга в пределах периода. Механизмы периодических расчетов задают порядок и взаимосвязь вычислений и хранение их результатов. Типичный пример — расчет заработной платы (начисления и удержания) с месячной периодичностью, когда результаты одних видов расчетов зависят от других.

Ведение периодических расчетов обеспечивают объекты конфигурации **Планы видов расчета** и **Регистры расчета**; они связаны между собой.

## 12.2. Основные понятия

**Период.** Для расчетов период задается датой начала и датой окончания. При заданной периодичности регистра для описания периода (действия, регистрации) достаточно указать любую дату — по ней вычисляется дата начала периода; это позволяет оптимизировать запросы по периоду.

**Периодичность расчетов.** Задается в свойстве «Периодичность» регистра расчетов. По ней (для периодического регистра) определяется период действия записи. Например, при периодичности «Месяц» период действия записи вычисляется по дате документа (например, 01.11.2008 за ноябрь 2008 г.).

**Период регистрации** — дата начала периода, в котором зарегистрирован расчет (вычисляется по дате документа-регистратора). Пример: в июне 2008 г. начислен оклад за май 2008 г.; май — период действия (в БД 01.05.2008), июнь — период регистрации (в БД 01.06.2008).

**Период действия** — начальная дата периода по свойству «Периодичность» регистра. Для даты «май 2008 г.»: при периодичности «Месяц» — 01.05.2008, при «Квартал» — 01.04.2008.

**Период действия расчета** — период, за который выполняется расчет (дата начала и дата окончания). Например, больничный за май 2008 г. имеет период действия 01.05.2008, а период действия расчета — с 06.05.2008 по 15.05.2008.

**Вытеснение (конкуренция за период действия).** Связь видов расчетов по периоду: несколько видов не могут действовать одновременно в одном периоде; выбирается один расчет. Настраивается в описании вида расчета в разделе «Вытесняющие (виды расчетов)». Пример: «Оплата по больничному листу» вытесняет «Оплата по окладу» — за период больничного оклад не начисляется.

**Фактический период действия.** При наличии вытесняющих видов фактический период — совокупность непересекающихся периодов, в которых данный расчет не был вытеснен. Это вторичные данные; пересчет выполняется при любой записи набора, в том числе пустой (отмена проведения). Результат расчетов не зависит от последовательности ввода данных.

**Базовый период** — период, за который выбираются результаты расчетов, используемых как базовые для данного вида. Пример: премия за май 2008 г. может опираться на Оклад, Доплату, Отпуск за некоторый период; этот период — базовый для премии. Связь по базовому периоду настраивается в разделе «Базовые (виды расчетов)» вида расчета.

## 12.3. Планы видов расчета

Объекты этого вида задают виды расчетов, используемые в регистрах расчета. **Ссылка на вид расчета** — одно из основных свойств записей регистра расчета, задающее качественное различие учетных записей. Редактирование выполняется в окне редактирования и в палитре свойств.

### 12.3.1. Категория свойств «Расчет»

**Зависимость от базы.** Если свойство не «Не зависит», видам расчета можно задать список базовых видов расчета; состав задается свойством «Базовые планы видов расчета».

**Базовые планы видов расчета** — список планов видов расчета, виды которых могут быть базовыми для видов расчета данного плана.

**Использует период действия.** Если установлено, план может быть назначен регистру расчета с периодом действия; для каждого вида расчета задается список вытесняющих видов расчета, определяющий механизм вытеснения регистра. В формах вида расчета при «Зависимость от базы» = «Зависит от периода действия (регистрации)» можно разместить табличное поле для выбора базовых видов расчета; при включенном «Использует период действия» — табличное поле для вытесняющих видов расчета.

### 12.3.2. Предопределенные виды расчета

План видов расчета может содержать предопределенные виды расчета (создаются при конфигурировании). В режиме 1С:Предприятие их можно редактировать. Пометка удаления и удаление видов расчета (включая созданные в конфигураторе) регулируются правами доступа. Редактирование списка предопределенных — палитра свойств, закладка «Прочее», кнопка «Предопределенные»; состав реквизитов (базовые и ведущие) зависит от свойств плана в категории «Расчет».

В форме вида расчета: закладка «Основное» — основные свойства; при зависимости от базы по периоду действия (регистрации) — закладка «Базовые» (указываются базовые виды расчета); при «Использует период действия» — закладка «Вытесняющие» (виды расчета, вытесняющие данный по периоду действия); закладка «Ведущие» — виды расчета, задающие механизм перерасчетов регистра. Пользователь может создавать собственные виды расчета; в конфигураторе нужна форма элемента с табличными полями для базовых, ведущих и вытесняющих видов в соответствии со свойствами плана.

## 12.4. Регистры расчета

### 12.4.1. Общая информация о регистрах расчета

**Регистр расчета** — объект конфигурации для учета результатов вычислений, выполняемых расчетами по плану видов расчетов (например, начисления оплаты труда, больничных, отпусков). Записи регистра могут влиять друг на друга двумя способами: конкуренция за период действия и зависимость по базовому периоду.

**Конкуренция за период действия.** Движения не могут относиться к одному и тому же промежутку времени (например, оклад и отпуск). Реализуется через период действия записи; у каждой записи есть фактический период действия — набор непересекающихся промежутков. Пример: отпуск 03.03.2008–13.03.2008; оклад может иметь фактический период 01.03.2008–02.03.2008 и 14.03.2008–20.03.2008. Поддержка — механизм вытеснения (состав вытесняющих видов расчета, даты периода действия и т.д.).

**Зависимость по базовому периоду.** Изменения записей одного вида в некотором периоде требуют пересмотра зависимых записей (например, средний заработок зависит от оклада в базовом периоде). Задается **базовый период записи**; изменения базовых записей в этом периоде приводят к необходимости перерасчета зависимой записи.

Работа с регистрами расчета — ветвь «Регистры расчета» дерева конфигурации. При редактировании задаются: план видов расчета; поддержка периода действия и базового периода; периодичность; структура — наборы измерений, ресурсов и реквизитов; при необходимости — экранные и печатные формы движений. В качестве типа измерения регистра расчета не допускаются УникальныйИдентификатор, ДвоичныеДанные, строка неограниченной длины.

#### 12.4.1.1. Механизм вытеснения

При включенном свойстве **Период действия** взаимное влияние движений носит конкурирующий характер: движения исключают друг друга во времени; ввод одного приводит к исключению другого. Регистр с периодом действия имеет ограничение на количество записей с одинаковыми измерениями, периодом регистрации и видом расчета. При большом числе одинаковых записей, участвующих в расчете фактического периода действия, возможна ошибка нехватки памяти (ограничение на число записей в одном отрезке времени); на практике не проявляется.

**График** — ссылка на регистр сведений с временной схемой исходных данных (доступно при «Период действия»). Указывается для расчетов, зависящих от данных, распределенных по правилу внутри периода (график рабочего времени, часы и т.п.). **Значение графика** — ресурс этого регистра сведений (например, признак «рабочий день» или число рабочих часов). **Дата графика** — измерение типа «Дата» того же регистра для привязки к значениям ресурса.

#### 12.4.1.2. Механизм перерасчетов

**Базовый период.** При установленном свойстве взаимное влияние движений связано (например, средний заработок от начислений в базовом периоде).

**Периодичность** — период регистрации движений и период, в пределах которого движения влияют друг на друга (для регистров с периодом действия).

**Перерасчеты** — подчиненные объекты регистра, задающие правила взаимного влияния движений. В свойстве «Измерение регистра» (группа «Связь») указывается основное измерение текущего регистра, пересчитываемое при изменении данных ведущих регистров из свойства «Данные ведущих регистров» (например, перерасчет удержания при изменении начислений). При включенном свойстве «Базовый период» формирование данных перерасчетов выполняется автоматически; иначе — вручную (нужна форма ввода перерасчетов и механизм выполнения).

### 12.4.2. Редактирование регистра расчета

Свойства регистра расчета и подчиненные объекты редактируются в окне редактирования. **План видов расчета** — основная характеристика регистра; для регистра расчета указывается только один план видов расчета.
