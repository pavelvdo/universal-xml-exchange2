## ADDED Requirements

### Requirement: Инициация загрузки пользователем или регламентом

На базе-приёмнике загрузка SHALL инициироваться пользователем через форму обработки `ргУниверсальныйОбменXML` или регламентным заданием по профилю. Допускается программный вызов экспортного API серверного модуля.

#### Scenario: Ручной запуск обмена

- **WHEN** пользователь выбирает профиль обмена и запускает обмен из формы обработки
- **THEN** система выполняет полный цикл получения и загрузки данных

#### Scenario: Регламентный запуск

- **WHEN** срабатывает регламентное задание, связанное с профилем
- **THEN** система выполняет обмен по настройкам этого профиля без участия пользователя

### Requirement: Чтение настроек профиля

Система SHALL считывать настройки из выбранного элемента справочника `ргНастройкиОбменаXML`, включая параметры табличной части и учётные данные.

#### Scenario: Подготовка сеанса по профилю

- **WHEN** начинается сеанс обмена для элемента справочника
- **THEN** используются URL, логин, имя макета и параметры из этого элемента

### Requirement: Вызов веб-сервиса и передача параметров

Система SHALL устанавливать соединение с веб-сервисом источника, используя сохранённые адрес и учётные данные, и передавать имя макета и структуру параметров из табличной части профиля.

#### Scenario: Успешный запрос данных

- **WHEN** приёмник вызывает `GetData` с корректными настройками
- **THEN** получен ответ Base64 с ZIP-архивом выгрузки

### Requirement: Распаковка и загрузка XML

Полученный ZIP-архив SHALL распаковываться во временный каталог; загрузка данных из XML SHALL выполняться обработкой `ргУниверсальныйОбменXML` расширения (единый движок на всех базах участников обмена).

#### Scenario: Загрузка после получения архива

- **WHEN** ZIP успешно получен и распакован
- **THEN** данные загружаются в информационную базу-приёмник через `Обработки.ргУниверсальныйОбменXML.Создать()`

### Requirement: Обработка ошибок соединения

Система SHALL обрабатывать ошибки соединения с веб-сервисом и некорректный ответ: информировать пользователя или записывать ошибку в журнал регистрации при регламентном запуске.

#### Scenario: Недоступность веб-сервиса

- **WHEN** при вызове веб-сервиса соединение не установлено или возвращена ошибка
- **THEN** сеанс обмена завершается с сообщением об ошибке; загрузка не выполняется
