# Глава 16. Работа с различными форматами данных

> **Agent ref:** XML (XDTO), JSON, двоичные данные, PDF, регулярные выражения, речь, архивы. Ключевые разделы: XDTO (фабрика, типы, сериализация), JSON (простые/объектные/потоковые техники), ЧтениеДанных/ЗаписьДанных, БуферДвоичныхДанных, ЗаписьPDF/ЧтениеPDF, ЭЦП, regex (ICU), распознавание/синтез речи, ЗаписьФайлаАрхива/ЧтениеФайлаАрхива.

## 16.1. Механизм XDTO

XDTO (XML Data Transfer Objects) — универсальное представление данных для обмена с внешними системами и Web-сервисами. Модель типов и значений позволяет манипулировать данными в 1С и прозрачно преобразовывать их в XML.

**Назначение:** обмен между конфигурациями с разными схемами; обмен по произвольным XML-схемам; описание типов параметров и возвращаемых значений Web-сервисов.

### 16.1.1. Фабрика XDTO

**Фабрика XDTO** — описание всех типов системы. Создаётся на основе набора схем XML (объект `НаборСхемXML`) через конструктор `ФабрикаXDTO`. Глобальная фабрика доступна как `ФабрикаXDTO` в глобальном контексте.

- Типы сгруппированы в **пакеты XDTO** (аналог схемы XML). Фабрика самодостаточна: типы могут ссылаться только на типы той же фабрики.
- Получение фабрики: из файла XSD (ЧтениеXML → ДокументDOM → построитель схемы → СхемаXML → НаборСхемXML → ФабрикаXDTO); из нескольких источников (файлы + пакеты конфигурации); из объекта данных XDTO.
- При формировании графа объектов XDTO используется один экземпляр фабрики; смешивание объектов/значений от разных фабрик недопустимо. Экземпляр фабрики значения: `ЗначениеXDTO.Фабрика()`.

### 16.1.2. Типы данных XDTO

- **ТипЗначенияXDTO** — простые неделимые значения (строки, числа, даты). Определяется: ограничением (БазовыйТип + Фасеты), объединением (ТипыЧленовОбъединения), списком (ТипЭлементаСписка). Фасеты: Длина, МаксДлина, МинДлина, МаксВключающее/МаксИсключающее, МинВключающее/МинИсключающее, Образец, Перечисление, ПробельныеСимволы, РазрядовВсего, РазрядовДробнойЧасти. Предопределённые типы — примитивы XML Schema (URI http://www.w3.org/2001/XMLSchema).
- **ТипОбъектаXDTO** — экземпляры с состоянием (совокупность свойств). Свойства: БазовыйТип (только тип объекта), Открытый, Абстрактный, Упорядоченный, Последовательный, Смешанный, Свойства (список СвойствоXDTO).
- **СвойствоXDTO:** Имя, Тип, ВерхняяГраница/НижняяГраница, ВозможноПустое, ЗначениеПоУмолчанию, Фиксированное, Форма (Текст/Элемент/Атрибут), ЛокальноеИмя, URIПространстваИмен.

### 16.1.3. Значения и объекты XDTO

- **Значение XDTO** — немутабельное простое значение. Создание: `ФабрикаXDTO.Создать(ТипЗначенияXDTO, значение | лексическое представление)`; чтение из XML.
- **Объект XDTO** — мутабельный; свойства — значения XDTO или ссылки на объекты XDTO. Создание: `ФабрикаXDTO.Создать(ТипОбъектаXDTO)`. Методы: Тип(), Установить(Выражение|Свойство, Значение), Получить(), Сбросить(), Добавить() (для открытой модели), Установлено(), Последовательность(), Проверить(). Цепочки ссылок не должны образовывать циклов.
- **Последовательность XDTO** — порядок элементов и текста в XML. Методы: Количество(), ПолучитьСвойство(Индекс), ПолучитьЗначение(), УстановитьЗначение(), Добавить(), Вставить(), Удалить().
- **Список XDTO** — список значений для свойств с ВерхняяГраница > 1. Методы: Количество(), Получить(), Установить(), Добавить(), Вставить(), Удалить().

### 16.1.4. XPath

Подмножество XPath для навигации: шаги через «/», шаг — имя свойства или «.» / «..». Путь от корня — начинается с «/». Для свойств с множественным значением — индекс в квадратных скобках (0-базированный или 1-базированный), поиск по значению свойства: `Список[ИмяСвойства='СтрокаПоиска']`.

### 16.1.5. XML-сериализация на основе XDTO

**СериализаторXDTO** создаётся на основе фабрики XDTO. Сериализация значений конфигурации в XML и обратно (в т.ч. ссылок на метаданные): `СериализаторXDTO.ЗаписатьXML()`, `СериализаторXDTO.ПрочитатьXML()`. Можно сериализовать все объекты вида метаданных (например, весь справочник).

### 16.1.6. Рекомендации по схемам XML

- Без анонимных типов.
- Для сложных типов — только модель sequence (не all/choice); minOccurs/maxOccurs по умолчанию.
- Свойства предпочтительно в виде элементов (атрибуты: только типы-значения, одно значение, возможны искажения при полиморфизме).
- Не использовать mixed content без необходимости. ФормаЭлементовПоУмолчанию = Квалифицированная.

### 16.1.7. Проверка фабрики XDTO

Ошибки кодируются префиксами: model (фабрика), package (пакет), valueType (тип значения), objectType (тип объекта), facet (фасет). Правила проверки пакетов: уникальный URI пространства имён, корректные директивы импорта, типы и свойства по правилам. Правила типов значения/объекта: имена, базовые типы, границы свойств, наследование (ограничением или расширением). Правила фасетов — согласованность с базовым типом (длины, числовые границы, пробельные символы и т.д.).

## 16.2. Работа с JSON

Формат JSON (RFC 7159): объект {}, массив [], значение (строка, число, объект, массив, true/false/null). Сериализуемые типы 1С: Строка, Число, Булево, Дата (формат задаётся), Неопределено→null, Массив, Структура/Соответствие→объект JSON. Типы не из списка — исключение при сериализации (можно задать функцию преобразования).

**Форматы даты (ФорматДатыJSON):** ISO, JavaScript (new Date(ms)), Microsoft (/Date(ms)/). Вариант записи даты: ВариантЗаписиДатыJSON (локальная, со смещением, UTC). Настройки: НастройкиСериализацииJSON, ПараметрыЗаписиJSON.

### 16.2.1. Простая техника

`ЗаписатьЗначениеJSON()` — сериализация значения в строку JSON. `ПрочитатьЗначениеJSON()` — десериализация из строки. Дата в простой технике не обрабатывается автоматически (преобразование вручную).

### 16.2.2. Объектная техника

Запись: объект (Структура/Соответствие), ЗаписьJSON, НастройкиСериализацииJSON → `ЗаписатьJSON()`. Чтение: ЧтениеJSON → `ПрочитатьJSON()` в Структуру или Соответствие. Ключи структуры — по правилам имён 1С; соответствие — любые ключи. Функция преобразования при записи — для несериализуемых типов. Функция восстановления при чтении — для полей (например, дат); можно передать список реквизитов, требующих восстановления. Функции — с ключевым словом Экспорт; в управляемой форме — в контекстной части (&НаКлиенте/&НаСервере).

### 16.2.3. Работа с XDTO-объектами в JSON

Сериализация/десериализация объектов XDTO в JSON через те же объекты записи/чтения и настройки.

### 16.2.4. Потоковая техника

Запись/чтение по токенам без загрузки всего документа в память. ЗаписьJSON/ЧтениеJSON: ЗаписатьНачалоОбъекта/ЗаписатьКонецОбъекта, ЗаписатьИмяСвойства, ЗаписатьЗначение, ЗаписатьДатуJSON и т.д. Навигация полностью на разработчике.

### 16.2.5. Совмещение техник

Возможность комбинировать потоковую и объектную техники в одном сценарии.

## 16.3. Работа с двоичными данными

Хранение — объект **ДвоичныеДанные** (только чтение/разбиение; модификация — через потоки и буферы). Источники: **Поток** (ФайловыйПоток, ПотокВПамяти), **ЧтениеДанных**/ **ЗаписьДанных** — структурированное чтение/запись чисел, строк. Схема: получить поток → создать ЧтениеДанных/ЗаписьДанных → операции → закрыть; ЧтениеДанных/ЗаписьДанных можно создавать сразу из ДвоичныеДанные.

**БуферДвоичныхДанных** — фрагмент с побайтовым доступом, фиксированный размер. Чтение/запись чисел с указанием порядка байтов (little/big endian). Подходит для асинхронного чтения на клиенте (чтение в буфер — асинхронно, работа с буфером — синхронно).

**Чтение двоичных данных:** открыть файл/поток, создать ЧтениеДанных (при необходимости — порядок байтов), читать символы/числа. **Запись:** ЗаписьДанных, ЗаписатьСимволы/ЗаписатьЦелое32 и т.д.

**Строки и двоичные данные:** длина строки — в символах (СтрДлина); в методах ПрочитатьСимволы/ПрочитатьСтроку/ЗаписатьСимволы/ЗаписатьСтроку размеры — в символах. Преобразование: ПолучитьДвоичныеДанныеИзСтроки(), ПолучитьСтрокуИзДвоичныхДанных(); для буфера — ПолучитьБуферДвоичныхДанныхИзСтроки(), ПолучитьСтрокуИзБуфераДвоичныхДанных() (кодировка, BOM).

**Шестнадцатеричная/двоичная строки:** ЧислоИзШестнадцатеричнойСтроки(), ЧислоИзДвоичнойСтроки(). **Логические операции с буфером:** ЗаписатьПобитовоеИ(), ЗаписатьПобитовоеИли(), ЗаписатьПобитовоеИсключающееИли(), ЗаписатьПобитовоеИНе(), Инвертировать(). **С числами (до 32 бит):** ПроверитьБит(), ПроверитьПоБитовойМаске(), УстановитьБит(); ПобитовоеИ(), ПобитовоеИли(), ПобитовоеНе(), ПобитовоеИНе(), ПобитовоеИсключительноеИли(); ПобитовыйСдвигВправо(), ПобитовыйСдвигВлево(). Операнды — 32-разрядные беззнаковые; бит 0–31.

**Особенности:** не совмещать изменение позиции потока с операциями ЧтениеДанных/ЗаписьДанных между двумя операциями чтения/записи — возможно исключение или некорректное поведение (буферизация).

## 16.4. Работа с форматом PDF

Сохранение: ТабличныйДокумент.Записать(), ПакетОтображаемыхДокументов.Записать() — параметры ТипФайлаТаблицы/ТипФайла (PDF, PDF/A-1, PDF/A-2, PDF/A-3). Ограничения PDF/A: версия 1.4 для A-1, без двусторонней печати, без прозрачности (A-1), сохраняется информация о версии платформы и дате создания.

**Вложения:** ЗаписьPDF — добавление вложений (КоллекцияВложенийPDF.Добавить()); замещение по имени (ВложениеPDF.ИмяФайла). Удаление вложений из файла — ЗаписьPDF.ЗаписатьУдалениеВложений() (по именам; физически вложение помечается удалённым). Чтение — ЧтениеPDF.ПолучитьВложения().

**ЭЦП в PDF:** утверждающая (несколько в документе, опциональная блокировка) и сертифицирующая (одна, разрешения на изменения). Добавление: ОписаниеПодписиPDF (Сертификат, ТипПодписи, БлокировкаДокумента/РазрешенияДоступаИзменения; для отображения — НомерСтраницы, Лево, Верх, Ширина, Высота), объект криптографии, ЗаписьPDF.ЗаписатьПодпись(). Проверка: ЧтениеPDF.ПолучитьОписанияПодписей(), ЧтениеPDF.ПроверитьПодписи() / ПроверитьПодпись(). Представление подписи — ОписаниеОтображаемогоОбъектаPDF (Объект — Картинка или ТабличныйДокумент).

**ДокументPDF:** чтение/запись документа, работа с вложениями, страницами, подписями. Просмотр — Показать(); печать — Напечатать(); КоличествоСтраниц(). Преобразование в оттенки серого — ПреобразоватьВОттенкиСерого() (ЗаписьPDF, ДокументPDF). Изменение человекочитаемой части после создания не поддерживается.

## 16.5. Регулярные выражения

Платформа использует диалект ICU (на базе Perl, с поддержкой Юникода). Методы: СтрНайтиПоРегулярномуВыражению(), СтрНайтиВсеПоРегулярномуВыражению(), СтрЗаменитьПоРегулярномуВыражению(), СтрПодобнаПоРегулярномуВыражению(). Параметры: ИгнорироватьРегистр, МногострочныйПоиск. Результат поиска — РезультатПоискаПоРегулярномуВыражению (Значение, НачальнаяПозиция, Длина); группы — ПолучитьГруппы() → ГруппаРезультатаПоискаПоРегулярномуВыражению.

**Операторы:** | (ИЛИ), * + ? {min} {min,} {min,max} (квантификаторы), *? +? ?? (ленивые), *+ ++ (захватывающие), () (?:...) (?>...) (?#...), (?=...) (?!...) (?<=...) (?<!...), (?ismwx...) флаги. Флаги: i — игнорировать регистр, x — пробелы в шаблоне, s — . совпадает с переводом строки, m — ^/$ по строкам, w — \b по Unicode UAX 29.

**Метасимволы:** \a \A \b \B \d \D \s \S \w \W \n \r \t \xhh \x{hhhh} \uhhhh \Uhhhhhhhh \p{...} \P{...} \Q...\E \num (обратная ссылка) и др. Символьный класс [...], оператор - (диапазон), ^ (отрицание).

Примеры использования: поиск версий 8.3.20.20xx с (?m)^8\.3\.20\.20\d{2}\t; даты dd.mm.yy — [0-3]\d\.[01]\d\.\d{2}; группы для версия–дата; проверка телефона, URL, IPv4, IPv6; кириллические слова — \b\p{Cyrillic}+\b; разбор конфигурационного файла (группы [], пары ключ=значение, комментарии ;/#).

## 16.6. Работа с речью

Две части: распознавание речи и синтез речи. Доступ — свойство глобального контекста РаботаСРечью.

**Распознавание:** преобразование аудио в текст. Модель (нейросеть) + грамматика + акустика. Результаты — гипотезы с вероятностями. Режимы: локально (файл/кластер) или облачный сервис; онлайн (поток с микрофона) или офлайн (файл). Не поддерживается: ИБ в режиме совместимости 8.1; веб-клиент по HTTP (не localhost); Chrome ≤65, IE, Firefox, Safari, Chromium ≤65; автономный сервер. Только сервис: Apple (файловая ИБ), учебная версия, мобильная платформа/клиент. Только офлайн+сервис: Windows XP / Server 2003. Подключение: локальная модель из файла или подключение к серверу работы с речью. Программный интерфейс: подключение к серверу, распознавание (поток/файл), модели и грамматики, дополнительные грамматики, перенос данных распознавания в другую ИБ.

**Синтез речи:** текст → аудио. Методы воспроизведения и Синтезировать() (возврат ДвоичныеДанные для WAV). Обработчик окончания синтеза получает WAV в параметре РезультатСинтеза.

## 16.7. Работа с архивами

Поддержка форматов: извлечение — BZIP2, GZIP, RAR, 7-ZIP, TAR, XZ, ZIP; создание — только ZIP.

**Запись:** ЗаписьФайлаАрхива. Создание на основе файла, потока или неинициализированное (инициализация Открыть() или ПолучитьДвоичныеДанные()). Параметры: пароль, ТипФайлаАрхива.ZIP, Комментарий, МетодСжатия (BZIP2, Сжатие/Deflate, Копирование), УровеньСжатия (Максимальный, Минимальный, Оптимальный), МетодШифрования (AES128, AES192), Кодировка имён (UTF8, КодировкаОСДополнительноUTF8, Авто). Добавление файлов — Добавить(ИмяФайла|маска, МетодСохраненияПутей, РекурсивнаяОбработкаПодпапок). Режимы путей: НеСохранятьПути, СохранятьОтносительныеПути, СохранятьПолныеПути. Завершение — Записать() или ПолучитьДвоичныеДанные().

**Чтение:** ЧтениеФайлаАрхива. Открытие по файлу/потоку, опционально Пароль, ТипФайлаАрхива (если не по расширению). Свойства: Комментарий, Элементы (ЭлементыФайлаАрхива). ЭлементФайлаАрхива: ВремяИзменения, Зашифрован, Имя, ИмяБезРасширения, ПолноеИмя, Путь, Расширение, РазмерНесжатого, РазмерСжатого, ТолькоЧтение, Невидимый, ИсходноеИмя и др. Извлечение: ИзвлечьВсе(каталог, РежимВосстановленияПутейФайлаАрхива), Извлечь() — для одного элемента (поддержка индивидуального пароля на файл).
