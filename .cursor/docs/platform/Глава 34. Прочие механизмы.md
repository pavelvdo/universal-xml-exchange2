## 34.1. Решение систем линейных алгебраических уравнений (СЛАУ)

### 34.1.1. Общая информация

Объект **РасчетСистемыЛинейныхУравнений**. Входные данные: две таблицы — свободные коэффициенты и коэффициенты при переменных. Алгоритмы: итерационный и прямой.

- **ИсточникДанныхУзлов** — таблица свободных коэффициентов.
- **ИсточникДанныхСвязей** — таблица коэффициентов при переменных.
- **КолонкаУравненияВУзлах**, **КолонкаУравненияВСвязях**, **КолонкаПеременныеВСвязях** — имена колонок с номерами уравнений и переменных.
- **ОписанияСистем** — для каждой системы указывается колонка свободных коэффициентов и колонка коэффициентов при переменных.

Результат решения: таблица значений с колонками **НомерУзла** (номер переменной) и **Решение1**, **Решение2**, … (значения решений).

При одновременном решении нескольких СЛАУ системы приводят к одинаковой размерности; для каждой дополнительной системы в источники добавляется новая колонка; имена колонок задаются в **ОписанияСистем**. При нескольких системах с одной размерностью и связанных коэффициентах предпочтительно решать одним вызовом (внутренние оптимизации).

**Параметры объекта РасчетСистемыЛинейныхУравнений:**

- **ТребуемаяТочность** — максимальная погрешность решения (разница левой и правой части при подстановке). Для i-го знака после запятой рекомендуется 10^-(i+1). Сначала менять точность, при необходимости — **КоличествоИтераций**.
- **КоличествоИтераций** — число итераций итерационного алгоритма; подбирать минимальное для достижения **ТребуемаяТочность**.
- **ИспользованиеВычислительныхРесурсов** — число потоков; не больше ядер процессора. Для частичной загрузки можно задать число, равное числу решаемых систем (или кратное).
- **ГраницаИзмененияАлгоритма** — приоритет итерационного vs прямого алгоритма (меньше — чаще итерационный, больше — прямой).

**Алгоритмы:** итерационный — O(n²) на крупных матрицах, точность задаётся; прямой — точное решение, быстрее на малых матрицах, O(n³) на крупных.

### 34.1.2. Компоненты связности графа

Перечисление **СпособПолученияКомпонентСвязностиРасчетаСистемЛинейныхУравнений**:

- **КомпонентыСлабойСвязности** — максимальные подграфы, путь между любыми вершинами есть; ориентация рёбер игнорируется.
- **КомпонентыСильнойСвязности** — то же с учётом ориентации рёбер.
- **КомпонентыСильнойСвязностиБезТребованияСвязиВнутриКомпонент** — для вершин из разных компонент гарантируется отсутствие взаимных путей; внутри компоненты гарантий нет. Зависит от порядка уравнений и переменных.

Метод **ПолучитьКомпонентыСвязности(Способ)** возвращает таблицу значений с компонентами. Слабая связность — простой способ разбить СЛАУ на независимые части (решать/менять по частям). Сильная связность используют, когда слабая даёт мало компонент; затем анализируют связи между сильными компонентами и при необходимости упрощают модель или данные.

## 34.2. Интеграция с системой «1С:Аналитика»

Публикация ИБ на веб-сервере с флажком **Публиковать систему аналитики**. Доступ: `http://host/base/ans` (клиент), `http://host/base/ans/data` (данные). Право **КлиентСистемыАналитики**. Аутентификация — средствами 1С:Предприятие (веб-клиент или клиентское приложение).

Глобальный контекст: **СистемаАналитики**. Методы: **УстановитьАдресСервераСистемыАналитики()** / **ПолучитьАдресСервераСистемыАналитики()**; **Открыть()** — переход к объекту системы аналитики.

## 34.4. Активация программных лицензий

Менеджер **ПолучениеЛицензий** (глобальный контекст). Этапы: параметры лицензии, данные владельца, привязка (компьютер или HASP), запрос, активация.

**Обычная лицензия:** номер комплекта и пинкод из формуляра; владелец вводится при активации; привязка к компьютеру или ключу. **Лицензия для разработчиков:** учётная запись на портале (developer.1c.ru или 1c-dn.com); владелец с портала; привязка только к компьютеру. Проверка возможности активации лицензии разработчика: **ПолучитьДоступностьИспользованияЛицензииРазработчика()**. Перед активацией лицензии разработчика — согласие с лицензионным соглашением: **ПолучитьЛицензионноеСоглашениеРазработчика()**; язык зависит от **ПорталРазработчиков** (developer.1c.ru — русский, 1c-dn.com — английский).

**ЗапросНаПолучениеЛицензии:** для разработчиков — ИмяУчетнойЗаписи, ПарольУчетнойЗаписи; для обычной — РегистрационныйНомер, Пинкод (без разделителей), СтарыйПинкод при обновлении, данные владельца, ДляВсехПользователей, ИмяФайлаЛицензии, ПараметрыПривязки (ПараметрыПривязкиККомпьютеруПолученияЛицензий или ПараметрыПривязкиККлючуПолученияЛицензий), АдресАгента, Порт. Сохранение/загрузка запроса: **СохранитьЗапросНаПолучениеЛицензии()**, **ЗагрузитьЗапросНаПолучениеЛицензии()**.

**Параметры привязки:** **ПолучитьПараметрыПривязкиККомпьютеру()** (текущий компьютер или кластер), **ПолучитьДоступныеКлючи()** — массив ключей HASP.

**Проверка и активация:** **ПолучитьДоступностьПолученияЛицензии(Запрос)** проверяет возможность сохранения файла лицензии; **ПолучитьЛицензиюАвтоматически(Запрос)** — автоматическая активация. По телефону: **СоздатьЗапросНаПолучениеЛицензииПоТелефону()**, **СформироватьКонтрольнуюСуммуСтрокиКодаПолученияЛицензии()**, **ПолучитьЛицензиюПоТелефону()**. По носителю: **СоздатьЗапросНаПолучениеЛицензииПоНосителе()**, **ПолучитьЛицензиюНаНосителе()**.

**Центр лицензирования (код страны — пустая строка, kk/kk_KZ, ru/ru_RU, uk/uk_UA):** АдресЦентраЛицензирования(), АдресЭлектроннойПочтыЦентраЛицензирования(), ПредставлениеЦентраЛицензирования(), НомерТелефонаЦентраЛицензирования(), ПолучитьДоступностьЦентраЛицензирования(). **ПолучитьДопустимыеСтраны()** — коды стран для ЗапросНаПолучениеЛицензии.Страна.

## 34.5. Уведомление клиентского приложения со стороны сервера

**УведомленияКлиента** (глобальный контекст). Отправка с сервера/толстого клиента: **ОтправитьУведомление(Ключ, Данные, Адресаты)**. Ключ — строка (или Неопределено для обработчиков по Неопределено). Данные — произвольное значение, передаваемое на клиент. Адресаты — массив номеров сеансов (СеансИнформационнойБазы.НомерСеанса; для фонового задания — ФоновоеЗадание.НомерРодительскогоСеанса). Уведомление сохраняется в хранилище; доставка — сервисом кластера (клиент-сервер), клиентским приложением (файловая с прямым доступом) или расширением веб-сервера (файловая через веб).

На клиенте: **ПодключитьОбработчик(Ключ, Обработчик)** — Обработчик типа ОписаниеОповещения; при Ключ = Неопределено обработчик вызывается для любых уведомлений. **ОтключитьОбработчик(Ключ)** — отключить по ключу или все при Неопределено. Обработчик отключается при закрытии формы, в модуле которой он подключён.

## 34.6. Пользователи, аутентификация и встроенный язык

### 34.6.1. Общая информация

**ПользователиИнформационнойБазы** — менеджер; **ПользовательИнформационнойБазы** — объект пользователя. **ДополнительныеНастройкиАутентификации** — настройки паролей и аутентификации. Операции: создание пользователя, получение по имени/УникальныйИдентификатор, список пользователей. Требуемые права — в синтакс-помощнике.

### 34.6.2. Работа с пользователями

**СоздатьПользователя()** возвращает пустой **ПользовательИнформационнойБазы**; заполнение и запись. Роли — свойство **Роли** (коллекция РолиПользователя). **ТекущийПользователь()**, **НайтиПоИмени()**, **НайтиПоУникальномуИдентификатору()**, **ПолучитьПользователей()**. Свойства пользователя: **КлючиСопоставленияПользователя** (для OpenID Connect), **ПарольНеСоответствуетТребованиям** (только чтение), **РазделениеДанных** (при использовании РД).

### 34.6.3. Сохранение аутентификации

**ДополнительныеНастройкиАутентификации**: **ПолучитьНастройкиАвтоматическогоСохраненияАутентификации()**, **УстановитьНастройкиАвтоматическогоСохраненияАутентификации()**. Объект **НастройкиАвтоматическогоСохраненияАутентификации**: ВремяЖизниСохраненнойАутентификации, РазрешитьСохранение, СохранятьПоУмолчанию.

### 34.6.4. Политика паролей

**ПолитикиПаролейПользователей** (глобальный контекст). Объект **ПолитикаПаролейПользователей** — свойства как в диалоге параметров ИБ, плюс **Имя**; методы Записать(), Удалить(). Менеджер: **ПолучитьПолитики()**, **НайтиПоИмени()**, **СоздатьПолитику()**, **ПроверитьСоответствиеПароляПолитике(Пароль [, Политика])** — возвращает массив **РезультатПроверкиСоответствияПароляПолитике** (пустой — пароль подходит): НеСоответствуетТребованиямМинимальнойДлины, НеСоответствуетТребованиямОграниченияПовторенияСредиПоследних, НеСоответствуетТребованиямПроверкиРаскрытия, НеСоответствуетТребованиямСложности. Индивидуальная политика: создать политику, найти пользователя, установить **ИмяПолитикиПаролей** у пользователя, записать.

**ДатаУстановкиПароля** (чтение для администратора), **ЗаписываемаяДатаУстановкиПароля** — логика обновления при записи пользователя (с правом Администрирование или без, при смене пароля). После записи для актуальной даты — перечитать пользователя.

**Установка пароля:** **Пароль** (только запись), **СохраняемоеЗначениеПароля** (хеш, запись/чтение); при записи обоих используется Пароль. **ГенераторСлучайныхПаролей** — СлучайныхПароль(ЗапоминающийсяПароль). **ВычислитьСохраняемоеЗначениеПароляПользователя()**, **ПроверитьСоответствиеПароляПользователяСохраняемомуЗначению()**. **ТипАлгоритмаХешированияПароля** — производное от пароля и параметров ИБ.

**Изменение требований к паролям ИБ:** Получить/Установить методы для ДействиеПриНесоответствииПаролейПользователейТребованиямПриАутентификации, ОграничениеПовторенияПаролейПользователей, МаксимальныйСрокДействияПаролейПользователей, МинимальнаяДлинаПаролейПользователей, МинимальныйСрокДействияПаролейПользователей, ПроверкаРаскрытияПаролейПользователей, ПроверкаСложностиПаролейПользователей, СрокПредупрежденияОбИстеченииСрокаДействияПаролейПользователей.
