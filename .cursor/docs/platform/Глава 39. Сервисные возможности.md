# Глава 39. Сервисные возможности

> **Agent ref:** Сервисные режимы конфигуратора: управление окнами, параметры (общие, тексты, модули, отладка, запуск 1С, сравнение/объединение, формы, справка), синтакс-помощник, шаблоны текста, калькулятор, сравнение файлов, встроенная справка.
> **Ключевые разделы:** Управление окнами · Параметры конфигуратора · Шаблоны · Синтакс-помощник · Сравнение/объединение
> ---

## 39.1. Управление окнами

### 39.1.1. Состояние (режим размещения) окна

Режимы размещения:
- **Обычное** — окно только внутри рабочей области конфигуратора;
- **Свободное** — окно перемещается по всему экрану; всегда поверх окон в других режимах;
- **Прикрепляемое** — окно прикрепляется к границам рабочей области или к другому прикрепляемому окну; возможно совмещение (закладки);
- **Прячущееся** — окно сворачивается в ярлык сбоку рабочей области, показывается при наведении.

Переключение режима: контекстное меню заголовка окна. Для **Обычное** и **Свободное** доступно свойство **Соединяемое**; при включённом соединении окна можно соединять (для Обычное — с нажатием Shift при перемещении). Для **Прикрепленное** и **Прячущееся** свойство Соединяемое всегда включено.

Прикреплённое окно: изменение размера — тянуть свободную границу; перетаскивание за заголовок переводит в Свободное. Совмещение: перетащить заголовок над заголовком другого прикрепляемого окна. Окна у одной границы можно располагать слоями или последовательно.

Прячущееся: строка ярлыков сбоку рабочей области; окно видно пока активно; при активации другого окна — прячется; показ — наведение на ярлык без щелчка.

**Восстановление положения:** контекстное меню заголовка окна (или панели окон) → **Восстановить положение окна** — сброс сохранённых параметров, окно переоткрывается с начальными настройками.

**Режим окон для форм объектов** (категория свойств **Окно** формы): Состояние окна; Соединяемое окно; Положение окна (Авто / Центрировать / Не перекрывать владельца); Положение прикрепленного окна; Изменять размер (НеИзменять не действует для Прикрепленное и Прячущееся).

### 39.1.2. Диалог настройки окон

**Окна → Окна…** — список открытых окон (только в состоянии Обычное). Модифицированные окна помечаются «*». Кнопки: Активизировать, Сохранить, Закрыть окна, Расположить (Горизонтально / Вертикально / Подряд), Соединить / Разъединить.

### 39.1.3. Новое окно

**Окна → Новое окно** — копия окна редактора форм или текстового/табличного редактора; содержимое синхронно во всех копиях. Удобно для одновременного просмотра страниц формы, модуля и диалога формы, разных частей текста/макета.

### 39.1.4. Служебные окна

Сообщения, результат поиска во всех текстах, шаблоны текста, синтакс-помощник. Первые два показываются автоматически при необходимости; шаблоны и синтакс-помощник — по вызову. Управление: контекстное меню в заголовке (поведение окна) и внутри окна (содержимое). Окно сообщений: очистка — команда **Очистить** в контекстном меню.

### 39.1.5. Закрытие окон

Закрытие: кнопка закрытия окна или **Shift+Esc**, **Ctrl+F4**.

### 39.1.6. История переходов

**Окна → Переместиться назад / Переместиться вперед** — навигация по истории переходов (например, возврат из «место определения» к исходному окну).

## 39.2. Настройка параметров конфигуратора

**Сервис → Параметры** — картотека закладок с группами параметров.

### 39.2.1. Общие

**Редактирование конфигурации для режимов запуска:**
- **Управляемое приложение** — скрываются элементы, относящиеся только к обычному приложению;
- **Управляемое приложение и обычное приложение** — полный интерфейс редактирования для обоих режимов.

Параметр хранится в разрезе ИБ. При загрузке конфигурации с основным режимом «Обычный» (или при смене свойства) автоматически выставляется режим «управляемое и обычное».

**Скрываемые при режиме «Управляемое приложение» элементы:** ветка метаданных Интерфейсы; свойство Основной интерфейс; выбор основного интерфейса в пользователе; закладка Форма в параметрах конфигуратора; список Основной режим запуска на Модули → Проверка; свойства «Использовать управляемые формы в обычном приложении» и «Использовать обычные формы в управляемом приложении»; закладка Интерфейсы в метаданных; пункт «Открыть модуль обычного приложения»; признак управляемой формы в конструкторе; Конструктор выходных форм; свойство общего модуля «Клиент (обычное приложение)»; закладка Дополнительные у формы; обработчики интерактивной активации у точки маршрута БП типа Действие.

**Рефакторинг при изменениях в метаданных и формах:**
- При удалении выполнять проверку в модулях;
- При переименовании выполнять изменение в модулях;
- При изменении в формах выполнять проверку по всей конфигурации (в модуле формы проверка всегда выполняется).

**При наличии возможного использования ссылок:** Задавать вопрос / Продолжать операцию / Отменять операцию.

### 39.2.2. Тексты

Редактор текстовых документов: перетаскивание текста (drag&drop); разрешить курсор после концов строк; отображать пробелы и табуляцию (символы Пробел/Табуляция); заменять табуляцию при вводе; ширина табуляции; автоотступ; шрифт; цвет фона результатов поиска; **Автозамена** (Включить / Включить с подсказкой / Только подсказка) — замена по шаблону после Пробел или Enter.

### 39.2.3. Модули

**Редактирование:** выделение цветом синтаксических конструкций (константы, идентификаторы, операторы, комментарии; Текущий идентификатор, Выбранный идентификатор, Границы блока); запретить выделение цветом; автозамена (аналогично Текстам) и «Использовать автозамену в комментариях»; заменять табуляцию при вводе; ширина табуляции; автоотступ (в т.ч. Синтаксический); шрифт (рекомендуются моноширинные); цвет фона результатов поиска и контекстной подсказки.

**Проверка:** выбор среды исполнения (общие и специализированные модули по режиму). **Основной режим запуска** задаёт набор режимов проверки. Режимы по умолчанию: Управляемое — Тонкий клиент, Сервер; Мобильное — Тонкий, Мобильный клиент, Сервер, Мобильное приложение‑клиент; Обычное — Толстый клиент (обычное приложение). Проверка выполняется для модулей, существующих в выбранном режиме. Команда **/CheckModules** с параметрами: `-ThinClient`, `-WebClient`, `-MobileClient`, `-MobileClientStandalone`, `-Server`, `-ExternalConnection`, `-MobileAppClient`, `-MobileAppServer`, `-ThickClientOrdinaryApplication`, `/ExtendedModulesCheck`.

**Проверять автоматически** — при записи и закрытии. **Включая расширенную проверку** — проверка обращений «через точку» и параметров строк для части типов.

**Группировка:** для каждой синтаксической конструкции — Группировать (только второй уровень вложенности) и Сворачивать при открытии.

**Контекстная подсказка:** автоматическая при наборе «.»; при «=» и «<>» для перечислений; при кавычке для параметров методов; при «(» или «,» для параметров процедур/функций; включать шаблоны и ключевые слова в список.

**Рефакторинг:** преобразовывать модальные вызовы; преобразовывать синхронные вызовы расширений и внешних компонент.

### 39.2.4. Отладка

Выбор протокола отладки (в т.ч. HTTP и параметры). **Сервер отладки:** локальный (для файловой ИБ; при нескольких сетевых картах — указать IP в «Адрес локальной сетевой карты»; диапазоны портов через «:» и «,»); удалённый (адрес и порт); сервер отладки кластера (кластер запускается с ключом /debug). **Имя ИБ:** по умолчанию (имя базы в кластере) или указанное. После смены параметров — перезапуск конфигуратора.

### 39.2.5. Запуск системы «1С:Предприятие»

**Приложение:** Выбирать автоматически (по основному режиму конфигурации и пользователю) или вручную — тонкий/толстый/веб/мобильный клиент. Для веб-клиента — выбор браузера (по умолчанию, Chrome, Edge, IE, Firefox, Safari, Яндекс.Браузер; список зависит от ОС). Для мобильного — режим обычный или с автономным режимом (Авто по составу автономной конфигурации). При веб-клиенте возможна проверка версии публикации.

**Пользователь:** Текущий / Имя (аналог /N) / Использовать аутентификацию ОС (/WA).

**Дополнительно:** Разделение данных (/Z, без кавычек в поле); Параметр запуска (/C → ПараметрЗапуска); Низкая скорость соединения (/O); Имитация задержки при вызовах сервера (/EmulateServerCallDelay: -Call, -Send, -Receive); Файл служебных сообщений (/OUT, -NoTruncate); Язык интерфейса (/L); Код локализации сеанса (/VL); Не выдавать предупреждения при запуске (/DisableStartupMessages); Установить режим разрешения отладки (/debug -tcp или -http); Начинать отладку при запуске (для HTTP аналог /debug -http -attach). При запуске через **Отладка → Начать отладку** отладка включается независимо от флажков. Проверять циклические ссылки встроенного языка (/EnableCheckScriptCircularRefs). Отображать показатели производительности (/DisplayPerformance). Режим технического специалиста (/TechnicalSpecialistMode) — пункт «Функции для технического специалиста»: форма с деревом объектов конфигурации и поиском (Ctrl+F, отмена Ctrl+Q; поиск по началу слова, буквы и цифры — отдельные слова). Формы из неё открываются во вспомогательном окне. Стандартные функции — просмотр активных пользователей, удаление помеченных и т.д. В режиме техспециалиста: **Информация для технического специалиста** в контекстном меню заголовка формы, панели открытых, меню функций; **Открыть в Конфигураторе** — при включённой отладке. Проверять серверные вызовы в обработчиках событий (/EnableCheckServerCalls). Запускать в пакетном режиме. **Автоматизированное тестирование:** Не использовать / Запускать как менеджер (TestManager) / как клиент (TestClient, порт или TestClientID для веб).

**Мобильное приложение (закладка):** Использовать Android Debug Bridge для передачи конфигурации; каталог Android SDK; мобильная платформа (архив с 1cem-arm.apk, 1cem-x86.apk в каталоге Android). При отсутствии устройств: Ожидать подключения / Запустить эмулятор / Отказаться. Гиперссылка на AVD Manager.

### 39.2.6. Сравнение/объединение

Внешние программы для сравнения/объединения модулей и конфигураций. По умолчанию перечислены: Araxis Merge (Windows), KDiff3 (Windows, Linux), TortoiseMerge (Windows), Perforce P4Merge (Windows, Linux), DiffMerge (Windows) — для каждой указаны режимы: сравнение двух файлов, объединение двух, трёхстороннее объединение. Путь к исполняемому файлу задаётся в настройках; при незаполненном пути программа в списке неактивна.

**Модули:** механизм для «Показать различия без учета структуры» и для «Настройка объединения при объединении двух конфигураций» — встроенный или внешняя программа (с указанным путём и режимом).

**Обновление конфигурации на поддержке:** режимы «Взять из…»/«Объединить с приоритетом…» и «Объединить с помощью внешней программы» — выбор встроенного или внешнего средства (для трёхстороннего). Если для внешней программы указано «Не используется», режим «Объединить с помощью внешней программы» убирается из настроек, а сохранённые такие режимы заменяются на «Взять из новой конфигурации поставщика». **Устанавливать по умолчанию** — при сохранении выставляется режим «Объединить с помощью внешней программы».

Добавление своей утилиты: наименование, путь к исполняемому файлу. **Командная строка** — параметры для операций: Сравнение двух файлов; Объединение двух файлов; Трехстороннее объединение; Автоматическое трехстороннее объединение. Подстановки: `%baseCfg`, `%secondCfg`, `%oldVendorCfg`, `%merged`, `%baseCfgTitle`, `%secondCfgTitle`, `%oldVendorCfgTitle`.

### 39.2.7. Формы

Начальное отображение окна предпросмотра редактора формы. **Назначение использования** и **Интерфейс** (для ПК — выбор интерфейса приложения; для мобильного — только мобильный предпросмотр). **Персональный компьютер:** Масштаб, Разрешение, Поворот (Авто / Горизонтально / Вертикально). **Мобильное устройство:** выбор устройства, вариант масштаба, Поворот. Пользовательские устройства добавляются в форме «Описание устройства»: Наименование, ОС (iOS/Android/Windows Runtime), Высота/Ширина экрана, DPI, Используются экранные кнопки; порядок в списке «Дополнительные описания устройств».

### 39.2.8. Справка

Вид справочной информации: в одном окне / в разных окнах. Синтакс-помощник: горизонтально/вертикально; использовать оба языка. Фильтр дерева содержания и индекса по доступности для режима запуска. **Отображать нерекомендуемые объекты** — показ объектов, недоступных или устаревших в выбранном режиме совместимости (например, СтрШаблон() при совместимости 8.3.5; устаревшие с 8.3.14). При смене режима совместимости дерево синтакс-помощника перестраивается.

## 39.5. Шаблоны текста

Сохранение фрагментов текста и вставка по краткому имени (шаблону). Автоподстановка включается отдельно для модулей и для текстов в окне параметров конфигуратора. Шаблоны хранятся в файлах: стандартные — в каталоге bin установки платформы; пользовательские — в любом каталоге. Набор подключённых шаблонов сохраняется по варианту встроенного языка. **Сервис → Шаблоны текста**; настройка — **Действия → Настройка шаблонов**. Редактирование стандартных шаблонов невозможно. Упорядочивание файлов: Переместить вверх/вниз; удаление — Удалить. Редактирование файла — **Действия → Изменить**.

**Реквизиты шаблона:** Название; Автоматически заменять строку (замена по Пробел или Enter); Текст шаблона; Включать в контекстное меню. Группы создаются для удобства.

**Управляющие конструкции:**
- **Произвольный запрос** — `<?"Подсказка">`; при вставке запрос на ввод; одинаковые конструкции — один общий запрос.
- **Установка курсора** — `<?>`; курсор ставится на первую такую конструкцию после вставки.
- **Специальный запрос** — `<?"Подсказка", КлючевоеСлово[, Параметр1, …]>`: выбор объекта метаданных, предопределённых данных и т.д. Ключевые слова (рус/англ): БизнесПроцесс/BusinessProcess, ВариантВыбора/VariantChoice, ВыборТипа/TypeChoice, Документ/Document, ЖурналДокументов/DocumentJournal, Задача/Task, ЗначениеПеречисления/EnumValue, ИмяПользователя/UserName, КонструкторОписанияТипов/TypeDescriptionConstructor, ПолноеИмяПользователя/UserFullName, ИмяПользователяХранилищаКонфигурации/ConfigurationStorageUserName, Константа/Constant, КритерийОтбора/FilterCriterion, Обработка/DataProcessor, Отчет/Report, Перерасчет/Recalculation, Перечисление/Enum, ПланВидовРасчета/ChartOfCalculationTypes, ПланВидовХарактеристик/ChartOfCharacteristicTypes, ПланОбмена/ExchangePlan, ПланСчетов/ChartOfAccounts, предопределённые данные планов (ChartOfCalculationTypesPredefinedData и т.п.), ОбъектМетаданных/MDObjectsSubset, Последовательность/Seq, регистры (AccountingRegister, AccumulationRegister, CalculationRegister, InformationRegister), Справочник/Catalog, СправочникПредопределенныеДанные/CatalogPredefinedData, ТекстЗапроса/QueryText, ДатаВремя/DateTime с форматной строкой, ФорматнаяСтрока/FormatString. Вставка конструкций — кнопка «Вставка…» в редакторе шаблона.

## 39.6. Синтакс-помощник

**Справка → Синтакс-Помощник.** Две части окна (разделение по вертикали или горизонтали): дерево/индекс/поиск — сверху (или слева), описание — снизу (справа). Закладки: **Содержание** (иерархия элементов встроенного языка), **Индекс** (поиск по наименованию), **Поиск** (по тексту описания, с учётом морфологии и операторов). Фильтр по режиму запуска — как в параметрах Справка. Раскрытие дерева: «+», «-», «*» на цифровой клавиатуре. Панель: история просмотра, кнопка «Найти текущий элемент в дереве». Копирование элемента в редактор: буфер обмена или drag&drop из дерева. **Ctrl+F1** на элементе в модуле — открыть описание (не зависит от флажка нерекомендуемых); при нескольких вхождениях — окно «Выбор главы». Контекстное меню редактора → «Поиск в синтакс-помощнике» — открытие на закладке Индекс. На закладке Поиск — ввод строки, список глав; Enter — показать описание. Гиперссылка «Методические материалы» — переход на материалы 1С. Внизу статьи — информация о версиях: с какой версии доступен объект; с какой объявлен устаревшим и рекомендуемые замены (для устаревших с 8.3.14); «Описание изменено в версии». Режим совместимости конфигурации: если объект недоступен — причина выделяется красным (ещё не реализован / уже недоступен).

## 39.7. Сравнение файлов

**Файл → Сравнить файлы.** Поля «Сравнить» и «С» — имена файлов (ввод, выпадающий список или диалог выбора). **Как** — вид сравнения: текстовый документ, табличный документ, внешняя обработка, двоичное сравнение. При разных типах файлов они приводятся к выбранному виду. Для внешних обработок — стандартная процедура сравнения; флажок **Устанавливать соответствие по именам объектов** — соответствие сначала по именам, затем по внутренним идентификаторам. **ОК** — запуск сравнения.

## 39.8. Встроенная система справочной информации

**Справка** или **F1** — раздел справки по текущему режиму. **Содержание** или **Shift+F1** — общее содержание справки по Конфигуратору. Навигация и поиск — панель инструментов в верхней части окна; ниже — содержание.
