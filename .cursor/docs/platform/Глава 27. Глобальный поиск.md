# Глава 27. Глобальный поиск

> **Agent ref:** Глобальный поиск — механизм поиска по ИБ, навигация, вычисление выражений. API: `ГлобальныйПоиск` (МенеджерГлобальногоПоиска), план (ПланГлобальногоПоиска, ЭлементГлобальногоПоиска), события `ПриГлобальномПоиске`, `ПриВыбореРезультатаГлобальногоПоиска`, `ПриВыбореДействияРезультатаГлобальногоПоиска`. Стандартные виды — перечисление СтандартныйВидГлобальногоПоиска. Ограничения: строка до 1000 символов, до 32 подстрок; результаты по видам — 10–30 элементов.
> **Ключевые разделы:** 27.1 Общая информация, 27.2 План и стандартные виды, 27.3 Выполнение и формирование результатов, 27.4 Обработка строки и свой вид поиска, 27.5 Обработка выбора результата и действия, 27.6 Подсказка, описание, НачатьПоиск().

---

## 27.1. Общая информация

Глобальный поиск позволяет выполнять поиск по данным ИБ, вычислять арифметические выражения и переходить по навигационным ссылкам (объекты конфигурации, URL). Разработчик может доработать план поиска: по префиксу в строке (например, «шк 12345678») ограничивать поиск конкретным реквизитом (например, Штрихкод справочника Товаров).

**Состав механизма:**
1. **План глобального поиска** — что и где выполняется.
2. **Методы встроенного языка** — реализация действий.
3. **Событие изменения плана** — модификация плана по введённой строке.
4. **Механизм поиска** — выполнение плана и формирование результата.
5. **Событие выбора результата** — обработка выбора элемента из результата.

**Точка входа:** свойство глобального контекста `ГлобальныйПоиск` (тип `МенеджерГлобальногоПоиска`). Вызовы: `ГлобальныйПоиск.ИмяМетода()`.

**Основные методы менеджера:**
- `НачатьПоиск(СтрокаПоиска)` — запуск поиска (строка подставляется в поле, поиск выполняется как при вводе пользователем).
- `УстановитьПлан()` / `ПолучитьПлан()` — план по умолчанию.
- `УстановитьОписание()` / `ПолучитьОписание()` — текст справки по поиску (гиперссылка «Что искать» в истории поиска).
- `УстановитьПодсказкуВвода()` / `ПолучитьПодсказкуВвода()` — подсказка в поле глобального поиска.

В веб-клиенте: если поле ввода глобального поиска недоступно, вызов `НачатьПоиск()` не запускает поиск.

## 27.2. План глобального поиска и стандартные виды

`ПолучитьПлан()` возвращает объект типа `ПланГлобальногоПоиска`, состоящий из элементов `ЭлементГлобальногоПоиска`. Каждый элемент задаёт одно действие: **стандартное** (платформа) или **заданное в коде** (метод на встроенном языке).

### Стандартные виды (перечисление СтандартныйВидГлобальногоПоиска)

| Вид | Где выполняется | Лимит результатов | Примечание |
|-----|-----------------|-------------------|------------|
| **Выражение** | Клиент | — | Строка вычисляется как выражение. |
| **НавигационнаяСсылка** | — | — | Строка начинается с `http://` или `https://` — переход по ссылке. |
| **ИзбранноеРаботыПользователя** | Клиент | — | Поиск по избранному. |
| **ИсторияРаботыПользователя** | Клиент | — | Поиск по истории. |
| **ГлобальныеСтандартныеКоманды** | Клиент | — | Поиск по командам меню «Сервис и настройки» (избранное, калькулятор, параметры, справка и т.д.). При выборе — выполнение команды, обработчик выбора не вызывается. |
| **ФункцииДляТехническогоСпециалиста** | Сервер (фоновое задание) | ≤10 | Только при включённом диалоге «Функции для технического специалиста». По представлениям из этого диалога. При выборе открывается форма объекта метаданных. |
| **МенюФункций** | Сервер (фоновое задание) | — | По представлениям команд, групп команд и подсистем (кроме «См. также»). Подстрока по началу слова; по группам/подсистемам — без учёта совпавшей подстроки. Учитываются и скрытые пользователем функции. |
| **ОбсужденияСистемыВзаимодействия** | Сервер системы взаимодействия | ≤15 | Обсуждения и пользователи по подстрокам (разделитель — пробел). |
| **СообщенияСистемыВзаимодействия** | Сервер системы взаимодействия | ≤15 | Сообщения неконтекстных обсуждений по подстрокам. |
| **Данные** | Сервер (фоновое задание) | ≤30 | Сначала полнотекстовый поиск исходной строки; если результатов <30 — дополнительно поиск по строке с «*» после каждого слова (если слово уже оканчивается на «*», второй «*» не добавляется). Исключаются дубли с первым поиском. |
| **Справка** | Сервер (фоновое задание) | ≤10 | Аналогично строке поиска в окне справки. |

### Стандартные условия поиска (для видов с «обычными условиями»)

- Строка разбивается на подстроки по **пробелу**.
- Ограничения: длина строки ≤ 1000 символов, подстрок ≤ 32.
- Поиск подстроки — **по началу слова**.
- Буквы и цифры: последовательность только букв и только цифр считаются отдельными подстроками; в цифровой подстроке лидирующие нули могут быть значащими или незначащими.
- В иерархии: элемент найден, если в нём есть все подстроки либо часть в нём, остальные — в родителях.

### Порядок видов в плане по умолчанию (сортировка результатов)

Чем меньше значение **Порядок** у элемента плана, тем выше его результаты в списке. По умолчанию: Выражение 10, НавигационнаяСсылка 10, ИзбранноеРаботыПользователя 20, ИсторияРаботыПользователя 30, ГлобальныеСтандартныеКоманды 40, ФункцииДляТехническогоСпециалиста 50, МенюФункций 60, ОбсужденияСистемыВзаимодействия 70, СообщенияСистемыВзаимодействия 80, Данные 90, Справка 100.

## 27.3. Выполнение поиска и формирование результатов

1. Анализ плана, разделение на **клиентские** и **серверные** виды.
2. **Серверные** — один асинхронный вызов: синхронные виды выполняются последовательно, для фоновых запускаются фоновые задания; на клиент возвращаются результаты синхронных видов.
3. **Клиентские** виды выполняются синхронно.
4. Результаты показываются по мере поступления; порядок вывода определяется свойством **Порядок** каждого вида.

**Правила формирования списка результатов:**
- Вывод начинается после завершения любого вида с минимальным порядком. Результаты видов с большим порядком не показываются, пока не завершены все виды с меньшим порядком.
- После вывода всех результатов с текущим порядком выводятся результаты со следующим (по возрастанию) порядком. Цикл до завершения всех видов или до нового поиска / выбора уже отображённого элемента.
- В рамках одного вида порядок задаётся самим видом; общее переупорядочивание после завершения не выполняется.

Поиск запускается по Enter в поле глобального поиска или по таймауту после прекращения ввода. При изменении строки предыдущие фоновые задания отменяются, запускается новый поиск.

**Рекомендации:** синхронные виды должны быть быстрыми и иметь меньший порядок, чем асинхронные; «тяжёлые» синхронные виды — увеличить порядок. В одном порядке последовательность выполнения видов не определена.

**Метод пользовательского вида поиска:** может выполняться на клиенте или сервере; на сервере — синхронно или асинхронно (фоновое задание). В метод можно передать дополнительные параметры. Результат имеет задаваемое положение в списке (значение сортировки).

## 27.4. Обработка поисковой строки и свой вид поиска

Событие **ПриГлобальномПоиске** (модуль приложения): вызывается при вводе строки. Параметры — `СтрокаПоиска`, `ПланПоиска`. В обработчике нужно только сформировать план (добавить/удалить элементы); сам поиск выполнять не нужно. Виды, которые должны быть всегда, задаются через `УстановитьПлан()` при начале работы системы.

Добавление своего вида в план по умолчанию (пример в `ПриНачалеРаботыСистемы`): `План = ГлобальныйПоиск.ПолучитьПлан()`; `План.Добавить(ИдентификаторВида, ИмяМодуля, Синхронность)`; `ГлобальныйПоиск.УстановитьПлан(План)`.

Для вида, заданного в коде, в указанном модуле реализуется метод поиска. Характеристики метода: контекст выполнения (клиент/сервер), синхронность/фоновое задание на сервере, дополнительные параметры, порядок в результатах.

Обработка выбора результата и дополнительных действий — в событиях п. 27.5.

## 27.5. Обработка результатов поиска

### 27.5.1. Выбор результата поиска

Каждый вид возвращает набор результатов — объект типа `РезультатГлобальногоПоиска` из элементов `ЭлементРезультатаГлобальногоПоиска` (представление, значение). Свойство `ВидПоиска` заполняется платформой для идентификации вида; платформа его не использует.

Если значение «известно» платформе (например, ссылка на объект ИБ), форма открывается автоматически. Для пользовательского вида нужен обработчик **ПриВыбореРезультатаГлобальногоПоиска** (модуль приложения). Параметры: `ЭлементРезультата`, признак стандартной обработки. В обработчике выполняется анализ результата и нужное действие (открытие формы, навигация и т.д.).

Для стандартных видов **ФункцииДляТехническогоСпециалиста** и **ГлобальныеСтандартныеКоманды** обработчик не вызывается.

### 27.5.2. Выбор действия в результате поиска

К элементу результата можно добавить действия (свойство `Действия` у `ЭлементРезультатаГлобальногоПоиска`). Обработка — в обработчике **ПриВыбореДействияРезультатаГлобальногоПоиска** (модуль клиентского приложения). Параметры: результат глобального поиска, значение выбранного действия. Если обработчик не определён, выбор действия ничего не делает (кроме действий, добавленных платформой).

Данные, необходимые для отображения и обработки результата/действия, должны быть доступны на клиенте (структура значения и т.п.).

## 27.6. Другие особенности

**Ошибка при выполнении вида поиска на сервере:** в свойстве `Значение` элемента результата передаётся объект типа `ИнформацияОбОшибке`. В списке для таких результатов: цвет текста и картинки — `ЦветОсобогоТекста`; при отсутствии представления — текст «Ошибка поиска», при отсутствии картинки — `ДиалогСтоп`. При выборе такого результата обработчик выбора не вызывается, показывается стандартный диалог ошибки.

**Подсказка в поле:** по умолчанию — «Поиск» и сочетание клавиш. Замена — `УстановитьПодсказкуВвода(Текст)` в `ПриНачалеРаботыСистемы`.

**Справка по поиску:** текст по гиперссылке «Что искать» (в истории поиска или вместо неё до первого поиска) задаётся методами `УстановитьОписание()` / `ПолучитьОписание()`.

**Программный запуск поиска:** `ГлобальныйПоиск.НачатьПоиск(СтрокаПоиска)` — строка подставляется в поле и запускается поиск; результат отображается как при вводе пользователем. Пример использования — команда «Искать везде» в контекстном меню (таблица формы, поле табличного/форматированного документа): выделенный текст помещается в поле глобального поиска и запускается поиск.
