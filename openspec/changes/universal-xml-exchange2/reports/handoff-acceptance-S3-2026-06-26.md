### 1. Что реализовано

Срез S3 закрывает отказные пути обмена и журналирование. Большая часть логики (отказы GetData/Confirm, ошибка выгрузки → «Ошибка», отсутствие Confirm при ошибке загрузки) уже была в коде S2 и сверена с tasks. В этой сессии добавлено централизованное включение пары префиксов профиля в текст записей журнала регистрации.

1. [x] S3.1 — отказ GetData при статусе ≠ «Новое»
2. [x] S3.2 — ошибка выгрузки после «Выполняется» → «Ошибка» + журнал
3. [x] S3.3 — «профиль не найден» / «>1» → ошибка + журнал
4. [x] S3.4 — Confirm: «Выполняется»→«Выполнено», «Выполнено» no-op, иной статус — ошибка
5. [x] S3.5 — префикс `[Префикс/ПрефиксБазыПриемника]` в тексте журнала
6. [x] S3.6 — ошибка загрузки на приёмнике без Confirm, предупреждение пользователю

### 2. Что проверить СЕЙЧАС

1. На источнике установить профилю статус «Выполнено» (или «Ошибка») → вызвать GetData → ожидается отказ без выгрузки, запись в журнале с префиксом профиля.
2. Вернуть профилю «Новое» → запустить обмен с заведомо ошибочными правилами/данными → после перевода в «Выполняется» статус должен стать «Ошибка», в журнале — текст ошибки с префиксом `[Префикс/ПрефиксБазыПриемника]`.
3. (Опционально) Confirm при статусе «Новое» → ошибка; повторный Confirm при «Выполнено» → успех no-op.
4. (Опционально) Ошибка загрузки на приёмнике → Confirm не вызывается, форма показывает предупреждение.
