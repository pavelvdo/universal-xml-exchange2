# Simplifier — шаблоны промптов

Шаблоны для делегирования `Task(subagent_type="onec-code-simplifier")`. Используется из `/review` для устранения REFACTOR-замечаний (запахи кода, читаемость), без изменения поведения. Общие правила, выбор модели, обработка ошибок — `SKILL.md` (навигатор), Shared instruction blocks — там же.

---

## Simplifier (рефакторинг кода)

```
Task(
  description="Рефакторинг кода [scope]",
  prompt="## 1. ЗАДАЧА
         Выполни рефакторинг кода в файле [путь] для устранения 'запахов кода' и улучшения читаемости.

         ## 2. ЗАМЕЧАНИЯ (REFACTOR)
         Для каждого замечания:
         - ID: [порядковый номер]
         - Type: CODE
         - Severity: [HIGH/MEDIUM/LOW]
         - Procedure: [имя процедуры]
         - Anchor: [фрагмент кода для поиска]
         - Issue: [описание 'запаха кода']
         - Fix: [рекомендация ревьювера]

         [Список замечаний с Action=REFACTOR]

         ## 3. ОГРАНИЧЕНИЯ
         - Behavior Preservation: СТРОГО сохраняй исходное поведение кода. Меняй только структуру, имена и организацию.
         - Не добавляй новую функциональность и не удаляй существующую логику.
         - Если рефакторинг требует выноса логики в новую функцию — проверь AP-007 (не перезаписывать параметры: ввести локальную переменную).
         - Правь только в рамках указанных процедур/замечаний.

         ## 4. GATES
         [Вставить блок DATA_CONTRACT_GATE]
         [Вставить блок INTEGRATION_CONTRACT_GATE]
         [Вставить блок CONDITIONAL_ACTION_GATE]
         [Вставить блок EXTENSION_GUARD — если файл в cfe/]

         ## 5. СТАНДАРТЫ
         Следуй .cursor/docs/1c-coding-standards.md.

         ## 6. САМОКОНТРОЛЬ
         - Поведение кода до и после рефакторинга идентично?
         - Читаемость улучшилась?
         - Когнитивная нагрузка снизилась?",
  subagent_type="onec-code-simplifier"
)
```
