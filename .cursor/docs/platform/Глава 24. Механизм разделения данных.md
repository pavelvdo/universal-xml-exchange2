# Глава 24. Механизм разделения данных

> **Agent ref:** Разделение данных — общие реквизиты-разделители (свойства, состав), режимы «Независимо» и «Независимо и совместно», условное разделение (константа/реквизит), установка разделителей (Z, Zn, default.vrd), безопасный режим, удаление области данных.
> **Ключевые разделы:** 24.1 Общая информация, 24.2 Свойства общего реквизита (использование, значение, разделение пользователей/аутентификации/расширений, условное разделение), 24.3 Определение возможности действия, 24.4 Установка значений разделителей, 24.5 Безопасный режим, 24.6 Удаление области данных, 24.7 Поведение объектов системы.
> ---

## 24.1. Общая информация

Механизм разделения данных позволяет разделить хранимые данные и работу прикладного решения на отдельные части. Если у общего реквизита свойство **Разделение данных** = **Разделять**, то:
- объекты конфигурации, входящие в состав этого общего реквизита (разделителя), меняют поведение;
- для каждого разделителя в ИБ задаются текущее значение и признак использования;
- такие объекты называются разделёнными; конфигурация — разделённой; ИБ — разделённой ИБ;
- данные, доступные для выбранных значений разделителей плюс неразделённые объекты — **область данных**.

При **чтении** платформа отбирает только записи, где значения используемых разделителей совпадают с текущими. При **записи** проверяется, что значения разделителей в записываемых данных равны текущим или значениям по умолчанию для типа; иначе — исключение. Если в данных указаны значения по умолчанию для разделителей, в объект подставляются текущие значения используемых разделителей сеанса. Для неиспользуемых разделителей: при чтении отбор не выполняется; при записи используются значения из данных.

В состав разделителя могут входить: константы, справочники, документы, последовательности, журналы документов, ПВХ, планы счетов, задачи, регистры (сведений, накопления, бухучёта, расчёта), планы обмена, регламентные задания, пользователи ИБ. Последовательности и журналы документов считаются разделёнными только если в их состав входят разделённые объекты; состав разделителей для объектов внутри них должен быть идентичным. Разделитель в режиме «Независимо и совместно» может использоваться в ограничениях доступа к данным. Тип разделителя не может быть составным.

**Ограничение:** разделитель с типом **Строка** не допускает смену языка региональных настроек ИБ при работе с Microsoft SQL Server.

Пример записи с проверкой разделителя:

```bsl
Объект = СсылкаНаОбъект.ПолучитьОбъект();
ПараметрыСеанса.АбонентЗначение = 2;  // смена разделителя в сеансе
Объект.Записать();  // исключение: значение разделителя не совпадает
```

## 24.2. Свойства общего реквизита

### 24.2.1. Использование разделяемых данных

**Независимо:** при неиспользуемом в сеансе разделителе запрещено обращение к разделённым объектам (кроме планов обмена). Разделитель недоступен в объектной модели, в запросах, в выгрузке XML/XDTO и в RLS. При записи разделённых данных система подставляет текущие значения разделителей сеанса. В этом режиме расширения могут дополнять свои объекты и создавать собственные хранимые объекты. Ссылки в БД могут совпадать для объектов из разных областей данных.

**Независимо и совместно:** разрешена работа с разделяемыми объектами при любом использовании разделителя; доступна область данных по значениям используемых разделителей. Разделитель доступен в объектной модели, запросах, XML/XDTO и RLS. Невозможно создание предопределённых элементов для объектов разделителя. При использовании не всех разделителей объекта работа с ним менее эффективна. Расширения данных не поддерживаются. Ссылки разных областей данных не могут совпадать.

### 24.2.2. Значение разделителя и признак использования

- **Значение разделения данных** — параметр сеанса, хранящий значение разделителя (тип совпадает с типом разделителя).
- **Использование разделения данных** — параметр сеанса (Булево): Истина = разделитель используется.

Менять эти параметры можно только при наличии у пользователя прав на их изменение; платформа не гарантирует целостность данных после смены. При изменении параметра сеанса, на который ссылается разделитель: очищается кеш объектов; удаляются повторно используемые значения на клиенте и сервере. Значение разделителя можно задать: в командной строке (параметр **Z**), в строке соединения (параметр **Zn**).

### 24.2.3. Разделение пользователей

Определяет, какая часть списка пользователей доступна в зависимости от использования разделителя. **Не использовать** — единый список для всех значений. **Разделять** — список зависит от разделителя (используется при программном доступе и в конфигураторе). Таблица доступности (по одному разделителю):

| Разделитель в сеансе используется | Разделитель в сеансе не используется |
|-----------------------------------|--------------------------------------|
| **Значение для пользователя задано** → доступен при равенстве значения разделителя | доступен |
| **Значение не задано** → недоступен | доступен |

При редактировании пользователя можно задать значения для всех разделителей конфигурации; они используются при установке разделителей при входе (если у пользователя есть права на изменение параметров сеанса).

### 24.2.4. Разделение аутентификации

**Не использовать** — уникальность имён пользователей по всей ИБ. **Разделять** — допускаются одноимённые пользователи в разных областях данных (различаются значениями разделителей). Пример при разделителе «Абонент» (Число), разделение аутентификации включено: пользователи «Администратор» с значениями «Не задано», 1634, 2245, 1245 допустимы. При отключённом разделении аутентификации имена должны быть уникальны (например, АдминистраторИБ, Администратор1634 и т.д.).

### 24.2.5. Разделение расширений конфигурации

**Разделять** — пользователю доступны только расширения, подключённые с тем же значением разделителя, что и у пользователя. **Не использовать** — значение разделителя не учитывается при подключении и доступности расширений. При смене с «Разделять» на «Не использовать» разделитель перестаёт учитываться для уже загруженных расширений. При смене с «Не использовать» на «Разделять» все существующие расширения становятся доступны в области «отключённого» разделителя. Рекомендуется отключать разделение расширений для разделителя в режиме «Независимо и совместно».

### 24.2.6. Условное разделение

Позволяет использовать конфигурацию и в разделённом, и в неразделённом варианте (например, автономно без разделителей или с несколькими абонентами). Для разделителя или объекта в составе разделителя задаётся объект (константа или реквизит типа Булево), хранящий состояние разделения; при выключении используется значение по умолчанию для типа разделителя. Значение условного разделения не может прямо или косвенно зависеть от значения самого разделителя. Поддерживается рекурсивное условное разделение по цепочке разделителей; зацикливание невозможно.

Способы задания: для всех объектов разделителя — свойство разделителя **Условное разделение**; для отдельных объектов — колонка **Условное разделение** в составе разделителя; возможна комбинация.

**Объект для хранения значения условного разделения:** константа типа Булево, не входящая в данный разделитель; либо реквизит типа Булево у ссылочного объекта, который не входит в данный разделитель и является типом единственного другого разделителя.

При условно выключенном разделении: при чтении/записи данных используются признак «используется» = Истина и значение разделителя = значение по умолчанию типа; для пользователей ИБ — признак «не используется» и значение не используется. Если значение условного разделения не прочитано или неоднозначно — исключение. Если разделитель, участвующий в условном разделении другого, не используется, тот тоже считается неиспользуемым. При выключении всех разделителей (условно) становятся доступны действия с ИБ, недоступные при использовании хотя бы одного разделителя. Условное разделение в конфигураторе не действует (реквизиты считаются условно включёнными).

**Константа:** тип Булево, не разделяется этим разделителем. При первом запуске после добавления условного разделения значение константы по умолчанию — Ложь (разделение выключено).

**Реквизит объекта (другой разделитель):** позволяет разное поведение по областям данных (пример: НСИ общая для группы компаний или своя по организациям — реквизит типа Булево у справочника-типа разделителя).

**Условное разделение для объекта в составе разделителя:** отключает разделение только для выбранных объектов, а не для всего разделителя. Управление — константой или реквизитом.

**Распределённая система:** изменение условного разделения в главном узле влияет на всю систему. В разделённом сеансе объект, управляющий условным разделением (если входит в состав разделённого плана обмена), при автоматической регистрации изменений можно менять только в неразделённом сеансе или когда текущее значение Ложь (можно только включить условное разделение, выключить в автоматическом режиме нельзя). Доступ к изменению реквизита/константы условного разделения целесообразно ограничить администратором.

## 24.3. Определение возможности выполнения действия или набора данных

При нескольких разделителях для каждого разделителя, разделяющего объект, определяется возможность действия так, как для одного разделителя (с учётом режима «Независимо» или «Независимо и совместно»); итог по всем разделителям объединяется по И. Пример: два независимых разделителя (Абонент, Организация), справочники Номенклатура и Контрагенты разделены обоими. Если Абонент используется, Организация — нет: список Номенклатуры получить нельзя (Истина И Ложь = Ложь); список Контрагентов при использовании обоих — можно (Истина И Истина = Истина). Метод `ПолучитьЗначенияОтбораЖурналаРегистрации()` по колонке «Пользователь»: для используемого разделителя — пользователи по значению разделителя; для неиспользуемого — все пользователи; итог — пересечение списков (И).

## 24.4. Установка значений разделителей при запуске

При запуске значения разделителей можно задать в командной строке (параметр **Z**) или в строке соединения (**Zn**). Не указанные в командной строке разделители берутся из настроек пользователя. Если значение в командной строке и у пользователя не совпадают: при наличии прав на изменение параметров сеанса используются значения из командной строки; иначе аутентификация не выполняется. При отладке клиента из конфигуратора разделители задаются в диалоге настройки запуска; список пользователей ограничен разделителями, разделяющими аутентификацию. При аутентификации параметры сеанса разделителей устанавливаются до выполнения кода (в т.ч. модуля сеанса); при обращении к ним не вызывается УстановкаПараметровСеанса. Значение разделителя по идентификатору определяется без учёта других разделителей; при неоднозначности (например, два элемента справочника с одинаковым кодом) — исключение.

### 24.4.1. Командная строка клиентского приложения

Параметр **Z**, синтаксис: `"[<Признак>][<Значение>][,[<Признак>][<Значение>]]"`. Порядок пар соответствует порядку разделителей в конфигурации. Пары разделяются запятыми; пропуск разделителей недопустим.

**Признак:** «+» (по умолчанию) — разделитель используется, значение записывается в параметр сеанса; «-» — разделитель не используется (в параметр использования — Ложь), при указании значения оно всё равно записывается.

**Значение (текст):** Булево — 0/1; Число — канонический вид; Дата — ггггммддччиисс (14 цифр); ссылочные типы — Код или Номер (уникальные); справочники с длиной кода 0 и документы с нумераторами не допускаются; предопределённые элементы по коду/номеру не допускаются до их создания. Символ «+» или «-» в начале значения удваивается; запятая в строке значения удваивается. Если указан только признак без значения — используется значение по умолчанию типа. Конвертация при неоднозначности (например, «1») выполняется по типу разделителя.

### 24.4.2. Веб-клиент / тонкий клиент через веб-сервер

Параметр **Z** в URL (формат аналогичен командной строке). Недопустимые в URL символы кодируются по RFC 1738 (UTF-8, %XX). Значения разделителей можно задать через элемент `<zones>` в файле default.vrd (привязка к адресу ИБ).

### 24.4.3. Внешнее соединение и строка соединения

Параметр **Zn** в строке соединения; формат совпадает с параметром Z командной строки. Пример: `Srvr=localhost;Ref=Demo82Srv;Zn="...,+,---3"`. Возможна настройка в default.vrd при публикации на веб-сервере.

## 24.5. Безопасный режим разделения данных

Используется для гарантии, что при доступе через веб (безопасный сеанс) нельзя перейти в другую область данных. Метод `УстановитьБезопасныйРежимРазделенияДанных()`. Эквивалент для разделителя — атрибут `safe="true"` элемента zone в default.vrd; если безопасный режим включён через default.vrd, отключить его из кода нельзя.

В безопасном режиме запрещено: выключать использование разделителя (если разделение не условно выключено); менять значение используемого разделителя; изменять объекты, управляющие условным разделением (для разделителя и для объектов в его составе). Режим задаётся по каждому разделителю отдельно; число включений должно совпадать с числом выключений. При выходе из процедуры/функции система автоматически выключает безопасный режим по незавершённым включениям. Если выключений больше, чем включений — исключение. Проверка состояния — метод `БезопасныйРежимРазделенияДанных()`.

Пример обхода областей с безопасным режимом:

```bsl
Для Каждого ЗначениеРазделителя Из СписокЗначенийРазделителей Цикл
    УстановитьЗначениеРазделителя(ЗначениеРазделителя);
    УстановитьБезопасныйРежимРазделенияДанных("Абонент", Истина);
    ОбработатьДанныеОбласти();
    УстановитьБезопасныйРежимРазделенияДанных("Абонент", Ложь);
КонецЦикла
```

## 24.6. Удаление области данных

Метод глобального контекста `УдалитьДанныеИнформационнойБазы()` или параметр **/EraseData** пакетного режима Конфигуратора. Удаляемая область задаётся параметрами сеанса (значениями разделителей) при вызове метода или параметром **/Z** при пакетном запуске.

- **Ни один разделитель не используется** — удаляются все данные ИБ; состояние как после создания пустой ИБ и загрузки конфигурации.
- **Хотя бы один разделитель используется** — удаляются полностью разделённые данные текущей области. При нескольких разделителях с общими объектами конфигурации для хранения параметров объект считается входящим в текущую область, если разделён хотя бы одним из них. После удаления область в том же состоянии, что после первой установки набора значений разделителей до операций с данными.

Удаляются: таблицы объектов, входящих в состав разделителей (включая предопределённые данные и данные расширений); хранилища настроек; история работы пользователя; список пользователей ИБ; при неиспользуемых разделителях — часовой пояс ИБ (неопределён или удалён в зависимости от наличия независимых разделителей); при неиспользуемых разделителях — все сохранённые политики паролей, настройки политики паролей сбрасываются. Режим совместимости 8.3.2 и младше: предопределённые данные получают значения из конфигуратора или по умолчанию; при отключённом режиме — создаются при первом обращении. Планы обмена: реквизиты элемента ЭтотУзел — по умолчанию. Константы: неразделённые — по умолчанию, разделённые — удаляются. Данные регистрации ИБ в сервисе взаимодействия удаляются. **История данных:** при неразделённой ИБ или при неиспользуемых разделителях — удаляются все версии и настройки истории, настройки версионирования по метаданным; при одновременном использовании независимых и совместных разделителей — удаляется история для полностью разделённых данных, настройки версионирования и история метаданных не удаляются; при только независимых разделителях — удаляется история для полностью разделённых данных, настройки использования истории и история метаданных для текущей комбинации разделителей.

Требования: право **Администрирование**, монопольный доступ к ИБ (рекомендуется `УстановитьМонопольныйРежим()` до вызова; при ошибке получения монопольного доступа метод завершается с ошибкой). После выполнения метода рекомендуется сменить значения разделителей. При пакетном режиме Конфигуратора при ошибке монопольного доступа выполнение завершается с ошибкой. Разделитель с предопределёнными данными в составе нельзя удалить до удаления этих данных.

## 24.7. Особенности поведения объектов системы

При использовании разделителей поведение объектов системы (запись, чтение, отбор, планы обмена, регламентные задания, пользователи, история данных, хранилища настроек и т.д.) подчиняется правилам, изложенным в разделах 24.1–24.6: область данных определяется текущими значениями и признаками использования разделителей, условным разделением и безопасным режимом.
