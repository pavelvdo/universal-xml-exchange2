## Ручная конфигурация (пошагово)

Расширение: **рг_УниверсальныйОбменXML**. После каждого блока — **Конфигурация → Выгрузить конфигурацию в файлы** (или ваш обычный способ выгрузки cfe).

### S2.3 — веб-сервис `rgExchangeService`

**Где:** Конфигуратор → расширение → **Веб-сервисы** → `rgExchangeService`.

**Общие свойства (проверить):**
- **URI пространства имён:** `http://www.rushydro.ru/exchangesrv`
- **Имя файла публикации:** `ws1.1cws` (as-built)

**Операция `GetData`:**
1. Открыть операцию `GetData`.
2. **Возвращаемое значение:** `base64Binary` (уже есть в выгрузке).
3. Вкладка **Параметры** → добавить два параметра:

| Имя | Тип XDTO | Направление |
|-----|----------|-------------|
| `Префикс` | `string` (Строка) | Входной |
| `ПрефиксБазыПриемника` | `string` (Строка) | Входной |

4. **Процедура модуля:** `GetData` — сигнатура должна совпасть с BSL:
   `Функция GetData(Префикс, ПрефиксБазыПриемника)` → делегат в `рг_УниверсальныйОбменXMLСервер.ПолучитьДанныеОбмена`.
   Если Конфигуратор предложит обновить модуль — **не перезаписывать** тело процедуры, только проверить совпадение имён параметров.

**Операция `ConfirmDataReceipt`:**
1. **Возвращаемое значение:** `string` (as-built).
2. Те же два **входных** параметра: `Префикс`, `ПрефиксБазыПриемника` (`string`).
3. Модуль: `Функция ConfirmDataReceipt(Префикс, ПрефиксБазыПриемника)` — в выгрузке BSL уже корректно; в `rgExchangeService.xml` сейчас `<ChildObjects/>` пустые у обеих операций — после правки в XML должны появиться узлы параметров.

**Публикация (вне метаданных, на сервере источника):**
- Администрирование → **Публикация на веб-сервере** → включить `rgExchangeService`.
- Аутентификация: **HTTP Basic** (логин/пароль = учётные данные профиля-приёмника на источнике).

**Проверка после выгрузки:** в `WebServices/rgExchangeService.xml` у `GetData` и `ConfirmDataReceipt` непустые `ChildObjects` с параметрами `Префикс` и `ПрефиксБазыПриемника`.

### Форма `УправляемаяФорма` — поле `ПрофильПриемника`

**Контекст:** реквизит **`ПрофильПриемника`** уже на **объекте** обработки (`СправочникСсылка.рг_НастройкиОбменаXML`). Команда «Выполнить обмен» и BSL `ВыполнитьОбмен` читают **`Объект.ПрофильПриемника`**. В `Form.xml` **нет элемента** для выбора профиля — пользователь не может заполнить поле в UI.

**Где:** **Обработки** → `рг_УниверсальныйОбменДаннымиXML` → **Формы** → `УправляемаяФорма`.

**Шаги:**
1. На подходящей странице (например, верх **«Выгрузка данных»** или отдельная группа «Обмен по профилю») добавить **поле ввода**.
2. **Имя элемента:** например `ПрофильПриемника`.
3. **Путь к данным:** `Объект.ПрофильПриемника` (не отдельный реквизит формы — привязка к реквизиту объекта).
4. **Заголовок:** «Профиль приёмника».
5. **Быстрый выбор / параметры выбора** (отбор списка):
   - `Источник` = `Ложь` (только профили-приёмники).
6. Разместить поле **рядом с кнопкой** «Выполнить обмен» на командной панели или над ней — чтобы сценарий Primary был очевиден.

**Не создавать** второй реквизит `ПрофильПриемника` на форме — достаточно элемента, привязанного к `Объект.ПрофильПриемника`.

**Проверка после выгрузки:** в `Forms/УправляемаяФорма/Ext/Form.xml` — `InputField` (или аналог) с `DataPath` `Объект.ПрофильПриемника`.
