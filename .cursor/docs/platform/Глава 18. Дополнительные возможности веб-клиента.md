# Глава 18. Дополнительные возможности веб-клиента

> **Agent ref:** Интеграция веб-клиента и сайта (iframe), обмен сообщениями сайт ↔ приложение. Ключевые сущности: глобальный контекст `ОкноВнешнегоСайта`, JavaScript API `WebClient1CE`, режим основного окна (MainWindowMode/EmbeddedWorkplace), PWA — `ПрогрессивноеВебПриложение`. См. разделы 18.1 (интеграция, интерфейсы 1С и сайта), 18.2 (PWA, установка, режимы).
> ---

## 18.1. Интеграция веб-клиента и сайта

### 18.1.1. Условия интеграции

- Прикладное решение доступно через веб-клиент (опубликовано на веб-сервере).
- Веб-сервер настроен для встраивания (iframe).
- Используются только браузеры, поддерживаемые веб-клиентом 1С.

Встраивание: на странице сайта выделяется область (например через `<div>`), прикладное решение реализует интерфейс на встроенном языке для обмена с сайтом, сайт — интерфейс на JavaScript; при необходимости задаётся режим основного окна и права доступа.

### 18.1.2. Интерфейс прикладного решения (ОкноВнешнегоСайта)

Глобальный контекст: **ОкноВнешнегоСайта**.

- **ОкноВнешнегоСайта.Доступно** — признак работы веб-клиента внутри внешнего сайта. Все действия с менеджером окна внешнего сайта выполнять только при значении Истина.
- **ПодключитьОбработчикСообщений()** — регистрация обработчика входящих сообщений от внешнего сайта.
- **ОтключитьОбработчикСообщений()** — отключение обработки сообщений от внешнего сайта.
- **ОтправитьСообщение()** — отправка сообщения внешнему сайту.

Режим основного окна задаётся: свойством конфигурации «Режим основного окна клиентского приложения», параметром командной строки MainWindowMode или методом УстановитьРежимОсновногоОкна(). Необходимы соответствующие права доступа.

### 18.1.3. Интерфейс веб-сайта (JavaScript, WebClient1CE)

Сайт и приложение согласовывают собственный протокол и формат обмена.

На сайте:

1. Задать контейнер для веб-клиента — элемент `<div>`; веб-клиент сам создаёт в нём подчинённый `<iframe>`.
2. Подключить скрипт Embedded WebClient API:  
   `<script src="%АдресВебСайта%/%ИмяИБ%/scripts/webclient1ce.js"></script>`  
   После загрузки доступен объект **WebClient1CE**.

**WebClient1CE(webClientContainerID, config)**

- **webClientContainerID** (String) — идентификатор элемента `<div>` для встраивания iframe. Обязательный.
- **config** (Object) — настройки веб-клиента. Обязательный.

**Методы WebClient1CE:**

- **gotoURL(navigationURL)** — переход по навигационной ссылке (аналог ПерейтиПоНавигационнойСсылке()). Параметр **navigationURL** (String) — навигационная ссылка.
- **postMessage(message)** — отправка текста в веб-клиент; в приложении вызываются обработчики, подключённые через ОкноВнешнегоСайта.ПодключитьОбработчикСообщений(). Параметр **message** (String).

**Свойства объекта config:**

- **webClientURL** (String) — строка запуска веб-клиента (URL, включая команды и параметры).
- **width** (String) — ширина области iframe с единицей измерения (необязательно).
- **height** (String) — высота области iframe с единицей измерения (необязательно).
- **events** (Object) — обработчики событий от веб-клиента (необязательно).

**Свойства events:**

- **onStart** (Function, без параметров) — вызывается после запуска приложения (после ПриНачалеРаботыСистемы).
- **onEnd** (Function, без параметров) — вызывается перед завершением (после ПриЗавершенииРаботыСистемы).
- **onMessage** (Function) — при получении сообщения из приложения (ОкноВнешнегоСайта.ОтправитьСообщение()). Параметры: **message** (String), **origin** (String — протокол, домен, порт веб-клиента).
- **onFormOpen** (Function) — при открытии формы (после ПриОткрытии). Параметры: **url** (String, навигационная ссылка или пустая строка), **formName** (String, имя формы для английского варианта).
- **onFormClose** (Function) — при закрытии формы (после ПриЗакрытии). Параметры: **url**, **formName** — те же.

### 18.1.4. Пример встраивания

Минимальные шаги на сайте: контейнер `<div id="webClientContainer"></div>`, загрузка webclient1ce.js, создание WebClient1CE с config (webClientURL с MainWindowMode=EmbeddedWorkplace, при необходимости width/height, events.onMessage). Подстановки: %АдресВебСайта% — адрес публикации (протокол, домен, порт), %ИмяИБ% — имя ИБ из диалога публикации. Пример: при публикации http://mysite.org/dbName: %АдресВебСайта% → http://mysite.org, %ИмяИБ% → dbName.

Скрытие заставки при загрузке: задать контейнеру `style="visibility: hidden"` и в events зарегистрировать **onStart**, в котором выставить контейнеру `visibility: visible`.

Проверка origin в onMessage: при нескольких веб-клиентах на одной странице по **origin** определять источник сообщения. Параметры аутентификации можно передавать в webClientURL (строка формируется до запуска веб-клиента).

## 18.2. Прогрессивное веб-приложение (PWA)

### 18.2.1. Условия и название

Веб-клиент может работать как PWA при подключении по HTTPS и при наличии действительного сертификата на веб-сервере. Поддержка зависит от браузера (см. системные требования).

Название PWA (до 45 символов, при превышении обрезается с многоточием) используется при установке/удалении, в меню и на иконке. Задаётся (в порядке приоритета):

1. Параметр командной строки запуска веб-клиента **ProgressiveWebApplicationName** (удобно при разделении данных).
2. Настройки публикации веб-клиента или файл default.vrd публикации (если название должно отличаться от представления конфигурации).
3. Если не задано — используется представление конфигурации.

После смены названия нужно переустановить PWA.

### 18.2.2. Программный интерфейс (ПрогрессивноеВебПриложение)

Менеджер доступен через свойство глобального контекста **ПрогрессивноеВебПриложение**.

- **Поддерживается()** — проверка возможности установки PWA на устройство.
- **Установлено()** — установлено ли PWA на устройстве.
- **НачатьУстановку()** — инициирует установку; браузер показывает диалог подтверждения. Тот же диалог доступен по кнопке (+) в адресной строке или через меню веб-клиента «Сервис и настройки» → «Отображать как отдельное приложение».
- **ПодключитьОбработчикОкончанияУстановки()** — регистрация обработчика обратного вызова после ответа пользователя в диалоге установки.
- **ОтключитьОбработчикОкончанияУстановки()** — снятие обработчика.
- **ТекущийРежим()** — режим работы:
  - **РежимПрогрессивногоВебПриложения.ВОтдельномОкне** — PWA установлено и открыто в отдельном окне;
  - **РежимПрогрессивногоВебПриложения.ВОкнеБраузера** — установлено и открыто во вкладке браузера;
  - **Неопределено** — PWA не установлено или приложение запущено не как PWA.
