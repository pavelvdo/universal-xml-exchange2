# Глава 28. Разработка для мобильных устройств

> **Agent ref:** Мобильная платформа, мобильный клиент, автономный режим, сборщик приложений, файловая ИБ на устройстве, синхронизация, ограничения метаданных и форм, нативный API (SMS, геолокация).
> **Ключевые разделы:** 28.1 Варианты развёртывания, 28.2 Ограничения и особенности, 28.3 Программный интерфейс и конфигурация.
> ---

## 28.1. Варианты развёртывания

Система «1С:Предприятие» поддерживает разработку приложений для мобильных устройств (Android, iOS, Windows с сенсорным экраном).

**Термины:**

- **Мобильная платформа** — мобильная версия «1С:Предприятие» (движок и возможности для мобильных устройств).
- **Мобильное приложение** — исполняемый пакет (apk / ipa / appx). Сборка и распространение — через сборщик приложений для мобильных устройств.

**Три варианта:**

| Вариант | Описание | ИБ на устройстве | Соединение | Сложность адаптации |
|--------|----------|-------------------|------------|----------------------|
| Приложение на мобильной платформе | Клиентская и серверная части на устройстве; файловая ИБ локально | Да | Синхронизация с основной ИБ — отдельная реализация | Высокая (новое или сильно доработанное решение) |
| Мобильный клиент | Доступ к ИБ через веб-сервер; серверный код на сервере приложений | Нет | Обязательно | Адаптация существующего решения; поддерживаются расширения конфигурации |
| Мобильный клиент с автономным режимом | Работа без подключения к ИБ | Кэш/подмножество данных | Синхронизация при подключении | Средняя: выбор метаданных для автономного режима, формы автономного режима, механизмы синхронизации и разрешения конфликтов |

**Интерфейс:** аналог Такси; на экране обычно одна форма. Главное меню (жест слева или кнопка ≡) — разделы и команды. Навигация зависит от типа формы (без владельца / с владельцем / блокирующая). Начальная страница настраивается.

## 28.2. Ограничения и особенности разработки

**Мобильная платформа:** ограниченный набор метаданных и конструкций встроенного языка; особенности работы с файлами, картинками и печатью; управление подключениями и обновлениями.

**Мобильный клиент:** ограничения по формам и элементам управления; серверные вызовы и контекст выполнения; совместимость с веб-клиентом; доступ к возможностям ОС через мобильные библиотеки и приложения (SMS, звонки, геолокация и т.п. — через API платформы для мобильных устройств).

**Автономный режим:** выбор метаданных для автономной работы; синхронизация при подключении; разрешение конфликтов.

## 28.3. Программный интерфейс и конфигурация

Доступ к возможностям мобильного приложения — через свойства и методы платформы: определение среды выполнения, вызов нативных функций, работа с локальным хранилищем и синхронизацией.

В конфигурации должно быть создано **мобильное приложение** (объект метаданных): настройки для магазинов приложений и параметры сборки пакетов.
