## 6.1. Общее устройство командного интерфейса

### 6.1.1. Разделы и подразделы основного окна приложения

Командный интерфейс формируется платформой автоматически по правилам конфигурации. На его состав влияют права пользователя, функциональные опции и настройки пользователя. Декларативное описание обеспечивает согласование команд с правами и модификацию интерфейса при неполном внедрении.

Глобальные команды объединяются в разделы. Панель разделов строится по подсистемам верхнего уровня с признаком **Включать в командный интерфейс**. Команды текущего раздела отображаются в панели навигации и панели действий. Подразделы в панели навигации соответствуют подчинённым подсистемам с тем же признаком (например, в разделе Торговый учёт — подразделы Розничная торговля, Оптовая торговля).

### 6.1.2. Виды команд

**Классификация по охвату:** независимые глобальные команды, параметризуемые глобальные команды, локальные команды формы. Ниже рассматриваются только глобальные команды.

**Классификация по происхождению и роли:** стандартные (добавляются платформой), навигационные, команды действия, команды, созданные в конфигурации.

#### 6.1.2.1. Стандартные команды

Платформа добавляет стандартные команды для объектов конфигурации. Сводка по объектам:

| Объект | Стандартные команды |
|--------|---------------------|
| Общая форма | Открыть общую форму; форма редактирования константы |
| Справочники | Список, новый, новая группа; ввод на основании; список с отбором по владельцу |
| Документы | Список, новый; ввод на основании |
| Журналы документов | Список |
| Отчеты, Обработки | Основная форма |
| ПВХ, Планы счетов, Планы видов расчета | Список, новый, новая группа (где применимо); ввод на основании |
| Перечисления | Список |
| Регистры сведений | Список |
| Регистры накопления, бухгалтерии, расчета | Список; список с отбором по регистратору |
| Бизнес-процессы, Задачи | Список, новый; ввод на основании; для БП — список задач с отбором по БП |
| Планы обмена | Список, новый; ввод на основании |
| Критерии отбора | Список |

**Условия появления:** команды списка и создания — при включённом свойстве **Использовать стандартные команды**. Команда отчёта — при заданной основной схеме компоновки или основной/дополнительной форме. Обработка — при основной или дополнительной форме. Ввод на основании — при заданном свойстве **Ввод на основании** у объекта. Список с отбором по владельцу — для подчинённых справочников; с отбором по регистратору — для регистров, подчинённых регистратору.

**Размещение стандартных команд:**

| Панель | Команды |
|--------|---------|
| Панель навигации | Открытие списков |
| Панель действий | Форма редактирования констант; формы новых объектов; формы отчётов и обработок |
| Панель навигации формы | Список с отборами |
| Командная панель формы | Ввод на основании |

#### 6.1.2.1.2. Параметризуемые стандартные команды

Параметр команды: ввод на основании — ссылка на объект-основание; список с отбором по владельцу — ссылка на владельца; по регистратору — ссылка на документ-регистратор; список критерия отбора — ссылка на значение критерия. Тип параметра может быть составным (несколько оснований/владельцев); от него зависит набор форм, в которых команда доступна.

#### 6.1.2.2. Независимые и параметризуемые глобальные команды

Независимые команды не требуют параметра (открытие списка, журнала, формы отчёта, нового элемента). Параметризуемые требуют контекста: например, список подчинённого справочника (параметр — ссылка на владельца) или список записей регистра по регистратору (параметр — ссылка на документ).

#### 6.1.2.3. Навигационные команды и команды действия

**Навигационные** — переход к форме в том же окне (основном или вспомогательном). Размещаются в панели навигации. Примеры: переход к списку справочника/журналу в основном окне; в форме элемента — переход к подчинённому списку (например, регистр курсов валюты, движения документа). Отказ от открытия формы не прерывает команду; откроется пустая форма.

**Команды действия** — обычно открывают новое вспомогательное окно (создание документа и т.п.). Размещаются в панели действий или в командной панели формы.

Глобальные команды в командной панели формы отображаются перед «Открыть справку». Группа **Важное** — в панели; остальные (в т.ч. **Создать на основании**) — в подменю. Глобальные команды не используют свойство **Пометка** (оно только у команд формы).

#### 6.1.2.4. Команды, созданные в конфигурации

Разработчик может создавать команды, задавать группу размещения и действие на встроенном языке (объект конфигурации **Команда** и свойства команды).

#### 6.1.3. Группы команд

Четыре категории размещения глобальных команд:

| Категория | Назначение |
|-----------|------------|
| Панель навигации | Независимые навигационные команды |
| Панель навигации формы | Параметризуемые навигационные команды из формы |
| Панель действий | Команды, открывающие вспомогательное окно |
| Командная панель формы | Параметризуемые команды в форме, открывающие вспомогательное окно |

Стандартные группы: Панель навигации — Важное, Обычное, См. также; Панель действий — Создать, Отчеты, Сервис; Панель навигации формы — Важное, Перейти, См. также; Командная панель формы — Важное, Создать на основании. Дополнительно можно создавать группы команд (Общие → Группы команд) с указанием категории (свойство **Категория группы команд**).

Сводка размещения по типу команды:

| | Независимые | Параметризуемые |
|--|-------------|-----------------|
| Навигационные | Панель навигации | Панель навигации формы |
| Команды действия | Панель действий | Командная панель формы |

По умолчанию глобальные команды не попадают в формы групп иерархических справочников и ПВХ; их можно добавить вручную в редакторе формы.

## 6.2. Построение глобального командного интерфейса

### 6.2.1. Подсистемы

Основа интерфейса основного окна — дерево подсистем с признаком **Включать в командный интерфейс**. Подсистемы первого уровня задают разделы в панели разделов; подсистемы ниже — подразделы в панели навигации. Если ни у одной подсистемы признак не установлен, панель разделов не показывается; при этом на рабочий стол попадают все команды, редактируемые вручную (дерево команд в редакторе рабочего стола скрыто). После появления первой подсистемы, включённой в интерфейс, автоматическое добавление команд на начальную страницу отключается — команды размещаются явно. Режим «Обычное приложение» отключает автоматическое размещение на начальной странице.

Раздел показывается в панели разделов, если: 1) у подсистемы установлено Включать в командный интерфейс; 2) права доступа разрешают подсистему текущему пользователю; 3) подсистема не отключена функциональными опциями; 4) не отключена пользователем в настройке панели разделов. При отсутствии доступных команд раздел всё равно может отображаться.

Принадлежность объекта к подсистемам определяет появление его команд в соответствующих разделах. Новые объекты по умолчанию не входят ни в одну подсистему — команды не попадают в интерфейс, пока объект не отнесён к подсистемам. Объект может входить в родительскую и дочернюю подсистемы или только в дочернюю.

### 6.2.2. Команды

Команду (объект **Команда**) можно создавать под: Справочники, Документы, Журнал документов, Отчеты, Обработки, ПВХ, Перечисления, Планы счетов, Планы видов расчета, Регистры сведений/накопления/бухгалтерии/расчета, БП, Задачи, Планы обмена, Критерии отбора. Также существуют общие команды (Общие → Общие команды).

### 6.2.3. Параметризация команды

Параметризуемая команда получает значение параметра из данных формы. Свойство **Тип параметра команды** задаёт состав форм, в которых команда доступна (например, СправочникСсылка.Товары — форма элемента/списка товаров, строка табличной части и т.д.). Задавать тип параметра имеет смысл только для команд групп «Командная панель формы» и «Панель навигации формы».

**Режим использования параметра:** Одиночный — в команду передаётся одно значение; Множественный — всегда массив (при множественном выделении в таблице первый элемент массива — текущая строка, порядок остальных не гарантируется). Если текущая строка не входит в выделение, первый элемент массива не определён однозначно. При нескольких таблицах формы, связанных с одним реквизитом типа ДинамическийСписок, не определено, какая таблица используется для параметра.

Фрагмент командного интерфейса формы формируется по типу основного реквизита; разработчик может дополнять его в редакторе формы. Команды с типом параметра, не совпадающим с основным реквизитом (например, команды по производителю в форме товара), по умолчанию не включаются — их можно добавить вручную. Для формы группы иерархического справочника/ПВХ параметризуемые глобальные команды в интерфейс по умолчанию не попадают.

На размещение влияют: Тип параметра команды, Группа, принадлежность команды (или владельца) к подсистемам. Независимые команды привязаны к подсистемам владельца или к подсистемам самой команды (общие команды). Параметризуемые — к объектам, чьи типы указаны в Типе параметра. Группа уточняет панель (навигация/действия для независимых; навигация формы/командная панель формы для параметризуемых).

### 6.2.4. Формирование командного интерфейса по умолчанию

Текстовые представления берутся из свойств Синоним и Имя; на них можно влиять через представления объекта и списка.

### 6.2.5. Свойство «Командный интерфейс»

Свойства **Командный интерфейс** (подсистемы), **Командный интерфейс** и **Командный интерфейс основного раздела** (конфигурация) позволяют редактировать состав и порядок команд раздела, группы команд и видимость по умолчанию. Одна и та же команда в разных разделах может быть в разных группах и с разной видимостью. Редактирование всех разделов — в редакторе «Все подсистемы». Состав команд формы — на закладке **Командный интерфейс** редактора формы.

### 6.2.7. Ролевая настройка видимости команд по умолчанию

Видимость по умолчанию задаётся при автоматическом формировании и может быть изменена в разрезе ролей. Команда видна по умолчанию, если видимость установлена хотя бы для одной роли пользователя. Видимость имеет смысл только при доступности команды для роли; для сильно ограниченных ролей скрывать команды дополнительно обычно не требуется. Ролевая настройка задаёт начальную «насыщенность» интерфейса в первую очередь для пользователей с широкими правами.

**По умолчанию видимы:** команды открытия списков (справочники, документы, журналы, ПВХ, планы видов расчета/счетов, регистры сведений, БП, задачи, планы обмена), формы отчётов и обработок, форма констант, переход к списку подчинённого справочника/логически подчинённого регистра сведений, ввод на основании, пользовательские команды.

**По умолчанию скрыты:** списки перечислений, регистров (подчинённых регистратору), накопления, бухгалтерии, расчета; формы нового объекта/новой группы; переход к подчинённым регистрам и критериям отбора.

## 6.3. Сервисные возможности навигации

### 6.3.1. Навигационные ссылки

Навигационная ссылка — текстовая ссылка на раздел интерфейса, отчёт, обработку или объект базы (документ, элемент справочника и т.д.). Используется для перехода, избранного, передачи другому пользователю, мобильных сценариев.

**Получение ссылки:** меню Сервис и настройки → Получить ссылку или кнопка формы; Ctrl+F11 — ссылка на текущий объект. В диалоге: Показать ссылку (текст ссылки), Внешняя ссылка (с префиксом адреса базы), Копировать название (ссылка + представление), Копировать и закрыть, Добавить в избранное. Для разделения представления и ссылки используется строка «: e1c». Стандартные функции не поддерживают навигационные ссылки; для быстрого вызова — поле глобального поиска. Внешняя ссылка — URL при публикации на веб-сервере; переход по ссылке в том же окне браузера или в новом. Для тонкого клиента ссылка может передаваться параметром URL командной строки.

**Программное формирование:** семейство методов **ПолучитьНавигационнуюСсылку…()**; **ПолучитьНавигационнуюСсылку()** — для объектов конфигурации; в расширениях форм — специализированные методы по основному реквизиту; **ПолучитьНавигационнуюСсылкуИнформационнойБазы()** — адрес базы (аналог тумблера «Внешняя ссылка»). Параметр **ВнешниеДанныеНавигационнойСсылки** — структура, помещаемая в хранилище внешних параметров (параметр extdata в ссылке). При открытии формы по ссылке с extdata: создаётся форма, из хранилища читается структура, пары ключ–значение переносятся в **Параметры** формы, форма открывается с этим набором. В обработчике команды (**ОбработкаКоманды**) параметры приходят в **ПараметрыВыполненияКоманды.Параметры**.

**Обработчики при получении ссылки / добавлении в избранное:** 1) Формируется список навигационных ссылок: при активной таблице с динамическим списком — обработчик таблицы **ОбработкаПолученияСпискаНавигационныхСсылок()**, затем платформа формирует список по умолчанию для формы, к нему добавляются ссылки из обработчика таблицы, затем вызывается обработчик формы с объединённым списком. 2) Разработчик может менять список в обработчиках. 3) Пользователь выбирает ссылку. 4) Вызывается **ОбработкаПолученияНавигационнойСсылки()** (таблицы или формы — в зависимости от источника выбранной ссылки). Параметры обработчика: **Ключ**, **СписокВыбораНавигационнойСсылки** (коллекция **ОписаниеЭлементаСпискаВыбораНавигационнойСсылки**: Ключ, Представление, ЗапрашиватьПользовательскоеПредставление). **КлючПоУмолчанию** задаёт вариант по умолчанию в диалоге. В **ОбработкаПолученияНавигационнойСсылки()** можно менять **НавигационнаяСсылка** и **СтандартнаяОбработка**: Ложь — используется переданная ссылка; Истина и Ключ из **ВариантСтандартнойНавигационнойСсылкиФормы** — стандартная ссылка; иначе — исключение. СтандартнаяОбработка = Истина, если Ключ стандартный или (Ключ = Неопределено и **АвтоНавигационнаяСсылка** = Ложь); СтандартнаяОбработка = Ложь, если Ключ добавлен в ОбработкаПолученияСпискаНавигационныхСсылок или (Ключ = Неопределено и АвтоНавигационнаяСсылка = Истина). Для параметризации формы: в ОбработкаПолученияСпискаНавигационныхСсылок добавить элемент с уникальным Ключом; в ОбработкаПолученияНавигационнойСсылки сформировать структуру параметров, поместить в структуру с ключом ЗначенияПолей, передать в **ПолучитьНавигационнуюСсылку(…, ВнешниеДанныеНавигационнойСсылки)** и вернуть ссылку. При переходе по ссылке структура читается из хранилища и передаётся форме в параметре ЗначенияПолей (использование в **ПриСозданииНаСервере**).

Список по умолчанию для таблицы: при **РазрешитьПолучатьНавигационнуюСсылкуТекущейСтроки** = Истина — один элемент (ОбъектТекущейСтрокиСписка или ЗаписьТекущейСтрокиСписка). Для формы: форма списка — Список, СписокСТекущимиНастройками, СписокСТекущимиНастройкамиИСтрокой; форма отчёта — Отчет, ВариантОтчета, ОтчетСТекущимиНастройками; прочие формы объекта/записи — один элемент (Объект/Запись).

**Переход по навигационной ссылке:** обработчик **ОбработкаПереходаПоНавигационнойСсылке** (модуль клиента). Вызывается при интерактивном переходе, **ПерейтиПоНавигационнойСсылке()**, глубинных ссылках мобильного приложения, внешней ссылке. Параметр **ДанныеПереходаПоНавигационнойСсылке**: Запуск (Истина — приложение запущено для обработки ссылки), БазоваяНавигационнаяСсылка (адрес базы), ОтносительнаяНавигационнаяСсылка (внутренняя часть ссылки), ПараметрыНавигационнойСсылки (Соответствие с ref, extdata, stgng и т.д.), ДополнительныеДанныеПереходаВМобильноеПриложение (Intent Android или Неопределено). **СтандартнаяОбработка** = Ложь отключает переход платформой (для своей обработки).

**Свойства формы:** **АвтоНавигационнаяСсылка** — автоматическое формирование ссылки формы; при Ложь используется значение **НавигационнаяСсылка** (платформа его не меняет). Учитывается в **ОкноКлиентскогоПриложения.ПолучитьНавигационнуюСсылку()** и при интерактивном получении/избранном. При открытии формы с параметром НавигационнаяСсылка это значение записывается в свойство формы, АвтоНавигационнаяСсылка = Ложь; при последующей установке АвтоНавигационнаяСсылка = Истина свойство НавигационнаяСсылка игнорируется. У таблицы: **РазрешитьПолучатьНавигационнуюСсылкуТекущейСтроки** — разрешает ссылку на текущую строку динамического списка (кроме регистра сведений, подчинённого регистратору); при сбросе ссылка на текущую строку не формируется.

### 6.3.2. Оповещения пользователя

Оповещения информируют о действиях системы или приложения. Создаются платформой при записи/изменении объекта или вызовом **ПоказатьОповещениеПользователя()**. Окно оповещения зависит от статуса, клиента и ОС. При заданном действии при нажатии оповещение ведёт себя как гиперссылка.

**Статусы:** Информация — можно перемещать и менять размер, положение запоминается; 1 с без затухания, исчезновение через 5 с; в веб-клиенте — в рамках основного окна при активном приложении; пропущенные не сохраняются. Важное — фиксированные размер и положение; отображаются при свёрнутом окне; 10 с (в веб-браузере при пропуске — запись в центре сообщений); лимит одновременных зависит от клиента (тонкий/толстый/IE — до 3; Chrome — до 3; Firefox — неограниченно; Edge, Safari — последнее или по настройкам).

**Действие при нажатии:** параметр **ДействиеПриНажатии** — Строка (навигационная ссылка) или **ОписаниеОповещения** (вызов обработчика). Важное — нажатие по любой области; Информация — только по гиперссылке. **КлючУникальности** — обновление существующего оповещения с тем же ключом (закрытие старого, показ нового с обновлённой информацией); используется только в интерфейсе Такси. Системные оповещения — статус Информация, ключ = навигационная ссылка объекта. Оформление текста в оповещениях недоступно.

### 6.3.3. Панель состояния

**Состояние()** — отображение хода длительного процесса; вызывается только на клиенте. Второй параметр — абсолютное значение индикатора прогресса (0–100); при отсутствии индикатор не показывается. Панель исчезает через короткое время после завершения вызвавшего кода; при наведении курсора остаётся до закрытия или увода курсора. Модальные окна выводятся поверх панели. В веб-клиенте панель — отдельное окно; обновление: IE — после каждого вызова; Firefox — при серверном вызове и по окончании блока кода; Chrome, Safari — после окончания блока кода на встроенном языке. Для отображения прогресса серверного процесса нужно выполнять его порциями с инициацией с клиента и вызовом Состояние() при старте каждой порции.

### 6.3.4. Сообщения

Механизм сообщений (**СообщениеПользователю**) связывает сообщение с реквизитом объекта; при отображении сообщение может привязываться к элементу формы. Панель сообщений — внизу рабочей области, до 5 строк, с прокруткой; высота не меняется. Рекомендуется использовать для информирования об ошибках и сопровождать отказом от закрытия формы при действии, подразумевающем закрытие, иначе сообщения могут не быть видны. Для информирования о событиях предпочтительнее **ПоказатьОповещениеПользователя()**.

**Привязка к полю:** **ПутьКДанным** — реквизит формы, хранящий данные объекта; **Поле** — реквизит объекта (формат: ИмяРеквизита, ИмяТЧ[Индекс].ИмяРеквизита, [Индекс].ИмяРеквизита для наборов записей). **ИдентификаторНазначения** — к какой форме привязать сообщение. При **КлючДанных** не равном ключу основного реквизита текущей формы открывается форма объекта по КлючДанных; если ПутьКДанным не задан, подставляется основной реквизит открытой формы; сообщения с тем же КлючДанных переносятся в новую форму с установкой ПутьКДанным при необходимости. Соответствие объект–форма задаётся вызовом **УстановитьСоответствиеОбъектаИФормы()** (второй параметр — структура с ИдентификаторФормы, Реквизит) или устанавливается формой при **РеквицитФормыВЗначение()**. Для использования привязки в сообщении вызывается **УстановитьДанные()** объекта СообщениеПользователю. Получить соответствие — **ПолучитьСоответствиеОбъектаИФормы()**. При стандартных действиях (проведение и т.д.) расширение формы само создаёт объект и соответствие. При записи нового объекта в транзакции и отмене транзакции в сообщениях очищаются ссылки на этот объект (КлючДанных). Если сообщения создаются при серверном вызове из формы/общего модуля/модуля общей команды, вывод блокируется до возврата на клиент; список непоказанных — **ПолучитьСообщенияПользователю()**. При привязке к строке табличной части изменение порядка строк не влияет на привязку; при удалении строки сообщение привязывается к элементу табличной части. При поиске в таблице (ДанныеФормыКоллекция и т.д.), если строка с сообщением скрыта, поиск отменяется и сообщение показывается; при программном отборе (ОтборСтрок) отбор не отменяется, привязка — к таблице в целом.

### 6.3.5. Работа сочетаний клавиш

В независимом вспомогательном окне работают: сочетания панели навигации и панели действий главного окна (в т.ч. для невидимых команд) и сочетания самого окна (команды формы и панели навигации формы). Скрытые команды панели навигации формы и скрытые элементы формы в вспомогательном окне сочетаний не имеют. В блокирующих окнах — только сочетания самого окна. В тонком клиенте возможен переход в окно сообщений; в других клиентах — нет.

### 6.3.6. Работа со списком форм

**ПолучитьОкна()** возвращает коллекцию **ОкнаКлиентскогоПриложения**. Окно с **Основное** = Истина — основное окно приложения, **Содержимое** пусто. **НачальнаяСтраница** = Истина — начальная страница (рабочий стол), **Содержимое** — формы на ней. Остальные — вспомогательные окна, **Содержимое** — список форм (первый — основная форма окна). Порядок элементов в коллекции не определён.

### 6.3.7. Списки переходов (jump list)

В ОС с поддержкой (например, Windows, Linux Unity) список переходов даёт быстрый доступ к объектам приложения (правый клик по кнопке на панели задач). В 1С в списке — информационная база; список синхронизирован с последними базами в окне запуска. Для закрепления в списке переходов в Windows используют команду «Закрепить на панели задач» у ярлыка 1С:Предприятие.

### 6.3.8. Кнопки окна предпросмотра панели задач

В Windows 7+ в эскизе окна на панели задач можно размещать кнопки через **ПанельЗадачОС.УстановитьКнопки(Массив)**. Каждый элемент массива — структура: Картинка, Заголовок, Действие (**ОписаниеОповещения** на обработчик в клиентском общем модуле). Ограничения: не более 7 кнопок; после первого создания нельзя увеличить число кнопок (только менять/скрывать); рекомендуется .ico с размерами 16×16, 20×20, 24×24; все три свойства обязательны. В Linux, Windows XP кнопки в эскизе не отображаются.

### 6.3.9. Управление значком панели задач

В Windows 7+ доступны: **ПанельЗадачОС.УстановитьНаклейку(Число, УдалятьПриВыборе)** — значок или число на кнопке (УдалятьПриВыборе = Истина — снять при выборе приложения); **ПанельЗадачОС.УдалитьНаклейку()**; **ПанельЗадачОС.Сигнализировать(КоличествоМиганий)** — мигание кнопки. Иконки на кнопке в Windows видны при выключенном «Использовать маленькие значки» в настройках панели задач. В Ubuntu Unity при запуске под root эффекты могут не отображаться.

## 6.4. Порядок разработки командного интерфейса

Типичная последовательность: 1) Определить структуру конфигурации по предметной области (дерево подсистем). 2) Задать роли. 3) При создании объектов относить их к подсистемам, назначать права. 4) При необходимости редактировать свойство **Командный интерфейс** подсистем и состав команд форм (размещение и видимость по умолчанию для параметризуемых команд — переход к подчинённому списку, ввод на основании и т.д.).

В результате в разделах появятся команды открытия списков и при включённой видимости — команды создания. Пример: документ Расход товара в подсистемах Торговый учёт и Бухгалтерия, роли Менеджер по продажам и Администратор, ввод на основании Счет и Заказ покупателя, регистратор регистра Остатки товаров на складах — команда «Расход товара» в панели навигации раздела Торговый учёт; в формах Счет и Заказ покупателя — команда «Создать на основании» для расхода. Представление команды списка — свойство **Представление списка** документа; представление команды «создать на основании» — **Представление объекта**. В редакторе командного интерфейса подсистемы (или «Все подсистемы») команду можно перенести в группу Важное и настроить видимость по умолчанию по ролям. Пользователь может дополнительно настраивать видимость команд для себя.
