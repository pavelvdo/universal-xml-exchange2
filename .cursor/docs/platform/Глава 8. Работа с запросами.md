# Глава 8. Работа с запросами

> Agent ref: Язык запросов, источники данных, виртуальные поля, выполнение запросов и временные таблицы.
> Ключевые разделы: 8.1 Общая информация, 8.2 Источники данных, 8.4 Язык запросов, 8.5 Выполнение запросов
> ---

## 8.1. Общая информация

Для формирования и выполнения запросов к таблицам базы данных используется объект **Запрос**. Запрос удобен для сложной выборки, группировки и сортировки. Примеры: сводка по состоянию регистра на момент времени, отчёты по нескольким таблицам.

## 8.2. Источники данных (таблицы) запросов

Источники данных языка запросов — таблицы базы данных. Классы таблиц:

- **Реальные** — хранятся в БД; могут содержать вычисляемые поля (функция нескольких реальных полей).
- **Виртуальные** — не хранятся в БД; при обращении система собирает данные из реальных таблиц. Могут быть параметризованы (параметры задаются в тексте запроса).

Имя таблицы задаётся на русском или английском (например, `Справочник.Товары`). Имена таблиц и полей не могут совпадать с ключевыми словами языка запросов.

**Объектные таблицы** — реальные таблицы для хранения объектов (справочники, документы и т.п.). Каждой объектной таблице соответствует тип объектов; у каждой записи есть поле типа «Ссылка» на текущую запись. Иерархические таблицы имеют поле **Родитель** (ссылка на родительскую запись).

Поле таблицы может быть обычным полем или **вложенной таблицей** (табличная часть документа/справочника — тип РезультатЗапроса с заданным набором колонок). Значения полей могут быть NULL (поле не определено или не применимо для данной записи).

## 8.3. Виртуальные и обычные поля

**Обычное поле** — формируется из данных СУБД (возможна обработка средствами СУБД).

**Виртуальное поле** — формируется из данных СУБД с дополнительной обработкой платформой.

Виртуальными в таблицах запросов являются:

- Поля представлений (поле Представление таблицы или функция ПРЕДСТАВЛЕНИЕ).
- ВерсияДанных, ИмяПредопределенныхДанных, МоментВремени.
- ТипЗначения (таблица плана видов характеристик); Вид (план счетов); Тип (журнал документов); ВидДвижения (регистры бухгалтерии и накопления).
- Поля типов: ХранилищеЗначения, УникальныйИдентификатор, ссылка внешнего источника данных.

Особенности виртуальных полей:

- Не все можно «обратно» преобразовать в значение СУБД (например, представление, ТипЗначения ПВХ).
- Не все виртуальные поля с одинаковым именем из разных таблиц можно сравнивать между собой (ВерсияДанных, ИмяПредопределенныхДанных, МоментВремени, Тип журнала документов).
- Для виртуальных полей строкового типа не поддерживаются операции со строками и сравнение с типом Строка.

## 8.4. Язык запросов

### 8.4.1. Общая информация

Текст запроса — инструкция на языке запросов: какие таблицы и поля используются, правила группировки, сортировки и т.д. Состоит из секций, предложений, ключевых слов, функций и комментариев.

### 8.4.2. Синтаксическая диаграмма конструкций

В правилах: угловые скобки — конструкции языка; ЗАГЛАВНЫЕ — ключевые слова/функции; квадратные скобки — необязательное; вертикальная черта — альтернатива.

```
ЭТО_КЛЮЧЕВОЕ_СЛОВО <Конструкция>, <Конструкция> ЭТО_ФУНКЦИЯ(<Конструкция>) [НЕОБЯЗАТЕЛЬНОЕ] [<Необязательная конструкция>]
ЛИБО_ЭТО | ЛИБО_ЭТО  <Либо эта> | <Либо эта>
```

### 8.4.3. Комментарии

Комментарий — часть строки с `//` до конца строки. При выполнении игнорируются.

```bsl
// Это комментарий
```

### 8.4.4. Предопределённые данные конфигурации

В запросе можно использовать: значения перечислений; предопределённые данные справочников, ПВХ, планов счетов, планов видов расчётов; пустые ссылки; точки маршрута БП; системные перечисления (ВидДвиженияНакопления, ВидСчета, ВидДвиженияБухгалтерии).

Обращение — литерал функционального типа **Значение**(<ПредставлениеЗначения>):

- Перечисление: `<ИмяСистемногоПеречисления>.<Значение>`
- Предопределённое: `<ТипПредопределенногоЗначения>.<ИмяОбъектаМетаданных>.<Значение>`
- Пустая ссылка: `Значение(Справочник.Города.ПустаяСсылка)`
- Точка маршрута: `БизнесПроцесс.<ИмяОбъектаМетаданных>.ТочкаМаршрута.<ИмяТочкиМаршрута>`

Типы предопределённого значения: Справочник (Catalog), ПланВидовХарактерик (ChartOfCharacteristicTypes), ПланСчетов (ChartOfAccounts), ПланВидовРасчета (ChartOfCalculationTypes), Перечисление (Enum).

Примеры:

```bsl
ГДЕ Город = Значение(Справочник.Города.Москва)
ГДЕ ТипТовара = Значение(Перечисление.ВидыТоваров.Услуга)
ГДЕ ВидДвижения = Значение(ВидДвиженияНакопления.Приход)
```

### 8.4.6. Основные секции текста запроса

```
<Текст запроса> = <Описание запроса> [<Объединение запросов>] [<Упорядочивание>] [АВТОУПОРЯДОЧИВАНИЕ] [<Описание итогов>]
```

Обязательна только секция описания запроса. В ней задаются поля результата, источники (ИЗ), условия (ГДЕ), группировка (СГРУППИРОВАТЬ ПО), условия на группировку (ИМЕЮЩИЕ).

### 8.4.7. Описание запроса

**Синтаксис:**

```
ВЫБРАТЬ [РАЗРЕШЕННЫЕ] [РАЗЛИЧНЫЕ] [ПЕРВЫЕ n] <Список полей>
  [ПОМЕСТИТЬ|ДОБАВИТЬ <Имя таблицы>]
  [ИЗ <Список источников>]
  [ИНДЕКСИРОВАТЬ ПО [НАБОРАМ] <Список полей>]
  [ГДЕ <Условие>]
  [СГРУППИРОВАТЬ ПО <Поля группировки>]
  [ИМЕЮЩИЕ <Условие>]
  [ДЛЯ ИЗМЕНЕНИЯ [<Список таблиц>]]
```

- **РАЗРЕШЕННЫЕ** — только записи, на которые у пользователя есть права (учитываются RLS, не право на саму таблицу). Только в предложении ВЫБРАТЬ верхнего уровня.
- **РАЗЛИЧНЫЕ** — исключить повторяющиеся строки.
- **ПЕРВЫЕ n** — ограничить число строк результата (целое число).
- **ДЛЯ ИЗМЕНЕНИЯ** — блокировка считываемых данных в транзакции до её завершения (файловый вариант — таблицы, клиент-сервер — выбранные записи).

Пример:

```bsl
ВЫБРАТЬ Документ.РасходнаяНакладная.Ссылка ИЗ Документ.РасходнаяНакладная
```

**Пакетный запрос** — несколько запросов в одном тексте, разделённые `;`. Выполняются последовательно; связаны через временные таблицы. Каждый запрос пакета выполняется на СУБД отдельно. Поддерживается в системе компоновки данных.

Пример пакета с временной таблицей:

```bsl
ВЫБРАТЬ УчетНоменклатурыОстаткиИОбороты.Номенклатура, ... ПОМЕСТИТЬ УчетНоменклатуры
ИЗ РегистрНакопления.УчетНоменклатуры.ОстаткиИОбороты(, , Авто, ,) КАК ... ;
ВЫБРАТЬ УчетНоменклатуры.Номенклатура, ... ИЗ УчетНоменклатуры КАК УчетНоменклатуры
```

### 8.4.8. Поля выборки и источники

- **Список полей выборки**: `<Поле> [, <Поле> ...]` или `*` (все поля источников; виртуальные поля не включаются).
- Поле: `<Описание поля> [[КАК] <Псевдоним>]`. Описание — разыменование поля, выражение или агрегатная функция.
- **ПУСТАЯТАБЛИЦА.(Псевдоним1, Псевдоним2, ...)** — для объединения запросов, когда в одной части есть вложенная таблица, в другой — нет.
- **Вложенная таблица в выборке**: поле может ссылаться на табличную часть; результат — тип РезультатЗапроса. Группа полей: `(Поле1 КАК Псевдоним1, Поле2)` или `*`.

**Источники (ИЗ):** список через запятую. Источник: `<Таблица> [[КАК] <Псевдоним>]`. Таблица — имя таблицы (для виртуальной — с параметрами) или вложенный запрос в скобках. ИЗ можно опустить, если источники заданы в полях выборки.

**Соединения:** при нескольких источниках без соединения получается декартово произведение. Типы соединений:

- **ВНУТРЕННЕЕ СОЕДИНЕНИЕ** … **ПО** <условие> — только пары, удовлетворяющие условию.
- **ЛЕВОЕ [ВНЕШНЕЕ] СОЕДИНЕНИЕ** — все записи левого источника + подходящие правого; при отсутствии — NULL по полям правого.
- **ПРАВОЕ [ВНЕШНЕЕ] СОЕДИНЕНИЕ** — симметрично.
- **ПОЛНОЕ [ВНЕШНЕЕ] СОЕДИНЕНИЕ** — все записи обоих; при несовпадении — NULL.

В списке источников могут быть вложенные таблицы (ТЧ документов/справочников) и вложенные запросы.

### 8.4.9. Временная таблица

Временная таблица создаётся и используется в контексте сеанса или объекта Запрос (или МенеджерВременныхТаблиц). Служит для промежуточных данных и ускорения запросов.

- **Создание:** `ВЫБРАТЬ ... ПОМЕСТИТЬ ИмяВременнойТаблицы ИЗ ...` Типы колонок задаются неявно; при несовпадении типов при ДОБАВИТЬ возможна ошибка — при необходимости использовать ВЫРАЗИТЬ/ВЫБОР.
- **Внешний источник:** в ИЗ указать параметр `&Параметр` (таблица значений, ТЧ или результат запроса). У таблицы значений должны быть заданы типы колонок. В таком запросе нельзя использовать объединения/соединения и реквизиты полей таблиц источника.
- **Добавление:** `ВЫБРАТЬ ... ДОБАВИТЬ ИмяВременнойТаблицы ИЗ ...` Структура должна совпадать. У поля перед ДОБАВИТЬ должен быть псевдоним.
- **Индексация:** `ИНДЕКСИРОВАТЬ ПО Поле1, Поле2` или `ИНДЕКСИРОВАТЬ ПО НАБОРАМ ((Поле1, Поле2), (Поле3) УНИКАЛЬНО)`. Только для повышения производительности, не для контроля уникальности. При создании нескольких индексов первый в SQL Server — кластерный.
- **Удаление:** `УНИЧТОЖИТЬ ИмяВременнойТаблицы`. Отсутствующая таблица — ошибка.

Временная таблица живёт до: явного УНИЧТОЖИТЬ; уничтожения объекта Запрос (или смены текста запроса); уничтожения менеджера временных таблиц.

### 8.4.10. Фильтрация, группировка, объединение, упорядочивание, итоги

- **ГДЕ** <Условие> — отбор записей из источников. Поле из ГДЕ не обязано входить в список выборки.
- **СГРУППИРОВАТЬ ПО** <Поля> — группировка; в выборке допустимы только поля группировки и агрегатные функции. В списке выборки при группировке не допускается использовать поле и в группировке, и внутри агрегатной функции (ошибка по разным СУБД).
- **ИМЕЮЩИЕ** <Условие> — условие на результаты группировки; только агрегатные функции и поля группировки.
- **СГРУППИРОВАТЬ ПО ГРУППИРУЮЩИМ НАБОРАМ ((Набор1), (Набор2), ...)** — несколько группировок за один обход (один набор агрегатных функций для всех наборов). Поле должно входить хотя бы в один набор.
- **ОБЪЕДИНИТЬ [ВСЕ]** <Описание запроса> — объединение результатов; количество и порядок полей должны совпадать. По умолчанию дубликаты строк убираются; ВСЕ — оставить все строки.
- **УПОРЯДОЧИТЬ ПО** <Поле1> [ВОЗР|УБЫВ|ИЕРАРХИЯ], ... Упорядочивание по иерархии — только по полю-ссылке на справочник; для корректного результата источник должен быть таблицей справочника (при необходимости — соединение со справочником).
- **АВТОУПОРЯДОЧИВАНИЕ** — автоматическое формирование полей упорядочивания (по УПОРЯДОЧИТЬ ПО, по ИТОГИ или СГРУППИРОВАТЬ ПО, иначе по умолчанию для таблиц).
- **ИТОГИ** [<Список итоговых полей>] **ПО** [ОБЩИЕ] <Контрольные точки> — расчёт итогов. Контрольная точка — выражение; по каждой комбинации значений добавляются итоговые строки. **ПО ОБЩИЕ** — одна итоговая строка по всему результату. **ПО Поле ИЕРАРХИЯ** — итоги по иерархии справочника; **ТОЛЬКО ИЕРАРХИЯ** — только иерархические итоги. **ПЕРИОДАМИ**(ВидПериода, Начало, Конец) — для полей типа Дата, дополнение датами периода. При совместном использовании с группировкой без списка агрегатных функций в ИТОГИ он берётся из полей выборки.

### 8.4.11. Выражения, агрегатные и встроенные функции

**Выражение** может быть: разыменованием поля `[Таблица.]Поле[.Поле...]`; агрегатной функцией; встроенной функцией; операцией выбора (ВЫБОР КОГДА … КОНЕЦ); приведением типа ВЫРАЗИТЬ; константой или параметром `&ИмяПараметра`; бинарной/унарной операцией над выражениями.

**Агрегатные функции:** СУММА(<Выражение>), СРЕДНЕЕ(<Выражение>), МИНИМУМ(<Выражение>), МАКСИМУМ(<Выражение>), КОЛИЧЕСТВО([РАЗЛИЧНЫЕ] <Выражение> | *). Используются в полях выборки, ИМЕЮЩИЕ, ИТОГИ, УПОРЯДОЧИТЬ ПО. В списке выборки — только для получения результата, не для сравнения в условии (разное поведение на СУБД).

**Встроенные функции (кратко):** АВТОНОМЕРЗАПИСИ() — уникальный номер строки во временной таблице (только при создании ВТ, не с ОБЪЕДИНИТЬ). ЕСТЬNULL(Выр1, Выр2) — подстановка при NULL. ВЫРАЗИТЬ(Выр КАК Тип) — приведение типа. НАЧАЛОПЕРИОДА(Дата, Период), КОНЕЦПЕРИОДА(Дата, Период) — период: Секунда, Минута, Час, День, Неделя, Месяц, Квартал, Год, Декада, Полугодие. ДОБАВИТЬКДАТЕ(Дата, ТипПриращения, Величина). РАЗНОСТЬДАТ(Дата1, Дата2, ТипРазности). ПОДСТРОКА(Строка, Начало, Длина). ЛЕВ(Строка, n), ПРАВ(Строка, n). ВРЕГ/НРЕГ — верхний/нижний регистр. ПРЕДСТАВЛЕНИЕ(Выр) — строковое представление. ТИПЗНАЧЕНИЯ(Выр) — тип значения. УНИКАЛЬНЫЙИДЕНТИФИКАТОР(Ссылка). ГОД, МЕСЯЦ, ДЕНЬ, ЧАС, МИНУТА, СЕКУНДА, КВАРТАЛ, НЕДЕЛЯ, ДЕНЬНЕДЕЛИ, ДЕНЬГОДА. ОКР(Число, Разрядность). СТРОКА(Значение). СГРУППИРОВАНОПО(Поле) — Истина, если по полю выполнена группировка (для группирующих наборов). Алгебраические: EXP, LOG, LOG10, POW, SQRT. Тригонометрические: SIN, COS, TAN, ASIN, ACOS, ATAN (аргумент в радианах).

### 8.4.12. Приведение типа, константы, параметры

**ВЫРАЗИТЬ**(<Выражение> **КАК** <Тип>) — приведение к примитивному или ссылочному типу (<Имя таблицы>). Если в составном типе есть требуемый — результат значение или NULL для остальных; если типа нет в составном — ошибка.

**Константы:** Истина, Ложь; числа; строки в кавычках; ДАТАВРЕМЯ(год, месяц, день [, час, минута, секунда]); ТИП(ИмяТипа) — тип примитива или ссылки на таблицу; НЕОПРЕДЕЛЕНО, NULL. Параметры: &ИмяПараметра.

### 8.4.13. Условия в языке запросов

Условия (ГДЕ, ИМЕЮЩИЕ, ПО соединения): логические выражения — простые или соединённые **И**, **Или**, **Не** (приоритет: Не, затем И, затем Или; скобки для изменения порядка).

**Логическое выражение:** выражение булева типа; сравнение (<, >, =, >=, <=, <>); **В** (список) или **В** (запрос) / **В ИЕРАРХИИ** (список или запрос); **МЕЖДУ** Выр1 **И** Выр2; **ЕСТЬ [НЕ] NULL**; **ССЫЛКА** ИмяТаблицы; **НЕ** ПОДОБНО шаблону [СПЕЦСИМВОЛ символ].

**В ИЕРАРХИИ** — для ссылки: вхождение в множество или иерархическое подчинение одной из групп из множества.

**ПОДОБНО:** шаблон — `%` (любая последовательность), `_` (один символ), `[...]`, `[^...]`; для литерала спецсимвола — СПЕЦСИМВОЛ.

**Правила сравнения:** при разных типах — порядок по приоритету типа (NULL, Булево, Число, Дата, Строка, ссылки). Сравнение с NULL даёт NULL; в условии NULL трактуется как ложь. Строки — без учёта регистра, концевые пробелы в сравнении не учитываются (в отличие от встроенного языка). Не допускается сравнение полей неограниченной длины (строка неограниченной длины, ХранилищеЗначения, ТипЗначения ПВХ). Операция **В** по нескольким полям: (Выр1, Выр2, ...) **В** (ВЫБРАТЬ Выр1, Выр2, ...) или **В** (&Параметр) — таблица значений с типами колонок.

## 8.5. Выполнение и работа с запросами во встроенном языке

### 8.5.1. Основные приёмы

Объекты: **Запрос** (формирование и выполнение), **РезультатЗапроса** (данные), **ВыборкаИзРезультатаЗапроса** (обход записей). Тип колонки результата определяется по типам выражений и включает NULL.

Типичный фрагмент:

```bsl
Запрос = Новый Запрос("ВЫБРАТЬ Товар.Наименование, Товар.Родитель.Наименование ИЗ Справочник.Товары Товар");
РезультатЗапроса = Запрос.Выполнить();
Выборка = РезультатЗапроса.Выбрать();
Пока Выборка.Следующий() Цикл
    Сообщить(Выборка.Наименование + " " + Выборка.НаименованиеРодителя);
КонецЦикла
```

**Способы обхода:** Выбрать() — линейный; Выбрать(ОбходРезультатаЗапроса.ПоГруппировкамСИерархией) — по уровням иерархии; Выбрать(ОбходРезультатаЗапроса.ПоГруппировкам) — по группам (итоги как детальные). У текущей записи можно получить вложенную выборку (Выбрать(...)) для дочерних. Методы: Следующий(), СледующийПоЗначениюПоля(ИмяПоля), НайтиСледующий(); Уровень(), ТипЗаписи() (групповой итог, итого по иерархии, детальная, общий итог).

### 8.5.2. Временные таблицы

Работа обеспечивается синтаксисом языка запросов (ПОМЕСТИТЬ, ДОБАВИТЬ, УНИЧТОЖИТЬ) и объектом **МенеджерВременныхТаблиц**. Менеджер задаёт время жизни временных таблиц; при откате транзакции ВТ, созданные в ней, удаляются. Создание: Новый МенеджерВременныхТаблиц. Связь с запросом: Запрос.МенеджерВременныхТаблиц = МенеджерВременныхТаблиц. Чтобы использовать одну и ту же ВТ в разных запросах: установить один и тот же менеджер в оба запроса. Закрыть менеджер — Закрыть(). Для отладки: ВыполнитьПакетСПромежуточнымиДанными(); доступ к таблицам по имени или индексу через МенеджерВременныхТаблиц.Таблицы.

### 8.5.3. Пакетные запросы

Выполнение: Запрос.Выполнить() — выполняется весь пакет, возвращается результат последнего результативного запроса (создающего/обновляющего/удаляющего ВТ или последнего). ВыполнитьПакет() — массив результатов по каждому запросу пакета по порядку; запрос на УНИЧТОЖИТЬ даёт Неопределено в массиве.

### 8.5.4. Программная работа с запросами

**СхемаЗапроса** — формирование и модификация запроса из кода: ПакетЗапросов, УстановитьТекстЗапроса(), ПолучитьТекстЗапроса(), доступ к операторам, источникам, полям, отбору, порядку, итогам. Режим компоновки данных: СхемаЗапроса.РежимКомпоновкиДанных = Истина (свойства ВыраженияИтогов, КонтрольныеТочкиИтогов, ОбщиеИтоги недоступны; при установке текста с ИТОГИ — исключение).

**КонструкторЗапроса** — интерактивный редактор запроса во время работы: Показать(ОбработчикОбратногоВызова), свойство Текст (редактируемый/отредактированный текст), РежимКомпоновкиДанных для режима СКД.
